Overview

Use the Import feature if you want to set-up and configure contracts and their budgets off-line. I.e. bypass the quote/accept process where budgets are normally built.

A contract may be updated by importing a compatible .csv file. Sample CSV File

[edit]

Notes about the csv file

  • The file header names are specific to this routine and should not be changed.
  • Date fields must be formatted as follows YYYY-MM-DD or DD/MM/YYYY.
  • Numeric fields cannot be greater than 2 decimal points e.g.: 56.325 will fail, 56.33 will import successfully.
  • Zero values can be entered as '0'.
  • "fm_con_fuel_consumption_lower_limit" cannot exceed "fm_con_fuel_consumption_upper_limit"
  • The repeating columns, "fm_conb_posting_class_code" and "fm_conb_amount_net" can be inserted or deleted as required. They should reflect the budgets used in the contract.
  • Only one Redbook or Glasses code can imported at a time. If both fields are populated, the import will fail.

Contact Catch-e to get detailed support for this feature.

Import Process

  1. When your csv file is ready, go to the relevant contract record in the Contracts screen
  2. Click on the Import button
  3. Browse to find the relevant file (the naming is not important)
  4. Upload - the system will check that the contract id in the import file is the same as the system contract id. The system will display a screen of proposed changes which the user can accept or cancel. Registration changes are included in the import function and create the appropriate RC contract event when importing.
  5. If there are validation errors, close down the page and fix the errors. Once the validations have all passed "Save"
