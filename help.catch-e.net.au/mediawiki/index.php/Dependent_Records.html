Overview

Use a UUID placeholder to import dependent records without needing to populate the parent key field into the import file for the child records.
This feature is available in the following places:

  1. Setup / Import Table Data
  2. APIs → Import / Export → Request a table data import
  3. APIs → Import / Export → Insert or update a record with auditing


Below is a summary of the steps required without this function, using approvals as the example. The same principle can be used for any table relationship. eg. [fm_drivers]driver_id and [fm_drivers_assets]driver_id.

  1. Import [fm_approvals] records.
  2. The [fm_approvals]approval_id fields are populated during the import.
  3. Export the detail of the [fm_approvals] record you have just imported, , or identify the [fm_approvals]approval_id some other way.
  4. Update the [fm_maintenance] records you want to import to contain the right [fm_maintenance]approval_id value to link the dependent records to their parents.
  5. Import the updated [fm_maintenance] records.

Below is a summary of the same example as above, but using this function.

  1. Populate each [fm_approvals]approval_id field with a unique UUID value. Populate each [fm_maintenace]approval_id field with the same UUID value as it's parent.
  2. Import the [fm_approvals] records. During the import, the [fm_approvals]approval_id fields are populated and the [fm_approvals]approval_id and the original UUID value are stored in [gb_import_table_field_translations].
  3. Import the [fm_maintenance] records. During the import, the [fm_maintenance]approval_id fields are populated with their parents value. The details stored in [gb_import_table_field_translations] enables this.

Detailed Example of using a placeholder value to import dependent records without reference

You want to create a batch of three approval records. Each of them has at least one corresponding maintenance row.
To do this, you need a universally unique identifier (UUID) for each approval record you are creating.
A UUID can be generated in a number of ways. One way is by generating them using a generator tool such as: https://www.uuidgenerator.net/ (use Version 4 for best results).
The format of the UUID must be: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx. The UUID is not case sensitive.

Once you've generated the required UUID values, put the UUID into the place of the table's Primary Key (the Unique ID of the table).
The syntax in the field is 'ref(UUID)'. E.g. ref(xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)

Here's a simple example:
