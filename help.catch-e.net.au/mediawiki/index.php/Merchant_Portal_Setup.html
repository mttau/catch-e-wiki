URL Access

Standard Merchant Portal URLs are as follows:

https://CLIENTNAME.portal.catch-e.com/ for Live
https://CLIENTNAME.portal.support.catch-e.com/ for Support
https://CLIENTNAME.portal.staging.catch-e.com/ for Staging
https://CLIENTNAME.portal.test.catch-e.com/ for Test

[edit]

Create Franchise role

The franchise role is not a standard role and needs to be created if a Merchant Portal is being set-up for your business.

[edit]

Set-up access to Contracts

Contracts can only be accessed by franchisers if the contract is active (i.e. there is no "Suspend Date" entered) and the Contract Types / Details “Allow External Approvals” check box is selected.

Access can be further restricted to only allow access to contracts with a service profile by setting [gb_controls]approvals_franchise_service_profile_required_flag to 'yes'.

Your portal can also be configured to require the selected contract to be linked to an active card. This is useful if you want to use a card record as a key to manage access.

[edit]

VMRS Codes available on Portal

VMRS Codes are only available on the Franchise Portal where

  • the 'external_role_access_flag' is set to 'yes'

and

  • approval_job_type_id is populated
  • approval_item_job_type_id is optional.

Note: A new VMRS Code 'UNEXP' must be created (if not already existing).

[edit]

GST Calculations

GST calculations in the portal screens use the same business rules as the Approvals screen.
The GST calculation is determined by the GST flag on the Contract Types / Expenses-Approvals screen for the "Class" that is attached to the approval row.
The "Class" should be pre-set in the VMRS codes / Details screen for all VMRS codes you make available to franchisers.
If there is no "Class" attached to the approval row, the system assumes the GST flag is 'Y' and will calculate it using the rate stored in [gl_gst_codes]gst_rate.

[edit]

GST Tolerance Checks

GST tolerance checks are not performed in portal screens.
They are performed internally when all approvals go through the Check/Post process.

[edit]

2 Factor Authorisation (2FA)

To use 2 factor authentication (2FA), follow the steps below. Also review the 2 Factor Authentication Setup page for an overview of this feature.

  1. Go to Setup / Reference Data and select the gb_controls table. Find the record for "2factor_auth_enabled_flag" and set this to 'yes'.
  2. Check and update System / Roles for 'franchise'. If the "Mandatory" check box is left un-checked, the franchiser can optionally choose to use 2FA. If it's checked, the franchiser is required to use 2FA and will be taken to the enrolment screen on login.
  3. If the franchiser has a problem using 2FA, internal users can unenroll the franchiser in the Users / 2 Factor Authentication screen.
