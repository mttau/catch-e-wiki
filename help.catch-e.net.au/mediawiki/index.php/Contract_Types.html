Overview

Contract types are used to define the business rules of particular products or services offered.
A product is typically a type of lease or service such as ‘Fully Maintained Operating Lease’ or ‘Fleet Managed’ service.
The contract type defines the transaction classes allowable, GST defaults per class, recharge flags per class and general ledger codes applicable per class.

[edit]

Entering GL Codes

The Accruals tab and those to the right of it are used to store the GL Codes to use for each transaction class.
The GL Codes recorded in the "Debit", "Credit", "Debit (R)" and "Credit (R)" fields are used to write accounting transactions when records are posted.
The GL Codes recorded here are not validated against the records stored in Accounts / General Ledger / GL Codes.
If the records match, the GL Codes / Details "Description" can be used on accounting map reports.

[edit]

Tabs

Screen Description
DetailsThis screen contains the description of the contract type and default information, such as whether inertia billing is allowed, whether GST applies to residual values and an Approval Note which appears when a new approval is raised.
AccrualsContains the allowable lease budgets used on the lease screen defined by Classes. Also contains the GL codes allocated during the initial and periodic billing processes which creates the general ledger journal entries associated with billing.
BillingContains the type of billing allowable for this type of contract, for example Fleet Management Fees, Initial lease charges or Periodic charges. These different types of billing allows control over the invoice template and hence the content of the invoices.
Initial FeesContains the fees allowed on initial billing. These are linked to the client defaults options. Typically a Novated Lease will have and establishment fee while other lease types may not.
FeesContains the fees allowed on any periodic billing. These are linked to the client defaults options. Typically these fees are applied for the first period of billing where an initial invoice is not used. This enables fees in the first billing period to be included in the periodic billing invoices without needing to run an individual initial invoice. The fees are shown on seperate rows on the periodic invoice.
Expenses-ApprovalsContains default values used when raising approvals.
Expenses-OtherContains default values used by batch entry processes, e.g. registration renewals.
SaleContains default values used by the Sale Management module, e.g. for generating sale car invoices.
PayoutContains default values used by the Sale Management module, e.g. for generating payout transactions including Recipient Created Tax Invoices.
BookingContains default values used by the Bookings module. This is currently in development.
RentalContains default values used by the Bookings module. This is currently in development.
PaymentsEnables the set-up of Payment Schedules in the Contracts/Payment screen.
