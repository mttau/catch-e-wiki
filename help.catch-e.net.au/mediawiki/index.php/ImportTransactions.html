Overview

This job will import and post card service files (e.g. fuel transactions) by automating the steps defined in Card Transactions.
It targets import file(s) stored in your DMZ area.
Each file is removed once it has been processed.

Related page: WEX Motorpass

Process Pages

Import a card transactions file
[edit]

Field Descriptions

Field Type Description [table]field
Parameters
Card Interface Is the interface_code from fm_interfaces for the card supplier providing the files. A search icon is provided to find search for this. Card Interface cannot be blank.
File Pattern A regular expression pattern used to match the card file(s) within the DMZ Subfolder. e.g. ^FMFP[0-9]{4}.txt$ will match FMFP0122.txt, FMFP0123.txt... File Pattern cannot be blank.
DMZ Subfolder Is the subfolder in the DMZ area where the card file(s) are to be found. If left blank, the DMZ home folder will be used. Subfolders can be nested.
Example : 'fuel/out'
Queue
Recycling Check Box Check to recycle the job when it completes successfully. We recommend this field be checked for this type of job.

Import Process Outcomes

When the Scheduler job is triggered, one of the following outcomes will occur.

Import Result Cause Comments / Action
Failed Import File does not import Incorrect file format Confirm file received is in expected file and content format. If not, please contact your fuel supplier representative.
File does not import Duplicate File File has been previously imported
File does not import .zip file This scheduled process does not accept .zip files
Import with validation warnings Pending status Incorrect product A batch has been created and can be retrieved from the Card Transactions screen.
To resolve an incorrect product, follow this process.
Pending status Incorrect gl code A batch has been created and can be retrieved from the Card Transactions screen.
To resolve an incorrect gl-code, follow this process
Pending status Failed card A batch has been created and can be retrieved from the Card Transactions screen.
To resolve a failed card validation, follow this process.
Pending status Failed date A batch has been created and can be retrieved from the Card Transactions screen.
To resolve a failed card validation, follow this process.
Posted status Expired Card A batch has been created and can be retrieved from the Card Transactions screen.
To resolve a failed card validation, follow this process.
Posted status Duplicate transaction A batch has been created and can be retrieved from the Card Transactions screen.
To resolve a failed card validation, follow this process.
Successful import Posted status A batch has been created and can be retrieved from the Card Transactions screen.


[edit]

Set-up Notes

If [gb_control]fuel_treat_inactive_cards_as_valid_flag is set to 'yes', card transactions will be processed and imported even when the card's "Status" is 'inactive'. The default setting is "no".
