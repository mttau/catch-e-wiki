Overview

The with Transactions version of the Balance screen uses mapping tables to present budgets, actuals and variances in your stored groupings.
The total of each Actual spend, the posted Receipts and the Current funds balance incl GST amounts are shown as hyperlinks.
Users can click on these to navigate into a transactions screen showing detailed records for the selected total.
Users can also generate excel or pdf reports for the transactions.
By default this screen is configured to display the Budget, Actual and Variance totals that are also reported in the Funds Check pop-up screen. This is Catch-e's recommended set-up.
However, this can be changed to match the cost selection rules used in the Driver Report if you prefer.

Transaction Details

To view the transaction details for each reported budget type, click on the hyperlink amounts in the Actual column or the Funds received, Less actual spend or Current funds balance incl GST field amounts.

  • When an Actual column value is clicked, all transactions for that budget are displayed with a total for the selected budget.
  • When the Funds received amount is clicked, all posted receipt transactions are displayed with the total.
  • When the Current funds balance incl GST amount is clicked, all transactions for all budgets and receipted amounts are displayed. When all transactions are selected here, users can select to output a file of these transactions using: 'Output to PDF' and 'Output to Excel'.

Users can click on the Columns... button and customise which columsn they want to show or hide on the transactions screen. 'Output to PDF' and 'Output to Excel' report all available columns, even if the user has hidden them on the screen. The columns your users can see initially depend on how your report has been configured. Your organisation can also hide specific columns if desired.
