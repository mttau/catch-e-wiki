Overview

glmyob_export_pl_surname_in_description_flag 'yes'
glmyob_export_pl_transaction_dateSet this control to ‘invoice_date’ to populate the PL export file fields ‘Date’ and ‘Shipping Date’ with the stored [gl_transactions]date. This is like choosing an accrual accounting method.

Set the controlto ‘posted_date’ to populate the PL export file fields ‘Date’ and ‘Shipping Date’ with the stored [gl_transactions]posted_date. This is like choosing a cash accounting method.
If the control is set wrongly, the PL export file will default to using 'invoice_date'.

invoice_date
glmyob_export_summarise_data
  • When both 'sl' and 'pl' are set to 'no', MYOB SL & PL export files are created with the multiple transaction lines that have been invoiced to the client or paid to the supplier.
  • When set to 'yes', the SL export file is created by grouping same transactions to individual line(s) eg: Where recharges are included in 'Combined' Billing, they will be itemised separately to the Lease Charges.
  • The PL export file will group each transaction based on gl_code to one line.
Once a file has been exported, the format cannot be altered by changing the gb_control setting alone.

Contact Catch-e support to clear the data from the gb_exports table for a particular export batch and re-set [gb_control]myob_export_summarise_data.
You can then re-export the batch in the preferred format. Re-export the batch immediately to avoid possible data corruption.

Default settings: sl 'no', pl 'no'

Refer to General Ledger Export To MYOB for further details. Valid entries are: none, surname,contract_id,reg_no,client_order_no.'surname'
