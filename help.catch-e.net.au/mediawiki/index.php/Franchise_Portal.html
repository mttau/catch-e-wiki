Overview

The franchise role does not have access to the primary system and is used in a separate system called the Merchant Portal.

The franchise role is not a standard role and is only created if a Merchant Portal is being set-up for your business.

The Merchant Portal has been set up to allow users to Raise, Edit and Finalise Approvals without the need to phone a Maintenance Controller for authorisation. This is done by configuring Client and Contract approval limits to allow approvals to gain automatic approval if they pass the validation checks. Edit and Finalise tolerance limits can be set to allow for small variations to the original approval.

[edit]

Work Flow

Approvals are created by franchise user in a 2-step process
1. The user raises a new approval and enters an estimate for each job being undertaken.

Users can create a new approval estimate when a correct Registration number is entered. They cannot look up vehicles in the system via a search – they must know the information. The purpose of their access is to raise approvals for work they plan to do on a vehicle booked in for scheduled or unscheduled maintenance work.

If the approval estimate progresses and is set to an Active status, the work may proceed. If a validation issue occurs, the user must ring your Maintenance Controllers to get the approval set to Active, or be advised that their approval request has been rejected either on the phone by a Maintenance Controller or by email.

2. On completion of the work, the final values are entered.
External users can review their approvals on an “Update Approvals” screen and either “Edit” their approvals to modify the estimates values, or “Finalise", which updates costs on the approval rows and finishes the approval.

Capped Price Services

Capped Price Services can be set up using the Service Groups (Parts Explosion) feature where the VMRS code from the 'VMRS Group' has the Capped Price Service flag checked.
The table [fm_vmrs_group_items] which holds the relationship and values between a Service Group and its individual parts is optional and does not require the Capped Price Service flag to be checked. A VMRS Group [fm_vmrs_groups] record must exist however for this feature to work in the Franchise Portal.

By adding a “Cap Price Service” flag to the VMRS Group Code, when Finalising the Approval, all amount and quantity fields will be read-only. The Total will be prepopulated and equal to the Amount entered when requesting the New Approval.

Here is a useful Query and it's results showing the 'VMRS Group' and its Capped Price Service flag along with it's individual parts/items and its Capped Price Service flag(s). 

SELECT t2.vmrs_code AS 'VMRS Group', t2.description AS 'VMRS Group Description', t2.capped_price_service_flag
, t4.vmrs_code AS 'Group Items', t4.description AS 'Group Items Description', t4.capped_price_service_flag
FROM fm_vmrs_groups t1
INNER JOIN fm_vmrs_codes t2 ON t2.vmrs_code_id = t1.vmrs_code_id
INNER JOIN fm_vmrs_group_items t3 ON t3.vmrs_group_id = t1.vmrs_group_id
INNER JOIN fm_vmrs_codes t4 ON t4.vmrs_code_id = t3.vmrs_code_id
WHERE t2.capped_price_service_flag = 'yes'
AND t2.vmrs_code = '10C';

Validation Rules

Contracts available on Portal

  • Contracts must be active (i.e. there is no "Suspend Date" entered)
  • The Contract Types/Details “Allow External Approvals” check box is selected

Access can be further restricted to only allow access to contracts with a service profile by setting Global Controls field approvals_franchise_service_profile_required_flag to 'yes'. This is useful if you are using the Service Groups feature.

VMRS Codes available on Portal
VMRS Codes are only available on the Franchise Portal where

  • the external_role_access_flag is set to 'yes'

and

  • approval_job_type_id is populated
  • approval_item_job_type_id is optional.

Approval Validation Rules
An Approval may be automatically approved if it meets the following conditions:

  • Client Approval Defaults Approval Limit. If total approval estimates entered is less than Approval Limit.
  • Contract Vehicle Approval Limit. If total approval estimates entered is less than Approval Limit.
  • “Job Type”, ”Description” and “Est. Retail Cost excl GST” must all be completed. Incomplete rows are discarded.
  • Interim Service?
  • Duplicate Service?
  • Duplicate VMRS Code?
  • Excluded VMRS Codes from Approval Defaults list
  • If one row contains a Posting Class not authorised for the Supplier entering the Approval the approval is automatically set to Requested. See Approvals:Authorised Suppliers

Additional Finalise Validation Rules

  • If Maintenance Controller has approved a previously 'Requested' approval the controller will have also set an Approval Approval Limit. Finalise amounts entered will be validated against this limit, along with revalidating against Client & Contract Approval Limits.
  • Labour tolerance checks
  • GST tolerance checks
  • All amounts must be positive
  • Reference number populated
