Enter claims

Qualifiers

  1. You have the Scheduler Jobs set up for processing claims.
  2. You have received documented claims for an employee's expenses that can be deducted from their pay (and then paid)
  3. The employee has an Employees / Item record where
    1. Benefit Type is 'Exempt'
    2. Status is 'Active'
    3. Disbursement Type is 'Claim'
    4. Contract ID is populated
    5. the linked Contracts / Lease contains pending periods


Process

  1. Navigate to Contracts / Claim
  2. Click on
  3. Date - select today's date (this is not important - a Scheduler job updates this later to show the selected pay run budget date)
    1. Budget - if the claim is being reimbursed to the Employee, choose 'Claim excludes GST'
    2. Budget - if the claim is being reimbursed to the Employer who holds a Tax Invoice, choose 'Claim includes GST'
  4. Description - enter a description of the expense
  5. Qty - enter a quantity to push the required expense into a future payroll period. This must be a value between 1 and 5. If you enter anything else, the claim will be added to the next period (I.e. the next pending)
    1. 0 or 1 will update the next pay run period deductions. E.g. 02/08/2021
    2. 2 will update the next pay run period + 1. E.g. 09/08/2021
    3. 3 will update the next pay run period + 2. E.g. 16/08/2021
    4. 4 will update the next pay run period + 3. E.g. 23/08/2021
    5. 5 will update the next pay run period + 4. E.g. 30/08/2021
  6. Total - enter the total including GST.
  7. GST - GST will calculate if required and can be edited to match if the rounding is not correct. It will set to 0 if the claim excludes GST.
  8. Click on
  9. Click on
  10. Attach the supporting file you have received to the claim
  11. Click on
  12. Expect to see a message that says 'Current funds balance will be -$###.## after this payment and is now Pending Funds


Next Steps

A scheduler job will run and

  1. Add the amount of the claim to the specified Contracts / Lease Period PRETAX budget record.
  2. Date
    1. If the the Contracts / Lease Period date is before or within the current month, the claim row Date is updated to match
    2. If the the Contracts / Lease Period date is beyond the current month, the claim row Date is updated to show as the last day of the current month
  3. Odo is updated to show the matching Contracts / Lease Period
  4. Qty is updated to show as 0
