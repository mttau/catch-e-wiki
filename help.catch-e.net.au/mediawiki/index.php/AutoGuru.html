Overview

AutoGuru's FleetGuru integration can be set-up if you engage them to manage vehicle servicing and tyres for you.
When arranged with the supplier, the ongoing process consists of the following two steps

  1. Create and send a daily file of vehicles to manage with required details
  2. Receive and process a file of completed bookings (Approvals).


For more information visit: https://www.autoguru.com.au/about/catch-e-fleetguru-integration-partner

Client Set-up

  1. Suppliers / Detail - create a supplier record for AutoGuru
  2. Contract Types / Details "Autoguru Export" - check on for each contract type to include in AutoGuru processing.
  3. Contract Types / Expenses-Approvals "Class" - records are present for the classes M, S and T.
  4. Posting Classes / Details "Autoguru Export" - review the posting class list and check that only the appropriate records are checked. By design, only M, MAINT, S and T should be checked.
  5. Contracts / Vehicle "Autoguru Export" - update existing contracts as checked for each contract to include in AutoGuru processing.

Screen Aliases

Create records in [gb_screen_aliases] for the following three fields to change the field name "1Link Export" to "AutoGuru Export"

  1. Contract Types / Details
  2. Posting Classes / Details

[edit]

DMZ Access

  1. Catch-e SysAdmin creates SSH public and private keys with the public key issued to AutoGuru
  2. Catch-e run API's to set the file destination access:
    1. Use createDmzUploadFileDestination to create a new File Destination. This method requires the DmzUploadFileDestinations:Create permission to be associated with your role
    2. Use setDmzUploadFileDestinationEncryptionKey to update encryption key (private key) stored within the configuration in an existing File Destination. This method requires the DmzUploadFileDestinations:SetEncryptionKey permission to be associated with your role.

Useful Queries

# Check Screen Alias details
 # There should be 3 records
SELECT * FROM gb_screen_aliases WHERE field = '1link_export_flag';

# Contract Types
# 1link_export_flag = 'yes' if being exported
select posting_map_group, posting_map_code, pm.name, 1link_export_flag
from gl_posting_maps as pm 
order by 1link_export_flag, pm.posting_map_group, pm.posting_map_code
;
# Posting Classes
# M, MAINT, S and T should be set to 'yes'. All others should be 'no'
select rg.name as category, pc.posting_class_id, pc.posting_class_code, pc.name, pc.1link_export_flag
from gl_posting_classes as pc
LEFT JOIN gl_reporting_class_maps AS rcm ON rcm.posting_class_id = pc.posting_class_id
LEFT JOIN gl_reporting_groups AS rg ON rg.reporting_group_id = rcm.reporting_group_id
ORDER BY pc.1link_export_flag, rg.name desc, pc.posting_class_code
;
# Contract Types with count of contract settings
select posting_map_group, posting_map_code, pm.name, pm.1link_export_flag as contract_type_export, c.1link_export_flag as contract_export, count(c.contract_id) as contracts
from fm_contracts as c
inner join gl_posting_maps as pm on pm.posting_map_id = c.posting_map_id
where c.suspend_date = '0000-00-00' and c.contract_start != '0000-00-00' and c.reg_no != 'ORDER'
group by pm.posting_map_code, c.1link_export_flag
order by pm.1link_export_flag, c.1link_export_flag
;
# Contract Listing by setting
select posting_map_group, posting_map_code, pm.name, pm.1link_export_flag as contract_type_export, c.contract_id, c.reg_no, c.1link_export_flag as contract_export
from fm_contracts as c
inner join gl_posting_maps as pm on pm.posting_map_id = c.posting_map_id
where c.suspend_date = '0000-00-00' and c.contract_start != '0000-00-00' and c.reg_no != 'ORDER'
order by pm.1link_export_flag, c.1link_export_flag, c.contract_id desc
;
