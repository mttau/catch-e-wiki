Overview

This screen is available for use in the Driver Portal and the Employee Portal.
Sometimes salary deductions do not match the required Employee Contribution needed for the year, so ECM needs to be done via Contributions.
In this case, users can be prevented from making portal claims and must instead raise an ECM record. The Catch-e system contains an ECM event. Values can be stored in this event to record employee contributions.
These sum of these records are added to any ECM values posted in contract billings and used in FBT reports and calculations as the Employee Contribution amount.

A companion event called ECMR will allow users to create an event that defines the Employee Contribution that is required in a given FBT year.
The creation of this event triggers business rules in the Driver Portal to direct the driver to record an Employee Contribution where one is required.

To use this feature the follow set-up is required. Open Driver Portal Employee Contributions for screenshots and illustrations;

  1. A Posting Class called ECMR must exist. You can create this if you want to use this feature.
  2. Create a contract event record for each contract where an ECM shortfall is required to be made up by the driver. The Type is ‘EmployeeContribution Required’. The Date must be within the FBT year where there is a shortfall. Enter the shortfall into the Amount field.
