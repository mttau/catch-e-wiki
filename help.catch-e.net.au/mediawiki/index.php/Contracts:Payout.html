Overview

The Payout tab is designed to enable users to:

  • Create payout request records which can have a calculated payout value (open the Payout Calculation spreadsheet for details of the calculation).
  • Enable an event to be created when a payout record has been created
  • Enable emails to be sent that support the payout process
  • Record various types of payout amounts: a financiers payout, price to purchase the car, early term the car or record a total loss amount.
  • Visit Payout Set up for details of how to set this feature up.
[edit]
Role Access

The following roles have been configured to provide payout information and have full access to this screen: Admin, Accounts, Disposals, Senior Manager and Manager.

All other roles are been configured to request payout information. The following fields are hidden from these users and they have a limited ability to update record statuses:

  • Remaining
  • Calculated Value
  • Financier Value
  • Valid until

Remaining Number This field shows the remaining 'pending' budgets periods for the contract in months. If it displays as '0', all budgets have been invoiced.
The calculation for this field is:
(YEAR(IF([fm_contracts]residual_due_date = '0000-00-00', [fm_contracts]contract_end, [fm_contracts]residual_due_date)) - YEAR([fm_contract_budgets]payout_date)) * 12 + MONTH(IF([fm_contracts]residual_due_date = '0000-00-00', [fm_contracts]contract_end, [fm_contracts]residual_due_date)) - MONTH([fm_contract_budgets]payout_date)
If this calculation returns a negative number of periods, this field will be set to '0'. and “Calculated Value” = $0.00 		[fm_contract_payouts]remaining_billings
Calculated Value Number This field displays the ‘present-value’ calculation that was generated when the record was created. A value of zero can occur when all budgets have been invoiced. Status Listbox New records default to a status of 'Pending'. Change to 'Requested', 'Financier', 'Review', 'Open', 'Sent', 'Accepted' and 'Invoice Sent' to support your workflows. 'Invoice Sent' is only available to users with full access.

'Expired' is not available to users - it is used to expire records in a scheduler job which runs daily at 11:00pm. It updates records to 'Expired' if the status is not 'Accepted', 'Invoice Sent' or 'Deleted' AND the "Payout Date" is older than 30 days, or if the "Valid Until" is older than 30 days.

PendingDefault status when creating a record.Record saved, no further actions are performed.

  • Financier Value Mandatory flag = yes,
  • Contract is within 12 months of Finance Start Date,
  • "Calculated Value" > $40,000
    If a Financier Value is not available, save the record at this status.
 Validation is performed for these conditions. If any fail validation, you will be warned and will be left in Edit mode to update the Financier Value & Valid to date, or update Status to 'Financier'.
Review Optional Status to allow another user to review the payout record content. Record saved, no further actions are performed.
OpenYour Finance department will use this status when the Payout value has been saved and is ready to be sent to your Client.When the Payout Status has been set to "Open", if [gb_control]payout_notifications has been set up, an email will be sent to the Payout record creator when the Payout Notification Open field is set to yes.
SentUse this status to show that you've sent the Payout quote to the Client.Record saved, no further actions are performed.

Invoice SentUsers with full access can update to this status to record that an invoice has been sent to the ClientRecord saved, no further actions are performed.

CancelledUse this status to show a cancelled Payout request.Record saved, no further actions are performed.

Action buttons

Field Description
Edit is enabled once you checks one # column checkbox. Only one record at a time can be edited. Once a record is in 'Accepted' status, only an Admin or Senior Manager role users can edit the record.

All users without full access can edit records at 'Pending', 'Open' and 'Sent' status, but the editable fields are limited to "P2P", "ETF", "TL" and "Status".

Only Admin or Senior Manager role users can use the Delete button.

Output Options

Form NameDescription
Payout ReportA default customised report is available for use. The Payout Report will only work if the # checkbox is checked to identify the record that is to be printed.

From this, you can create a form that suits your processes. For example, you can advise your client or driver of the payout details. For this reason the report cannot be generated when the record is at 'Pending' or 'Requested' status because the data is not complete.
