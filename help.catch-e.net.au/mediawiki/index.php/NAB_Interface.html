Overview

The NAB interface is designed to import fuel transactions, but not other services.

[edit]

File Details

This file should be in .csv format.
There is no header row in this file.

File Content

Column A –Client name (data not used, column must be present)
Column B – Cardholder name
Column C - is the card number
Column D – is the date of the transaction
Column E – is the amount – inclusive of GST
Column G – is the supplier address

[edit]

Interface Treatments

Odometer readings are not provided in this file. Maintenance records show "Odo" as blank and invalid.
Quantities are not provided in this file. Maintenance records show "Qty" as 0.
GST is not provided in this file. GST is calculated from the amount in column E.
[edit]

Set-Up

[edit]
Interface Mappings

All items should be mapped to product type FUEL

[edit]
Other Services

Should your client hold a card specifically to pay for servicing, tyres etc. that card can set as a record in Fleet / Card Services / Cards. The card would not be allocated to a contract and the field “posting enabled” is unchecked.
