[edit]

Overview

GST claimable is calculated based on the linked Vehicle Type (e.g. the vehicle may be exempt from GST limits) and the maximum GST claimable. This amount can be edited in each quote.
Contract Types / Details "ITC/Residual GST Code" records whether GST is claimable and applicable for the Residual Value of a vehicle being quoted.
Visit ATO Cars and Tax for more details.
Also visit the LCT Limits page to see the current claimable gst amount that is applied here.

[edit]

GST Claimable calculation rules

  1. If Contract Types / Details "ITC/Residual GST Code" = 'N' then "GST Claimable" = 0.00
  2. If Quotes / Vehicle "Vehicle Type" is LCT exempt, then "GST Claimable" = Quotes / Vehicle "Total On Road Price - GST"
  3. If Quotes / Vehicle "Vehicle Type" is not LCT exempt, then "GST Claimable" = Quotes / Vehicle "Total On Road Price - GST", but limited to the value of [gb_controls]maximum_gst_claimable


[edit]

Identifying your "Vehicle Types"

Vehicle Type details are stored in the table [qt_vehicle_types]] and each record is tagged as either being LCT exempt or not.
To find out which vehicle types are LCT exempt in your system, export the table data for 'qt_vehicle_types'

  1. Module - 'qt | Quotes'
  2. Table - 'qt_vehicle_types'

If the record's [qt_vehicle_types]lct_exempt_flag is 'yes', then it is LCT exempt.

[edit]

Vehicle Type

Each variant has two links to the table that identifies the "Vehicle Type".

1. Quotes / Vehicle Specifications "Variant ID" - hyperlinks to Setup / Reference Data → Detail to see the first "Vehicle Type".
This is the "Vehicle Type" that your data provider (RedBook or Glasses) allocated to the variant.
This is stored as [qt_variants]vehicle_type_id.

2. Quotes / Vehicle Specifications "Vehicle Type"
The vehicle type shown here determines if Luxury Car Tax, Luxury Car Charge and a GST Claimable cap should be applied to quotes created for this variant.
It is also used is Purc. Stamp Duty calculations for NSW.
This is stored as [qt_variant_profiles]vehicle_type_id.

If you don't agree with the allocation from your data provider (RedBook or Glasses), you can update the variant profile to record the one you want to use for system calculations.
Go to Setup / Reference Data → qt_variant_profiles and update the selected variant.

[edit]

Change the variant profile "Vehicle Type"

Commercial vehicles may be exempt from LCT and you can update your variant's settings to reflect this.
Once updated, the "GST Claimable" value of new quotes for these variants will not be limited.

  1. Go to Setup / Reference Data → qt_variant_profiles
    1. Choose the "Make", "Model" etc details to find the variant you want to change
    2. Or enter the "Variant ID" to go straight to it
  2. Edit the variant record and update the "Vehicle Type" to match the lct_exempt_flag setting you want
  3. "Vehicle Type" needs to match the field [qt_vehicle_types]vehicle_type_code
  4. "Data Source" needs to match the field [qt_vehicle_types]data_source_id


[edit]

Amount Financed is out of balance Alert

You may be alerted to update the Claimable GST when saving a quote.
If you see the error message, it is likely your administrator has changed the method of calculating Option GST recently.
Please follow the on screen instructions to fix your quote.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.
[edit]
Quotes / Finance - "GST Claimable" is limited for an exempt vehicle

If you are quoting a variant that is LCT exempt and the "GST Claimable" amount is being limited, you can change the variant's "Vehicle Type" so this does not happen.
To do this, go to the GST Claimable page and follow the update steps listed there.

[edit]
Quotes / Finance - "GST Claimable" is not correct

If you are quoting a variant that has GST that is more than can be claimed, then the "GST Claimable" amount is reduced to the allowable amount.
This amount populates into the Quotes / Finance "GST Claimable field by default from the control [gb_controls]maximum_gst_claimable.
If this default is not correct, update the global control. Go to 'Setup / Reference Data → gb_controls and update the control 'maximum_gst_claimable'.
