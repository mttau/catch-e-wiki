Main Page → Fleet / Administration / Contracts → Payments → Payments Setup → Sundry Payment

[edit]

Overview

Use the "Payment Type" 'Sundry Payment' to set-up a payment schedule to make payments for Sundry Payment.
This is only available when it has been set up for use in the Contract Types / Payments tab.
You can set-up multiple 'Sundry Payment' records per Contract Type.
Go to the Payments Setup page for full details.

[edit]

Set-up

Two new table fields, payment_flag and supplier_id are added to the [gl_posting_classes]

payment_flag: Controls visibility of payment types. It can be set with values as:

 *'yes' = active and selectable.

 *'no' = hidden from new entries but selectable for existing records.

supplier_id: Sets default supplier for each payment type. Used as fallback for non-sundry types if primary supplier is missing.

Following [table]fields are affected and updated as below:

  1. [gl_posting_class_maps]category_type
  2. [fm_contract_payment_structures]payment_type
  3. [pl_payables]payment_type
  4. [gl_batches]invoice_type
  5. [gl_transactions]invoice_type

Following workflows are to be impacted with the 'sundry payment' type change:

  1. Main Page → Fleet / Controls / Accounts → Posting Classes / Details
  2. Main Page → Fleet / Controls / Accounts / Contract Types → Payments
  3. Main Page → Fleet / Administration / Contracts → Payments → Sundry Payments
  4. Main Page → Workflow → Fleet Management → Payments → Print a Payments (ABA) file
  5. Main Page → System / Scheduler → Job → Method → Payments → Periodic
