Overview

The Ampol Star Fleet interface (CALNP) imports non-fuel transactions (e.g. servicing, tyres etc).
The Interface Code reflects the previous name of Ampol, which was Caltex.


The interface files are prefixed with CALNP and have a .txt extension. The file includes fuel transactions restricted to Ampol outlets.
The layout is similar to CALNS but includes the PUMP PR and LOCN fields.

The interface files CALNP and CALNS can include all transactions. However, the non-fuel transactions need to be excluded and imported using the CALSF interface in order to obtain enough detail of the work carried out.
These transactions are filtered out by excluding the specific product codes 'STARFLEET', 'S/FLEET' and 'S/FLEET P', 'FLEETCRDP' and 'FLEETCARD' when the data files are imported.
This means you need to run two imports to bring in all Ampol transactions into Catch-e: one for fuel and one for maintenance.
The product codes 'FLEETCRDP' and 'FLEETCARD' were introduced when Ampol changed it's name from Caltex.

Ampol often supplies this file as an .xls format. Ask Ampol to provide you with a .txt file instead.


If you open the supplied file in excel, the last digit of the card number may be converted to a zero due to the field length. This renders the field unreadable.
To avoid this, view the file from Notepad or Wordpad. Alternatively, save as a txt file and use the excel import wizard to bring the data into excel correctly.
