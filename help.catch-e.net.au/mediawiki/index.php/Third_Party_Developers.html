Overview

Third party developers can be engaged for the purpose of creating features and workflows outside of the Catch-e application using the eligible Catch-e APIs.

E.g. A Novated Lease Calculator on your website.

[edit]

System Access for Individuals

Third party developers require access to the System to enable use of the documentation in the Catch-e Help wiki.
Create user logins for your supplier staff using the 'demo' role and check on the "Read Only Access" check box.
This will give them access to this Help wiki content but they cannot update data.
Key fields from a sample record is shown below:

User
Role 'demo'
User Login 'Jim Beam'
Organisation 'ABC Web Development
 Read Only Access

[edit]

API Access for your Developers and Suppliers

Create a separate role and user login for each API function you are using.

[edit]
Roles

Review the API permissions on the Roles / API tab.
To optimise security, only enable permissions for the individual APIs that you need.

[edit]

Separate Roles by Supplier

Create separates role for each developer group you want to give access to the system to (in-house or third parties)
Catch-e staff will create new roles for your different API functions and developers as requested.
These roles have access to run APIs, but cannot login to the System ('Access denied!') or the Portal

  1. Enable only the required APIs to be enabled for each group
  2. Keeps logins separate for each of your suppliers
[edit]

Logins

Create a separate login for each function to This makes it easier to identify issues by user.
You can decommission an API process or function by suspending the related user login without affecting other system areas.
