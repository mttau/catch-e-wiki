Overview

An uplift fee can be added to imported card transactions for any nominated Fuel Supplier

Access to this feature can be controlled through the use of Role Restrictions.

With the introduction of the Card Fee at client level, when entered, any fee entered against the product code in the Mappings tab will be replaced. For example, if the nominated fee for the product in the interface map = $2.00 and an entry of $1.50 exists in Clients → Card fees, the figure carried forward to add to the transaction is $1.50. Where an entry doesn't exist in Clients → Card fees, the current process using the interface map fee will remain.

A positive or negative figure is allowed, with a minimum fee of -$99.99 and a maximum of $99.99. Where negative, the entered amount will be deducted from the product value in the supplier’s fuel file. Where positive, the amount will be added on.

When a supplier and product code have been selected to have a fee allocated, the product can only be selected once. Where a user nominates a supplier and attempts to select a pre-selected product code, an alert will be displayed advising the user that the proposed selection already exists. They will not be able to proceed.

  • New entries are started by clicking on the "New" button. Select a Fuel Supplier from the interface dropdown, a product from the product dropdown and enter the fee value.
  • Click on "Create" to save your entry.
  • When the screen is in ‘Edit’ mode to an existing product, only the Fee field will be editable. If a zero value is entered, then a warning will appear.
  • When ‘Delete’ has been selected, a warning will show, giving the user an opt out option.

Set-up Notes

If you want to use this feature, firstly, decide which user roles should be able to access the tab. Then contact Catch-e support.

Catch-e will update your role restrictions to make the Card Fees tab visible in the Client tab for the selected roles. This is done by updating the bit array.
