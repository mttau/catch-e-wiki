Whitelist an IP address

Jump to: navigation, search

Main Page → Workflow → System Administration → System Configuration → Whitelist an IP address

Qualifiers

  1. You are trying to log in to Catch-e, use the Driver Portal or run APIs from a country other than AUS, NZ, UK or USA.
  2. You have a static IP address, or a range of static IP addresses you will use for system access.
  3. Only static IP addresses can be whitelisted.
  4. Allow about 2 weeks lead time to whitelist an IP address, as this must go through the standard development and deployment processes.


Request Change

  1. Send Catch-e Support an email that notes the following
  2. The IP address or addresses you will be using
  3. The access method you will be using:
    1. System logins by users
    2. APIs
    3. Both System logins and APIs
  4. Catch-e Support will raise a task for this and confirm this back to you.

UAT Updates

  1. Catch-e Support will invite you to UAT the changes when they are available
  2. Email back a confirmation that the IP address is working

Deployment

  1. The changes will be deployed to the system during the next scheduled deployment
