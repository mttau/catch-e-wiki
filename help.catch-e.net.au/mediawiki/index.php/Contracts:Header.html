Field Type Description [table]field

ClientTextDisplays the name of the Client that the contract belongs to. This field hyperlinks to Clients:Contact*. [fm_clients]name
DriverTextDisplays the name of the Driver that the contract belongs to, showing Last Name, First Name. If you hover your cursor over the Driver's Name extra information appears in the bottom left of the screen. The fields come from the Drivers/Contact screen and display (if entered): "Work Phone", "Mobile" and "External Code". This field hyperlinks to Drivers:Contact*.[fm_drivers]surname, [fm_drivers]given_name
Type Text-Num-Num Displays the Contract Type, Term and Contract Kms of the selected contract. This field hyperlinks to Contract Types*. [gl_posting_maps]posting_map_code,[fm_contracts]term, [fm_contracts]contract_kms

StartDate, NumDisplays the Start Date of the selected contract and the Start Kms.[fm_contracts]contract_start
EndDate, NumDisplays the scheduled End Date of the contract and the Contract Kms.[fm_contracts]contract_end
LastDateDisplays the date and odometer reading of the last valid recorded transaction
Column Four
Tyres? Num/Num Tyre Allowance / Tyres Used.
The number of tyres allowed for in the contract budgets is shown here along with the number of tyres used so far.
Recharged tyres are not included in the count.
Where the number of tyres used exceeds the allocated amount, this field is highlighted in green.

Tyre Allowance less Tyres Used is used as the Autoguru daily vehicle file 'TyreBalance' field amount.

[fm_contracts]tyre_allowance
ProjectedNumDisplays the projected kms of the vehicle based on existing information. See Estimated Odometer Calculation for details of this calculation.
DivisionTextDisplays the division that the vehicle is linked to. This is created or changed using Contracts / Events.
IconView Edit History is a pop-up screen that lists all the field changes in the selected record.
IconPerformance Indicators is a pop-up screen that displays a summary of budgets and actual costs that are useful for maintenance control.
IconFunds Check is a pop-up screen that displays a summary of budgets, actuals, receipts and available funds. This is useful for determining if reimbursements can be paid or not.
Message RowTextAt the bottom of the Header area a red message row may appear. This is populated with information based on these rules:
  1. If a "Terminated" date has been entered in the Approvals / Contract or the Contracts / Contract tab, display the terminated date. E.g. “Terminated 28/06/2018”.
  2. If "Terminated" is blank and a "Suspend Date" has been entered in the Approvals / Contract or the Contracts / Contract tab,, display the suspend date. E.g. “Suspended 16/06/2018”.
  3. If “Terminated” and “Suspend Date” fields are blank and a "Replacement Order ID" exists in the Contract / Terminate tab, display the replacement contract. E.g. “Replacement Order ######”.
  4. If “Terminated”, “Suspend Date” and "Replacement Order ID" fields are blank suspended, display the first 40 characters of the "Notes:" field. If this is blank, no message will display.

Buttons

Button Description
Click to create a new contract. You will be navigated to the Contracts / Contract tab in edit mode.
Click to copy an existing contract. Use this to create a new contract that replicates the detail of the existing contract.

The following actions will occur:

  1. A new contract record is created
  2. "Reg Number" is set to ‘Order’
  3. "Driver" is removed.
  4. "Cost Centre" is removed.
  5. "Division" is removed. No contract event is created.
  6. Billing budgets are copied to the new contract and set to 'Pending' status. Note: there are no linked invoices.
  7. Maintenance records are not copied to the new contract
  8. Contracts / Other records are not copied to the new contract
Click to Import a contract may be updated by importing a compatible .csv file. Contact Catch-e to obtain a detailed file specification. When importing a contract the system will check that the contract_id in the import file matches to an existing system contract id. The system will display a screen of proposed changes which the user can accept or cancel. Registration changes are included in the import function and create the appropriate 'Registration Change' contract event when importing.


  • Hyperlinks will only work if your Role has access to the linked module.
