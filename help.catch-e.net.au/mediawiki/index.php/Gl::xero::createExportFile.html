Gl::xero::createExportFile

Main Page → System / Scheduler → Job → Method → createExportFile

[edit]

Overview

This job performs the same function as navigating to Accounts / Export → Xero, selecting a batch and clicking on Export to get the file.

This job can do one or both of the following;

  1. Send an email with an attached Xero export batch file
  2. Store a Xero export batch file in a nominated DMZ folder

The file is copied from the most recently created batch file that is in 'pending' status.
If the [gb_exports]data field is blank, the file will be re-created in this job.
The job also then updates the batch status to 'exported_ok', (so you know it has been sent).

Make this a child job of gl / xero / createExportBatch which creates the 'pending' export batch this job needs.

  Catch-e recommends using this job to store the file in a DMZ folder only, or directly downloading files as required from Accounts / Export → Xero. Sending files as email attachments is not secure.


[edit]

Parameters

The following parameters are displayed when this method has been selected.

Parameter Notes
Ledger Type Select the ledger that you want to create an export batch for: Purchases, Sales or Journals.
Company This parameter is only displayed if you are accounting for more than one company. This is controlled by [gb_controls]multi_company_export_flag.
DMZ Subfolder If you want to store a copy of the file in a DMZ folder, enter the folder path where the file is to be stored. E.g. 'xero/in'
