Overview

Invoices can be selectively "re-printed".
Note that invoices are re-generated on this screen, so if you have changed system branding or details on a contract the invoice may not reflect the actual original invoice. If the actual original invoice is required you can obtain that from the mail queue.

A range of filters are available to search for and choose invoices for re-printing. The system defaults to 90 days of recent invoices. The output is PDF format using the template the invoice was originally created in.
Once identified, an invoice can be viewed in PDF by clicking on the PDF symbol at the left of screen.

Invoices can also be printed to .csv for use in spreadsheets. Use the filters in the top section of the screen to find the range of invoice(s) you wish to download (e.g. by client code and date range) and click on the Print button at bottom right of screen to create the .csv file of transactions. The output will be in the .csv version of the 'combined' invoice which gives contract description as well as transactions descriptions.
Invoice type 'Sale Car' are excluded from this screen and need to be reprinted from the Sale Management screens.

The Billing / Re-print function is also replicated as an API: billingReprintWebService.

Action Buttons

Button Description
Print a CSV output of the selected records. This contains details for each invoice row and includes extra useful fields such as Contract ID, Registration No, Employee ID, Driver Name and FBT Value.
Click to unpost an invoice or batch.

The Un-post button can be enabled by setting the expected filters and running the search.
Populate: Either "Invoice No" or "Batch No" (but not both).
Leave blank: "Channel Code", "Group Code", "Client Code", "Reg No", "Created"

Emails: If a pending emails exist for invoices or batches that you unpost, then the emails are deleted and will not send.
If a pending email has a range of invoices attached to it, then email will still send. The unposted invoices will be generate without data. I.e. they will be blank.
If sent emails exist for invoices or batches that you unpost, you will need to contact the client to communicate about the reasons for unposting the invoices.

Access: 'admin' role users can unpost exported invoices. Other roles can be given permission also if this suits your business.
Check on the Roles / Details "Unpost Exported Invoices" check box for each role you want to give this authority to.

Exports: If exported invoices or batches are unposted, the linked export sales batch is updated to remove the unposted invoice(s).
An updated MYOB sales export batch file can be created.

Employee Transactions If the unposted invoice was a Pay Run invoice, the linked employee transactions are also unposted. They are no longer visible on the Employees / Transactions tab.

System Locks

System locks are invoked when the billing process is occurring to prevent data corruptions.

If possible, run your billing process after hours or use the scheduler billing jobs to do this.

The locks listed below will be in place:

1. Any updates in Contracts / Billing and Contract / Lease tabs will not be saved and the user will be warned.
2. Whole table import on tables* used in the billing process will not be saved and the user will be warned.

Tables*: fm_billing, fm_contract_budgets, fm_maintenance, bk_booking_billing_items and sl_receipt_item_lines.

Some Role "Session Timeout" values are deliberately set to a short timeframe and may be shorter that some POST process run times. Therefore, during the POST process, your session timeout value will be temporarily reset to the greater of your Role "Session Timeout" value or 8 hours. Once the POST process is complete your session timeout value will revert back to your Role "Session Timeout" value.

Users - do not shut down your browser screen when you are in the middle of a process or have a records open in Edit mode. This causes orphaned locks.
[edit]

System Locks

Records are locked by the system if a user is editing the record or running a process that updates the record (E.g. Billings, Payments, Receipts)
If you try to access a record that is locked, you will alerted and prevented from using the locked record.
The alert shows the login of the user who has locked the record.

Alert Comments
Edit
Table {table_name} already locked by user: {user_login} The table cannot be edited while another user is editing it. Wait and try again later. Visit System Locks for more details.


Suggested steps:

  1. Wait a short time for the user to finish their editing process.
  2. Contact the named user and ask them to finish the work they are doing and/or exit the record they have open.
  3. If the user is editing or processing a different record and you still get the lock message, they have probably orphaned that lock.


[edit]

Orphaned Locks

Records can be orphaned if they are locked, but the user is no longer editing that record.

Orphaned locks happen for two main reasons:

  1. A record was in edit mode and the user closed down their browser window instead of logging out of the system. Please DO NOT do this.
  2. The user's internet connection dropped out. Even a lost connection which lasts only an instant may mean the system is unable to find related locks related to the session concerned.


Users - do not shut down your browser screen when you are in the middle of a process or have a records open in edit mode. This causes orphaned locks.


To fix orphaned locks:

  1. The user should logout of the system correctly and then log back in again. This should remove the orphaned lock/s.
  2. Logout by clicking on the Logout icon. This creates a logout record and closes out the users current session.
  3. If the user is running two session, they will need to repeat this process for both of their sessions.
  4. Orphaned locks on contracts and quotes are usually tidied up by the system, by a process which runs every 5 minutes.
  5. If you have tried to remove the locks by logging back in and you have waited 10 minutes and the lock has not be removed, please escalate to your System Administrator.
  6. You System Administrator may email Catch-e Support if required to check and remove the locks for you.
