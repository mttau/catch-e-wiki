The banking details used to create the Receipts (ABA) file for direct debits are stored in the [gb_controls] table.
You can update these details yourselves.
Go to Setup / Reference Data → gb_controls to update the details shown below.

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gb bank_accounts Receivables
Financial Institution - the three letter code for your bank
Company Name - the name to use in your ABA file.
Bsb - the BSB to use in your file.
Account Number - the account number to use in your file.
gb becs_details Company Name Short - the short name to use in your ABA file
Identification Number - the identification number to use in your ABA file.
Aba File Description - the description to use as your referent in your ABA file.
User Credits Only - set to no to add the self balancing row to the ABA file.
gb aba_identification_number_debit Value - enter the identification number which is assigned by APCA and User Financial Institutions for transacting debit ABA files.
gb aba_user_debits_only_flag Value - is used to determine if your organisation is a full ABA user (allowing debits and credits) or a debit user only.
If your organisation is debit only, any items in the batch that result in a credit will be excluded from the ABA file. 		yes
