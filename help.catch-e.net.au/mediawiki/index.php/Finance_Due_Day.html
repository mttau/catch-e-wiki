[edit]

Overview

When you enter or edit the Contract / Finance “Finance Start” date, the financier's Suppliers / Account “Pay Due” value is looked up. Based on what is stored in "Pay Due", "Finance Due Day" is updated.

You can enter "Pay Due" with one of three formats to support the update logic you want;

  1. a number, e.g. 19
  2. multiple numbers separated by a comma, e,g. ‘5,15,30’
  3. ‘DOM’ which stands for 'day of month'

If Suppliers / Account “Pay Due” is not entered in a format that supports the three choices above, then "Finance Due Day" needs to be entered manually.

Single Day

  1. If Suppliers / Account “Pay Due” contains a single day, e.g. 19, "Finance Due Day" becomes ‘19th’.
  2. If Suppliers / Account “Pay Due” contains a single day, e.g. 28, "Finance Due Day" becomes ‘28th’.

Multiple Days
If Suppliers / Account “Pay Due” contains multiple days, e.g. ‘5,15,30’ additional logic is applied. The day in the list that is greater than the “Finance Start” DAY is used. E.g.

  1. If Contract / Finance “Finance Start” = '19/11/2019', "Finance Due Day" becomes ‘30th’.
  2. If Contract / Finance “Finance Start” = '05/11/2019', "Finance Due Day" becomes '15th'.
  3. If Contract / Finance “Finance Start” = '01/11/2019', "Finance Due Day" becomes '5th'.

Day of Month
If Suppliers / Account “Pay Due” contains ‘DOM’ (day of month), additional logic is applied. The “Finance Start” DAY is used. E.g.

  1. If Contract / Finance “Finance Start” = '07/11/2019', "Finance Due Day" becomes '7th'.
  2. If Contract / Finance “Finance Start” = '23/11/2019', "Finance Due Day" becomes '23rd'.


If populated, "Finance Due Day" becomes the default value for the Contracts / Payments "Due Day" field when new finance payments are being set-up.
The Contracts / Payments "Due Day" field is part of the default Contracts / Payments "1st Payment Due" date field logic.
