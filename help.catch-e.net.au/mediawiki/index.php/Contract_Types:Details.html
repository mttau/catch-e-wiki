Overview

This screen contains the description of the selected contract type and default information, such as whether inertia billing is allowed, whether GST applies to residual values and an Approval Note which appears when a new approval is raised.

The details here govern the behaviour of quotes, contracts and all of their subsequent transactions.

GST on Finance PaymentList 3 values are available in this field.

1.'Standard' - This is the default setting and calculates the Monthly GST as a % (Global Control 'gst_rate') of the rental amount.
2.'GST on Interest' - If selected GST is calculated on quotes with the contract type in question, calculated on the interest on the amount financed. This applies for quotations post 1/7/2012. This field is used to comply with the "GST – Hire Purchase Reforms" implemented by the Australian Government as at 01/07/2012. This setting is for Australian Commercial Hire Purchase Contracts.
3.'Straight Line' - The monthly Finance Budget GST (Quotes/Lease Finance budget GST) field calculation is based on the purchase price GST less the RV GST straight lined over the term of the deal. This setting is for New Zealand Finance Lease Contracts.
See attached spreadsheet for detailed calculation.
NOTE: When 'GST on Finance Payment' is set to "Straight Line", the Global Control 'total_lease_gst_calc_flag' must be set to "no" as the calculations are incompatible.

Quote Residual ReserveNumericEnter an amount here to create a default value for the Quotes / Lease Residual Reserve budget. This would be used to buffer shortfall on the Residual value of a vehicle and would be used on operating Contract Types.

To set-up this feature;

  1. Go to Posting Classes / Details and check that "Class Code" 'RRES' exists. Event check-boxes should be un-checked
  2. Go to Contract Types / Accruals and add "Class" 'RRES' to the records here.
  3. Enter this default value.

Quote Financier Lookup Enter a financier here to create a default entry.

When creating new quotes, Quotes / Finance "Financier" will be populated in the following preference order

  1. If Clients / Quote Defaults "Financier" is populated, use this financier.
  2. If Contract Types / Details "Quote Financier" is populated, use this financier.
  3. Lookup or directly enter a financier.

Visit the Financier page for configuration details.

Quote Payload MaximumNumeric Populate this field if to stop users from selecting variants with a payload that is greater or equal to this value in a quote.
If this field is blank, no restrictions will occur.
The payload of a variant can be seen in the Quotes / Vehicle "Payload" field (in the Specifications section) and can be edited in Setup / Reference Data.
The restricted variants are still shown in the Quotes / Vehicle "Variant" list and are shaded in pink.
'Admin' users are warned but can save quotes, but other users are prevented from selecting the variant.
The ability to select restricted variances can be given to other roles by updating their Role Restrictions.

This field is read-only for all users except 'admin'. Access can be changed in Role Restrictions.

This payload maximum is also applied in the getQuickQuoteWebService API.
Variants with a payload above this value will not calculate and return values in this API.

[gl_posting_maps]quote_payload_maximum

If selected quotes prepared will 'capitalise' any budget amounts (all running costs and fees but not Finance payments). A Posting Class 'BF' is required. For further information refer to Budgets Financed page.
Payout TabCheck BoxThe Contracts/Payout screen is available for these contracts if this flag is selected.

Funds Check on ClaimsCheck BoxIf selected, only an 'admin' role user can post a claim in the Contracts/Claim screen if doing so would make the Current Funds Balance (incl GST) fall below zero. If Funds Check is not selected, no alerts or restrictions will occur.
Auto Post on Claims Check Box This feature allows users to place claims that do not pass the Funds Check validation into a queue that is checked nightly. Once the claim passes the Funds Check validation, it is then posted and will be paid in the next run of the ABA file automatically. It is only available if Contract Types/Details "Funds Check on Claims' is checked.

If in use, a claim status will change to 'Active' and sub status to 'Pending Funds' if the Funds Check validation fails during the Check validation, unless over-ridden by the user.

Funds Check on Reg/CTPCheck BoxIf selected, shows funds balance on the alert for contracts which are due to be suspended or ending it in next six weeks on Fleet/Maintenance/Reg or CTP Renewals screen.
