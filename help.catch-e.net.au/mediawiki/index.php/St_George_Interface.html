[edit]

Overview

This interface allows submission of a finance application to St George Sovereign system. A successful application submitted from Catch-E has a status of 'Draft' in Sovereign. Using the Application Reference returned on success by Sovereign, an authorised Sovereign user can login to Sovereign system to view and further edit this application.

Currently 2 types of finance application can be made to St George. Each application is identified by a St George Product Type. The Product Type code used for an application is determined by Catch-E Contract Type to St George Product Type mapping.

[edit]

Product Type currently supported

1) FLE = Finance Lease. This product type is typically used for both an Individual Driver and Company finance application.
2) OLE = Operating Lease. This product type is typically used for a Company finance application.

The option to use the St George interface is given when you set the Financier code to St George (recognised by the appropriate supplier_id advised to Catch-e).


[edit]

Submitting a new Finance Application

An "Apply" button appears on the Quote/Finance and/or Contracts/Finance screens when certain criteria are met:

a) An interface_type of 'st_george' needs to be set on Fleet / Administration / Suppliers / Account
b) The Quote Application Status in the Finance Application section must not be "successful"
c) Login credentials for the Sovereign service must have been set up for the relevant user_id in gb_user_logins, typically matched to the user logged in (further details may be given in your Wiki)
d) The Contract Type must be mapped to a St George Product Type in qt_financier_posting_map_defaults with applicable values stored in product_type_code, xml_template & finance_apply_from_type fields. See St_George_Setup.

When clicked the system sends the application details and retrieves a reply from Sovereign with a message (i.e. successful or unsuccessful) and an Application Reference which is then stored on the Quote/Finance or Contract/Finance screen. This process takes only a few seconds. An application may be done only once per quote. A copied quote will retain an application. You can login to Sovereign to view and edit the application if required.

If an application fails the user will be given an error message and the Application Reference may be set to "Not Assigned".

To use the new interface, Sovereign logins, Originators and Client Codes need to be supplied to Catch-e for storing in the system and the web service activated. The apply button will not appear unless this is done.

[edit]

Submitting a Finance Application for an existing Sovereign Client

An applicant (Individual Driver or Company) who has previously made a successful finance application to St George will have a 10 digits reference number called 'Sovereign Client ID'. This reference number can be stored as 'Applicant Reference' in Catch-E on Driver/Finances for individual driver or Client/Financiers for Company.
If you have made a successful finance application from Catch-E, you can get the 'Applicant Reference' from St George by providing the 'Application Reference' you received on successful application. If an 'Applicant Reference' for Driver or Company is present in Catch-E any subsequent finance application made for this driver or company will use this 'Applicant Reference'. When an application is submitted using an Applicant Reference ONLY following information are sent to St George.
1) Vehicle Details
2) Financial Details about the Vehicle

And following information are NOT sent,
1) Applicant's identity details such as Contact Details and Driver License No or ABN/ACN (for Companies)
2) Bank Account Details
3) Employment Details (for individual driver only)
4) Financial Details such as Incomes, Expenses, Assets and Liabilities (for individual driver only)

Sovereign looks up these details using the Applicant Reference supplied. If these details in Sovereign are out of date, DO NOT POPULATE Applicant Reference in Catch-E to force a new application.
