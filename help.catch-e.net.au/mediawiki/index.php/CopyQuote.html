Synopsis

This API replicates the system function of copying a quote using the Copy button.
Visit Quotes → Copy to see the business rules in detail.

[edit]

Permissions

To run this API, the nominated 'web-services' role needs to be given permission.
If you are not actively using the API, leave the permission off for better security.
Go to Roles / Apis and check on Quotes:Copy.

This API is not configured for external use but can be made available on request.
Contact Catch-e Support to request this update.


[edit]

Authentication

Authenticate with the API before running this API.

[edit]

HTTP Method

Use the HTTP Method 'POST' for consuming this web service.

[edit]

URL Examples

https://api.catch-e.com/qt/quotes/153103/copy
[edit]

Parameters - Path

Key Format Notes Default Value Mandatory
quote_id string Quote to copy. Yes
[edit]

Headers

Key Format Notes Default Value Mandatory
Audit-User-Id string Pass a user_id you want to use in audit records. To use this key, the authenticated user's role must have the permission Audit:UserIdOverride. No
Time-Zone string Used to set the [qt_quotes]quote_accepted date. No
[edit]

Body - form-data

Key Format Notes Default Value Mandatory
quote_status_id string If this key is not passed, the quote status of copied quotes will be populated from [gb_controls]qt_quotes_status_flag_default.
Use this input field to pass a different quote status. Only records where [qt_quote_statuses]status_flag is 'active' can be passed. 		No
override_warnings_flagstringOverride warning validation. If not specified, defaults to 'no' No
