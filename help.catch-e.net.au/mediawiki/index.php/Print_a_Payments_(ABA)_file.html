Print a Payments (ABA) file

From Help wiki

Jump to: navigation, search

Main Page → Workflow → Fleet Management → Payments → Print a Payments (ABA) file

Qualifiers

  1. You want to create an ABA file for a supplier
  2. You have posted the batch you want to create the file for
  3. The posted batch Payment Type is Comprehensive Insurance, Finance, Management Fee or Roadside Assistance

Process

  1. Go to Payments / Periodic
  2. Status - 'Posted'
  3. Payment No - Enter or lookup the payment no
  4. Bank Account - if you have multiple bank accounts, select the one that was used to create this payment.
  5. Click on
  6. Check the seletion is correct
  7. Output File - 'Text(ABA)'
  8. Click on
  9. Click on
  10. Save the generated ABA file
  11. Upload to banking software for payment


Note: 'Text(ABA)' is not available if the posted batch is for Payment Type 'Sundry Payment' and/or if the posted batch has an allocated UUID for the field[fm_contract_payment_structures]payee_payment_method_id. Warning alert is seen as follows - 

 "Cannot export as Text(ABA)!
 Either a) no bank account selected, or b) wrong payment type(s) detected. Refer to online help for further details."

Note: 'Text(ABA)' is available but won't be able to print if the posted batch contains 'Sundry Payment' payment type. Warning alert is seen as follows - 

 Print failed, selection contains 'Sundry Payment' payment!


Next Steps

  1. Export SL and GL files to your financial system using the Accounts/Export/General Ledger option
