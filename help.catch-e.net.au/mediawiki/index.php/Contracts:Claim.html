Overview

Raise expense records here and attach supporting documents. These records are known interchangeably as approvals and claims.
The Claim and Approval tabs have been designed for different purposes. The sections below summarise their features.

[edit]
Claim

Use the Contracts / Claim tab when

  1. The expense is for a driver reimbursement or a packaging disbursement (the Payment and Banking details are shown)
  2. The expenses are simple and you prefer to enter a GST inclusive amount
  3. You want to use the Funds Check function to check if claims should be posted and reimbursed.
  4. You want to use the autoPostClaims process to manage and post claims where they have previously failed the Funds Check.
[edit]
Approval

Use the Approvals / Approvals tab when

  1. The expense is for a vehicle contract supplier (the Supplier's Payment and Banking details are not shown)
  2. You want to use VMRS Codes and/or Job Numbers
  3. The approval header has a Maintenance and Tyre pop-up to show usage information
  4. Each cost line allows for discount amounts and is entered initially as GST exclusive amounts

Claim Status

Status Sub Status Description
Requested Requested claims have been created by drivers and employees in the Driver Portal.
The maintenance rows for claims in this status are set as pending.
Active Created but not yet posted. If you review and save a requested claim, it will save into 'Active' status.
The maintenance rows for claims in this status are set as pending.
Active Pending Funds If you are using the Funds Check feature and you Check a claim, it can automatically update to a sub status of 'Pending Funds' if there are not enough funds available to reimburse the claim.
This can be used in conjunction with the autoPost feature, or you can periodically review claims with a 'Pending Funds' sub status to see if they can be posted and paid.

The maintenance rows for claims in this status are set as pending.

Active Posting Failed Usually, the autoPost job posts claims once funds are available, or leaves them in 'Pending Funds' if they are still short on funds.
But sometimes a claim will be updated to 'Posting Failed' if something has changed and the autoPost could not post the claim.
This sub status happens rarely.
The maintenance rows for claims in this status are set as pending.
Posted The claim has been finalised. It can now be exported to your accounting system and banking system for payment.
Cancelled Created but cancelled without saving. Cancelled claims are not visible on the Contracts / Maintenance tab.
Deleted Claims can be deleted by clicking on Delete. This is only possible if the claim has not been posted.
Deleted claims are not visible on the Contracts / Maintenance tab.

To see these claims, go to the Approvals / Search tab and filtering on this status.
Click on the hyperlink to get to the claim. The maintenance row is shown in grey.

History Only Active claims can be set to history only by clicking on History.
These claims will be shown in Driver Reports but will no be added to accounting exports.
This status is available for adding legacy records, but is not recommended for general use.
Maintenance rows for this status are shown with a grey background.
