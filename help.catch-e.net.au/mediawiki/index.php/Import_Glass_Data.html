Overview

Use the Import Data / Glass Data feunction to create new variants and update existing variants where required.

[edit]

Scheduler

Glass data is usually processed by using scheduled jobs to import data from their SFTP site.
Go to the qt / glass / importVehicleData page for the set-up details of this job. The job is run 'Monthly' as standard.

By arrangement with Glasses, weekly files can be supplied. If done, a second job is required to run the 'Weekly' import process.

[edit]

Import File Access

To access import files directly, set-up software such as FileZilla and use the login provided by Glasses to access their hosted SFTP folder.

[edit]

Quotes / Vehicle "List Price" data treatment

Glass provide a 'Recommended Retail Price' which includes LCT, when it is applicable at this point to show the RRP as being inclusive of taxes like LCT and GST.

When imported, the following steps are performed

  1. The RRP is stored as [qt_variants]amount_gross (this is not shown in system screens).
  2. List Price Exc GST: [qt_variants]amount_net is derived by removing a calculated LCT amount from [qt_variants]amount_gross.
  3. List Price GST: [qt_variants]amount_gst is calculated from the [qt_variants]amount_net.
  4. List Price Total: [qt_variants]amount_net + [qt_variants]amount_gst.

Thus explaining why, sometimes, the database fields amount_net, amount_gst and amount_gross do not match.

Useful Queries

from gb_audit_import as ai
inner join gb_audit_fields as af on af.audit_field_id = ai.audit_field_id
ORDER BY ai.timestamp desc
limit 10
;
