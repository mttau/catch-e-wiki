Overview

The password auto recovery feature can be enabled or hidden on the Portal Login screen.
When enabled - the user sees hyperlink text for Forgot Username? and Forgot Password?
Forgot Username? returns a message.
Forgot Password? navigates the user to the Forgot Password? page.

  1. User enters their "Username" and "Email Address"
  2. An email is sent with a one-time-use URL link.
  3. The user enters a new password on the Reset password page.


Note: This feature is similar to the Reset Password feature that internal users can use. The two processes share some set-up data.

[edit]

Set-up Notes

Follow the following set-up steps to active the password reset function.
Note that some controls here are shared with the Reset Password feature that internal users can use.

[edit]
User, Role and Expiry Details
  1. The system user called '_password_reset_' is linked to the emails that are sent in this process. Check and update the record "Name" and "Email" details are suitable.
  2. This user is linked to the role 'password_reset'. Check and update the "Session Timeout" of the role 'password_reset' as this role determines how long a password recovery email url will work for. The default setting is 2 days.

[edit]
Email Message

Run the set-up changes in your Staging environment and review the email message. If you want to change the text of the message, email the changes to Catch-e Support.
This message is also used by the Reset Password feature, so check that the wording is suitable for both your staff and for driver portal users if using both features.

An email message needs to be configured for the URL email that is sent. Visit Email Setup for standard configuration information for emails;

[gb_templates]name this must be populated as pdf_password_reset
[gb_templates]mail_sender_name can be left blank, as the reset email is being sent by the user '_password_reset_', it will use that "Name". Like other emails, a default name can be entered that will override '_password_reset_' "Name".
[gb_templates]mail_sender_address can be left blank, as the reset email is being sent by the user '_password_reset_', it will use that "Email". Like other emails, a default can be entered that will override '_password_reset_' "Email".
[gb_templates]mail_subject enter a suitable subject
[gb_templates]mail_message enter a suitable message, but it needs to contain the placeholder "#password_reset_url#" for the reset URL link to appear. You can also add the URL's timeout timestamp "#signon_timeout#" if that suits.

Event Logs

A log of password reset requests are stored in the gb_eventlog table.

Additional information can be found in the gb_user_access_log table, where all reset attempts are recorded and defined as 'reset'.

Useful Queries
SELECT * FROM gb_controls WHERE name LIKE 'password_%';
SELECT full_name,email FROM gb_users WHERE user_login = '_password_reset_';
SELECT mail_sender_name,mail_sender_address FROM gb_templates WHERE name = 'pdf_password_reset';
# template details can be blank, the email will use _password_reset_ Name and Email

SELECT * FROM gb_eventlog WHERE narrative LIKE 'RESET%' ORDER BY timestamp DESC LIMIT 10;
SELECT * FROM gb_user_access_log WHERE access_type IN ('reset','fiel','fail-hash') ORDER BY timestamp DESC LIMIT 10;
