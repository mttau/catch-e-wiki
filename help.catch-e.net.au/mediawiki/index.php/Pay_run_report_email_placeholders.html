[edit]

Overview

The pay run report email placeholders below can be used in pay run report email.
They use the # syntax to mark them as placeholders. E.g. #invoice_date#.

[edit]

Pay Run Report Email Placeholder Values

placeholder_value Description (Location of data in system screens) [table]field
client_name Pay Runs / Pay Run "Client Code" → Clients / Contact "Name" [fm_clients]name
pay_cycle_code Pay Runs / Pay Run "Pay Cycle". PROPER ( [gb_pay_cycles]pay_cycle_code )
invoice_date Pay Runs / Pay Run "Period Date". Presented in dd/mm/yyyy format. [fm_billing]invoice_date
pay_run_id The pay run that matches the selected "Client Code", Pay Cycle" and "Period Date" combination. [sp_pay_runs]pay_run_id
pay_run_total Pay Runs / Pay Run total for the selected "Client Code", Pay Cycle" and "Period Date" combination.

This does not round to 2dpi. E.g. if the total is 727.80, you will see '727.8'.

SUM ( [fm_billing]amount_net + amount_gst )
funds_due_date Will only display if the date has been populated into the [sp_pay_runs] table.
This date is added by the scheduler job SP - sp_pay_runs - add pay_run_dates.
The added date can be derived from the Clients / Contract Defaults "Billing Terms" of the client.

If [sp_pay_runs] is NULL, an odd date will be shown in emails. E.g. '30/11/-0001'

[sp_pay_runs]funds_due_date
payment_method Clients / Contract Defaults "Payment Method" [fm_clients]payment_method
invoice_no If you have created multiple invoices, only one invoice_no will be displayed.
This placeholder limits your flexibility to create multiple invoices to update pay runs in process.
For a fixed reference for your pay run, use pay_run_id instead. 		[fm_billing]invoice_no
posted_date Date the pay run invoice was created. Presented in dd/mm/yyyy format.
If you have created multiple invoices and posted on different days, only one posted_date will display. 		[posted_date]posted_date
