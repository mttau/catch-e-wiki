Overview

Use this profile to add an Extended Warranty to the lease quote.

If the quote's "State" or "Effective Date" are changed, Extended Warranty will re-calculate.
If the quote's "Type Code" (Contract Type) is changed, and the new "Type Code" (Contract Type) does not have an EW class accrual, Extended Warranty will be un-checked and the values removed from the quote.

If the quote's "Term" is changed, Extended Warranty is not changed.
The Extended Warranty "Months" is selected separately of the quote's "Term" and can be different. E.g. a driver may choose a cover that is longer than the lease term.

Extended Warranties can be set-up to work with a Swann Interface.

Extended warranties can be budgeted or financed. If financed, the premium is added as a brokerage item on the Quotes / Finance tab.
Calculations are made by Warranty Plan, Category, Months and Kilometres using the tables qt_warranty_plans, qt_warranty_categories and qt_warranty_plan_categories. The profile requires the Contract Type to have a budget class of "EW". [qt_warranty_plans]status_flag must be 'active' to appear in drop down list of Plans ('inactive' plans will not be able to be selected).

Extended Warranty (EW)

Field Type Description [table]field
Include? Check if you want to add an Extended Warranty budget to this quote.
Plan Drop List If "Include?" is checked, select the plan you want.
Category Drop List Select the category you want.
Months Drop List Select the number of months the warranty is to cover. This can be different to the quote's "Term".

Premium The premium stored for the selected warranty will populate here. It can be edited by the user.
Monthly Budget Number The "Monthly Budget" is calculated by dividing the "Premium" by the number of billing periods there will be.

Financed? If your system is configured to finance this product this check box will be checked by default and cannot be edited.
If the premium is being financed, the total will be stored and displayed in the Quotes / Finance Broker "Extended Warranty" field.

Set Up

Enable the Extended Warranty profile as follows.

[edit]
Global Controls
Module Name Description Value
qt extended_warranty_type Determines if the Extended Warranty "Premium" will be budgeted or financed. 'budgeted' or 'financed'


[edit]
Posting Classes

For the Set Up data to appear in the quoting process, 'Extended Warranty' must be created as a Posting Class. Go to menu Fleet / Controls / Accounts / Posting Classes / Details Tab.

To create a new posting class,

1. Select 'New'
2. Enter class code 'EW' - this must be 'EW' to ensure that there is consistency through out quotes and reporting.
3. Enter Name 'Extended Warranty'*
4. Check ON the box next to 'Event View Flag?' if you wish to create contract events
5. Select Quote Profile Table 'qt_quote_extended_warranty' from the list to enable the Profile to be created when quoting.

If a posting class already exists, make sure that the Quote Profile Table is set to ' qt_quote_extended_warranty'.

If you want to create an event for Extended Warranty, but not have it link up to Quotes, simply leave the posting class screen reference 'Quote Profile Table' blank.

  • You can nominate another name if you prefer. Refer to Posting Classes for more information and what impact this has on screen field names.
[edit]
Contract Types

Go to Fleet / Controls / Accounts / Contract Types

1. Enter or search for Contract Type Code
2. Go to Accruals Tab
3. Select Edit
4. From the drop list select 'EW'
5. Enter required set up details in the right of screen and save
  • Any new quotes will have the product available. Existing quotes, will need to be re-calculated to introduce the new posting class eg: re-setting the contract term will have this affect.
[edit]

Rename

  1. Quotes / Lease - Extended Warranty profile
    1. Navigate to Accounts / Posting Classes
    2. Class Code - enter or lookup 'EW'
    3. Click on Edit
    4. Name - enter the name you want

Useful Queries

select * from qt_warranty_plans;
select * from qt_warranty_plan_categories;
select * from qt_warranty_categories;

select wp.name, wc.category, wpc.*
from qt_warranty_plan_categories as wpc
inner join qt_warranty_plans as wp on wp.warranty_plan_id = wpc.warranty_plan_id
inner join qt_warranty_categories as wc on wc.warranty_category_id = wpc.warranty_category_id
where wpc.status_flag = 'active'
