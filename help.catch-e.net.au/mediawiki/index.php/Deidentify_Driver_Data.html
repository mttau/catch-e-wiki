Qualifiers

Deidentify personal data when

  1. there is no requirement to keep them from the business
  2. you want to delete personal data of driver / employee having no linked quotes, items, or contracts or/and are not edited in more than 2 years
  3. you want to improve system performance by reducing data volume



Overview

This process is designed to be run annually or on a regular time frame with assistance from Catch-e staff.
Catch-e will offer a policy recommendation on the retention time, but clients can decide their own timeframe and business rules.

When run, the updates made in a DeidentifyDriverData job are not reversable.
Test and check your updates carefully before running in your Live environment.


Set-up Steps

  1. Review the selection query used for this process. This is detailed in the Deidentify Driver Data Query page.
  2. If you want to change the query to suit your business, contact Catch-e Support to raise a task for your requested changes.
  3. Determine the timing of the update
  4. Test the process in your Staging and Support environments
  5. Train staff on the process
  6. Arrange for the job to be scheduled in Live

Update Steps

  1. Scheduler job runs using a query to target drivers for this update.
  2. For each field in the DeIdentify Driver Data Field List
    1. Field values are replaced with un-usable data, numeric fields are set to 0.
    2. Audit record written for each table updated
    3. Audit records are updated to replace before and after values with un-usable data, numeric fields are set to 0.
  3. A driver event records the time of the update.
  4. The driver record is now set as read only.
