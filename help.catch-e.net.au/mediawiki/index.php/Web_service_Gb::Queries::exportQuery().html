Overview

Execute a query known by query_id and return the resultset as XML. This web service executes a query stored in the table qt_queries, and can include queries with derived fields.

If you don't need to use any derived fields in your query, use the Print a report query API.
Contact Catch-e Support if you need assistance with the set-up.


[edit]

Input Fields

  Query string key
Format
Notes
Mandatory
1 query_id
integer
Unique identifier for a query
yes
2 output_format
string
Request a specific output format, options include 'xml' and 'text'. Defaults to 'xml'.
no
3 limit
integer
Add a LIMIT clause to the query
no


  The limit clause will only work with simple queries, e.g. it won't work with queries involving UNIONs. If in doubt, please contact Catch-e Support.


[edit]
Input URL Example
https://yourname.catch-e.net.au/services/gb/queries/exportQuery?query_id=100001

[edit]

Error Codes

Error Codes Comments
CATCH_E_ERROR_BAD_QUERY The SQL was badly formed and not executable
CATCH_E_ERROR_BAD_QUERY_TYPE The SQL was not a type that returns a resultset
CATCH_E_ERROR_UNKNOWN_QUERY The query_id is not known
CATCH_E_ERROR_TOO_MANY_REQUESTS Rest call requests exceeded the limit set in [gb_controls]rest_rate_limit.
403 Forbidden If you are using the exportQuery function in your web site, Catch-e will need to whitelist the IP addresses you are using so this function is allowed.
