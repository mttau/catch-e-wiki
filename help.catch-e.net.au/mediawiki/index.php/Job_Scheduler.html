Job Types

Scheduler Job Types

[edit]

Run Times

The Scheduler runs in the background all the time. It only processes one job at a time. The remaining jobs are queued up in chronological order which are processed when its earliest job has finish running. Any job that take 3 hours or more on single run will be killed and made inactive. The maximum frequency you can run a job is once every minute, the minimum is once a year. All other combinations of dates and times therein can be set.

[edit]

Methods

Various methods are available to create jobs that perform a range of functions, such as

  1. Generating reports and
    1. storing them in the Report Queue for uses to pick up
    2. emailing them to users, clients and drivers
  2. Creating event records and
    1. setting the due date and completion fields for a process
    2. sending an email (possibly with a report attached)
  3. Updating system data when certain conditions are met
    1. e.g. set quotes to 'expired' based on date.
  4. Transferring files via SFTP
  5. Run a chain of jobs to form a process by linking jobs to a parent. Jobs then run consecutively.

My job is inactive

If a job is inactive, and there is no audit record of this, check to see if the job is attached to a parent job.
If a a parent job is made inactive, the child jobs are also made inactive.

Handy hint: If you need to update a chain of jobs to make them active again, do the following

  1. Find the Parent Job on the Scheduler / Schedule tab
  2. Show all Children check this on
  3. Job ID - find and hyperlink to the last job in the chain
    1. Edit this job
    2. Parent Job ID - select and hyperlink to the parent
  4. Continue until you are back and the original job
  5. Recheck all jobs in the chain again to confirm


[edit]
My job is active but does not run

If a job is active, and is not a child job, the Run Times settings may be counteracting each other.

  1. If a job should run on a certain day of the week, e.g. every Wednesday
    1. Days: should be checked ON.
    2. Months: should be checked ON.
    3. Weekdays: should be un-checked and should contain selections. E.g. Wed
  2. If a job should run on a certain date/s in the month, e.g. 15th of each month
    1. Days: should be un-checked and should contain selection. E.g. 15ths
    2. Months: should be checked ON.
    3. Weekdays: should be checked ON.
  3. If a job should run on a certain date in a month or selected months. E.g. 1st July each year
    1. Days: should be un-checked and should contain selections. E.g. 1st
    2. Months: should be un-checked and should contain selections. E.g. July
    3. Weekdays: should be checked ON.


[edit]
My job has failed

Find and review the failed job on the Scheduler / Schedule tab. The "Last Message" often contains a message about the failure.
Visit the Message page for a list of these messages and explanatory comments that may help you diagnose the problem.

[edit]
My Attachment Formatting did not work

If your scheduler job is using 'Office(xlsx)' as the "Mail Attachment", you cannot use a customised format.
Go to the Scheduler / Job and change the output format to be 'Office(xls)' if you want to create and use a customised format.
