Overview

Raise expense records here and attach supporting documents. These records are known interchangeably as approvals and claims.
The Claim and Approval tabs have been designed for different purposes. The sections below summarise their features.

[edit]
Claim

Use the Contracts / Claim tab when

  1. The expense is for a driver reimbursement or a packaging disbursement (the Payment and Banking details are shown)
  2. The expenses are simple and you prefer to enter a GST inclusive amount
  3. You want to use the Funds Check function to check if claims should be posted and reimbursed.
  4. You want to use the autoPostClaims process to manage and post claims where they have previously failed the Funds Check.
[edit]
Approval

Use the Approvals / Approvals tab when

  1. The expense is for a vehicle contract supplier (the Supplier's Payment and Banking details are not shown)
  2. You want to use VMRS Codes and/or Job Numbers
  3. The approval header has a Maintenance and Tyre pop-up to show usage information
  4. Each cost line allows for discount amounts and is entered initially as GST exclusive amounts

Workflow

Internal Users

The recommended sequence for creating a claim is:

  1. Select the required contract id that the claim is to be allocated against
  2. Select the Claim tab
  3. Press if creating a new claim, if making changes to an existing claim, that has not been exported
  4. Enter invoice number, you may wish to use the receipt number
  5. Enter nominated driver BSB and account number details, if different to the default set up against the driver id (this will be saved back to the Driver's record).
  6. Select the "Stop Becs" checkbox if you wish to stop the reimbursement claim from proceeding (refer to Stop BECS for more information on this feature).
  7. Select the date and enter the odometer reading (normal odo validations apply)
  8. Select the Budget Account that the claim is to be expensed to
  9. Select the Classification (if only one exists, this will populate automatically)
  10. Enter the Qty and Amount details
  11. Select
  12. Select to confirm all required details are entered. If there are any incomplete items, a warning will appear, prompting a revision of the entered data. If items are complete, a prompt will appear to send a Reimbursement Notification
  13. Select to confirm posting. This will update the maintenance records and the data can now be exported to your accounting system.


Drivers

The recommended sequence for creating a claim is:

1. Select the required contract id that the claim is to be allocated against
2. Select the Claim tab
  • A Claim Instruction box will appear, which will instruct your Driver’s on how to proceed with the claim reimbursement.
3. Select “Continue” once the instructions have been read and understood
4. Select for a new claim.
5. Enter the information, selecting the most relevant descriptions as you work across the page.
6. Select
  • The Driver will be asked to confirm that the information you have entered is true at the time and correct at the time of raising the claim.
7. If you agree, select OK, if you need to review, select cancel and you will return to the claim screen
  • A “Claim Saved” Instruction box will appear, to help you through the final stages of the claim process.  :9. Select "Continue" to exit the “Claim saved” screen
8. The Driver can select to attach all supporting documents relating to the claim.
9. If the Driver is trying to retrieve a previously lodged claim, then select the search function next to the “Approval” box. In the Search pop up box, select the Approval that is required and this will return the Driver back to the Claim screen.
10. The claim can be seen, and any related documents, if needed, can be attached, using the button.
11. Should the Driver choose not to claim a reimbursement, then the button can be used to remove it, and it will be retained in catch-e as a deleted item. This can only be done, when the item/claim is at a “Pending” status. This will be identified by the text being red. If the item has been processed, black text, the function will not be available.


Each Driver, under their individual login details, will have the ability to extract reports from Catch-e that can show what reimbursements have been created.

1. Go to the Maintenance Tab.
2. In the bottom right of the screen select the 'Maintenance' from the drop box.
3. Select the format in which you want the data to be exported - PDF or .csv.
4. Select

All maintenance entries will appear in this report when run, however the reimbursement entries can be identified where Supplier is equal to 'Reimbursement'.
