Overview

The Approvals tab is used to display, create and manage approvals.
Various rules governing the approvals process can be setup for a client on their Client / Approval Defaults screen.
Approval process emails can also be configured.

These records are also known as claims. The Approval and Claim tabs have been designed for different purposes. The sections below summarise their features.

[edit]
Approval

Use the Approvals / Approvals tab when

  1. The expense is for a vehicle contract supplier (the Supplier's Payment and Banking details are not shown)
  2. You want to use VMRS Codes and/or Job Numbers
  3. The approval header has a Maintenance and Tyre pop-up to show usage information
  4. Each cost line allows for discount amounts and is entered initially as GST exclusive amounts
[edit]
Claim

Use the Contracts / Claim tab when

  1. The expense is for a driver reimbursement or a packaging disbursement (the Payment and Banking details are shown)
  2. The expenses are simple and you prefer to enter a GST inclusive amount
  3. You want to use the Funds Check function to check if claims should be posted and reimbursed.
  4. You want to use the autoPostClaims process to manage and post claims where they have previously failed the Funds Check.

Entering negative values

To enter a negative value in an Approval row, by typing a negative sign "-" a calculator popup appears. Type the required value (excluding the negative sign "-") into the calculator input field and click 'Use'. The negative value will be added into the approval.

Approval Status

Status Sub Status Description
Requested Requested approvals have been created by franchise users in the Merchant Portal.
The maintenance rows for approvals in this status are set as pending.
Active Created but not yet posted. If you review and save a requested claim, it will save into 'Active' status.
The maintenance rows for approvals in this status are set as pending.
Posted The claim has been finalised. It can now be exported to your accounting system.
If the maintenance row has the "R" column set to 'Y' for recharge, those rows will display as blue until the recharge invoice has been created.
Cancelled Created but cancelled without saving. Cancelled approvals are not visible on the Approvals / Maintenance tab.
Deleted Approvals can be deleted by clicking on Delete. This is only possible if the claim has not been posted.
Deleted approvals are not visible on the Approvals / Maintenance tab.

To see these approvals, go to the Approvals / Search tab and filtering on this status.
Click on the hyperlink to get to the claim. The maintenance row is shown in grey.

History Only Active approvals can be set to history only by clicking on History.
These approvals will be shown in Driver Reports but will no be added to accounting exports.
This status is available for adding legacy records, but is not recommended for general use.
Maintenance rows for this status are shown with a grey background.

Action buttons

Field Description
Click on New to create a new approval. Details of the approval can be entered immediately or over several entries using the Edit and Save action buttons.
Click on 'Attach' to attach a related document to a previously saved approval. The document can then be opened at any stage by selecting the paperclip icon located in the Approval Header area. It is also displayed in the Approvals/Maintenance tab and Contracts/Maintenance tab.
You can delete approvals if you no longer want them. They can only be deleted if unposted. Deleted approvals are still stored in the system as 'deleted' and will not be used for accounting or reporting purposes.
You can save an active approval with a status of History Only. Approvals saved at this status will not produce accounting transactions, export files or recharge invoices. Use this status when an approval is not going to be paid or processed further, E.g. a commentary type transaction, recording odometer readings for reporting.
If the check process is completed successfully, the Post button is enabled. Once posted, the approval transactions are available for Recharge Invoicing and Accounting Exports.
Posted approvals can be un-posted and edited. When another transaction or export file has been created using the approval data, Un-post becomes disabled.
Select Check to perform the data validations that are required before posting is allowed. If data is missing or incorrect, validation messages will guide you to correct these before being allowed to post. Once the check validation has passed, the Post button is enabled.
Select Edit to activate the screen for data entry and/or data changes and updates to any active approval. Select Save to keep your changes, or discard them by selecting Cancel.
Un-posting sets the approval status back to 'Active' and can then be edited.
An approval may be un-posted if:
1) the approval has not had a recharge invoice created
2) the approval has not been exported.
