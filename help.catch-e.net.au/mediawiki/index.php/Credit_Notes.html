Overview

You can configure your system to create separate credit notes for the transaction types detailed below.
This will only happen when the total of the transaction is a negative amount, not just one component of the transaction.
Like invoices, credit notes can be re-printed from the Billing / Re-print screen.

You can configure negative records to convert to a credit note automatically, or you can provide an alert asking the user to choose if the transaction is to be recorded and shown as a credit note or not.

[edit]

Credit Note records

If you use the credit note feature, the invoice header will show as 'Credit Note' instead of 'Tax Invoice'.
If more than one transaction is recorded as a credit note, these transactions will all be grouped onto one credit note.

Fuel Card and Booking transactions may contain line items that have a negative value.
These will not trigger a credit note to be generated, as the total amount is still a positive amount.

Credit Notes can only be created for the three transaction types listed below.

[edit]

Generating Credit Notes

Contracts / Lease - credit note transactions will be created if you edit an existing lease period or insert an adjustment entry.
The field [fm_contract_budgets]billing_type_flag is set to 'credit'.
You will be able to see that the transaction will be a credit as the "Note" column will display as 'Credit'.
These credit notes can be generated from Periodic or Combined billing screens and periodic or combined scheduler jobs.

Approvals / Approval - if the total of the approval's recharges are negative, they will become a credit note.
The field [fm_approvals]credit_note_flag will be set to 'yes'.
These credit notes can be generated from Combined, Recharge or Recharge / Fuel billing screens and periodic or combined, recharges or rechargesFuel scheduler jobs.

Billings for Recharge / Fuel or scheduler jobs for rechargesFuel will not generate separate credit notes for small credit amounts where the total charge still requires an invoice.
This is because the imported credits for fuel cards are normally small rebates where a credit note is not applicable.

Sale Management / Sale - if the sale transaction total is negative, this will become a credit note.
The field [fm_billing]credit_note_flag will be set to 'yes'.
These transactions are invoiced directly from the Sale area.

Billing / Re-print - Credit notes re-printed from the Re-print screen will show the same details.

Set-up Notes

Update the controls below to enable credit notes.
For the steps to follow to do this, go to the process page Update a global control.

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
fmbilling_credit_note_flagSet this to 'yes' if you want to generate separate credit notes for the negative transactions detailed above.

The default is 'no' and in this case, no separate credit notes are created.

no
fmbilling_credit_note_prompt_flagSet this to 'yes' if you want the user to choose if a credit note should be created when a transaction is negative.
The alert message is as follows: "Save as a Credit Note?".


The default is 'no' and in this case, credit notes are created automatically when the required conditions are met.

no
