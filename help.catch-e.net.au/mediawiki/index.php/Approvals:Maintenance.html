Overview

The Maintenance screen will show all Maintenance and Approval entries raised against the selected contract. The Fuel Performance and Maintenance Reports can be produced from this screen.

[edit]

Field Descriptions

Screen NameDescriptionType[table]field
Exclude Fuel & TollCheck BoxWhen checked, fuel transactions are excluded. This check box can be configured to exclude transactions for the posting classes you choose.
The Global Controls field contract_maintenance_exclude_checkbox_classes controls which posting classes are excluded.
All HistoryCheck BoxWhen checked any history from contracts with the same registration number will be displayed as 'History Only'. Contracts from separate States are excluded.
Odo ValidationCheck BoxWhen checkeded the odometer reading is treated as 'valid' and is used in reporting and current odometer reading calculations.
CTextClass: The transaction type is allocated to a particular Class, e.g. M = Maintenance. Refer to Posting Classes for further details.
RTextRecharge: Y = Yes, N = No. The recharge flag denotes if the transaction was recharged to the customer. The recharge flag defaults from the Contract Types table depending on the Class. The user may override the default at the point of data entry if permitted.
DescriptionText(40)Description of the transaction
QtyNumQuantity. Typically used to track the quantity of tyres used.
SupplierLinkSupplier code for the transaction. This link navigates you to the Supplier's record. When hovering over the link - the full name of the supplier is displayed in the message line at the bottom left of screen.
ReferenceLinkReference number of the transaction. If the transaction is an Approval the Reference is an Invoice Number and the link will navigate you to the relevant Approval. If the transaction is a Payment, the Reference is a Payment No and the link will navigate you to the Payment screen. Payment No is displayed in Payment Advice PDF's and can be used to located the transaction on the Fleet / Payments / Periodic screen.

Some Reference numbers do not display as a hyperlink and have been saved (or uploaded) directly to the fm_maintenance table. E.g. a Fuel transaction, an "Odometer" reading or "History only" maintenance transactions.

AttachmentDocuments attached on the Approvals screen can be opened on this screen by clicking on the . Users with external role can access the attachment in following two condition:

Firstly, if "Disclose Running Costs" check box is OFF and any Maintenance transaction linked to a particular Approvals attachment having Recharge flag as 'Y'. Secondly, If "Disclose Running Costs" check box is ON.

App #LinkApproval Number of the transactions. The link will navigate you to the relevant Approval.
CostNumNet of GST value of the transaction


[edit]

Output Options

Screen NameDescription
MaintenanceA report that displays the screen information in a PDF or CSV format. This report reflects the screen information. I.e. if "Exclude Fuel and Tolls" is checked ON, then the Maintenance report will also exclude Fuel and Tolls.

The vehicle's running costs will be hidden from external users if the "Disclose Running Costs" Check box of the Contract Type is checked OFF.

Fuel PerformanceA report that displays the fuel usage and summarises fuel performance in a PDF or CSV format. A fleet-level verion of this report is available at Reporting/Standard Reports/Fuel Performance


[edit]

Action buttons

Field Description
Select 'Print' to produce the reports listed in Output Options above.

Screen Legend

Field Description
PendingIndicates that transactions in red are pending and have not yet been posted. Pending Insurance transactions are not displayed in the Maintenance screen.
PostedIndicates that transactions in black have been posted
BillingIndicates the transaction will be recharged but the recharge invoice has not yet been raised
History only Indicates that transactions with a gray background are history only records and do not have accounting implications in Catch-e
