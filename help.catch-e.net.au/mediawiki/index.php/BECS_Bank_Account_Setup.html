Overview

The details on this page are for setting up the banking and aba file records for Reimbursements (ABA) and Disbursements (ABA).

Visit the Receipts (ABA) and the Payments (ABA) pages for their requirements, as they are different.

[edit]

Client Set-up Steps

Provide your Account Manager with the bank account details you want to use.
You can use multiple bank accounts to make reimbursements. In this case, each bank account will need to be linked to a separate Company, as the "Company" of each contract is used to filter the records.
Note: if you are only using one bank account, the field "Company" is not used in the selection process.

Below are the details that are needed for a bank account:

  1. Abbreviated Bank Name eg: NAB, WBC, etc.
  2. Account Name
  3. BSB
  4. Account Number
  5. Company (only required if using multiple bank accounts). E.g. the name that is entered into the "Company" field on the Contracts / Contract tab

Catch-e Set-up Steps

[edit]
Single Bank Account

Where only one bank account is used to pay reimbursements, check the following;

1. [gl_bank_accounts] has one completed record
2. [gl_bank_account_aba_mappings] conatins one record where [gl_bank_account_aba_mappings]process_type = 'becs'.
3. The field [gl_bank_account_aba_mappings]company_id should not be populated and is to remain as NULL. This means all reimbursements will be selected, regardless of their stored company_id.

Click here to view an example of required set up.

[edit]
Single Bank Account - Salary Packaged

1. The field [gl_bank_account_aba_mappings]company_id must be populated if you are salary packaging and using a company bank account.

[edit]
Multiple Bank Accounts

If you want to pay reimbursements out of more than one bank account, you will need to

1. Review all your contracts to make sure each one has their Contracts / Contract "Company" field populated correctly. Visit the Company page for more detail on how companies are set-up and linked to contracts.

2. Check and create new [gl_bank_account_aba_mappings] table records and link each of them to a bank account you are using.

3. Check and update the field [gl_bank_account_aba_mappings]company_id, as each record needs this to be populated and linked to a the [gb_companies] record that is in use for this bank account. I.e. if you are changing from using one bank account, the original record that has NULL set here must now be changed to link to a company.

4. Once this is done, users will need to choose which "Bank Account" they are creating a reimbursement file for. Only reimbursements for contracts linked to that account's company will be selected.

Click here to view an example of required set up.

Take care to makes sure each contract is linked to the right company. When a claim is raised and posted, the company_id is stored against the claim and cannot be changed later by the user, so it must be correct when the claim is raised.

gl_bank_accounts

Field Type Description
bank_account_idint(10) unique_id
gl_code_idint(10) Enter as 0 or add gl_code_id to link this to the gl_codes table for reporting. The field is not required for this functionality.
bankvarchar(3) Enter the bank code. E.g. ANZ, CBA.
account_namevarchar(26) Enter the account name. This is displayed on the BECS screen in the "Bank Account" field.
account_name_shortvarchar(16) Enter the short account name, this is used to populate aba files.
bsbtinyblob
account_numbertinyblob


[edit]

gl_bank_account_aba_mappings

Field Type Description
bank_account_aba_mapping_idint(10) Recorded as a unique_id
bank_account_idint(10) This must link to a valid [gl_bank_accounts] record. It is recorded as a unique_id
process_typeenum('payables','receipts','becs','bpay') Enter as 'becs'.
company_idint(10) Enter as 0 if you are only using one bank account. Enter a valid [gb_companies] record only if you are using multiple bank accounts. The [fm_contracts]company_id is used to filter payment records if multiple bank accounts are in use. This means you need to review and update your data to make sure that contract records all have a matching company_id and that each company that is being used has a matching bank account map record.
apca_numbervarchar(6) Enter the apca number.
aba_descriptionvarchar(12) Enter the description you want for the aba file]. E.g. Reimbursements
aba_self_balancing_flagenum('yes','no')This field determines if a balancing row is added to the aba file or not. If 'no', only Detail(Credit) rows will be created. If 'yes' then a Detail(Debit) row will also be added. The description of the added row is 'Reimbursements'. Click here to see illustrations of the two options.
