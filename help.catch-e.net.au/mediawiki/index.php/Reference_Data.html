Overview

This menu option allows various tables of data to be edited and created.
A module needs to be selected followed by a table name.
Once a table is selected a screen designed to allow the edit and creation of records within the table is displayed with relevant filters and functionality.

Tables

[edit]

qt_makes

Contains the Make Code, description, Type, Data Source and Status. The status flag can be 'active' or 'inactive'. The status needs to be set to 'active' for the make to be made available for selection on the Quote Vehicle screen.

[edit]

qt_models

Contains model information linked to a make. The status needs to be set to 'active' for the make to be made available for selection on the Quote Vehicle screen.

[edit]

qt_bonus_types

Contains descriptions of bonues types, e.g. 'Dealer Bonus', 'Manufacturer Bonus' etc.

[edit]

qt_insurance_rates

Base insurance rates per insurer based on location, vehicle type and lower-upper limits are stored in this table. Stamp Duty and Fire Service levels (if applicable) are stored in Global Controls in the fields isd_rates and fsl_rates per state.

[edit]

qt_rebate_types

Contains names of rebates, e.g. 'Confidential'

[edit]

qt_vehicle_types

This table contains an id which is included in qt_variants to nominate the vehicle type per variant. Fields in this table are:

  • vehicle_type_id (contained in qt_variants)
  • vehicle_type code (e.g. LC, PS, SV)
  • vehicle_type_description (e.g. Light Commercial, Passenger, Sports Vehicle)
  • lct_exempt_flag (used to nominate whether to apply Luxury Car Tax and GST Input Tax Credit limits in the Quote Financecalculation and the GST claimable calculation).
  • date
  • data_source_id (linked to qt_data_sources - e.f. RedBook, Glasses, User defined).

NOTE: This table is not visible in the Reference Data Screen and is noted here for information purposes only.
