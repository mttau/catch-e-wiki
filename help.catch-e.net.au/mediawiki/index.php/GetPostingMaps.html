[edit]

Overview

Enables you to get a list of the available Contract Types.

Supplementary SWAGGER documentation is available here: https://api.test.catch-e.com/docs/#/Posting%20Maps/getPostingMaps

[edit]

Permissions

To run this API, the nominated 'web-services' role needs to be given permission.
If you are not actively using the API, leave the permission off for better security.
Go to Roles / Apis and check on PostingMaps:Get

This API is not configured for external use but can be made available on request.
Contact Catch-e Support to request this update.


[edit]

HTTP Method

Use the HTTP Method 'GET' for consuming this web service.

[edit]

URL Examples

https://api.catch-e.com/gl/posting-maps
[edit]

Parameters - Query

Key Format Notes Default Value Mandatory
posting_map_code string Enter the Posting Map Code. no
posting_map_group string Enter the Posting Map Group (one of: novated, lease, fleet-managed, insurance, non-billing, salary-packaged) no
name string This field supports the wildcard % operator (See Wildcard % operator notes below). no
include_inactive_flag string Defaults to no. Set to yes to include inactive Contract Types. no
pagenumberReturns all pages by default. You can limit the results with this input value 1 no
page_sizenumber25 results per page are returned by default. You can limit the results with this input value 25 no


[edit]

Wildcard % operator notes:

If you are using "Postman" app, with the wildcard operator, the search string of either %ab% or %ba%, will give this response: 
"field_name": {"isEmpty": "Value is required and can't be empty"}.

If you need to search either %ab% or %ba%, please use %25ab%25 or %25ba%25 as required to return results as expected.
