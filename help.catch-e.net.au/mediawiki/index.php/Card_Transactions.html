Overview

This screen enables you to import, process, post and review card transactions created from files provided by you card services provider.
Visit Card Interfaces for a directory of the available interfaces with a link to detail page for each one.

Filters

Field Description
Supplier CodeThe supplier code related to the imported data
Batch NoThe unique batch number allocated per import file by the system
Card NumberThe card number allocated to each transaction
Group CodeThe client's group code if relevant
Client Code (R)The client code. Note that this code is the #Recharge Client code allocated by the card record, which may be different to the client code associated to the vehicle's contract id (the same relationship is relevant to Client Group above.
FromThe transaction date which can be used if searching for transactions in a particular date range
Contract IDThe contract id allocated by the card record
Reg No The registration number associated with the contract id. #Note that this may be different to the registration number contained in the imported transaction file
ToThe ending transaction date if searching in a date range.
Validation Pending batches can contain incomplete records. Visit the Validation page for details of each type of issue and steps to take to fix them.
Recharges / HOThese filters allow the users to include or exclude recharge and history only status transactions.
Status
  1. '--All--'. All statuses will be displayed.
  2. Deleted. A transaction may be deleted by the user or by the card record (i.e. a card set to 'Do not post'. Deleted transactions cannot be posted.
  3. Pending. The transaction has been imported and saved, but it has not yet been posted or deleted.
    1. This is the 'work in progress' status for the user and is set as the default when the user arrives at this screen.
    2. Pending transactions are displayed on the Contracts / Maintenance tab as pending transactions. These records are updated to match changes made here.
  4. Posted. The transaction has been posted to the maintenance file.

Top Buttons

Button Description
Click to execute your chosen filtering.
If the button is disabled, then the current filtering has been executed.
Click to import a card services file.
Click to purge unwanted data in a pending batch that you do not want to process.
Records will be deleted from the Maintenance / Transactions screen and the Contracts / Maintenance tab.

The purge button is enabled if

  1. "Batch No" is populated
  2. "Status" is 'Pending'
  3. the linked Contracts / Maintenance records are either 'Pending' or 'Deleted'

Purge is disabled if

  1. "Status" is 'Posted'
  2. the linked Contracts / Maintenance records are 'Posted', 'History Only' or 'Recharge'

Screen Legend

Field Description
Failed Transactions have a pink background. The transaction can't be processed.
Go to the Validate a card transactions batch process page for more details.
Warning Transactions have a yellow background. The transaction can be processed, but there is a card setting you should review.
Go to the Validate a card transactions batch process page for more details.
Pending Transactions with red text are pending. These records are also shown in the Contracts / Maintenance tab as Pending.
Posted Transactions with black text are posted. These records are also shown in the Contracts / Maintenance tab as Posted or Recharge.
Deleted Transactions with grey text are deleted. These records are not displayed in the Contracts / Maintenance tab.


[edit]

Action Buttons

Button Description
Prints the selected transactions into a PDF or csv file. Note that the csv file contains additional transaction details not necessarily displayed on the batch screen.
Click to check each transaction. The results of the checks are shown in the the "Validation" column. Click to re-run the check process. This will refresh the details in the "Validation" column.
The check process also validates the odometer readings against valid odometer readings stored in the maintenance table. All pending transactions (including card service transactions) in the maintenance table are included in the odometer validation. If the odometer reading is not zero and is not one of the error values (e.g. 777) and the validation method selected is the "3) Contract Km Method" (refer to Estimated Odometer Calculation) but fm_contract->contract_kms are set to 0, the odometer reading will be treated as valid.
You can post once the selection contain no failed, deleted or posted transactions. The Post button will permanently update the maintenance file and allow relevant transactions to be recharged via recharge billing. Transactions are changed from a status of 'pending' to 'posted' and cannot be re-posted.
A transaction can be deleted if you use the check box to select it and then click on the Delete button.
Deleted records are no longer visible in this screen, but remain in the records with a 'deleted' status.
To permanently delete transactions, use the 'Purge' feature.

A batch has already been created with a file that is an exact match for the file being imported.
Cancel this import.

Sometimes you may need to re-import a file if you have previously purged an imported file before it was posted. This is an unusual situation.
You should not import the file unless you are sure the records have not already been imported and posted.

Before proceeding:
1. Check recent batches stored in Card Services / Transactions to see if you can identify the existing batch.
2. If you cannot find the existing batch, open the import file in notepad and compare some of the transactions with posted maintenance records for the contracts to see if the imported records are present.
3. If the prior batch has definitely been purged before being posted, you can re-import the batch.

Please enter a valid 'Invoice Date'. To save a card import, the "Invoice Date" and "Invoice No" to be used for the records must be entered.
The format of the Import File does not match the selected Card Interface!
Please check your selections and try again 		Possible reasons for this alert are as follows:
  • You are trying to import a fuel import file with a format that does not match the Card Interface you have chosen.
  1. Once the Card Interface has been selected during the import process, the file format is often shown.
  2. If this is not shown, go to the Card Services page and click through to your provider's Card Interface page to see which file format is required.
  3. If you think the provider has changed the format of the import file, contact Catch-e Support to confirm the current format.
  • You are importing a csv file and this is the correct format
  1. You have saved the file using an incompatible CSV file type (there are many files with a *.csv format that are subtly different), E.g. 'CSV UTF-8 (Comma delimited)(*csv)
  2. Save again in excel using Save as type: CSV (Comma delimited) (*.csv)
