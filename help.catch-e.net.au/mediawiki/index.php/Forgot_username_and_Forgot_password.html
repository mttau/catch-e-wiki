“Forgotten Password”

This feature is accomplished with a Global Control [gb_controls]driver_portal_password_auto_recovery_flag.

  • When the control is set to ‘yes’ the feature will be available.
  • With the feature enabled, on the Driver Portal login screen to initiate a ‘Forgot Password’ action the user would click ‘Forgot Password?’.
  • The text will be a hyperlink that points to the ‘Forgotten password’ page.
  • They would be navigated to a ‘Forgotten password’ page where they enter their ‘user name’ and ‘email address’ for authentication and select ‘Reset’.

By clicking the forgot password from the driver portal , the user will get a message as below :-
Thank you. An e-mail will be sent to your registered e-mail address shortly with further instructions for resetting your password. If you do not receive an e-mail within 10 minutes please call your system administrator.''

  • The above message is stored in the in [gb_controls]password_forgotten_message and can be edited by the user.
  • If the email provided matches one stored in the users [fm_drivers]home_email field, an email will be sent to that address and will write the reset message to the gb_mail table.
  • If you find the mail in the screen Mail Queue, the field “Mail Type” can be filtered to show ‘Notification’ for these emails.
  • In the mail message a token will be generated.
  • This token is stored in gb_tokens. The token will have an action of ‘password_reset’. It will have a record_id linked to [gb_users]user_id.
  • The Created date will be the date of the password reset.
  • The token will expire after the time and date created + Session Timeout set for the role password_reset
  • The URL shown in the mail message will be the same format as any other portal link i.e http://demo-driver-portal.catch-e.net.au/en/login/TOKEN_VALUE
  • Using the link will navigate the user to a ‘Change password’ page. The user will enter a new password and confirm, on Save there will be a confirmation and log out,
  • User can then re-login and if enrolled for 2FA will be prompted for that.


"Forgot Username"

  • If the user is unable to remember the Username they would select ‘Forgot User Name’.
  • If ‘Forgot Username’ is selected display a message as below:-

“Please contact [client name] to reset your password”.

  • This message will come from [gb_controls]password_forgotten_user_id_message and can be configured by the user.
