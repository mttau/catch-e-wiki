Overview

Two-factor authentication (2FA) is an extra layer of identity checking you can use to strengthen access security.

The Drivers / Contact field is checked when the driver has enrolled for 2FA.
To reset a driver's 2FA, edit and uncheck this field.

gb 2factor_auth_revalidation_days Set this control for the number of days a user may login without performing a 2 factor authentication.
  1. The system will not prompt for an access code for the stored number of days if the user is logging into the same browser, ip address, user login and method when they opted to 'Remember' the access code.
  2. Each time a user logs in and one of these variables has changed, a 2FA code will be required (but each scenario can be remembered).
  3. E.g. using a different combination of browser, network, computer, user login and enrolment method ('Sms' or 'Auth App') triggers the demand for a 2FA code.

Mandatory or Optional

If 2FA is enabled, you can make 2FA Optional or Mandatory for Drivers and Employees.

  1. Go to Roles / Details
  2. Mandatory - check on to make 2FA mandatory, or leave un-checked to leave 2FA as optional.
    1. If Mandatory, all unenrolled drivers will be redirected to the 2 Factor Authentication screen when they next login to complete the enrolment process
    2. If Optional, drivers can click on the 2 Factor Authentication button and enrol if they want.
  3. Method - unless Catch-e has set-up an SMS option for your business, choose 'Auth App'.
    1. If 'Auth App is selected, recommend an RFC 6238 compliant 2 factor authentication application such as Google Authenticator or Authy to your drivers. Where â€˜Auth App' is the selected method, the driver will be prompted to enter an authentication code generated by the phone App.
    2. If Sms is chosen, the required access code will be sent via SMS to the driver's mobile number. The access code will be valid for 5 minutes from the time it was requested. A mobile number can only be used for one driver record.

Troubleshooting


Visit the main Troubleshooting page for a list of all the available problem-solving tips.


[edit]
Unenroll a driver

If a driver or employee has problems with their 2FA Enrolment, a System Administrator in your business can unenroll them from 2FA.
The driver can then re-enrol when they next log in.

  1. Go to Drivers / Contact
  2. 2FA Enrolled - un-check to unenrol the driver.
[edit]
Driver is locked out

If the entered app or sms code is valid, the user will have access into the Catch-e system.
If it is invalid, three attempts will be allowed before the user is locked out of the system.
When a user is locked out, their Users / Details "Status" will be 'locked'.

[edit]
Unlock a driver

If a driver's login has become locked, their login will need to be reset.
Follow the Reset a Driver Login process to do this.
