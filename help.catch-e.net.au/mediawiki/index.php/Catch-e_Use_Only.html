Overview

You can use 'Lease Inclusions' to add funded inclusions to a quote.

[edit]

Inclusions

You can set up more than one inclusion product. Do this when you are offering two different products that can both be selected on the quote OR you are replacing one product with another.

Go to Lease Inclusion Plans Setup for the set up details.

[edit]

Plans

Each inclusion needs to be linked to a plan. The plan is where the product pricing is stored. If you only need one plan/price, the plan name and "Retail" value will default into the quote from the stored table data.
If you need to offer more than one plan for an inclusion, the user will choose the desired plan from a listbox once the "Inclusion" has been selected.
When a plan has been chosen, the "Retail" value will populate from the stored table data. Do this when you are offering two or more choices for a product, i.e. only one of the choices can be selected on the quote. See Notes on adding new plans below for further details.

[edit]

Exclusions

You can prevent an inclusion from being offered to a particular client, or for particular makes, models or variants.

[edit]

Financed

Each inclusion is shown by name in the Quotes / Lease tab as an inclusion.
It is a financed product and the total value of all inclusions are displayed in the Quotes / Finance "Total Lease Inclusions" field and this forms part of the "Total Brokerage" and "Amount Financed" amounts.

Renaming

The field name "Total Lease Inclusions" shown in Quotes / Finance can be relabelled.

[edit]

St George Interface

If you use the St George NBI2 financier interface, Inclusions are mapped to the "Other Brokerage" field in Sovereign by default. You can map this to "Other Insurance" instead by setting [gb_controls]st_george_lease_inclusion_brokerage_flag to 'no'.

[edit]

Eric Interface

If a quote contains Eric Insurance products, the Eric Interface is invoked during the Save process and the quote is updated to contain their current product pricing.
When copying a Quote either via the Catch-e application or via the copyQuote token based API, the "Retail" price of any Inclusions plans are reset to $0.00 whereby the Eric Interface is invoked during the Save process and the quote is updated to contain their current product pricing.
If you want to find out more about this feature, visit the Eric Interface page.

[edit]

Save Validations

A quote will not save and the user will be alerted if;

  • The inclusion is selected and the plan remains blank.
[edit]

Validation Updates

There are 4 scenarios that will affect the Lease Inclusion Plan price when the Quote Effective Date is altered and either a Lease Inclusion Product is now available, or no longer available.

  1. A quote is being edited that has an 'active' inclusion however based on the quote's "Effective Date", another plan record with the same code and a newer "Effective Date" is found.
    1. The system will update the price of the inclusion and recalculate the quote. The quote will save. The user will not be alerted.
  2. A quote is being edited and the selected inclusion is 'active' but the plan is now 'inactive'.
    1. if there is another plan record available (i.e. it has the same plan code, it is 'active' and has a conforming date), then the available plan record will be added and the quote will be recalculated. The quote will save. The user will not be alerted.
    2. if there are no plan records available that can replace the stored plan, but there are other plans that can be selected on the inclusion, then the inclusion will be un-checked. The user will need to re-check the inclusion and choose a new type of plan.
    3. if no plans are available for the inclusion, the inclusion will be un-checked and the check box will be disabled.
  3. A quote is being edited that has a selected inclusion that is now 'inactive'.
    1. The inclusion is unchecked and disabled. No products are available in the list box and the “Retail” field is display only.
    2. If a quote like this is copied, the stored inclusion is retained, but when the quote is edited, the update described above will occur.
  4. A quote is being edited that has a selected inclusion that is now excluded by either Client, or Make / Model or Variant.
    1. The 'Lease Inclusion' will remain intact in the Quote unless it is manually removed by the user after which the 'Lease Inclusion' will no longer be available.
    2. If the Client Code, Make, Model or Variant are updated in the Quote and the respective Client, Make, Model or Variant is to be excluded the 'Lease Inclusion' will be removed from the Quote and cannot be added again.
