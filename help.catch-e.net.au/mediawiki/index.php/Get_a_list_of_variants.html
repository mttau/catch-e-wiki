[edit]

Overview

This method will list the makes and its detail.
This method requires the QuoteVariants:Get permission to be associated with your role.
By default only 'web_services' role is linked in [gb_role_api_permissions].

[edit]

HTTP Method

Use the HTTP Method 'GET' for consuming this web service.

[edit]

Input Fields (Header)

  • Header should contain the 'Accept' Key with the Value = 'application/json' or 'application/vnd.catch-e-api.v1+json'.
[edit]

Input Fields (Body)

FieldFormatNotes Default Value Mandatory
make_idstringMake id No
model idstringModel id No
variant_codestringVariant code.
This field supports the wildcard % operator (See Wildcard % operator notes below). 		No
variant_namestringVariant name.
This field supports the wildcard % operator (See Wildcard % operator notes below). 		No
body_type_idstringBody type id No
data_source_idstringData source id No
profile_required_flagstring If yes, display variants that have a qt_variant_profiles record with a 'date' valid for the current invocation date.
I.e. the [qt_variant_profiles]date must be on or before the date the API is run. 		yes No
include_inactive_flagstringSet to yes if inactive items required in the list. Defaults to 'no' no No
pagestring Page number. Defaults to 1 1 No
page_sizestring Page size. Defaults to 25 records per page. 25 No
[edit]

Wildcard % operator notes:

If you are using "Postman" app, with the wildcard operator, the search string of either %ab% or %ba%, will give this response: 
"field_name": {"isEmpty": "Value is required and can't be empty"}.

If you need to search either %ab% or %ba%, please use %25ab%25 or %25ba%25 as required to return results as expected.
