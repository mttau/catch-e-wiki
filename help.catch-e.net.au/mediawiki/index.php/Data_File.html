Overview

You can create new records or update existing records in a table with the data file you upload.

When you build the data file for your import, the file doesn't need to contain all of the fields that exist in the table

  • if changing existing records in a table, only add the fields you are changing to the file
  • if created new records, only put in the fields you had information for, other fields will be filled with the database's default values

If you are using executeImportTableData, the query you create needs to conform to the Data File requirements or the job will fail.

Recommended Steps

The steps below are useful when building a new data file

  1. Export sample data from a table to get a sample file of the table you want to update.
  1. Visit the Data File page to check the data requirements for your upload file.
  2. Visit the Table Data page where known required fields have been documented.

[edit]

Update existing records

  1. Create a base file that contains the records you want. Below are some approaches to this
    1. Export the entire table. (This is only suitable if the table is small).
    2. Run a report from Contracts / Search or Quotes / Search
    3. Run an existing scheduler job that contains the required fields
    4. Run a select query in MySQL Workbench
  2. unique_id(s) - locate the unique id for the table. This is usually at the start of the table.
    1. Some tables require more than one id in the data file. Visit Table Data to check a list of known tables like this.
  3. If there are records that you are not going to update in the file, find these and delete these rows. (This reduces the risk of updating other records by mistake).
  4. Locate the fields you want to update and remove all other fields from the file that are not needed. (This reduces the risk of updating other fields by mistake).
  5. Update the file fields you want to change.
  6. Save your file in CSV format
  7. Import data to a table
[edit]
Example update

You want to update the Contracts / Insurance "Start Date" and "Policy Due Date" for a selected list of contracts.

Module - 'fm | Fleet Management'
Table - 'fm_contracts'

File

contract_id insurance_start insurance_renewal
100000 01/15/2025 01/15/2026
100001 13/15/2025 13/15/2026
100002 20/15/2025 20/15/2026


  1. The contract_id is the unique_id here.
  2. Only the unique_id and the two fields being updated are also in the file.
  3. When the file has been imported, the Contracts / Insurance "Start Date" and "Policy Due Date" fields will have been updated for the selected contracts.
  4. The selected contracts will also have audit records recording those changes.
  5. No other fields in those contracts, or any other contracts will be updated by the import


[edit]

Create new records

  1. Create a base file that contains the records you want. Below are some approaches to this
    1. Export sample data from a table
    2. Run a select query in MySQL Workbench
  2. unique_id(s) - locate the unique id for the table. This is usually at the start of the table. Leave this blank.
    1. Some tables require more than one id in the data file. Visit Table Data to check a list of known tables like this.
    2. If you try to add new unique id number, you will get the 'Record does not exist' error on the Error Report.
  3. Locate the fields you want to update and remove all other fields from the file that are not needed.
    1. Fields can be removed from the file if you want to use the system's default value populate into the field.
  4. Update the file fields as required.
  5. Save your file in CSV format
  6. Import data to a table

Preparing your Import file

Field Type Type Required Comments
Header RowTextyesAll files must contain a header row that lists each field that is being created or updated.
E.g. To update roadside details in [fm_contracts], you could have this header;
contract_id, roadside_managed_flag, roadside_start_date
Primary KeyNumif presentIf the table has a Primary Key, the file must contain a column for this field. This is the field that uniquely identifies each record.
E.g. for [fm_contracts], it would be contract_id.

When updating existing records, the Primary Key column must be populated.

When creating new records, the Primary Key column must still be present. It's possible to upload the unique id as a blank field, null or NULL. We recommend you populate this field with the value 'null'.
Be careful to make sure you do not populate this column. This may result in over-writing existing records (if the key value already exists), or potentially creating future data corruptions.
During the import process, the [gb_unique_ids] table is checked and corrected if it is out of synch before the upload is done. This ensures that new records are given the correct next id.

Unique KeyVariousif no Primary KeySome tables do not have a Primary Key. In this case, the file needs to contain all of the fields that make up it's Unique Key.
Visit the Table Data page where known required fields are documented.
DateDateno Date fields must be in either 'YYYY-MM-DD' or 'DD/MM/YYYY' format.
E.g. '2019-12-24' or '24/12/2019'.

If you want to remove an existing date, the entry can be '00/00/0000', '0000-00-00', or null or NULL.

If the field is blank, or the date is invalid, the upload will fail and you will get an error. e.g. blank, 01/13/2019 or 2022-12-35 are all bad entries.


[edit]

File requirements

  1. The file must be in csv format. The maximum file size is 32MB (32768KB)
  2. The file must have a header row that contains field names from the import table and the field names must exactly match field names of the import table
  3. The primary key;
    1. Is a required field in all import files
    2. Should be 'null' for new records
    3. Should be populated for existing records
  4. If the table contains unique keys, those fields are required in all import files
  5. The file cannot contain blank rows
    1. If you are importing data using a file you have exported using the Export Table Data menu, check to see no extraneous line has been added to the end of the file.
    2. This can be achieved by converting the file to a .txt file.
    3. Be sure to remove the line and save to .csv again prior to importing file.
  6. The file you are uploading cannot be identical to one that has been imported previously
  7. The file can contain all or only some of the fields that exist in the import table
  8. The fields can be in any order
[edit]

Data updates for new records

  1. A primary key value is added to each new record.
  2. Fields that are not in the import file will be created using the database default.
  3. Audit records are written for all fields in the table, even if the field was not in the import file
[edit]

Data updates for existing records

  1. Fields are only changed if they are in the import file and the value is different to the stored value
  2. Audit records are only written if the import has changed the field's value
