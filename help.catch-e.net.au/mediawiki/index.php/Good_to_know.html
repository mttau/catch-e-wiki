Email

System emails can be generated that contain standard text, report PDF's and attachments.

  • Examples
    • Email a quote to a driver from Quotes / Contract
    • Email a quote request to a dealer from Quotes / Vehicle
    • Email a Driver Report to a driver from Contracts / FBT
    • Email a reimbursement notification to a driver from Contracts / Claim


  • Emailing is usually on in your LIVE environment and off in your TEST and SUPPORT environments. Visit Mail Send Block Flag for full details of this feature.
  • When the email option is selected the user is presented with a Send Email popup. The "From" address is displayed as an aid to see which Domain is being used for the senders address.
  • If your Domain is not enrolled and you have a fall back email address, the "From" address will display 'noreply@catch-e.com.au' and a "Reply To" address will also be shown.

Modal window

A modal window is a child window that you need to interact with before you can return to operating the parent application. This prevents you from changing the main screen until the secondary window details have been dealt with. We use this feature in the following places;

  • Contracts / Claim to Select Bank Account
  • Quotes / Contract tab to Create Driver
  • Bookings / Booking tab to Create Driver for Booking
  • Drivers / Finances to View Bank Accounts
  • Drivers / Item to View Bank Accounts or to Select Bank Account
  • Employees / Finances to View Bank Accounts
  • Employees / Item to View Bank Accounts or to Select Bank Account

Multiple Email Addresses

This feature enables you to email more than one person.
To enter multiple email addresses - separate each email addresses with a semicolon (;). E.g.,

mike.smith@work.com.au;mikes74@gmail.com


This feature works in the following locations

  1. Drivers / Contact "Email"
  2. Clients / Contact A/C Payable "Email" and CC "Email"
  3. Suppliers / Detail "Email"
  4. Suppliers / Account "Email"
  5. Users / Details "Email"

Selecting multiple records

When reporting, some filters can be selected as multiples. We'll use clients as an example, but classes are another common filter when multiple selections are possible.

To select particular clients, hold down the “Ctrl” key and use your mouse to select.

Selecting a range of records

To select a range of clients, hold down “Ctrl” and “Shift” keys and use either your mouse or arrow keys to select.

Right Click on Hyperlinks

When on a hyperlink, the right click menu is disabled. This is to prevent session cross-over problems that can be caused by selecting "Open in new window" to open and run a second Catch-e window with the same credentials.

The right click function is still available in all other contexts allowing users to continue using features such as Copy / Paste and Print.

If you want to run a second window, you can safely do so using the separate url that is available for this. Go to Setting up a second shortcut for details.

If you use the Right Click or Middle Click function when on a hyperlink, the following alert will be displayed;

This feature is not supported!
If a new window or tab has been opened please close it immediately and do not attempt to use it.
If you require multiple sessions then please refer to "Appendix C: Browser Configuration Guide" in the online help or as your support desk.

Do not use other keyboard shortcuts such as Ctrl N and Ctrl T, as they also open and run a second Catch-e window with the same credentials.
Because hot keys such as Ctrl N and Ctrl T are a function of the operating system and not the browser, they can't be disabled from the software and remain a pathway for session cross-over problems. Fortunately, this problem only affects legacy versions of Internet Explorer. Edge, Chrome and Firefox all open an empty window, rather than a second window.

Search

If the registration number of the contract the user wishes to access is known, it can by typed directly into the ‘Reg Number’ field. Pressing the Enter key will bring up the contract details. Alternatively a Contract ID or Approval number can be entered directly into the relevant fields.

A contract or approval can be searched for by clicking on the search symbol (magnifying glass): which will open a search window. The search By: field can be changed via a drop-down list to enable a number of alternative searches.



Partial text searches can be performed. If part of the beginning of the item is entered and the ‘Go’ button is clicked, the system will search for items beginning with the text entered. If the click box ‘Contains text’ button is ticked, the system will search for matches where the text exists anywhere in the relevant field. This is particularly useful when searching for items within the text field itself.

Once the results of the search have been displayed, the user can select the desired record by clicking on the contract id field which is displayed as a clickable hyperlink. Where more than one page of results is returned, the user can navigate between the pages by using the navigation arrows on the bottom right of the screen.

The check box "Close on Select" is on by default and the window closes automatically when a record has been selected.

If this is checked off, the window will remain open. When a record is selected in these circumstances, the system window will come to the front and the Search window will be hidden. Use <Alt> <Tab> to bring this to the front again.

?Search

The '?Search' feature enables the user to bring up a pop-up screen to search for the required data for a given field.
When 'Search' is available, the symbol will be displayed in the bottom left corner of the screen.
This is useful in areas where it is not suitable to use a .

The '?Search' feature can be seen in the following places:

Contracts / Events → Driver Change / Driver Name

Bookings → Invoice

Setup / Reference Data → qt_variant_profiles → "Vehicle Type"

View Edit History

The icon that is located in the top right corner of most screens enables you to reveiw changes that have been made to the fields within the screen or area.
Clicking on generates a pop-up of the audit information called View Edit History.
The 'X' in the right-hand column called "S" indicates that the record was updated by an external process. i.e. via a web service.

Visit Field Auditing for a full description of this feature.
