Overview

Quotes / Finance

When the Refresh Base Rate button is clicked, the system will compare the Financier Rates table for latest corresponding Base Rate for the Supplier (Financier) / Term / Lower Limit / Upper Limit combination against the equivalent quote fields.

The System Date is used to compare to [fm_financier_rates]date (not Quote Date or Quote Effective Date).
If the [fm_financier_rates]interest_rate is the same as the Quote “Base Rate”, a message will be displayed advising:
Nothing to update!
No further action is required.

If the [fm_financier_rates]interest_rate differs from [qt_quotes]base_rate:

If the Quote or Contract cannot be locked, or Billing is in progress, no changes will be made to the Quote or linked Contract and a message will be displayed:

Locked by: {user_login}

If the Quote or Contract can be locked, & Billing is not in progress:

The following fields will be updated & recalculated using the new “Financier Rate” (refreshed “Base Rate + “Client Margin”):

Screen Label [table]field
Quote Fields:
Base Rate [qt_quotes]base_rate,
Financer Rate [qt_quotes]finance_rate,
Field not shown in Catch-e application [qt_quotes]deferred_interest (if applicable),
Lessee Rate [qt_quotes]lessee_rate,
Finance Exc GST [qt_quotes]finance_pmt_net,
GST [qt_quotes]finance_pmt_gst,
Budget Net [qt_quote_budgets]amount_net,
Budget GST [qt_quote_budgets]amount_gst,
If Quote “FBT Type” = ‘operating’
FBT Value [qt_quotes]fbt_value,
Field not shown in Catch-e application [qt_quotes]fbt_value_yearly,
Depending on Global Control settings the following values may also update:
Commission Rate [qt_quotes]commission_rate,
Commission Fee [qt_quotes]commission,
Luxury Car Charge [qt_quotes]luxury_charge,
Redundancy Insurance [qt_quotes]redundancy_insurance.
Contract fields:
Initial Payment [fm_contracts]finance_pmt_initial,
Regular Payment [fm_contracts]finance_pmt_regular,
Deferred Interest [fm_contracts]deferred_interest,
Amount Financed [fm_contracts]amount_financed,
Interest Rate [fm_contracts]interest_rate,
Finance Rate [fm_contracts]finance_rate,
Total Brokerage [fm_contracts]total_brokerage,
Contract Payment Structure/Schedule fields (if created):
Total Payments [fm_contract_payment_structures]total_payments,
Payment Schedule Finance [fm_contract_payment_schedules]amount_net,
Payment Schedule GST [fm_contract_payment_schedules]amount_gst,
Contract Budget fields (for Posting Class Code ‘FIN’):
Finance (Next) Net[fm_contract_budgets]amount_net,
Finance (Next) GST [fm_contract_budgets]amount_gst,

The Quote/Contract Lock will be released once all updates are complete and a message will be displayed advising:
Base Rate updated.

Note: this differs from the Settle process because that process updates numerous other fields where this process targets the Finance Rental component.

[edit]

Audit History

Audit History of changes will be recorded.
A “Reference” comment of “Refresh Base Rate” will be added to the [qt_quotes] & [fm_contracts] fields updated.

[edit]

Validation Rules

The Refresh Base Rate calculations will work only when:

Rule 1: Contract Type / Accruals is linked to Posting Class Code ‘FIN’, AND
Rule 2: Quote is linked to a Contract, AND
Rule 3: a) Contract Start date is blank, OR b) Contract Status [fm_contract_statuses]finance_base_rate_refresh_flag = yes AND
Rule 4: There are no Contracts / Lease records at 'posted' status. I.e. [fm_contract_budgets] AND
Rule 5: There are no Contracts / Payments 'Finance' or 'Finance - Principal and Interest' records at 'posted' status.

The following validation messages are displayed when their corresponding rules fail:

Rule 1: Contract Type / Accruals is not linked to Posting Class Code 'FIN'
Rule 2: Quote is not linked to Contract
Rule 3: b) Base rate refresh disabled for Contract #{contract_id]
Rule 4: Linked Contract #{contract_id] has started
Rule 5: Contract #{contract_id] has posted budgets or payments

[edit]

Other Notes

Editing a Quote after “Refresh Base Rate” clicked:
If a quote is edited after “Refresh Base Rate” has been executed, users should note that updating the Effective Date, Contract Type, Term, Financier or Client will trigger the system to re-load the applicable Contract Type / Financier defaults (Setup / Reference Data -> table name qt_financier_posting_map_defaults) which may update the Base Rate.
This update will not automatically update the contract.
User should click Settle button to update contract fields.
