Packaging Fees being deducted in each pay run

Set-up the supplier with the following attributes:

Supplier / Detail
Supplier Code - a code that suits for your packaging fees
Name - a descriptive name for your packaging fees
Supplier / Account
Status - 'Active'
Supplier Type - 'Salary Packaging'
BSB Number - enter the BSB Number for your packaging fees
Bank Account Number - enter the Bank Account Number for your packaging fees
Payee - 'Supplier'
Pay Method - 'Direct Credit'
ABA Format Type - 'Supplier'

Set-up the item with the following attributes:

Deduction Details
Total Deduction - let this populate with a default standard annual deduction (set a default entry for your standard annual fee amount)
Compress - un-checked (this depends on your fee policy)
Inertia - checked
Cycles - let this populate with the remaining cycles for the current FBT year.
Disbursement Details
Disbursement Type - 'Recurring' (set this as the default entry)
Supplier - 'Packaging Supplier' (set this as the default entry, read-only)
Reference - leave this blank
Payee - 'Supplier' (this will default from the supplier's stored entry)
Status - 'Active'

The process steps for this type of item is as follows;

  1. Pay Runs / Pay Run - post the pay run record
  2. Receipts / Allocate - receipt and post the pay run
  3. Scheduler / Job - sp / approvals / autoBuildClaims runs automatically
  4. Contracts / Claim - the job creates a posted claim for the receipted amount
  5. BECS / Disbursements (ABA) - create a BECS batch and download the ABA file
    1. Fees are grouped into one record. This means you receive one banking payment for multiple fees.
  6. Banking Software - import the ABA file and confirm it for payment


When the pay runs and claims for this item are complete, no further deductions or disbursements will be made.
It can now be suspended.

Benefits being paid for by the employer

Supplier

Set-up a supplier record for the employer

Detail - Supplier Code - for ease of use, use the same code as the employer's Client Code (not required, but just a suggestions)
Detail - Name - employer's name
Account - Status - 'Active'
Account - Supplier Type' - 'Salary Packaging'
Account - BSB Number - enter the employer's BSB Number
Account - Bank Account Number - enter the employer's Bank Account Number
Account - Payee - 'Supplier'
Account - Pay Method - 'Direct Credit'
Account - ABA Format Type - 'Supplier' (if payments are to be aggregated), 'Payment Reference' (if each payment should be made separately)


Set-up the item with the following attributes:

Deduction Details
Total Deduction - enter the total amount that will be paid back to the employer.
Compress - un-checked
Inertia - un-checked
Cycles - enter the number of pay runs the benefit will be spread over.
Disbursement Details
Disbursement Type - 'Recurring'
Supplier - 'Employer'
Reference - leave this blank if the payments are to be aggregated. Enter a reference if payments are to be separate.
Payee - 'Supplier'
Payment Method - 'Direct Credit'

The process steps for this type of item is as follows;

  1. Pay Runs / Pay Run - post the pay run record
  2. Receipts / Allocate - receipt and post the pay run
  3. Scheduler / Job - sp / approvals / autoBuildClaims runs automatically
  4. Contracts / Claim - the job creates a posted claim for the receipted amount
  5. BECS / Disbursements (ABA) - create a BECS batch and download the ABA file
    1. Payments can be grouped into one record or paid separately depending on your client's preferences.
  6. Banking Software - import the ABA file and confirm it for payment


When the pay runs and claims for this item are complete, no further deductions or disbursements will be made.
It can now be suspended.

Benefits paid for by the employee

Set-up the item with the following attributes:

Deduction Details
Total Deduction - enter the total amount that will be paid back to the employee
Compress - un-checked
Inertia - un-checked
Cycles - enter the number of pay periods the employee wants to spread the payroll deductions over
Disbursement Details
Disbursement Type - 'Recurring'
Supplier - 'Employee Disbursement'
Reference - leave this blank or add a short description to show on the employee's bank statement
Payee - 'Employee
Attach the supporting documents to the item (E.g. Invoice, Receipt, Declaration)
Status - 'Active'

The process steps for this type of item is as follows;

  1. Pay Runs / Pay Run - post the pay run record
  2. Receipts / Allocate - receipt and post the pay run
  3. Scheduler / Job - sp / approvals / autoBuildClaims runs automatically
  4. Contracts / Claim - the job creates a posted claim for the receipted amount
  5. BECS / Disbursements (ABA) - create a BECS batch and download the ABA file
    1. Employee disbursements are shown as one record per claim.
    2. The employee will receive one banking payment for each item being paid.
  6. Banking Software - import the ABA file and confirm it for payment


When the pay runs and claims for this item are complete, no further deductions or disbursements will be made.
It can now be suspended.
