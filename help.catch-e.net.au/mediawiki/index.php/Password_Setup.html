Password Expiry

The default number of days allowed until expiry is set in the Roles / Details "Password Expiry Days" field.
This value will populate into the Users / Details "Password Expiry Days" field when a user record is created and can be edited at any time.

Once the Users / Details "Password Expiry Days" number has passed and the user goes to the Login screen,
they are navigated to the Users / Password tab and must change their password before they can continue to use the system.
The calculation is the current System Date less the date when password last changed in whole numbers. I.e. [gb_user_passwords]created.

The users password will never expire if;

  1. Users / Details "Password Expiry Days" is set to '0' or
  2. The Roles / Details "Single Singon" field is checked for the user's role.

Password Validation

The variables you can adjust for the password validation are listed below;

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gb password_exclude_dictionary_words_flag no
gb password_history_size 12
gb password_maximum_dictionary_word_size 4
gb password_minimum_length Set the minimum required length of an allowable password. The maximum this can be is 64 characters. 8
gb password_requires_letter_flag yes
gb password_requires_lower_case_letter_flag yes
gb password_requires_number_flag yes
gb password_requires_special_character_flag yes
gb password_requires_upper_case_letter_flag yes
