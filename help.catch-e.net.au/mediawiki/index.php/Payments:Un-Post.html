Validations and Alert messages when Un-Post is selected

AlertComments
This action will delete or modify payment no ###, are you sure? You'll see this message when the transaction has not been exported. Click OK to proceed, or Cancel to return to the screen (A payment is identified as not exported if export id = 0).
This action will delete or modify payment no ###.

This transaction has been exported and other systems may be affected, are you sure?

You'll see this message when the transaction has been exported and you have permission to Un-Post. Click OK to proceed, or Cancel to return to the screen.
This transaction has been exported and other systems may be affected.

Only authorised users can Un-Post

You'll see this message when the transaction has been exported and you don't have permission to Un-Post. Click OK to return to the screen.


The Un-Post function does not check or alert you if the payment's remittance advice has been emailed to the supplier. Consider this when un-posting.



[edit]

Role Restrictions

By default, only the 'admin' role can use the Un-Post button when a payment has been exported.
The button is still displayed so that users know the action is possible.

You can contact Catch-e Support if you want to give access to the Un-Post button to other roles in your business.
The access is controlled from a Role Restrictions setting.
