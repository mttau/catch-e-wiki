Overview

Sensitive database fields within Catch-e are encrypted to enable a higher level of security.
When running reports / queries from external database connections e.g. using the Query Browser or Crystal reports, these encrypted fields are not available.

[edit]

What is encryption?

This means the underlying data has been changed, using an algorithm and is unreadable unless the cipher key is known. See http://en.wikipedia.org/wiki/Encryption for more about encryption.

[edit]

Encrypted Fields Syntax

Reports run from the Scheduler or Web Services can access these fields using special syntax.
Encrypted fields can be accessed using special syntax.

Example

To decrypt and report on the [fm_drivers]income_gross field you can use this: 


SELECT AES_DECRYPT(income_gross, 'AES_CRYPT_CIPHER_KEY') AS income_gross
FROM fm_drivers
WHERE driver_id = '100000';

This will only work when the query is run by the Scheduler or via a Web Service call. If you try and run this in Query Browser / Workbench it will return a 'null' value.
