Overview

If you click on the Attach button, a pop-up will enable you to either Browse to find the files you want to attach, or you can drag and drop the files onto the marked section of the pop-up. The same function is used to import files.

Attach pop-up

[edit]

Attaching multiple files

If you want to attach multiple files, selected them in one go and drag them.
If you drag one file then another, you will keep over-writing the previous one as only one Attach manoeuvre can be done at a time.

[edit]

Adding to an attachment

If you Attach a second (or multiple or third set) of files, these will be appended to the first (or earlier files) and the attachment will now be a zip that contains all the added files.

[edit]

Overwriting an attachment

You cannot delete an attachment, but to get around this, you can attach a zip file by following the Attach process again. This completely overwrites the previous zip.

[edit]

Zip files

If the attach location accepts zip files, they can be in the following formats: .zip, .gz, or .tgz

[edit]

Downloading

You can downloaded and view the attachment file or zip by clicking on the paper clip icon .

[edit]

Notes: field

Notes entered in the "Notes" field may be displayed in the paper clip tooltip. This feature is not available on all screens.

Set-up

A five megabyte total size limit applies to all of the files attached to a record. This can be updated.
Go to Setup / Reference Data → gb_controls "attach_max_file_size" and adjust the value.
You can also check where the attachments are being stored by looking at the "asset_storage_provider" control.

Set-up Notes

Attachments are stored in the table gb_attachments.
The source location of these files is set in [gb_controls]asset_storage_provider.
The default storage is with Amazon, see the March 2015 newsletter for more details.
The maximum file size you can have for a file is controlled by [gb_controls]attach_max_file_size.
