Having successfully logged in, the user can then select which area of the system to go to from the menu options. These menu options are more fully described in the relevant sections of this document.

Navigation throughout the system is via mouse/menu systems and selection of relevant screens. Within the system are a number of Hyperlinks, which the user can use to quickly navigate to relevant areas of the system.

Once navigation commences a history of navigation is kept by the system. The back and forth arrow buttons (located on the top right of screen) can then be used to scroll forwards and backwards through the areas of the system visited. This can be beneficial for going back to certain parts of the system and displaying the data which was being viewed when last visiting that area.

The return button can be used to restore the current screen to a blank or initialised state.

Fleet Structure The fleet system is essentially a database and application consisting of many data tables and application programs. The data tables contain relevant records which are linked to other data tables by the system. When entering data it is recommended to create certain types of data in a logical sequence, for example client record should be setup before a driver record.

Typically data should be created in the following sequence:

  1. Company code (if multi company: contact catch-e to set up)
  2. Client Groups (i.e. clients with branches or sub clients)
  3. Clients (attached to client groups if applicable)
  4. Drivers and Employees (attached to clients)
  5. Suppliers (used for maintenance processing and vehicle suppliers for a contract)
  6. Class Codes (used to catagorise transactions and to set product business rules in contract types)
  7. VMRS Codes (maintenance transaction event codes)
  8. Work Codes (maintenance transaction summary codes)
  9. GL Codes (general ledger codes)
  10. Contract Types (sets product business rules; contact Catch-e to set up)
  11. Contracts (individual assets, e.g. motor vehicles)
  12. Salary Package items
  13. Approvals (authorisations to carry out work, e.g. vehicle servicing)
  14. Approval Posting (convert authorisation to maintenance transaction)
  15. Reg/CTP renewals (process renewals due)
  16. Billing: recharge (creates invoices to clients for recharges)
  17. Billing: initial (first lease rental)
  18. Billing: periodic (regular lease rentals)
