Overview

The "Access" checkbox enables employees to login to the Employee Portal.
This flag is only visible to back office users, Drivers, Employees and Fleet Managers cannot see the field.
Visit Drivers / Contact "Access" for the driver version of this page.

You can use the "Access" checkbox to enable employees to login to;

  1. A limited set of system screens listed in the Package Employee page.
  2. The Employee Portal.
  3. Custom screens delivering your own data screens from your web-designers using a single signon.

Process Pages

Issue an employee login
[edit]

Workflow

A valid email address must be available to activate login access.
A login access can be created for any employee, however the employee's vehicle data will only be visible if Contract Types / Details "Driver Contract Access" is checked.
The Employees / Password tab is only shown to internal users when "Access" is checked.

[edit]
Create a login
  • In View mode, check the "Access" flag to create the login. Either the "Employee ID" or the "Email" will be recorded as the "Driver Login".
  • Once checked, a password will be generated by the system and will be shown in an alert pop-up when created.
  • You will then have the option of sending the employee access email.
  • If you check on "Access" from here, you will see (Package Employee) displayed next to the checkbox. If it was issued from the Drivers / Contact tab, (Driver) will be displayed.
[edit]
Suspend a login
  • Un-check the "Access" flag
[edit]
Re-set a locked user

Users are locked out if their password fails 3 times in succession

  • Un-check the "Accesss" flag and the re-select it
  • You will have the option of sending the employee access email
[edit]
Access Email

Whenever you check the "Access" checkbox, you will be asked if you would like to send the employee an email. If you choose OK, an email will be created and displayed in a pop-up screen for you to review and send.

[edit]
Access Email Event

You can set-up a contract event email to send an access email instead of using the "Access" check box email.
This is useful if you want to provide the driver with contract information in the same email.
If this is done, contract event emails can be created, but will not send until the "Access" check box of the linked driver or employee has been checked.
If "Access" is un-checked, you will be alerted as follows; "Can not send email. Driver login access must be enabled before you can send this email."

To do this, set-up a Contract Event Email with the desired wording using the available Contract Event Email Placeholders.
Enter the Posting Classes / Details "Class code" of the access email event into [gb_controls]driver_access_check_posting_class.

Set-up Notes

[edit]
Driver Login

To use "Email" as the login, set [gb_control]driver_user_login_alias_enabled_flag = 'yes', the default using "Employee ID" is 'no'.
Note that "Employee ID" is always available as a login, even if email is enabled.

See the Employee Portal page for details of how employees can update their password.

For a contract to be visible to an employee, the Contract Types / Details "Driver Contract Access" checkbox must be selected.

[edit]
Email

The email subject and text for the driver access is stored in [gb_templates] using the name 'pdf_employee_login'.
If you have a Channel Portal, a specific email and User Guide should be configured. Create a second record and populate [gb_templates]channel_id.

The email can be configured in plain text or html. The sender can be either hard-coded or can display details of the Client / Contact "A/C Manager".

Note: if "Email" is used as the "Driver Login", you will need to change the login placeholder used on the email, replace the placeholder #driver_id# with #driver_login# in the mail_message.

Note: in some older systems, this email is in code, not gb_templates.

[edit]
Email Attachment

Create or update the [gb_template_attachments] record that is linked to the template above.
If you want to use a channel-specific attachment, create a second record and link it to the channel template it should be used with.

The content of the attachment can be managed through the File Import/Export menu item.

If the [gb_template_attachments]status_flag is set to 'inactive' or there is no attachment record for a template, the email will still send, but there will be no attachment.

Technical Notes about "Driver Login"

Using "Driver ID" as the login is the first method we developed and is in use by many clients. The value is stored in [gb_users]user_login as the login.

Using "Email" is the second method we developed and clients can switch to this anytime. The first email address stored in [fm_drivers]home_email is copied into [gb_users]user_login_alias and is displayed in the "Driver Login" field.

If the users "Email" has already been used as a "Driver Login" the system will revert to using the "Employee ID" as the "Driver Login". Edit and update the "Email" to use a different email address as the "Driver Login".

The login process checks both [gb_users]user_login_alias and [gb_users]user_login for a match, so either login will always work.
