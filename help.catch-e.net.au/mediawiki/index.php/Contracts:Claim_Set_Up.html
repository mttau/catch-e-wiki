Overview

Raise expense records here and attach supporting documents. These records are known interchangeably as approvals and claims.
The Claim and Approval tabs have been designed for different purposes. The sections below summarise their features.

[edit]
Claim

Use the Contracts / Claim tab when

  1. The expense is for a driver reimbursement or a packaging disbursement (the Payment and Banking details are shown)
  2. The expenses are simple and you prefer to enter a GST inclusive amount
  3. You want to use the Funds Check function to check if claims should be posted and reimbursed.
  4. You want to use the autoPostClaims process to manage and post claims where they have previously failed the Funds Check.
[edit]
Approval

Use the Approvals / Approvals tab when

  1. The expense is for a vehicle contract supplier (the Supplier's Payment and Banking details are not shown)
  2. You want to use VMRS Codes and/or Job Numbers
  3. The approval header has a Maintenance and Tyre pop-up to show usage information
  4. Each cost line allows for discount amounts and is entered initially as GST exclusive amounts

[edit]

Contract Type

Contracts / Claim is only visible on contracts where the Contract Type has been set up to allow claims.

  1. Go to Fleet / Controls / Accounts / Contract Types
  2. Select the required Contract Type record to land in the Contract Types / Details tab
  3. Claims tab - check this check box on to enable claims
[edit]

Create a reimbursement supplier

The reimbursement supplier defaults into new claims when they are created.
Only one supplier can be set as a reimbursement supplier.

[edit]

Create a reimbursement interface

If the contract is Salary Packaged - users can select any budget that is stored in the contract's linked Contract Types / Accruals tab

For all other contracts, to be available for selection in the Contracts / Claim tab and the Driver Portal must

  1. Be stored in the linked Contract Types / Expenses Approvals tab
  2. Have a stored reimbursement interface mapping

Catch-e Set Up

To assist your Driver's in making claims, instructional message boxes have been introduced to make this process easier. The content of these messages will be defined by you and supplied to Catch-e to upload into the system accordingly.
These settings only apply to the Contracts / Claims tab. Visit the Driver Portal Setup page for the settings used in the Driver Portal.

These messages are stored in Global Controls as follows;

1. claim_introductory_text appears when the Driver navigates into the Claims tab. Here, you can provide exact instructions on how the Driver can complete their claim and what they are required to have in order to submit it successfully.
2. claim_declaration_text is the second of the three messages that will appear. It occurs when the driver selects . It's expected that the driver will need to declare the information entered is true and correct.
3. claim_saved_text this will advise the Driver that their claim has been submitted. It provides you with an opportunity to explain next steps, such as attaching a supporting file of scanned receipts to the claim.
4 claim_suppress_gst_change_confirmation_flag defaults to 'no'. If an internal user updates the GST value of a claim row, an alert box asked them if they want to change the GST flag recorded for that row. This can be supressed if you change the default to 'yes'. In this case, the flag change is done silently in the background.
5 Refer to Reimbursement Notification page for set up notes regarding sending email notifications.

If you want to set-up a Claims process, contact Catch-e support to update your text information and flags.
