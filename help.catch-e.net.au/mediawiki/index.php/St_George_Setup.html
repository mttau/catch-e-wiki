Overview

This page summarises the various flags and tables within Catch-e that need to be setup and populated with data to enable the St George Interface. Contact your Account Manager for assistance with your St George Interface setup and implementation plan.

[edit]

Supplier record(s)

You need to create at least one Supplier record for St George. Set the Interface Type on the Suppliers Account page to 'st_george'.

Contract Type to St George Product Type Mapping

  The Interface will not be available if this mapping has not been set up for the Contract Type/Term and Effective Date combination in use. This means the ‘Apply’ button will not appear.

This mapping determines
1. St George Product Type for which this application is made.
2. Which XML template to use to build the application as different information are transmitted depending on type of application. An application made for an Individual (Driver) requires different information than that for 'Company' (Client).
3. Where in Catch-E the finance application can be made from.


FieldDescriptionAllowed Value
qt_financier_posting_map_defaults
product_type_codeSt George Product TypeFLE = Finance Lease

OLE = Operating Lease

xml_templateType of St George XML Template to useindividual

company

finance_apply_from_typeWhere the ‘Apply’ button is availableqt_quotes = Quote / Finance Page

fm_contracts = Contract/Finance Page

any = Both Quote and Contract Finance pages

effective_dateEffective Date of this mapping <= qt_quotes.date when finance_apply_from_type = qt_quotes

<= TODAY when finance_apply_from_type = fm_contracts

Originator and Sovereign Client Code

These codes are used by St George Sovereign system to identify and classify an application. These codes are set by St George.

Sovereign documentation defines this as The user in the header segment represents the user 'enrolled' to SOA. Sovereign User ID mapping for Catch-e users

  The interface will not be available if this mapping has not been set up for the logged in user. This means the ‘Apply’ button will not be available.

Each finance application made to St George is owned by a registered Sovereign user. This mapping maps a Catch-e user to a Sovereign user.


FieldDescriptionAllowed Value
gb_user_logins
user_idLinking to Catch-e user id in gb_users Example 10006
serviceService for which this login to be usedst_george
login10 character Sovereign User CodeEach client to get the list of valid users from St George. Example 0020009999
status_flagActive or Inactiveactive or inactive

Useful Queries

SELECT * FROM fm_suppliers WHERE interface_type = 'st_george';

SELECT * FROM gb_controls WHERE name LIKE '%st_george%';

SELECT t2.supplier_code AS 'Financier', t3.posting_map_code AS 'Contract Type', t1.term, t1.product_type_code,
t1.xml_template, t1.finance_apply_from_type, t1.effective_date
FROM qt_financier_posting_map_defaults AS t1
INNER JOIN fm_suppliers AS t2 ON t2.supplier_id = t1.supplier_id
INNER JOIN gl_posting_maps AS t3 ON t3.posting_map_id = t1.posting_map_id
WHERE t2.supplier_code = 'STG';

SELECT * FROM gb_st_george_originators;

SELECT * FROM gb_user_logins WHERE service = 'st_george';
