Overview

Use the Contracts / Events to record particular events of note for the contract.
Visit the Contracts Events page for a summary of Catch-e's standard event records.
Visit the Contracts Events Setup page for details on how to set-up or change contract events.

This tab is also available in the Driver module as Drivers / Vehicle Events.
Events can be reviewed and reported in bulk in the Fleet / Administration / Events / Contracts screen.
Event screens exist for Clients, Quotes, Contracts, Drivers, Employees and Suppliers.

[edit]

Workflow

New events are started by clicking on the New button. Click on Create to save the event.
Use the # check box to select an event to Edit or Delete.
Note that you need to have a role of Manager or above to edit or delete events. Other internal users may edit or delete events if they last edited the event today.

Use the # check box to select an event to Attach a document to. Attached documents can be downloaded for viewing by clicking on the paper clip icon .

Every event can be managed by flagging the required action, setting a due date to be completed by and also giving you the flexibility to mark the event as completed or incomplete, if nothing further is required. Any incomplete tasks are summarised in the KPI Home screen.

Fleet managers can create, edit or delete most events however individual drivers may only edit Log Book events (other events can be displayed to them).
You can control access to events for some external users in the Posting Classes / Detail tab. The "Event Create Flag" allows the user to create and edit the class, whilst the "Event View Flag" limits users to only view the event. If not checked, then users cannot view the event at all.

The database audits all events and records edited and deleted events with your user id and the date and time.

The Events remain in the order that they were created – Date, the Last Edit.

Attaching and Emailing

If there is a need to forward an attachment (such as a Toll or Infringement Notice) with outbound email created with an event this can be easily done by following this workflow:

  1. Select the appropriate event class, enter the appropriate date and value, then 'Create'.
    When the event is created, an email icon appears indicating an email can be sent but has not yet been sent:
  2. Tick on the check box of the event. This will enable the 'Attach' button.
  3. Click on the 'Attach' button and use the 'Browse' pop-up to find and attach the relevant file. The default size of attachment allowed is 5 MB which can be increased or decreased from the Global Control field [gb]attach_max_file_size.
  4. Click on the email icon which will launch the email screen. The email details (recipient, text etc) can be then modified and the email sent.
    Once the email has been sent, the email icon is changed to: which indicates that the email has been sent.
    The icon can be hovered over to see the details of the email or to modify and re-send the email.
  5. Once the user resend the email by click on the send button and closing the browser using the 'X' button can change the status flags in gb_mail as 'pre-save' and in gb_mail_recipients the status as 'pending' respectively

External users (i.e. Fleet Managers and Novated Drivers) can view email events, but do not have the ability to create and send them.

Events per contract can be reported on by selecting the options at the bottom right of screen. All events can be viewed and reported on at the screen found at: Fleet/Administration/Events.

[edit]
My event email shows an \ before the apostrophe

If you look at an event email in the Contracts / Events pop-up, a '\' may be displayed before the apostrophe.
This is a function of the code behaviour of this pop-up. The '\' is not stored in the email and does not display in the Mail Queue or the recipient's email.
To check this, find the email in the Mail Queue and review it there.
