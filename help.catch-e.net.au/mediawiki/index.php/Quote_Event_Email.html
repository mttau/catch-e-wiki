Overview

Create the contract event emails that you want using a standard report format. These can then be used and stored on the Quotes/Events screen.

Download this request form Quote Event Email Request and send it to Catch-e Support if you want to set-up a Quote Event Email.

[edit]

Client Set-up

1. Compose the email details you want, refer to the placeholder table below for the data fields that are available for use in your emails.

  • Sender (this can be either the user or a nominated generic name and email address (E.g. Catch-e Support, support@catch-e.com.au))
  • Recipients (you can have more than one, they can be To:, CC: or BCC:)
  • Subject Line
  • Body Text
  • HTML layout (if required)
  • Static attachments (if required)

Refer to the Quote Event Email Placeholders page for details of placeholders you can use in these emails.

[edit]

Catch-e Set-up

  1. Populate gb_templates with the mail message.
    1. Enter the related posting_class_code into the sub_name field
    2. Populate mail_sender_name and mail_sender_address if the sender details are to be generic, otherwise leave blank and the users details will be applied to the sent emails.
  2. Add the template_id into the selected [qt_quote_event_codes]template_id field
  3. Add the recipients to gb_template_recipients
  4. If attachments are being used, add them to gb_template_attachments
  5. Once the user resend the email by click on the send button and closing the browser using the 'X' button can change the status flags in gb_mail as 'pre-save' and in gb_mail_recipients the status as 'pending' respectively
