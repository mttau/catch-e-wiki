Overview

The Contracts tab displays a table of vehicles used by the driver.

On entering the screen, the table is sorted by assignment (Current/Previous) ascending, then by Start Date descending. Each column can be re-sorted by clicking on the heading (once for ascending, twice for descending order).

Drivers and Package Employees can only view current contracts.

Field Type Description
Contract Table
Contract ID NumA unique system number to identify the vehicle. This is a hyperlink that navigates the user to the Vehicle's Maintenance screen if clicked. Drivers and Employees can only view "Current" vehicles. Clients and Employers can also see "Previous" vehicles, but these are not hyperlinked.
TypeTextThe Contract Type determines budgets available, default GST flags, default Recharge flags and GL codes. Display only on this screen and Contract screen. Refer Billing screen to edit.
Start DateDateRefers to the date the driver started to use the vehicle. If no driver event exists, then the contract start date is displayed if known, otherwise the field will be blank.
End DateDateRefers to the date the driver stopped using the vehicle. If no replacement driver event exists, then the contract end date is displayed if known, otherwise the field will be blank.
DescriptionTextVehicle description. Also shown on the Vehicle screen.
Contract Legend
CurrentTextThe vehicle is active and the driver is the current driver
PreviousTextThe vehicle is active and the driver is not the current driver (i.e someone else is now driving the vehicle)
Current/SuspendedTextThe vehicle has been suspended and the driver is the current driver (i.e. was the last recorded driver of the vehicle)
Previous/SuspendedTextThe vehicle has been suspended and the driver is not the current driver (i.e. was a driver at some time, but was not the last recorded driver)
