Finance Application

Field Type Description [table]field
This section displays details of a submitted Finance Application. These are often done via a web-service (E.g. Macquarie), but can also be entered manually. Once an "Application Reference" exists, only an Admin user can edit these fields, e.g. the information might be deleted so you can re-submit an application.
Originator Drop List Choose an appropriate Originator if one has been set up. Only active originators can be selected. [fm_contracts]originator_id

Application DateDateThe date the application was made to Financier for funds approval [fm_contracts] application_date
StatusTextShows whether the submission of the finance application to the financier via the web-service has failed or whether it was successful [fm_contracts] application_status_flag

Apply / Resubmit Button

Field Description
The Apply button will be available if an interface is available to receive a credit application and the financier has been selected on the quote. We have interfaces set-up for;
  • Macquarie Leasing (Maclease Partner API)
  • BMW (DealerPoint)
  • Bank of Queensland (BOQ)

Visit the pages above to review use and set-up requirements of each interface.

Output Options

Form NameDescription
SettlementThe settlement form can be used to create a variety of documents.
If the contract's "Amount Financed" does not equal the linked quote's "Amount Financed", the user will be warned, but can continue to print the document, or can be navigated back to the quote to settle it. This alert is not invoked for contract's in the 'Fleet Managed' or 'Non Billing' Contract Types / Detail "Group".
Payroll AdviceThe Payroll Advice form can be used to advise employers of each employees salary sacrifice requirements.
DepreciationProduces a Depreciation report based on the current contract.
Screen ReportsIf set up a customised report can be generated from here.


Note that GST applicable on residual value and payout values is defaulted at the current GST rate as located in the table gb_controls. The [Contract Types] Details screen contains the flag which determines whether GST is applicable for the contract type (e.g. for ITC and Residual purposes).

[edit]
I can't edit the Financier field

Contracts / Finance "Financier" becomes a read only field when an active payment schedule is in place.
If you have created a payment schedule for the wrong financier, delete the payment schedule and then update the "Financier".
Process page: Delete a payment schedule
