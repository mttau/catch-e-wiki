Overview

Registration and CTP amounts in the quote are automatically calculated based on the "Delivery State" of the vehicle.
These costs may further defined by using the Clients / Quote Defaults "Reg CTP Type" field. This will bring different values into the quote depending on the setting ('Business' or 'Private'). If configured, these rates will over-ride the "Delivery State" values.
Even more specific rates can be stored by variant/state combination. If configured, these rates will over-ride "Reg CTP Type" and "Delivery State" values.

You can configure the Quotes / Vehicle "Initial Registration" and "Initial CTP" fields to return a '0' amount for 'Used' and 'Demonstrator' vehicles.
You can configure the Quotes / Lease "Annual Registration" and "Annual CTP" fields to return a '0' amount for 'Used' and 'Demonstrator' vehicles.

You can add an uplift percentage to increase the registration budget each year to allow for general cost increases.

Review the Registration calculations to see the various configurations work and also visit the Set-up section of this page.

These calculations are also used in the web service getQuote().

[edit]

Quotes / Vehicle

You can overwrite calculated values at any state. E.g. you are using a dealer quote.

The Reg/CTP values are calculated when a new quote is being created once the State, Client and Variant have been selected.

[edit]

Quotes / Lease

Registration costs are normally calculated from stored table values and settings first.
If the Quote / Vehicle tab "Initial Registration" and "Initial CTP" fields are updated during creation or editing, the values will normally copy here into "Annual Registration" and "Annual CTP" leading to a re-calculation of the "Monthly Budget".
You can edit these values to be different to the Quote / Vehicle tab amounts, but do this last.

Number of Renewals:
Quotes are calculated on the basis that the Quote / Vehicle tab "Initial Registration" and "Initial CTP" are included in the financed amount of the vehicle for 'New' vehicles and the first years registration does not need to be budgeted for. I.e. the calculated "Number of Renewals" is a year less than the term. E.g. a 36 month lease will require 2 registration renewals. Review the Registration calculations for expected results.

The behaviour of your screens may be different to what's described above depending on your system settings. You can change the behaviour of 'Used' and 'Demonstrator' vehicles, which can return a '0' amount for these calculations.

The field name for "Annual Registration" can be renamed.

[edit]

Editing "New or Used"

When a new Quote is created, "New or Used" defaults to 'New'. When "New or Used" is updated to 'Used' or 'Demonstrator':

Initial CTP/Registration on Quotes / Vehicle tab:

When Global Control quote_vehicle_reg_ctp_used_demo_calc_flag is set to 'yes', the Initial CTP/Registration will retain their current values,

When Global Control quote_vehicle_reg_ctp_used_demo_calc_flag is set to 'no', “Initial CTP” and “Initial Registration” values will be set to $0.00.


Annual CTP/Registration on Quotes / Lease tab:

When Global Control quote_lease_reg_ctp_used_demo_calc_flag is set to 'yes',

  • If Initial CTP/Registration > 0 the Initial CTP/Registraion values will be used for the respective Annual CTP/Registration values.
  • If Initial CTP/Registration = 0.00, the same calculations that would normally populate Initial CTP/Registration when New or Used = 'New', will populate Annual CTP/Registration values.

When Global Control quote_lease_reg_ctp_used_demo_calc_flag is set to 'no', “Annual CTP” and “Annual Registration” values will be set to $0.00.

When "New or Used" is updated from 'Used' or 'Demonstrator' to 'New', the system will keep the current values for Initial CTP/Registration and Annual CTP/Registration unless the current values are $0.00 in which case they will be recalculated.


[edit]

Editing "Variant", "Delivery State", "Effective Date or "Client"

An option will be given to recalculate Initial CTP/Registration and Annual CTP/Registration values when a "Variant", "Delivery State", "Effective Date or "Client" is changed.

Set-up Notes

Table Comment
[qt_state_reg_ctp]Set a default values by state here. This table contains a date field, so you can add new values when price changes occur. Quotes will only reliably return Registration and CTP values if a state-only record (i.e. where "type" = NULL) exists, so set these up first.
[qt_state_reg_ctp]If you want to set values for business and private use, set default values by state/type here. You can have a mixture of state only and state/type values in the table.
[qt_variant_state_reg_ctp]Store values here for more sensitive costings.
[gb_screen_aliases]The field Quotes / Lease "Annual Registration" can be renamed here.
