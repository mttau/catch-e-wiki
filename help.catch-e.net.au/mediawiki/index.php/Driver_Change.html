When Setting up a new Contract

A 'Driver Change' event is required
When a contract is created from a quote, the date value is blank. Any subsequent 'Driver Change' events created will use the system date by default, unless overwritten before save.
Put the delivery date in as the event date (for correct reporting and FBT calculations).

[edit]
If the Driver is new

Create the Driver in the Drivers / Contact tab.
Then a 'Driver Change' event can be created

[edit]
Pool Vehicles

If the vehicle is a Pool vehicle, we recommend a "Pool" driver be created. This enables a custodian to be allocated and the usage of the vehicle to be clear.
Be consistent in how 'Pool" vehicles are described, so that searching and report presentation is clear. E.g. Surname = Pool, Given Name = Client or Area (Surname is the default "Search" field so this will be fastest for locating Pool vehicles).
Put the custodians email address in (this enables email events to be sent to the Custodian).
