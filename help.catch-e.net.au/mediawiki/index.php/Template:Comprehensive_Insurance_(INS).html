Include? Check Box Select if you want to add or calcuate a Comprehensive Insurance budget. This was likely selected on the Quotes / Contact tab.qt_quote_insurance]included_flag
Policy OwnerListSelect who will be responsible for managing and paying for the Comprehensive Insurance of this contract.
A default entry may come from the Clients / Quote Defaults "Policy Owner" field if recorded, otherwise it will default to 'Driver Policy'.
This choice carries into the Contracts / Insurance "Policy Owner" field when a quote is accepted. 		[qt_quotes]policy_owner_type
InsurerListSelect an insurer for this vehicle. A default entry may come from the Clients / Quote Defaults "Insurer" field.
This choice carries into the Contracts / Insurance "Insurer" field when a quote is accepted.		[qt_quotes]supplier_id_insurer
LocationListSelect a Location or a Postcode to complete the insurance premium calculation.
The choice is comes from your system's insurance set-up.		[qt_quotes]insurer_location_id or [qt_quotes]insurer_location_postcode
Billing Type Drop List This defaults to 'Monthly'. 'Annually', 'Reimbursement' and 'Excluded' can also be selected.

When a quote is accepted AND "Policy Owner" is 'Internal Policy' THEN Contracts / Insurance "Billing Frequency" will be set as
'Annually' if this field is set as 'Annually', or 'Monthly' for all other selections.

[qt_quote_insurance]billing_type
Ins Type List New quotes will default the "Insurance Type" stored on the variant profile into this field. Go to the Setup / Reference Data Variant Profile record to update the default.

This value is used as a factor to calculate insurance premiums and carries into the Contracts / Insurance "Insurance Category" field when a quote is accepted.

When a variant does not have an insurance type recorded a warning is shown. Hover over the icon to see the tool tip message "Warning: No Insurance Type on Profile".
An insurance type can be added from the list contained in the profile window in order to calculate an insurance premium, but the warning will remain until an insurance type is added to the vehicle profile.

[qt_quote_insurance]insurance_type_id
Claim Excess Num This defaults from the [qt_insurance_types] table and can be edited.
This choice carries into the Contracts / Insurance "Claim Excess" field when a quote is accepted. 		[qt_quote_insurance]claim_excess
Annual Insurance Lock Check Box Check this to lock the amount entered or calculated into the "Annual Ins" field. Once locked, the amount remains fixed.
However, if one of the insurance variables gets updated, the lock will un-check automatically and "Annual Ins" will re-calculate. 		[qt_quotes]lock_comprehensive_insurance_flag
Annual Ins Number The annual insurance amount is calculated by the system if this is configured. This amount can be edited.
This amount will respect the upper and lower limit amounts in the set up to adjust accordingly if the value set in the control comprehensive_insurance_basis is adjusted into a different insurance bracket.
If one of the insurance variables gets updated, the annual insurance amount will be re-calculated.
The insurance variables are "Effective Date", "Term", "Client Code", "Location", "Variant" and "Amount Financed".		[qt_quote_insurance]annual_insurance
Fund 1st Year? Check Box If checked, the 1st year's insurance is included in the Quotes / Finance Broker "Comprehensive Insurance" field.
The "Fund 1st Year?" checkbox default is a Contract Type setting. 		[qt_quote_insurance]fund_1st_year_flag
