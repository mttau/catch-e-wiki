glpl_export_finance_reporting_flag This control is only applicable when
  1. you are exporting transactions to Xero
  2. you are using the "Payment Type" 'Finance - Principal and Interest'.

When the flag is set to ‘no’

  1. The export file will itemise the classes ‘PRI' (Principal) and 'INT’ (Interest). The class 'FIN' (Finance) is ignored.
  2. Contract Types / Payments "G" must be set to 'N' for INT, PRI and FIN for the "Type" 'Finance - Principal And Interest'
  3. Contracts /Payments - when a 'Finance - Principal And Interest' payment is saved, the GST setting is checked. If the 'FIN' class GST is populated, the record cannot be saved. You will see a validation message.

Where the flag is set to ‘yes’

  1. The export file will only include the ‘FIN’ class and if configured, will include GST.

gl xero_export_sl_invoice_date_for_pay_run Use this control to change the date that is used in sales ledger exports using MYOB Purchases and Xero Purchases export files.
This control only applies to pay run invoice transactions.

invoice_date - the export file will use [gl_transactions]date to show as InvoiceDate (this is the same date as [fm_billing]invoice_date)

posted_date - the export file will use [gl_transactions]posted_date to show as InvoiceDate (this is the same date as [fm_billing]posted_date)

invoice_date

gl xero_pl_export_invoice_date_for_reimbursement Use this control to change the date that is used in Xero Purchases export files.
This control only applies to claim reimbursements. I.e. where Contracts / Claim "Reimbursement" is checked.
Using 'posted_date' can be helpful if you close out accounting periods in Xero and older claims have been recently processed.

invoice_date - the export file will use [fm_maintenance]invoice_date as the InvoiceDate

posted_date - the export file will use [gl_transactions]posted_date as the InvoiceDate
