Field Descriptions

Screen NameDescriptionType[table]field
Column One Fields
SupplierSearchUser selects supplier from the search screen or by entering supplier code directly
DescriptionText (40)Vehicle description. Includes a Search icon in edit mode to select a variant from the Setup/Reference Data - Variants data. Search fields include Make, Model, Model Year, Body Type and Variant. When Saved the selected variant populates the screen with the standard variant data. The variant_id is populated in fm_contracts for us in functionality such as Approvals.
Model CodeText Model code identifier
ClassificationText Asset classification. this field is included on the Text(CSV) version of the inventory report
ColourText Vehicle exterior colour
Reg Managed/ExpiryTick /DateReg Managed tick box indicates the contract will appear on reg renewal reports. Registration renewal date is the first date entered by the user, subsequent dates are updated via Reg/CTP Batch entry.
The registration date of HO (History Only) vehicles can be edited by external fleet managers. Refer notes below*.
Rego StateListThe state which the vehicle is registered in. This field is used on the Registration Renewals Report. The state field on the Contract screen is used to identify the state location of the vehicle
DescriptionText (40)Vehicle description
New/UsedText Single character reference, e.g. N = New, U = Used
On Road CostNumTotal vehicle cost including GST, Registration, Purchase Duty
List PriceNumManufacturer's list price
FBT ValueNumOn Road Cost less reg/CTP/purchase duty
Start OdometerNumOdometer reading at delivery. If zero is required enter ".1" as "0" is ignored by FBT reports.
Start Odometer DateDateEffective date of odometer reading. Refer note in approvals section for full explanation of use in calculating estimated kilometres
Warranty KlmsNumUser enters warranty kilometre limit Can be numeric or alpha. Note the import function accepts only numeric or the text 'unlimited'
Warranty MthsNumUser enters warranty months limit
Warranty ExpiryNumUser enters warranty expiry date
LCT Exempt?Checkbox
1link Export FlagCheckboxIndicates that this contract will be exported to 1Link. Defaults to 'yes' when a new contract is created. This field is only editable if 1link Export Flag on the Contract Type is set to 'yes'. It is shown ticked/unticked as per its current value. But if this flag is set to 'no' on the Contract Type then it is a display only field with the checkbox showing as unticked.
Roadside ManagedTickRoadside Managed tick box indicates the contract will appear on Roadside Assistance reports.
Roadside SupplierLookupEnter or lookup the Supplier code for the roadside asswith the istance provider for this contract. If using quotes, this field will be updated with the provider used in the linked quote.
TermNumEnter the agreed term in months for roadside assistance.
Start DateDateEnter the start date of the roadside assistance provision.
End DateDateEnter the end date of the roadside assistance provision.
Approval LimitNumEnter the maximum amount allowed for Maintenance Approvals to be accepted, prior to client authorisation required. Maximum number accepted is 9,999.99 and this is the default when an unsupported number is entered[fm_contracts]approval_limit
Column Two Fields
Supplier RefText (10)Reference number such as supplier's invoice number
Build DateText (7)Build Date of vehicle in YYYY-MM format
Compliance DateText (7)Compliance date of vehicle in YYYY-MM format
Key NoText (20)Ignition key number
VINText (20)Vehicle Identification Number (17 characters typically but not for certain equipment and vehicles pre 1980). The user is warned if the VIN is not 17 characters long but the record can still be saved.
EngineText (20)Vehicle Engine Number
Man. Model CodeText (15)Manufacturer code number - supplied by manufacturer
Radio PinText (10)Radio PIN code
Fuel TypeListDrop down list of fuel types, e.g. ULP, Diesel, PULP, etc.
CO2 Emissionsmediumint(8) Carbon-dioxide emission value in g/km.
Fuel CapacityNum (10) Fuel tank capacity (litres). If using quotes, the "Tank Capacity" of the selected "Variant" is copied into accepted contracts, but this can be edited. The Variant's tank capacity can be viewed in the Specifications section of the Quotes/Vehicle screen.[fm_contracts]fuel_capacity
Fuel Consumption Upper Limit decimal(9,2) The field is auto-populated when Quotes is accepted with its vehicle "Fuel Metro" value. It is used for comparing expected L/100 consumption with actual L/100 consumption in Operating Cost and Fuel Service reports.
Fuel Consumption Lower Limit decimal(9,2) The field is auto-populated when Quotes is accepted with its vehicle "Fuel Country" value. It is used for comparing expected L/100 consumption with actual L/100 consumption in Operating Cost and Fuel Service reports.
Fuel Card PIN Text (8)Fuel card PIN number, if known
Service Interval (Kms)NumManufacturer intervals between services. Use in service due report
Service Interval (Mths)NumManufacturer intervals between services. Use in service due report
$/1000 kms decimal(9,2) It is used for comparing expected cost per 1000 kms with the actual in Operating Cost report.
Vehicle TypeListDrop down list of vehicle types, e.g. sedan, wagon etc.
Body ConfigurationListDrop down list of body configurations, e.g. Dual Cab, Hi Roof, Low Roof etc.
TransmissionTextTransmission type, e.g. Automatic, Manual etc.
Drive TypeListDrop down list of drive types, e.g. 4WD, Front, Rear etc. When included against the variant_id, this field is automatically updated on contract creation from the quote.[fm_contracts]drive_type
ANCAP RatingNumAncap Star Rating - 0-5[fm_contracts]ancap_rating
Column Three Fields
Manufacturer CodeText (4)Short Manufacturer code, e.g. 'HOLD'
Manufacturer NameText (40)Long Name, e.g. 'Holden"
Model NameText (30)Model or Family name, e.g. 'Commodore'
Variant NameText (20)Variant Name, E.g. 'Executive'
Red Book CodeText (10)Red Book Reference number
NVIC CodeText (10)If the Swann interface is enabled, "NVIC Code" will be an editable field with a lookup icon. If the Swann Insurance product "Extended Warranty" is included in the contract, you will need to fill this field in. Refer to Swann Interface - Extended Warranty for full details.
Engine CapacityText (10)E.g. 3.0 Litre
CylindersNum (3)E.g. 4
Tyre CodeText (15)Reference to tyre code (user defined)
Tyre NameText (20)Common tyre name
Tyre SpecText (15)E.g. 20570R16 V80
Tyres Per SetNum (3)E.g. 4
Total OptionsNum Value of options fitted to the vehicle
Interior ColourText (20)Vehicle's interior colour
DoorsListDrop down list 0-5
Pink Slip RequiredTick Boxindicates that this contract will require a pinkslip
Pink Slip ReceivedTick Boxindicates that this contract has received a pinkslip, this becomes unticked when a registration payment is processed for this contract
Rego Papers ReceivedTick Boxindicates that this contract has recieved a registration label, this becomes unticked when a registration payment is processed for this contract
CTP NumberTextCompulsory Third Party Insurance Policy Number.[fm_contracts]ctp_number]
CTP Expiry DateDateEnter the date the CTP Insurance Policy is due.[fm_contracts]ctp_expiry_date
Reg Account NumberTextEnter the Registration Account Number the client is listed under, with the relevant State Authority.[fm_contracts]state_authority_account_number
Reg Vehicle ReferenceTextEnter the Vehicle Reference number allocated by the issuing State Authority, if applicable.[fm_contracts]state_authority_vehicle_ref_number
