Overview

Pay Cycle Billing calculations are applied to quotes when the Quotes / Contract "Pay Cycle" is not 'Monthly'. e.g. 'Fortnightly' or 'Weekly'.

When a quote is accepted or settled, the Contracts / Billing "Periodic Interval" is set to 'Monthly' and the budget records will contain monthly values, or it can use values calculated from the quote's "Pay Cycle". This is determined by the settings in the client record.

A contract with monthly billing values can still be edited and changed away from 'Monthly' if needed later.

[edit]

Quotes

  1. The Quotes / Contracts "Pay Cycle" defaults in from the driver's record
  2. If no driver has been selected, or their "Pay Cycle" is blank, the Clients / Quote Defaults "Pay Cycle" is used
  3. If the "Pay Cycle" field is left blank, the Clients / Contract Defaults "Billing Interval" (which is 'Monthly') will be applied.

This field can be edited and the list of available pay cycles comes from the table [gb_pay_cycles].
If you create, copy, or edit a quote and the quote's "Pay Cycle" does not match the driver's "Pay Cycle", you will be warned when saving.

[edit]

Accepting or Settling quotes

  1. If the Clients / Contract Defaults "Contract Pay Cycle From Quote" is checked AND
  2. The selected Quotes / Contract "Pay Cycle" is set with gb_pay_cycles]billing_flag = 'yes' THEN
  3. The Quotes / Contract "Pay Cycle" will be used to populate the contract when a quote is accepted or settled
    1. the Contracts / Billing "Billing Interval" and "Total Billings" are created to match the calculated quote values.
    2. the Contracts / Lease budget amounts are calculated to match the selected pay cycle. See the section below for the calculation that is used.
  4. If Clients / Contract Defaults "Contract Pay Cycle From Quote" is unchecked, or gb_pay_cycles]billing_flag = 'no'
    1. The contract's billing values are created using monthly values.

Roundings for calculations

Values being calculated for "Total Billings" or for budget amounts are rounded by the factor that is stored in the Global Control "pay_cycle_billing_rounding_method".
Values may be 'floor','ceil','round' or 'none'. 'none' is not recommended.

[edit]

New Contract Recalculation (editing "Months Deferred")

If the contract has not yet invoiced, editing the "Months Deferred" value will re-calculate the pending budgets.

Example calculation;

  • "Lease Term" is '36'
  • "Periodic Interval" is 'Monthly'
  • "Months Deferred" is being changed from '0' to '2'
  • "Total Billings" is '36'


DescriptionCalculationExample
Calculate the new Total BillingsTerm - Months Deferred36 - 2 = 34
Calculate the existing Finance budget LTD TotalFIN Budget * Existing Total Billings$ 1,000.00 * 36 = $ 36,000.00 FIN LTD Budget
Calculate a new monthly BudgetROUND ( FIN LTD Total / New Total Billings , 2 )$ 36,000.00 / 34 = $ 1058.823529 FIN (Monthly raw)
Apply pay_cycle_billing_rounding_methodROUND ( FIN , 2 )ROUND ( 1058.823529, 2 ) = $ 1,058.82 FIN (Monthly rounded)

New Contract Recalculation (editing "Periodic Interval")

If the contract has not yet invoiced, editing the "Periodic Interval" value will update the contract and re-calculate the pending budgets.

Example calculation;

  • "Lease Term" is '36'
  • "Periodic Interval" is being changed from 'Monthly' to 'Fortnightly'
  • "Months Deferred" is '2'
  • "Total Billings" is '34'


DescriptionCalculationExample
Calculate the existing Total BillingsTerm - Months Deferred36 - 2 = 34 Total Billings (Monthly)
Convert the existing Total Billings into YearsTotal Billings (Monthly) / 12 34 / 12 = 2.83333 (Years)
Multiply by the new Periodic IntervalYears * Periodic Interval (Fortnightly) = Total Billings (Fortnightly) 2.83333 * 26 = 73.66667 (Fortnightly raw)
Apply pay_cycle_billing_rounding_method ROUND (Total Billings (Fortnightly), 0) ROUND (73.66667, 0) = 74 Total Billings (Fortnightly rounded)

New Contract Recalculation (editing "Periodic Interval" and "Months Deferred")

If the contract has not yet invoiced, you can edit both the "Periodic Interval" and "Months Deferred" fields to re-calculate the pending budgets.
However, changing "Months Deferred" requires the total budget value to be re-calculated before the "Periodic Interval" re-calculation can be performed.

Example calculation;

  • "Lease Term" is '36'
  • "Periodic Interval" is being changed from 'Monthly' to 'Fortnightly'
  • "Months Deferred" is being changed from '0' to '2'
  • "Total Billings" is '36'


DescriptionCalculationExample
Calculate the new Total BillingsTerm - Months Deferred36 - 2 = 36
Calculate the existing Finance budget LTD TotalFIN Budget * Existing Total Billings$ 1,000.00 * 36 = $ 36,000.00 FIN LTD Budget
Calculate a new interval BudgetFIN LTD Total / New Total Billings$ 36,000.00 / 74 = $ 486.4864865 FIN (Fortnightly raw)
Apply pay_cycle_billing_rounding_methodROUND ( FIN , 2 )ROUND (486.4864865, 2) = $ 486.49 FIN (Fortnightly rounded)

Existing Contract Recalculation (editing "Periodic Interval")

If the contract has commenced billing, changing the "Periodic Interval" to another interval will not recalculate the Budgets, but it will calculate the new "Total Billings" and "Remaining".

Example calculation;

  • "Lease Term" is '36'
  • "Periodic Interval" is being changed from 'Monthly' to 'Fortnightly'
  • "Months Deferred" is '2'
  • "Total Billings" is '34'
  • "Remaining" is '30' (4 months have been invoiced)


DescriptionCalculationExample
Convert the Billed Periods into Billed Periods (Fortnightly)Billed Periods / 12 * Periodic Interval(fortnightly) = Billed Periods (Fortnightly)4 / 12 * 26 = 8.66667
Apply pay_cycle_billing_rounding_method ROUND(Billed Interval Periods,2)ROUND(21.66667, 2) = 9 Billed Periods (Fortnightly)
Calculate Total BillingsTotal Billing(Fortnightly unmodified) - Billed Periods (Fortnightly) + Billed Periods (Monthly) 74 - 9 + 4 = 69 Total Billings (Fortnightly modified)
RemainingModified Total Billings - Monthly Billed Periods 69 - 4 = 65 Remaining

Timing and Data Entry Issues

If you change the rounding method you use in the system to calculate Billing Periods, timing issues can arise.

1. Quote rounding is applied based on quote effective date and effective date of [gb_control]pay_cycle_billing_rounding_method
2. Contract rounding is applied based on system date and effective date of [gb_control]pay_cycle_billing_rounding_method (used when setting up a pay cycle billing schedule)

This means an inconsistency can occur if quotes raised before the changeover are accepted and then have pay cycle billing applied (pre-change quotes that are set-up as monthly contracts are unaffected).
The outcome would be that the budget values/Lease Rate could potentially not match what was quoted on the Quote (i.e. 48 months 2 months deferred = 199.33 pds (none), 199 pds (floor), 200 pds (ceil) and thus different answers would result).

"Term" and "Months Deferred" are used to calculate "Total Billings". This is the number that is used to derive any budget recalculations. If you edit "Periodic Billings" directly, any budget recalculations will not match.

Set up Notes

  1. Check the list of records in the [gb_pay_cycles] table, these are all available for selection in the Quotes / Contracts "Pay Cycle" list box. Remove any that you do not want to use.
  2. Check which pay cycles you want to make available for billing.
    1. Set [gb_pay_cycles]billing_flag as 'yes' for these.
    2. Only these records can be selected in the Contracts / Billing "Periodic Interval" field listbox.
  3. Additional pay cycles can be added to [gb_pay_cycles] if needed, making sure [gb_pay_cycles]billing_flag is set to 'yes' to make them available in the "Pay Cycle" list box.
    1. Other fields in the gb_pay_cycles table will need to be updated to complete the set up. i.e.: number of pay cycles and/or tax cycles needed per year.
    2. They will still respect other settings when set up.
  4. If you are using Driver Reports, check that your selected pay cycles are available for driver reports.
    1. Driver Reports currently support the use of Weekly, Fortnightly ('fortnightly' & 'fortnightly2'), Monthly, 4 Weekly, Quarterly and Annual pay cycles.
    2. Bi-monthly and semi-annual pay cycles are not supported.
    3. For newly added pay cycles, please discuss this requirement with your Account Manager.
  5. If you want to use a second fortnightly choice to your system, 'fortnightly2' needs to be added to the gb_pay_cycles table to enable it's use.
    1. The calculations are identical to the 'fortnightly' record.
    2. This allows you to identify fortnightly contracts with different billing parameters.
  6. Check the Clients / Quote Defaults "Pay Cycle" defaults as required.
    1. The standard default is set as 'monthly', but can be updated in Setup / Reference Data → gb_controls "fm_clients_pay_cycle_default".
  7. Check the setting of Clients / Contract Defaults "Contract Pay Cycle From Quote" defaults as required.
    1. This is set to default as 'no' by default, but can be updated in Setup / Reference Data → gb_controls "fm_clients_contract_pay_cycle_from_quote_flag_default".
    2. Setting this to 'yes' brings the pay cycle of accepted quotes across to the new contract for billing.

Global Controls

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gbpay_cycle_billing_rounding_method This control is used to specify the rounding calculation that is to be applied to the calculation of [qt_quotes]pay_cycle_billing_periods when a rounding method is required. Values may be 'floor','ceil','round' or 'none'.

We do not recommend setting this to 'none' as this creates rounding issues in budget calculations. Use either 'floor', 'round' or 'ceil'. This can be updated at a particular effective_date if needed.

This control is also when recalculating Contracts / Billings "Total Billings" if "Periodic Interval" is changed away from a 'Monthly' setting.
Visit Pay Cycle Billing for more details.

'floor':ROUNDDOWN (billing_periods/12* pay_cycles_per_year) = pay_cycle_billing_periods e.g. 19-2 = 17 (months) 17/12*26= 36.83333 =36 (rounded fortnights).

'ceil':ROUNDUP (billing_periods/12* pay_cycles_per_year) = pay_cycle_billing_periods

'round':ROUND (billing_periods/12* pay_cycles_per_year, 0) = pay_cycle_billing_periods

'none': billing_periods/12* pay_cycles_per_year = pay_cycle_billing_periods to 5 decimal places

Useful Queries

List the Pay Cycles that can be used in your system when Clients / Contract Defaults "Contract Pay Cycle From Quote" is checked. 

SELECT pay_cycle_code AS billing_interval,pay_cycles_per_year 
FROM gb_pay_cycles
WHERE billing_flag = 'yes'
ORDER BY pay_cycles_per_year DESC
;
