Overview

Catch-e runs a number of system environments for you, their purpose and use are explained below.
See also the page Mail Send Block Flag for details of the emailing treatment on these environments.

[edit]

Primary Environments

LIVE, SUPPORT, STAGING and TEST are all available for you to use and example URL's are shown.

LIVE is available to users 24 hours a day.
STAGING and TEST are available 6am - 10pm EST.

TEST is only available if you have a VPN Connection to Catch-e running.

[edit]

URLS

Live: https://CLIENTNAME.catch-e.net.au/
Support: https://CLIENTNAME.support.catch-e.net.au/
Staging: https://CLIENTNAME.staging.catch-e.net.au/
Test: https://CLIENTNAME.test.catch-e.net.au/
[edit]

Logins

[edit]

Live

Live is available to you 24 hours a day.
Your Live environment has scheduler jobs and email functions working.
Use the Password Reset feature in this environment if you need to reset your password.

[edit]

Support

Your Support environment is an environment where a copy of your Live data has been copied into each night.
This means it has the same login credentials as Live, only the URL to access the environment is different.

The Scheduler and the Emailing functions do not run in Support
If you are testing in Support, you will need to

  1. run any jobs you need to see working manually
  2. go to the Mail Queue to see the emails have generated.
[edit]

Staging

Staging is available 6am - 10pm EST.
Staging has it's own login credentials, but you can maintain them to be the same as Live if desired.
If you have trouble when logging into Staging, it's possible that Staging has been refreshed from Live or Support data

  1. So try your Live login details and see if they work before escalating.
[edit]

Test

Test is available 6am - 10pm EST.
Test is only available if you have a VPN Connection to Catch-e running.

[edit]

Password Reset

  1. In Live, use the Reset Password process to reset your password if it has been forgotten.
  2. The Reset Password process will not work in other environments, as they do not send emails
  3. For Support, Staging or Test, ask one of your System Administrators to check and update your password for you.
  4. If you are locked out, ask one of your System Administrators to unlock your account.
[edit]

Emails

  1. Except for specific short testing windows, emails will only generate from the LIVE environment
  2. Go to the Email UAT Checklist page for suggested UAT steps for new or changed emails.
[edit]

Scheduler

  1. Except for specific short testing windows, scheduler jobs only run in the LIVE and TEST environments


[edit]
LIVE

This is your main operating environment where business is transacted and where users spend most of their time.

The url path for LIVE is as follows (i.e. replace "demo" with your own client name); 

https://demo.catch-e.net.au/bo.phpo?
[edit]
SUPPORT

This environment is a nightly copy of your LIVE data. It is available to your staff and Catch-e support as a "playground" to try things in a environment that is the same as LIVE, without risk to your operating data.

Your login to this environment are the same as LIVE (because it is a copy).

The url path for support is as follows (i.e. replace "demo" with your own client name); 

https://demo.support.catch-e.net.au/bo.phpo?
[edit]
STAGING

This environment is a 'point in time' copy of your LIVE data. It is available to your staff as an environment that is the similar as LIVE, and can be used for testing changes that may be more meaningful using "Live" data, without risk to your operating data.

Your login to this environment are the same as LIVE (because it is a copy). However if you change your password in LIVE that change does not carry over to STAGING

The url path for support is as follows (i.e. replace "demo" with your own client name); 

https://demo.staging.catch-e.net.au/bo.phpo?
[edit]
TEST

This environment provides a test system. The data is not copied or kept up to date from other databases. It is available to your staff and Catch-e to test new developments and changes that are not yet in LIVE. This is where we ask you to test new features in development and it is also used heavily during implementation processes.

To access TEST, you need a VPN Connection. Your Account Manager can help you set this up.

Your login to this environment is separate to LIVE, and may be different. Contact Catch-e support if you need a login set-up or reset for TEST.

The url path for TEST is as follows (i.e. replace "demo" with your own client name); 

https://demo.test.catch-e.net.au/bo.phpo?
[edit]

Ancillary Environments

[edit]
Disaster Recovery (DR)

This environment is a continuous copy of your Live system, so it always contains the same information. If there are any system problems in the LIVE environment, you will be moved onto DR until it is resolved.
Your login to this environment are the same as LIVE (because it is a copy).
The DR system is not available to users unless switched over by Catch-e staff during an outage.

[edit]
Other Environments

We also run other environments to provide additional services such as SNAPSHOT, REPORTING and PRINT (renders PDFs files and processes graphics).

The SNAPSHOT environment contains a daily snapshot of your database that you can copy to your own servers. Some clients do this to run their own reporting off-site, and as a risk reduction strategy. Contact Catch-e support if you would like to set this up.
