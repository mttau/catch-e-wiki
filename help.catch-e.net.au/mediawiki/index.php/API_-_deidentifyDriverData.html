[edit]

Overview

Allows you to remove personal details from Driver records where they are no longer active or linked to active Quotes, Contracts and/or Items.

[edit]

Permissions

This API is restricted to Catch-e internal 'super' users.
Once the data has been de-identified, it is no longer recoverable. For this reason, access to the API is restricted and is only accessible via a Scheduler job.


[edit]

HTTP Method

Use the HTTP Method 'POST' for consuming this web service.


[edit]

Input Fields (Body)

JSON Field Format Notes Mandatory
driver_classification_query_idStringSQL query_id used to identify applicable drivers for deidentification purposes.

The specified query id should return a driver_id column, identifying the drivers the de-identification process should apply to.

Yes
deidentifcation_methodStringDe-identification method to apply. Default value is 'expunge'. The related Scheduler Job has this fixed to 'wildcard-substitution'. Other methods include,
  • obfuscate
  • wildcard-substitution
  • wildcard-substitution-with-suffix
Yes
wildcard_substitution_characterStringSubstitution character to be used when method is set to wildcard-substitution or wildcard-substitution-with-suffix.

Defaults to asterisk (*). Other substitution options include,

  • Blank
  • Hash
  • Randomised
No
Audit-User-Id string Pass a user_id here to create audit records with this user's details.
This can only be done if the authenticated API user has Roles / Apis checked for the permission Audit:UserIdOverride 		No
