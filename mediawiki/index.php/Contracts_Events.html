Event Descriptions

Class Code Class Name Type Description
AAccident When an accident is created, a history-only maintenance record is created.
This allows the accident to appear on the Accidents Report.
Note that an amount can be recorded in the event screen, but it will not be shown in the maintenance screen or the Accidents Report.
Editing the event description will not update the maintenance description as there is no link between the event id and the maintenance id.
COCCertificate of Currency This event creates an email to the Driver with a Certificate of Currency document or similar attached.
This email may also contain supplementary static documents and a further document may be added to the email by using the "Attach" function.

You can also set-up channel specific emails that contain a customised coc.pdf attachment.

CCCost Centre Change Create a new event to record when the vehicle is assigned to a new Cost Centre.
Only Clients / Cost Centres records can be entered or selected. Use the ?Search function to find and select available Cost Centres.

The "Date" of the event should be when the vehicle changed to the new cost centre. This date is used to select the correct cost centre for date-driven reports.
The "Allocation(%)" field will default to '100.00'. Leave this at '100.00' unless you are using multiple cost centres.

A vehicle can be allocated to more than one cost centre. If multiple cost centres are recorded, the Cost Allocation' Report will assign the vehicles costs across the cost centres.
Entere each cost centre with the same "Date". Enter the required percentage of that cost centre in the field "Allocation(%)" field (this can be to 2 decimal places).
The total of the "Allocation(%)" amounts should be 100.00.
The last cost centre entered is the one shown in the Contracts / Contract tab and this one will be displayed when sorting or grouping general reports and invoices.

DIV Division Change Use this event type to assign or re-assign the contract to a division. Only active divisions stored in the Clients / Divisions tab can be selected.
You can enter the division name directly into the "Value" field or use the function to search for it.
If you enter a division name that doesn't match the available list and tab off the field, the selection pop-up window will launch.
The current division is shown on the Contracts / Contract tab.
If a contract is copied, the "Division" field on the new contract is blank and no event record is created.

A division can be added to quotes on the Quotes / Contract tab and this event is created when the quote is accepted.

DRIVERDriver Change The driver’s surname name may be entered in the 'Driver Name' field and will be completed when the user tabs off the field. A partially entered name will be auto populated if it is unique within the client code or client group, if it isn't unique a pop-up search box will appear.

The Driver Name can also be found by entering ? followed by the Tab key. See Search for more details of this feature.

When a new driver name is entered as taking possession of the vehicle from the entered date, an odometer reading is typically entered for FBT purposes. If an Odometer value is entered, the reading is validated and a maintenance ODO record is created when the event is saved.

- the effective date should be entered in the 'Date' field to ensure driver tracking for reportable benefits is calculated accurately. Blank dates will cause the FBT Per Driver and FBT Per Vehicle reports to show 'Unknown' as the driver. New contracts should use the contract start date.

PORTAL Driver Claim Portal These email events are created when a claim has been created in the Driver Portal and an email confirmation has been sent.
This is an optional Driver Portal setting.
ECM Employee Contributions If employees make after tax contributions to the running cost of the vehicle the amounts can be entered as an event.
This will populate the ECM field on the FBT screen for the relevant FBT year
EVCHARGE EV Charging Method Select a "Value" to record which claim method should be used in the Driver Portal for Fuel / Charging.
Choose from 'Actual Cost', 'Home Charging' or 'Home Charging Percentage'.
Visit the EV Charging Method illustrations page for illustrations of how various settings affect the Driver Portal screens.

Visit the Driver Portal Setup page for the required EVCHARGE settings.
FBTAFBT Declaration Advice EmailCreates an email message to the Driver advising them of an impeding requirement to furnish an FBT reading. See Fbt Declaration Email for full details of the FBT emailing process. The Drivers email address is checked when an email is created to detect missing or malformed addresses. If this occurs, the user is given an alert message "Driver email invalid!". The email address must be corrected before the email can be sent.

- An FBTA Contract Event can be created and sent even if the Contract FBT Email flag is set to 'No'. This flag only restricts the bulk email reporting process.

FBTDFBT Declaration EmailCreates an email message that contains a one-time URL to the Declarations page of the Contract. This enables the Driver to furnish an FBT reading without needing a login or password. The URL link is not "active" whilst in the event screen tool-tip, but will be active in the email when sent or re-sent. See Fbt Declaration Email for full details of the FBT emailing process. The Drivers email address is checked when an email is created to detect missing or malformed addresses. If this occurs, the user is given an alert message "Driver email invalid!". The email address must be corrected before the email can be sent.

- An FBTD Contract Event can be created and sent even if the Contract FBT Email flag is set to 'No'. This flag only restricts the bulk email reporting process.

FBTCFBT Confirmation EmailThis email is automatically generated when a Driver has lodged an FBT Odometer reading via an FBTD URL link. It contains the Odometer reading and the Date. See Fbt Declaration Email for full details of the FBT emailing process.
FCFuel Card Sent This event records the date the fuel card was forwarded and creates an advice email that can be sent to the driver.
INFInfringements It is recommended the amount field is used to record the number of demerit points. Other fields are free text. An Infringement system ‘note’ will appear to the user if a note has been added to the Client/Defaults screen
INFDInfringementsEmail to Driver
INFCInfringementsEmail to Client
LOCLocation Enter the vehicle's location in the event's "Location" field. This will also update the Contracts / Contract "Location" field to match your entry.
"Location" is a free text field in the event that can be up to 255 characters. A tooltip is available on this field to see the full content of the field.
If a location event is deleted, Contracts / Contract "Location" is also updated to reflect this.
LOCBK Location Change Booking When a Bookings / Condition "Current Location" or "Return Location" entry has been entered, a Location Change Booking event will be created and the Contracts / Contract "Location" field will be updated to match your entry.
You can also enter a Location Change Booking event directly in the Contracts / Events tab in the "Value" field.
"Value" is a free text field in the event that can be up to 255 characters. A tooltip is available on this field to see the full content of the field.
If a Location Change Booking event is deleted, Contracts / Contract "Location" is also updated to reflect this.
LOGLog Book entries Select either ‘Business’ or ‘Private’ from the "Value" list box and then describe the entry. Enter the end reading for each trip in the "Odometer" field. The results of Log Book entries can be manually reviewed to compile and enter the "Business Usage" percentage field on the Contracts/FBT screen. You can change the allowable values in the "Value" list box if desired. See details in the Field Descriptions section.
LRVLease Replacement This event can record the registration number of the replacement vehicle (value field). The number of days is to be entered in the amount field
NONNon Use Days If drivers have non use days whereby the vehicle is unavailable for their use the number of days can be recorded as an event. The ‘amount’ field is used to record the number of days, which is then used to populate the non use days field on the FBT screen for the applicable year
NOTENotes Use this for any notes you may wish to record regarding the contract
MRNRecall Notice When this event type is being created or edited, an alert message will appear if there is a message stored in the Clients / Contract Defaults "Recall Notice Note" field.
RCRegistration Change New contracts are created with a default registration number of ‘ORDER’ until this event is created. The Registration Number saved here also updates the field "Reg Number" on the Contracts / Contract tab.
AlertComments
You cannot save a blank 'Reg Number' for this contractA blank "Reg Number" will not save. Once you click OK, you will be returned to the record in edit mode to enter the "Reg Number".
Warning: {reg_no} Registration already used on Contract {contract_id}You are trying to enter a Reg Number that is already used by another active contract in the same state.

Click OK to over-ride the warning and save the contract, or Cancel to return to the record in edit mode.

REGLRegistration Label Use this event to record the date the renewal label was forwarded to the driver or client contact. You can also generate a letter to print and send with the registration renewal label. The default address for the letter is set ins the Clients/Contract Defaults screen. There are separate defaults for novated and non-novated contracts. This allows the letter to be addressed to either the driver (for a novated lease) or the fleet contact (for an operating lease). Go to Rego Label Sent for complete details of this feature.


RNReimbursement NotificationThis event is used to advise a driver when a reimbursement claim has been approved. It is created and sent when you post a claim in the Contracts / Claim tab or the Approvals / Approval tab. It will not send if the Drivers / Finances "Stop BECS" checkbox is checked. The event's Description field is populated with the related approval number.
SWANTRPNSwann Tyre & Rim Proposal NumberIf Swann Interface is in use, this event will be created when a Quote (with Tyre & Rim Inclusion) is accepted. The value field contains the next available proposal number. This event can be created manually, e.g. when phasing in the Swann Interface. Only one COC event can be created per contract.
SWANTRPSSwann Tyre & Rim Proposal SentWhen the Swann Interface scheduled service has sent the proposal to Swann Insurance's "Activate" system, the Date and Time is noted.
SWANTRPRSwann Tyre & Rim Policy ReceivedWhen the Tyre & Rim proposal has been accepted (and sent back by "Activate"), the Policy documents and number are stored in this event.
TLNToll Notice used to record toll notes (no email).
TLNDToll Notice Email to Driver
TLNCToll NoticeEmail to Client
TLNCToll Notice Statutory Declaration (no email).
