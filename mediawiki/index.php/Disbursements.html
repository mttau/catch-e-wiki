Overview

Create a Disbursements (BPAY) batch to record benefit payments being paid via Bpay.

[edit]

Record Selection

  1. Contracts / Contract "Company" is the same as the selected company or
  2. Contracts / Contract "Client" is the same as the selected client
  3. The contract is linked to an Employees / Item record
  4. The transaction is
    1. a claim or
    2. a payment
  5. The transaction "Payment Method" is 'Bpay'
  6. Suppliers / Account "BPAY Biller Code" is recorded.
  7. The transaction "Reference" is applied as the "Reference Number"
  8. The transaction is posted
  9. The transaction has not already been recorded in a previous bpay batch

Validations and Alerts

Below are alerts you may see during this process and the reasons why they have happened. In most cases, you should be able to identify a data error and once corrected, Save or Post the record.

Alert Comments
On navigation to the Accounts / Export / BPAY / Disbursements screen
BPAY details require set up! The BPAY requirements have not been set up in your system. Go to the Disbursements (BPAY) Setup page for details.
New
Nothing to disburse!There are no outstanding payments to disburse. Click on OK to be returned to the screen.
New - "BPAY Type" is 'Company'
No disbursements found for Bank Account {account_name} No records can be found to disburse.
  1. Check that there are benefit contacts have posted payment or claim records where the "Payment Method" is 'BPAY' and the disbursement has not yet been made.
  2. Check that the benefit contracts are linked to the same company as the selected "Bank Account".
Warning!

Disbursements cannot be expected for the following suppliers:
{supplier_code}

The identified Supplier has incomplete data to create the batch.
If you are sure there are transactions available to be paid for a company from your selected bank account, check the following.

  1. The Supplier’s Biller Code is blank.
  2. The payment structure linked to the payment in the batch does not have a ‘Supplier Reference’.

You can continue creating the BPAY batch by selecting OK and the payments that have correct and complete data will be included in the batch.

New - "BPAY Type" is 'Client'
No disbursements found for Client Code {client_code} No records can be found to disburse. Check the following;
  1. Clients / Packaging Defaults - a record has been created for the client.
  2. Clients / Packaging Defaults - "Client Bank Account" - is checked. If un-checked, the system will not look for these transactions.
  3. There are benefit contacts that have posted payment or claim records where the "Payment Method" is 'BPAY' and the disbursement has not yet been made.
Client Code {client_code} not valid!
  1. The entered Client Code is not recognised in the system.
  2. The entered Client Code has “Client Bank Account” unchecked.
  3. Confirm the Code you require via the search function to continue
Create
Transaction Date cannot be empty! A disbursement file cannot be created if this date is missing.
Invalid Date!

You must enter a valid date as DD/MM/YYYY or leave it blank.
A malformed date has been entered or the format is incorrect. It needs to be entered in this format: dd/mm/yyyy.
Re-select or re-enter a valid date.

Warning! There are disbursements due for export, but none can be exported due to insufficient data!

Suppliers:
{supplier_code}

[OK]

1. The listed suppliers do not have a stored biller code. Go to Supplier / Accounts “BPAY Biller Code” and enter the code.
2. Eligible claims or payments do not have a stored "Reference". Locate the records and add the correct reference.

Up to eight (8) records can be displayed on the alert message pop-up.
If more than eight records have missing bank details, they will show progressively as displayed records are corrected and the invalid list reduces.

If there are records that you cannot update at this time, you can still create a file for the valid records by clicking Create.
Payments that have missing BPAY details will be skipped.
The invalid records will be displayed again the next time you create a BPAY disbursements file.
Delete
BPAY batch ({becs_id}) will be deleted. BPAY files cannot be used and
disbursements will need to be re-issued.
[OK] [Cancel] 		This alert occurs when deleting a batch.
OK to confirm and proceed.
Cancel to return to the screen.
