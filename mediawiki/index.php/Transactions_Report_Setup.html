Where Funds Check is in place

If the Funds Check is already in use, the set-up steps for using the Transaction Report are as follows;

  1. Create a record in gl_posting_map_reporting for each Contract Type that should have this feature.
  2. Set the [gb_report_queries]status_flag for the 'transactions' report record to 'active'.
  3. Check the values stored in [gb_companies]name as it is used in the description field for Receipt and Budget transactions. There must be an entry in [fm_contracts]company_id for this to description to appear correctly on the report.

Visit Screen Reports for more details about [gb_report_queries].

Where the Funds Check is not in place

To make the Transactions Report feature available, you need to set-up a report map to identify which category each posting class belongs to and how that category should be treated.
The illustrated how the report map joins up all the information.
Your Account Manager will do this for you. Initial updates can be done with a standard script. The Funds Check normally uses the same mapping.
