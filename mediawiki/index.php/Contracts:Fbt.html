Overview

The FBT screen is designed show FBT information and can be seen by external users (Drivers and Clients). Clients are able to print and edit some fields on this screen, Drivers can only print their reports.
The Driver Report output is a key feature of this screen - it shows you the budget and FBT postition of the lease. The Whole of Life variation of the report is used during the termination process for novated leases.

Annualised kms

Annualised kms are calculated by comparing the Opening Reading with the Last Reading. Kms travelled per day over that period is then multiplied by the number of days in the current FBT year (365 or 366 - leap year). For example:

Opening Reading Date:24/05/2006
Opening Reading:2732
Last Reading Date:12/03/2007
Last Reading:18,476
Kms travelled:15,744
Days between readings:293 (12/03/2007 - 24/05/2006 +1)
Kms per day53.7338
Days this FBT year:365
Annualised Kms:19,613 (53.7338 * 365)



Note 1: Start Of Year Date:
The Contract Start date is the start of year date for the first FBT period. This date is substituted with the 3rd Party Start date if entered on the Contracts Screen. For the subsequent FBT periods the start date will be the 1st of April for the relevant year.

Note 2: Opening Km Reading
The opening Km reading is determined using the following logic:
If there is a Start Kms reading dated on or after the FBT year start use that,
Otherwise if there is a valid odometer in the Maintenance table dated on or before FBT year start use that,
Otherwise use any valid odometer dated after the FBT year start.
If there is more than one valid odometer reading on the opening or closing date the highest valid reading is used per date.

Refer to Estimated Odometer Calculation for information on business rules regarding odometer dates.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.


[edit]
Benefit Type is Operating and FBT Amounts show as Statutory

Operating Method FBT is not calculated and returned on this screen as a range of transaction data is required to do this.
To check the FBT calculations for Operating Method contracts, go to Fleet / Administration / Reports and run the FBT Operating report.

[edit]
Driver Report does not match the selected month

If a user (internal or external) does not have there browser caching setting set correctly, the Driver Report PDF output may not match the selected month. In Internet Explorer, ask the user to check their settings. They should be:

Tools/General/Settings

  • Check for newer versions of stored pages → Select "Every time I visit the webpage"

This is to prevent generated reports from being cached.

[edit]
Reports and Buttons are missing

Reports and buttons will be missing from this screen if the required report templates are not defined for the Contract Type.
Fix this by going to the Contract Types / Details tab and updating the two following fields

  1. "FBT Statutory Template"
  2. "FBT Operating Template"
