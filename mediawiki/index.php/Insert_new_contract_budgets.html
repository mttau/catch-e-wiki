Overview

Beta function

Allows you to insert a new contract budget period for a contract.

This API replicates the system process to adjust a lease period billing

[edit]

Permissions

The permission for this api is stored in [gb_api_permissions]permission as 'ContractBudgets:Insert'.
Only roles with a stored link to this permission in [gb_role_api_permissions] can run this api.
API access to this function is provided to the role 'web-services' as the default.

SELECT r.name,rap.permission
FROM gb_role_api_permissions AS rap
LEFT JOIN gb_roles AS r ON r.role_id = rap.role_id
WHERE rap.permission = 'SignonToken:Create'
ORDER BY r.name;
[edit]

HTTP Method

Use the HTTP Method 'POST' for consuming this web service. 

https://api.catch-e.com/fm/contract/budgets/insert
[edit]

Input Fields (Header)

Key Format Notes Mandatory
Authorization string authentication token Yes
Audit-User-Id string Value = [gb_users]user_id. Pass this if you want to record this user in the audit records as the creator of these records.

This permission should only be given to the 'web-services' role if required.		No
Content-Type string Value = 'application/json' or 'application/hjson'Yes

Input Fields (Body)

JSON Field Format Notes Mandatory
contract_idIntContract IDYes
billing_periodstringOptional. Must be specified if update_single_period is set to 'yes'.
Mandatory if Contract has not yet commenced Billing.
Yes
payment_methodstringUpdate the Payment Method preference for budgets. Choose from direct-debit, cheque, cash, other, credit-card, eft. If this is not passed, the method recorded in item 1 will be used.No
billing_datestringEnter the billing date. If this is not passed, the date recorded in item 1 will be used. No
budgetsstringEnter one or more budgets to insert values for. The "Class code" is used here.
If a "Class code" passed that is not stored in Contract Types / Accruals, the contract budget item will be created and will have a 0 value.		Yes
  amount_netstring Enter the amount, this can be 0. Yes
  amount_gststring Enter the gst amount.
If the gst amount is not passed, the value for amount_gst will be added as 0 and the "Overidden?" checkbox will be checked.
If the gst amount does not match the expected calculated value, the "Overidden?" checkbox will be checked.		No
