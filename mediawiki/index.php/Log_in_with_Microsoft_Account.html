Overview

If Log in with Microsoft Account is shown on your Login screen, click on it to go directly to the Home screen.
This feature uses your Office 365 account credentials to perform a single sign-on.
If you have 2 Factor Authentication enabled, you will be taken to the authentication screen once this sign-on has completed.

You may be able to continue to use the standard login method I.e.entering a "User ID" and "Password".
If your organisation wants to use the Log in with Microsoft Account method exclusively, then your password will be removed and the standard login method will no longer work.


[edit]

Set up

[edit]

Azure AD App Registration

In your Azure AD console, go to the "App Registrations" area and click "New Registration".

Provide a name for the App (eg. Catch-e) Set the Supported account types to "Accounts in this organizational directory only" (single tenant mode) - Preferred option.

In the Redirect URLs section, specify the following (depending on which Catch-e environments you need to access):

LIVE/PRODUCTION:
https://CLIENTCODE.catch-e.net.au/azure.phpo
https://CLIENTCODE2.catch-e.net.au/azure.phpo

SUPPORT:
https://CLIENTCODE.support.catch-e.net.au/azure.phpo
https://CLIENTCODE2.support.catch-e.net.au/azure.phpo

STAGING:
https://CLIENTCODE.staging.catch-e.net.au/azure.phpo
https://CLIENTCODE2.staging.catch-e.net.au/azure.phpo

STAGING2:
https://CLIENTCODE.staging2.catch-e.net.au/azure.phpo
https://CLIENTCODE2.staging2.catch-e.net.au/azure.phpo

TEST:
https://CLIENTCODE.test.catch-e.net.au/azure.phpo
https://CLIENTCODE2.test.catch-e.net.au/azure.phpo

   Substitute "CLIENTCODE" with your own environment name.


Then click the Register button.

In the top left of the next screen you should see:
Application (client) ID: The value is the "Client Id" for the Global Control (see below).
Directory (tenant) ID: The value is the "Tenant Id" for the Global Control (see below).
Object ID: Not used by Catch-e.

On the left hand menu click Certificates and Secrets. Click "New Client Secret". Give the secret a name, and configure an expiry. The "Value" is the "Client Secret" for the Global Control (see below).

Next go to API permissions and configure a single permission for the Catch-e app:
Microsoft Graph = openid
Type = Delegated
Description = Sign users in

Note: Azure AD configuration is beyond the scope of Catch-e, these notes should serve as a guideline only. Please consult your Azure AD administrator for changes applicable to your specific environment.

[edit]

Azure Login Details

Enter the appropriate details into [gb_controls]azure_ad_oauth2_login for your system.
Once done, the "Login with Microsoft Account" link will display on the Login page. Clicking on this link will use a single sign-on and will bypass the standard login method.

Tenant Id: Directory (tenant) ID
Client Id: Application (client) ID
Client Secret: Client Secret Value

[edit]

Create users

Create users as normal, you will still be required to issue the user a Password.

[edit]

Add Azure AD logins for users

For each user that will use this feature a record needs to be added to the [gb_users_logins] table. Go to Setup / Import Table Data to upload these records.

The key fields are as follows:

  1. [gb_users_logins]service = 'azure_ad'
  2. [gb_users_logins]login = the unique 'OID' value of each user from Azure AD.
[edit]

Disable passwords

If you do not want users to use the standard login method, go to Roles / Details and select the "Disable Passwords" check box for each role you want to disable.
Once checked, users will only be able to gain access by using Log in with Microsoft Account.

The "Disable Passwords" check box is disabled for roles where it is not suitable;
External and Portal roles continue to use the standard login method, the Web Services role can only access the sytem using an API.
Catch-e staff use the standard login method and are required to use 2 Factor Authentication whenever they log in to your system.

[edit]

Users who cannot use this feature

If you are using Log in with Microsoft Account feature, the icon will be shown on the Login screen, even if it cannot be used.

Any users that you cannot set up in Azure AD with credentials cannot use this feature.
This includes all external roles such as Drivers and Fleet Managers.

[edit]

Suspending users

  1. Catch-e: Go to Users / Details and update their "Status" to 'suspended'. In these statuses, the user cannot use Log in with Microsoft Account and will be returned to the Login screen with the message 'User Locked!'
  2. From Azure AD: Deactivate the user. Users who try to use Log in with Microsoft Account will get this message:
    "Azure AD OID is not mapped to Catch-e username. Please contact your Azure AD administrator."
    This message will also be shown if Catch-e has not been set up as an APP in Azure AD.
    Please note that unless you disable user passwords, users will still be able to use the standard login method to gain access if their Users / Details "Status" is still 'Active'.
