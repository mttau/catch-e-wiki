Overview

Administrators can update the Drivers User Guide, Quotes using html, Forms and Screen Reports by uploading new templates here. Driver user guides can be configured and customized for Channels
This is done by exporting and importing template files in the following formats: css, docx, pdf, html, xlsx and xml.
Your Account Manager will configure the file settings you need and help you build your first set of documents (if required). Once established, you can maintain your own documents.
If you cannot see a file you are using and want to change it, contact your Account Manager to configure it so that you can access it here.

We recommend that you;
  1. Export and save a back up copy of any template you are updating.
  2. The import process overwrites the file stored in your LIVE system and it cannot be recovered later by Catch-e staff.
  3. Before updating LIVE, import the updated file into your STAGING or SUPPORT environment to test it works before updating LIVE.

Hints and tricks for creating your import file

To create a template of a particular table, download the existing table using the Export Table Data function and remove the columns not required.

Dates must be in one of the following formats:
D/M/YYYY
D/MM/YYYY
DD/M/YYYY
DD/MM/YYYY
YYYY-MM-DD
0000/00/00 will also be accepted. This will result in the relevant screen date displaying as blank.

To import a NULL entry into a field, record the cell data as the word NULL in capitals.

I can't see the file I need

Some import or export files are marked as only editable by Catch-e staff.
If you cannot see the file you want to edit, contact Catch-e Support to update the access settings for the file.

Set-up Notes

[edit]
Adding new file type records
  1. Add a new [gb_file_types] record when required to set-up new files.
  2. Check that the [gb_file_types]directory address is set-up correctly. Files will not import if this is malformed. They must be set-up to point to one of the two folders detailed below;
    1. /clients/CLIENTNAME/../templates
    2. /clients/CLIENTNAME/../attachments
[edit]
Channels

Create a channel variation for aa file by created a new [gb_file_types] record for the channel and then the linked [gb_files] record.
The [gb_file_types] record must contain the correct [fm_channeles]channel_code in the directory path. E.g. /clients/CLIENTNAME/channels/CHANNEL_CODE/core/fleet/reports/attachments

[edit]
Adding new file records
  1. Add the required [gb_files] record that you want to use.
  2. Set [gb_files]status_flag as ‘active’
  3. The file can now be selected from the "File Name" drop-down list
[edit]
Hiding file records
  1. Hide files if you are no longer using them.
  2. To stop a file from appearing in the "File Name" drop-down list, update [gb_files]status_flag to ‘inactive’.
  3. Doing this will not prevent the files from being visible and available on the system screens. To removed them from user screens, the actual file must be deleted from the system folder. Contact your Account Manager to arrange this for you.
[edit]
Legacy files
  1. If you want to make changes to a file and cannot find it in the screen lists, you likely have a legacy file that is not stored in [gb_files].
  2. To enable changes, the old version must be removed from the application and replaced with a stored file here. Contact your Account Manager to arrange this for you.
[edit]
Catch-e only files

Access to some files are restricted to Catch-e staff only. This is the case when [gb_files]super_flag is set to 'yes'. When this flag is set to 'no', 'admin' users can manage the file.
