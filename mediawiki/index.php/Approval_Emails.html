Overview

Emails can be triggered to send on a change of Sub Status in the Approvals / Approval tab.

[edit]

Emails

[edit]
Credit Limit Exceeded

When approvals are saved and the client's credit limit has been exceeded and their "Send Approval Email" flag is on - a Credit Limit Exceeded email can be sent.
Visit Clients / Approval Defaults → Credit Limit for client settings. These are linked to particular Sub Statuses.

[edit]
Set-up
  • fm_approval_sub_status]template_id = [gb_templates]template_id
  • [gb_templates]name = 'pdf_email_review'
  • [gb_templates]sub_name enter the value of the linked fm_approval_sub_status]sub_status (for ease of use, but is not a coded link).
  • The list of available placeholders you can use in this template is here.
  • Create a gb_template_recipients entry for each recipient you want to send to.


[edit]
Unauthorised Supplier

If an approval has been raised using a Supplier/Posting Class combination that is not recorded as the client's preference, an email can be configured to advise the client. Visit Clients/Approval Defaults → Authorised Suppliers for client settings.

[edit]
Set-up
  • [gb_templates]name = 'approvals_authorised_supplier_class_exception_notification'
  • The list of available placeholders you can use in this template is here.
  • Create a gb_template_recipients entry for each recipient you want to send to. By default the email is sent to the Clients / Contact "Contact 1"
