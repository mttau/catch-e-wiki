Overview

Insurance rates and client default insurers can be stored in the system to allow insurance to be calculated systematically.
Refer to Insurance Premium for an illustration of this and also the Insurance Premium calculations model.

See also the details about the Insurance Fee (IFEE) budget. This enables you to add an additional charge to the quote for managing insurance.

Set-up Notes

[edit]
Insurance Type

New quotes will default the "Insurance Type" stored on the variant profile into this field.
If the default is not correct, go to the Setup / Reference Data Variant Profile record to update the default.

A bulk update and checking process can be created. Contact your Account Manager to arrange this.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.


[edit]
Fire Service Levy does not match expected value

Contracts / Insurance "Fire Service Levy" is populated from quotes when comprehensive insurance is calculated.
Check that Quotes / Contract "Delivery State" and the "Insurance Loc." state are the same.
Fire Service Levy is calculated using [qt_quotes]state.

[edit]
Stamp Duty does not match expected value

Contracts / Insurance "Stamp Duty" is populated from quotes when comprehensive insurance is calculated.
Check that the values in [qt_insurance_rates]base_rate and [qt_insurance_rates]cost are the same.
[qt_insurance_rates]base_rate is used to calculate stamp duty and [qt_insurance_rates]cost is used in other parts of the calculation.
