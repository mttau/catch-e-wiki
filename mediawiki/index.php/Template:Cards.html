Overview

Create, edit and view cards used in card interfaces here.
Visit Card Interfaces for a directory of the available interfaces with a link to detail page for each one.

Card details are used when importing transactions into the system.
During the import process, card details and business rules are applied to validate and process the incoming transactions.

The Contracts / Contract Card Services: page displays a contract's cards.
Each "Card Number" is a hyperlink to the record on this page.

The Import Table Data screen can be used to import new cards in bulk.
Visit the Table Data page for details of common import issues.

Cards can also be created using the Create cards API individually or in bulk.

Field Descriptions

Field Type Description [table]field
Header
Card Number Lookup Enter a card number or use the to look for a record using other fields to search.
Left hand side
Card IDNumber Unique id allocated by the system.
Supplier Lookup Supplier code from the Suppliers records. The relevant supplier type must be a "Card Services" type. Each relevant supplier also requires a Card Interface to be setup.
Card Number Number The card number as created by the Card Supplier. This number is used to match transactions imported to the details in the Card record. This number must be unique.
Account Number Number This is a text field that is not used in any system processes. It can be made read-only by applying a role restriction. If you would like to update access to this field, contact your Account Manager .
Contract ID Number The Catch-e contract id which has been matched with the card number. Transactions imported for the relevant card will be allocated to the contract id - which may have a different vehicle registration number to the registration number on the fuel card (if any). This feature is designed to allow posting of transactions to particular contracts when required.
Posting Enabled Check Box This is checked by default. Un-checking this field is not recommended by Catch-e
If this is un-checked, card transactions are set to 'deleted' when imported and checked. This feature allows unwanted card transactions to be automatically deleted by the system.
Reg Number Text Displays the Registration number associated with the contract.
Client Lookup Displays the client code and name associated with the contract.
Driver Text Displays the driver name associated with the contract.
Recharge? Check Box When a new record is being created and a "Contract ID" is selected, this field will default from the linked Contract Types / Expenses Other "R" field for the class 'F' (Fuel / Charging'). Edit or change this setting as required.

'Y' - checked, 'F' transactions will be set to recharge your client.
'N' - unchecked, 'F' transactions will be posted and will accrue against the budget.

For other classes, the Contract Types / Expenses Other "R" field setting determines if card transactions are set to recharge or not.

Recharge A/c Lookup The default client code to recharge is obtained from the Contract details but may be over ridden by the user. NOTE changing the client code on a Contract requires the client code to be changed here if applicable.
History Only Check Box If checked, then transactions will be posted with the status of "History Only". This means that recharge billing is not created, nor are accounting entries or any export transactions to accounting systems. The transactions are effectively displayed on the Maintenance screen for reporting purposes and are a useful way of obtaining odometer readings without generating other transactions. This defaults from Clients / Contract Defaults "History Only" but can be changed.
Date Activated Date Activation (i.e. issue date) of the card, which should match the Card Supplier's own activation date.
Date Inactivated Date Date the card is cancelled. This should match the Card Supplier's own cancellation date and triggers a warning on the fuel transaction.
Expiry Date Date Date the card is due to expire which should match the Card Supplier's expiry date. This date also triggers a warning on the fuel transaction if relevant.
Sub Status Drop List Displays the Card Sub Status. Sub Statuses can be used to show a reason for the Status. To set up this feature please contact Catch-e mailto:support@catch-e.com.au
Status Display active or inactive. Status is updated based on the Sub Status of the card. Transaction checks are done when transactions are imported for 'inactive' cards.
Notes: Text Free text notes which the user may populate. This should be used in instances of defaults being over-ridden by the user.
Right hand side
Contract Type Display Displays the contract type associated with the contract to assist in making decisions regarding business rules.
Fuel Budget Display Displays the fuel budget for the contract from the previous month and is also designed to assist the user to decide on appropriate business rules.
