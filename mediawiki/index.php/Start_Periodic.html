Overview

"Start Periodic" will populate with a calculated date if the following conditions are met.
Open the Start Periodic Date Model to see how this calculation works.
The date logic in this model is also used to calculate the Employees / Item "Start Date" for new recrods.

[edit]
Pay Cycle
  1. The contract's "Contract Start" and "Periodic Interval" fields have been entered
  2. The contract's "Periodic Interval" is one of the following types: Weekly, Fortnightly, Monthly, Bi-Monthly or 4-Weekly.
  3. The Clients / Quote Default "Pay Cycle", "Pay Cycle2 or "Pay Cycle3" matches the contract's "Periodic Interval"
  4. The Clients / Quote Default "Payroll Date", "Payroll Date2" or "Payroll Date3" is populated.


The following fields are hidden in the Contracts / Billing tab, if a contract has a Pay Cycle set-up:

Initial Billings
Start Period
Common Day
[edit]
Monthly

If the Pay Cycle requirements above are not met, the "Start Periodic" may still calculate if

  1. The contract's "Contract Start" and "Periodic Interval" fields have been entered
  2. The contract's "Periodic Interval" is 'Monthly'
  3. If "Start Period" equals '1', the "Start Periodic" defaults to show the "Contract Start" date
  4. if the "Start Period" is 2 or more, the "Start Periodic" displays the "Common Day" date of the "Start Period" month.

Visit Billing Schedule to see examples of how different billing schedules can be constructed.

[edit]

Recommended Steps to test

  1. Confirm the Clients / Quote Defaults "Pay Cycle" and "Payroll Date" are populated. e.g. Fortnightly, '08/12/2020'
  2. Check that the client's Quotes / Contract "Pay Cycle" is the same. e.g. Fortnightly.
  3. Accept the quote to create a new contract
  4. Add in a "Contract Start", the "Periodic Interval" should already be populated as 'Fortnightly'
  5. "Start Periodic" should populate

Set up Notes

  1. Go to the Pay Cycle Setup page to review the pay cycle records.
    1. Check which pay cycles you want to make available for billing.
    2. Set [gb_pay_cycles]billing_flag as 'yes' for these pay cycles.
    3. Only these records can be selected in the Contracts / Billing "Periodic Interval" drop box.
  2. Check the Clients / Quote Defaults "Pay Cycle" defaults as required.
