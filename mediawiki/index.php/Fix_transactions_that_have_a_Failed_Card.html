Fix transactions that have a Failed Card

From Help wiki

Jump to: navigation, search

Main Page → Workflow → Fleet Management → Card Services → Fix transactions that have a Failed Card

Qualifiers

  1. You have imported a fuel transaction file and it is at 'Pending' status
  2. The Transactions / Batches screen contains transactions with a Validation of 'Failed Card'

Why this happened

  1. The imported transaction belongs to a card that does not have a record in the system

Process

  1. Go to the Fleet / Card Services → Transactions / Batches tab
  2. Validation - choose 'Failed Card'
  3. Click on
  4. Click on the Card Number hyperlink of the first record to navigate to the Create Card pop-up.
  5. This pop-up is the Card Services / Cards screen in Edit mode with the "Card Number" populated.
  6. Contract ID - enter or lookup the required contract
    1. If the transaction has a Reg Number that matches the one in the transaction, the values above will be populated when this pop-up opens
    2. Review and change the Contract ID if it is not correct
  7. Date Activated - enter if known
  8. Date Inactivated - enter if known
  9. Expiry Date - enter if known
  10. Sub Status - this is left blank. The field is usually used in automated Card Ordering and Terminating processes
  11. Status - this should default to 'active' and cannot be edited
  12. Notes: - enter any notes you want to make about this record
  13. Click on to create this record
  14. You will be returned to the Transactions / Batches tab
