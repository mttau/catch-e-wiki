Overview

The billing screens are used to creating invoices for clients. Invoices can be generated in either PDF or CSV format.
Invoice numbers are allocated sequentially and being at 100000. This is done to avoid a conflict with the invoice sequence of your Accounting System, E.g. MYOB commences at ‘1’.

Automated Billing

We strongly recommend scheduling your bulk billings.Visit the scheduler billings page to find out more about how to do this. Your Account Manager can also assist you in setting this up.
This will be more efficient because

  1. Scheduled billings are 'set and forget'
  2. Emails are sent out in the scheduler process and do not require a second step
  3. Billings can be set to run out of hours, which reduces the likelihood of users being disrupted by the system locks that bulk billing creates. See Note.

Note - the billing process generates system locks that prevent users from editing records in many areas. Visit the System Locks page for full details about system locks.

Invoice Templates

Each billing screen has a different invoice template to suit the records being invoiced. See below for a brief description of each one.
If you have configured invoice templates for your Channels, these invoices will use a different invoice logo and email to reflect the channel's branding.
The Clients / Contract Defaults "Billing Type" setting determines which invoice formats are used for the client.
The Clients / Contract Defaults "Billing Terms" field determines which "Terms" are shown on the client's invoice footer.

Grouping, Sort Orders and Sub Totals

Each invoice is issued to a separate client.
A client's billings will be further separated if Clients / Contract Defaults "Billing By Client Order No?" is checked.
In this case, a new separate invoice will be created for each different instance of Contracts / Contract "Client Order No" there is in the client's selected records.

Invoices generate in the following order; Client Group, Client Code
Invoice records are sorted in the following order; Cost Centre, Registration Number
Invoices are sub totalled by Cost Centre.

[edit]

Emailing

Invoices will email if the Clients / Contract Defaults "Invoicing Preference" is set to 'Email'.
If Clients / Contract Defaults "Use Group Email?" is checked invoices will email to the linked Client Groups "A/C Payable" email address instead of the client's.
Choose which email attachments you want to send to the client by selecting from the Clients / Contract Defaults "Attachment Type" options (Pdf, Csv, or Pdf and CSV).

Suspended Contracts

If a contract has been suspended, billings that are generated from records located in the Contracts / Lease tab are invoiced if the "Period" date of the record is dated before the Contracts / Contract "Suspend Date".
Any records dated after the "Suspend Date" will not be invoiced. You can future-date a "Suspend Date" on a contract if this date is known in advance.
If you insert an adjustment record for a period that is dated before the "Suspend Date", it will be included in your billing selection.
When the Finalise button is used on a suspended contract, the finalise record is always dated the day before the "Suspend Date", so that it will be invoiced.

[edit]

Credit and Inertia Rentals

The "Suspend Date" rule is ignored if the Contracts / Lease "Notes" display as 'Credit' or 'Inertia'.
Inertia records are not entered by users in the system screens. They are updated by script or uploaded into the table fm_contract_budgets with a billing_type_flag of "inertia").

Credit and Inertia Rentals can be invoiced from the Periodic or Combined Billing screens.
If a contract contains Credit or Inertia billing records, the "Vary" button on the Contracts / Billing screen is permanently disabled.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.
[edit]
Missing a contract from the billing process

If you cannot see a contract you are expecting in a billing selection, check the following;

  1. The contract must have a Contracts / Billing "Start Periodic" date
  2. The contract must have a "Registration Number" that is not 'ORDER'.
  3. Packaged Contracts (contracts that are linked to an item) can only be viewed and invoiced from the Billing / Periodic screen if you are logged in as a Channel Accounts role.
[edit]
Suspended contract is not billing

Contracts being billed on or after the "Suspend Date" are not selected for billing. To invoice a suspended contract, do one of the following

  1. Enter a billing selection date that is before the "Suspend Date". E.g. If the "Suspend Date" is '24/09/2021', then the billing selection date must be on or before '23/09/2021'
  2. Use the Contracts / Lease Invoice Now hyperlink to invoice the selected period (this will only work if the period date is before the "Suspend Date".
  3. Remove the contract's "Suspend Date", run the billing and then re-enter the "Suspend Date".
[edit]
Problem creating Invoices (Error codes: 102, 103, 203)

When the "Invoice" button is clicked, the billing process locks relevant tables. If the session is interrupted or terminated (e.g. by closing the window) without using the "Abort" button the locks will remain in place, resulting in subsequent errors advising that billing is locked by the relevant user.

If the user locks are cleared a subsequent error such as:
Problem creating Invoices!Combined->createCombined:2 may result - if so log out of Catch-e entirely and log back in to process billing. Contact support if further problems are encountered.

If you encounter one of these :
Problem creating Invoices/Combined->createCombined:103
Problem creating Invoices/Combined->createCombined:102
Problem creating Invoices/Combined->createCombined:203

"Abort" the process and re-try.

[edit]
Problem creating Invoices (Error code: 202)

When trying to create an invoice and you see an error with the code '202' in it, this can happen if records in the table [fm_contract_budgets] have been incorrectly or incompletely updated by a web service or a script update from outside the user billing screens. E.g. [fm_contract_budgets]billing_id is populated, but the record is still in 'pending' status.

If you encounter an error like this :
Problem creating Invoices/Periodic->createPeriodic:202

"Abort" the process and contact Catch-e Support for assistance.
