Overview

The controls and tables that need to be updated for insurance calculations are detailed below.
See also the Insurance Premium calculations model to see how these variables work to calculate a premium.

[edit]

Global Controls

ModuleNameComment
gbcomprehensive_insurance_basisValues can be one of: fbt_value , on_road_price_total , depreciable_amount or financed_amount
gbinsurance_calculation_method'external' (default) or 'internal' (Refer internal calculation for further details).
gbfsl_ratesContains serialised Fire Service Levy rates for each state/territory.
gbisd_ratesContains serialised Insurance Stamp Duty rates for each state/territory.
gbinsurance_at_fault_rateActivated when "At fault incidents?" checkbox selected. Care should be taken when qt_insurance_loading_types & qt_insurance_loading_rates used.
gbinsurance_gst_calculation_method

Stamp Duty can be calculated using either the GST Inclusive premium or GST Exclusive premium.
Accordingly GST will then be calculated using either the Stamp Duty Inclusive premium or Stamp Duty Exclusive premium.
To calculate a) GST after calculating Stamp Duty and b) Stamp Duty before calculating GST use 'stamp_duty_inclusive' in this control.
To calculate a) GST before calculating Stamp Duty and b) Stamp Duty after calculating GST use 'stamp_duty_exclusive' in this control.
The calculation of GST is performed in qt_quote_budgets where the posting class = Comprehensive Insurance (posting_class_code = INS).

gbinsurance_location_by_postcode_flagInsurance rates held in qt_insurance_rates will refer to one of two tables based on the value in this control (insurer_location_id).
If set to 'no' qt_insurer_locations is used.
If set to 'yes' qt_insurer_locations_by_postcode is used.

If you change insurers and the location changes also (I.e. from yes to no, or from no to yes), create a new control record that uses the cut over date.
The first set of records created in the [qt_insurance_rates] table for the new insurer should use the same date.

Insurance Tables

Table NameComment
Required set-up records
fm_suppliers Suppliers:Account 'Insurance Broker Fee'. When populated the Insurance Broker Fee will be added to the Insurance Premium calculations.
qt_internal_insurers This table stores the details of insurance suppliers that you offer a managed insurance product for
qt_insurance_rates Base insurance rates per insurer based on location, vehicle type and lower-upper limits are stored in this table. Contact your Account Manager to set up process to create/update these rates.
qt_insurance_types Contains insurance type groups eg. 'Standard Sedan/Ute', '4WD', 'Prestige or European', 'High Performance'. Table also holds Claim Excess.
qt_insurer_locations Contains State/Territory's along with the supplier_id of the Insurer.
If the supplier_id is associated with an active qt_internal_insurers record, you must create an alternative Supplier for any Client policies (Policy Owner).
Location by Postcode (optional feature)
qt_insurer_locations_by_postcode Holds Post Codes and State details. Data is only required in this table if you are building insurance calculations that use a Postcode instead of a State as the location.
If you use this feature, create a matching [gb_controls]insurance_location_by_postcode_flag record set to 'yes'.

This is a restricted table. Contact Catch-e Support to update records into this table for you.

Insurance Loading (optional feature)
Contract Types / Details Insurance loadings are bought in to a quoted insurance premium when the Apply Insurance Loading flag is checked on as a Contract Type option.
qt_insurance_loading_types Used to store drop down content on Fleet / Administration / Quotes / Contract tab for fields "Licence", "Past Claims" and "Restrictions".
Based on the selection premiums may be 'loaded' by the corresponding rate found in qt_insurance_loading_rates. These field names (Licence, Past Claims & Restrictions) can be renamed.
qt_insurance_loading_rates Links the qt_insurance_loading_types and insurance_rate_id to a loading rate. If multiple loading types selected the rates are added up to determine the "Insurance Loading" %
