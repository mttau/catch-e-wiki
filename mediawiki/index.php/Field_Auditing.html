Overview

The icon that is located in the top right corner of most screens enables you to reveiw changes that have been made to the fields within the screen or area.
Clicking on generates a pop-up of the audit information called View Edit History.
The 'X' in the right-hand column called "S" indicates that the record was updated by an external process. i.e. via a web service.

Filters

Filter Type Description [table]field
User Name Drop List Choose a user to review only their entries. [gb_audit]user_id
Table Drop List Choose a specific table from the selection to review only entries for that table. [gb_audit_fields]table_name
Reference Drop List Choose a specific reference from the selection to review only entries for that reference.
If there are no references used, this will display as '--All--'
Field Drop List Choose a specific field from the selection to review only entries for that field. [gb_audit_fields]field_name

Multiple records

When a screen displays multiple records, audit records are separated and can be shown for the records separately.
In this case, the record itself is selected, then the audit record can be viewed.
This is the case on the following screens:

Contracts / Events
Users / Logins
[edit]

Lookup Key Data for an ID

Audit fields can be configured to display a linked description rather than the stored value if it is an id that refers to another table.
E.g. Contracts / Order contains an "Order Status" field that displays a description on screen, but the value returned in the Audit History is the order_status_id.
If the lookup value is configured, [fm_order_statuses]description is displayed instead of [fm_contracts]order_status_id.
The user sees a screen value, rather than a database number.

When a lookup is in use, the Audit History tooltip displays the lookup table, field and value that has been used to link the lookup value.
E.g. 'fm_order_statuses: order_status_id: 100006'

Note: This does not apply to bespoke records which use a different technique for display.
