Controls - Quotes (QT)

ModuleNameCommentDefault Value
qt quote_effective_date_expiry_days Quotes / Contact "Effective Date" validation

When a quote is saved, a check is run to make sure “Effective Date” is not older than a defined number of days. If it is, the user is alerted and has the option to cancel the save and update the quote. The days are stored in [gb_controls]quote_effective_date_expiry_days and it has a default value of 90 days. This can be changed or make blank if you do not want the check to occur.

qt default_posting_map_id Quotes / Contact "Type Code"

If this control is populated, the stored Contract Type will will default into the Quotes / Contract "Type Code" field of new quotes.
When a "Client Code" is added, Quotes / Contract "Type Code" will switch to the client's Clients / Quote Defaults “Contract Type” (if one is stored) when a new quote is being created.

Only Contract Types where Contract Types / Details “Status” is ‘Active’ will be used in the logic described above.
