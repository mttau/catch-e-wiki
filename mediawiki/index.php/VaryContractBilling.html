Overview

Beta function

Allows you to Vary Billing details for a Contract that has started billing.
Contract Budget Web Services use an alternative authentication protocol.

[edit]

HTTP Method

Use the HTTP Method 'POST' for consuming this web service.

[edit]

Input Fields (Header)

  • Header should contain the 'Content-Type' Key with the Value = 'application/json' or 'application/hjson'.
  • If an Authenticated API user has the Audit:UserIdOverride permission, an optional Key 'Audit-User-Id' can be passed that will create the associated Audit Records with the user_id passed in.
[edit]

Input Fields (Body)

JSON FieldFormatNotesMandatory
contract_idIntContract IDYes
inertia_flagstringEither yes or no are the only acceptable values to be passed. Determines the contract's FBT days calculations.No
billing_intervalstringEnter a valid pay cycle code. Records where [gb_pay_cycles]billing_flag = 'yes' are valid. No
remaining_billing_periodsstringNumber of billing periods to create or remove.No
next_billing_datestringDate of the next budget period to billNo
billing_common_daystringFor monthly billing, select the day of the month charges will be due (1-31 or 'eom').
This field is ignored if the contract's "Billing Interval" is not 'Monthly'.
Do not use the values 29, 30 and 31.
Although they will be applied successfully, the correct logic is not being used when the period being created or updated does not contain this DAY.
E.g. if 30 is used, the date '28/02/2020' is updated to '30/02/2020'.
No
payment_methodstringPayment method preference for budgets. Choose from direct-debit, cheque, cash, other, credit-card, eftNo

JSON Example

https://api.catch-e.com/beta/fm/contract/billing/vary

{
    "contract_id": 100000,
    "inertia_flag": "no",
    "billing_interval": "monthly",
    "remaining_billing_periods": 10,
    "next_billing_date": "YYYY-MM-DD",
    "billing_common_day": 1,
    "payment_method": "direct-debit"
}
