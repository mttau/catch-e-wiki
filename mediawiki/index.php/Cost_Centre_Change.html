[edit]

Overview

Cost Centres are used in many invoices and reports and how they are entered can affect those reports.
If a quote has a recorded cost centre, this will be populated into the contract when it is accepted. The "Date" field is not populated in these opening events.

Cost centres can be entered either a single item, or multiples can be created to allocate a percentage of costs to different cost centres.

Cost centre changes can also be created using the API changeContractCostCentre.

[edit]

Create a Cost Centre Change event

When a vehicle is assigned to a new cost centre, create an new event to record this.
Only active Clients / Cost Centres records can be entered or selected.
Use the ?Search function to find and select available cost centres.

Record the "Date" when the vehicle changed to the new cost centre in the event. This date is used to select the correct cost centre in date-driven reports.
The "Allocation(%)" field will default to '100.00'. Leave this at '100.00' unless you are using the multiple cost centre feature.

[edit]

Multiples

Multiple cost centres would be created if percentage cost allocation is desired for that contract and only affect the Cost Report.
The Cost Report uses cost centre information stored in a table called fm_contract_cost_centre_allocations to drive its calculations.

All other reports use the cost centre that is displayed on the Contracts/Contract screen.
If more than one cost centre entry exists for a contract - the entry with the most recent date will be displayed on the Contracts/Contract screen.
If multiple cost centre entries have been entered - the last entry made with most recent date will be displayed on the Contracts/Contract screen.

Enter Date
When a cost centre is entered without a date on Contract creation, it will be treated as covering the whole life of the contract.
If a new cost centre(s) is entered, the previous cost centre(s) records will be updated with an end date of the previous day. E.g. a new cost centre that is entered with a date of 05/08/2019 will then update 04/08/2019 as the end date of the previous cost centre in the fm_contract_cost_centre_allocations table and used accordingly in the cost report. Where new cost centres are created without a specified date, the end date of this cost centre will be the day prior to the saved date of the new cost centre. E.g. new cost centre is entered dated 23/03/2020, then the end date of the previous cost centre will become 22/03/2020.
The most recent cost centre(s) entered will be treated as covering the remainder of the contract until a new cost centre(s) entry changes this.

Multiple Cost Centres
Each Cost Centre of a reporting set needs to be entered with the same date, then a percentage allocation (up to 2 decimal places) is entered.
If the percentages entered do not total 100%, an unallocated entry for the difference will appear on the Cost Report. These entries will need to be edited to equal 100% to correct the report. This will apply in both under-allocation and over-allocation scenarios, E.g. allocation of 90% would yield an auto-entry of 10% and an allocation of 120% would yield an auto-entry of -20%.
