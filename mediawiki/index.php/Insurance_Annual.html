Process Steps

Use this process when you have had a price change from your Insurer.

[edit]
Update Internal Insurer Records

Update your internal insurer records so that the information is correct for the upcoming year.
When new contracts are accepted from quotes, these insurance details are added to the contract if the insurer is an internal insurer.

  1. Go to Setup / Reference Data
  2. Status - select 'Active'
  3. Click on
  4. Click on
    1. Policy Start Date - update.
    2. Policy Due Date - update.
    3. Policy Number - update.
  5. Click on


When quotes are accepted, the following updates are made in the Contracts / Insurance tab from these records.

  1. "Start Date"
  2. "Policy Due Date"
  3. "Policy Number"


[edit]
Update Quote Pricing

Overwrite or add new records to the table qt_insurance_rates. The values here are used to calculate quote insurance amounts.
Do this update when most of your vehicle contracts will be insured and settled after this price change.
I.e. the contract insurance rates will be correct for the first payment
E.g. Do this in say - early June.
You will need assistance from your Catch-e Account Manager if there has been a change in the structure of your insurance locations or categories.

  1. Generate the report Ins - current qt_insurance_rates data'
  2. Use this as a basis to create an import file for the new pricing for quotes.
    1. This can use the existing system records to overwrite the current rates with your updated rates.
      1. If the current rates are overwritten - quotes with an earlier effective date will always recalculate, but you will not retain all rate changes in system tables
      2. The disadvantage: quotes with an earlier effective date will not recalculate with the latest rates
    2. Alternatively, use the report to create new records by leaving the id field blank and adding a new effective_date
      1. The effective_date can be in the future
      2. Quotes will not use the new rates until the effective_date has been reached, but old quotes will not recalculate unless the effective_date is changed.
  3. When prepared, import the file in Setup / Import Table Data
    1. Module - choose 'qt | Quotes'
    2. Table - choose 'qt_insurance_rates'


[edit]
Update Contract Pricing Table

Overwrite the records in the table fm_insurance_rates. The values here are used to check the amounts of contract payments and for bulk pricing updates via scheduler jobs.
No coded updates are done from records in this table.
Do this step when all of the payments that need to be made at the old rates have been posted (usually dated in June).
You will need assistance from your Catch-e Account Manager if there has been a change in the structure of your insurance locations or categories.

  1. Generate the report Ins - data for fm_insurance_rates update
  2. The report uses data from the quoting table [qt_insurance_rates] to calculate the expected rates to be in use in the contract payments.
    1. Save as an excel file for sorting and checking
    2. fire_service_levy - is calcuated at 1% for NSW and 0% for all other states
    3. stamp_duty - is calculated at 11% for SA, 10% for VIC, WA, NT and TAS, 9% for QLD, 5% for NSW and 0% for ACT.
    4. check these values against your insurers rate charts.
  3. Prepare file for upload
    1. Save as CSV for the upload step
    2. Remove the first three columns: quote_rates_date, Insurer, comment
    3. File should contain these columns: insurance_rate_id, supplier_id_insurer, insurance_policy_no, insurance_renewal, state, insurance_category, lower_limit, upper_limit, insurance_base, fire_service_levy, stamp_duty, broker_fee_annual
  4. Import file
    1. Go to Setup / Import Table Data
    2. Module - choose 'fm | Fleet Management'
    3. Table - choose 'fm_insurance_rates'


[edit]
Update Market Value

Use this optional process to update the Market Value of your contracts.
Depending on your insurer, you may be able to use Market Value as the Vehicle Value factor which will the premium to pay.

  1. Update the insurance_market_value of contracts with contract insurance payment schedules.
  2. Generate the report Ins - data for fm_contracts Market Value update
  3. Use this information to prepare an update file to update the market value of selected contracts.
  4. The file would contain contract_id,insurance_market_value'
  5. When prepared, import the file in Setup / Import Table Data
    1. Module - choose 'fm | Fleet Management'
    2. Table - choose 'fm_contracts'


[edit]
Update Contract Pricing

Do this step when all of the payments that need to be made at the old rates have been posted (usually dated in June).
Use the process below to update the remaining pending payments (usually dated July onwards).

  1. Generate the report Ins - check contract data quality
    1. Review and correct the data in contracts where there is a reported Data_Check issue
  2. Generate the report Ins - current and suggested amounts
    1. This reports the current amounts for the next pending payment against the amounts in the fm_insurance_rates table. I.e. the new updated pricing.
    2. This report preferentially selects 'Market Value', then 'Depreciable Value' as the insurance value factor. Contact your Catch-e Account Manager to check and update this preference if required.
    3. Where the VAR fields show no value, no change in pricing has occurred.
    4. Review the report and investigate records where the VAR amounts are unexpectedly high.
    5. Make data corrections if required and re-run this report as required.
  3. Check the records in Setup / Reference Data 'qt_internal_insurers' is correct, as the data is used in the report below
  4. Generate the report Annual Insurance Update - suggested _sch_insurance_import
  5. Save and import this file, go to Setup / Import Table Data
    1. Module - choose '_sch | Scheduler'
    2. Table - choose '_sch_insurance_import'
  6. Run the scheduler update Ins - contract and payments update - this will update the contract insurance and payment records stored in the =_sch_insurance_import table (updated in the previous step).
    1. Optional step - re-run the report Ins - current and suggested amounts. If the update is successful, there should be no VAR amounts showing.
