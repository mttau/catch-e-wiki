The 'Billing Type' field enables monthly billing to be run in any order without compromising client’s preferences for “Combined” or “Periodic” invoices. The default selection for this field is “Any”. E.g. When Billing is run in the Recharge screen, no invoices will show or generate for clients who have their Billing Type flag set to “Combined”.

[edit]

Billing Type Scenarios

Billing Screen Combined Periodic Any
Recharge/Standardn/a Maintenance Recharges and Bookings Maintenance Recharges and Bookings
Recharge/Fuel n/a Fuel Recharges Fuel Recharges
Periodic n/a Periodic,Credit and Inertia Periodic,Credit and Inertia
Initial n/a Initial Lease Charge Initial Lease Charge
Combined All charges n/a All charges


[edit]

Set-up Notes

A default can be set-up to set the default entry of "Billing Type" to the one you prefer to use - fm.client_billing_type_default. You can also exclude options you don't want used - fm.client_billing_type_exclusions.
E.g. you could set a default of "Combined" and prevent the use of "Any".

- If clients are set with an "Any" billing flag - the order that billing is run will determine how the item is billed.
