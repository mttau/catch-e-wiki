Package Item

Use the details below as a guide for setting up Package Items where the employer is being paid.

[edit]

Supplier

Create a supplier record for the employer. See also Create a new Packaging Supplier.

Detail - Supplier Code - for ease of use, use the same code as the employer's Client Code (not required, but just a suggestion)
Detail - Name - employer's name
Account - Status - 'Active'
Account - Supplier Type' - 'Salary Packaging'
Account - BSB Number - enter the employer's BSB Number
Account - Bank Account Number - enter the employer's Bank Account Number
Account - Payee - 'Supplier'
Account - Pay Method - 'Direct Credit'
Account - ABA Format Type
'Supplier' (if payments are to be aggregated) or
'Payment Reference' (if each payment should be made separately)


[edit]

Items

Set-up the item with the following attributes:

Deduction Details
Total Deduction - enter the total amount that will be paid back to the employer.
Compress - un-checked
Inertia - un-checked
Cycles - enter the number of pay runs the benefit will be spread over.
Disbursement Details
Disbursement Type - 'Recurring' or 'Claim'
Supplier - 'Employer' (choose the supplier created to match the client record)
Reference - leave this blank if the payments are to be aggregated. Enter a reference if payments are to be separate.
Payee - 'Supplier'
Payment Method - 'Direct Credit'
[edit]

Processing

The process steps for this type of item is as follows;

  1. Pay Runs / Pay Run - post the pay run record
  2. Receipts / Allocate - receipt and post the pay run
  3. Scheduler / Job - sp / approvals / autoBuildClaims runs automatically
  4. Contracts / Claim - the job creates a posted claim for the receipted amount
  5. BECS / Disbursements (ABA) - create a BECS batch and download the ABA file
    1. Payments can be grouped into one record or paid separately depending on your client's preferences.
  6. Banking Software - import the ABA file and confirm it for payment


When the pay runs and claims for this item are complete, no further deductions or disbursements will be made.
It can now be suspended.

Set-Up the Package Item

Field Value Description
sp_package_items
package_item_code RAA Create a short code for the benefit.
package_name Rent - Employer Displays as Employees / Item "Package Item" and is shown in the selection list.
This field also defaults into the Employees / Item "Description" field (which can be edited by the user).
benefit_type
cap_type 'primary' Select the required cap_type. This is used in FBT calculations and reports. If the benefit is exempt, leave this field as NULL.
gst_flag 'yes', or 'no' Set to 'yes' if the benefit will contain GST and the Employer will hold GST Invoices. When an item is made 'Active', the contract budgets will contain gst.
add_gst_flag 'no' Set to 'no'. This feature is not currently in use.
tax_invoice_flag
sp_profile_table_name NULL Leave as NULL for these benefits.
employee_contribution_flag 'no' Employees / Item "Employee Contribution" is enabled if this is set as 'yes'.
Do this if an Employee Contribution or Post Tax amount can be set for the benefit. E.g. Novated Leases or Associate Leases.
sp_package_item_defaults
total_deduction_default 0.00 Defaults into Employees / Item "Total Deduction".
Leave as 0 in most cases. Set a value if a known annual amount can be applied. E.g. Packaging Fees.
compress_flag_default 'no' Defaults into Employees / Item "Compress".
Checked items are calculated for the remaining pay periods in the current FBT year.
inertia_flag_default 'no' Inertia functions are not suitable for these benefits.
cycles_default NULL Defaults into Employees / Item "Cycles".
Use this to set a default for specific items. E.g. an Annual Fee that is charged once a year. This is mostly left as 0.
disbursement_type_default 'Recurring or Claim Defaults into Employees / Item "Disbursement Type".
supplier_id_default NULL Defaults into Employees / Item "Supplier". Leave this as NULL, so the correct supplier record can be chosen for these benefits.
supplier_id_edit_flag 'yes' Set this as 'yes' as Employees / Item "Supplier" should be editable.
substantiation_flag_default enum('yes','no') Set this either way. Set as 'yes' if you want to create a Contract Substantiation Event when an item is made 'Active'.
