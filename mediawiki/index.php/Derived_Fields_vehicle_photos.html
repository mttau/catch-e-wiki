Overview

This library is used to obtain a single photo code per variant.

This library can also be used with the following scheduler job methods: executeQuery, mailQuery and eventQuery.
This library can also be used with the following APIs: printReportQuery.

[edit]

Query Header

The following lines must be included in your query header: 

# odometer
# library_type standard
# library_path /lib/PDF
# library_name vehicle_photos


[edit]

Input Fields

The following Input Fields need to be added to your query:

Field Source Comments Mandatory
variant_id qt_variants → variant_id_id Yes


[edit]

Derived Fields

The following derived fields can be added to your query using the Derived Fields syntax:

Field Description
photo_code [qt_photos] table may contain multiple photo_code values for a given variant_id (e.g. front, rear, etc.). 'Front' is treated as the default followed by Side, Interior and Rear respectively. If no photo_codes exist, the value 'no_photo' will be returned.
