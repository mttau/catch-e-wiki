Overview

The details on this page are for setting up the banking and aba file records for Payments (ABA).

Visit the Receipts (ABA) and Reimbursements (ABA) and Disbursements (ABA) pages for their requirements, as they are different.

[edit]

Client Set-up Steps

Provide Catch-e with the bank account details you are using.
You can use multiple bank accounts in the Payments area and each links to a "Company".

Below are the details that are needed for a bank account:

  1. Abbreviated Bank Name eg: NAB, WBC, etc.
  2. Account Name
  3. BSB
  4. Account Number
  5. Company (only required if using multiple bank accounts). E.g. the name that is entered into the "Company" field on the Contracts / Contract tab


[edit]

Catch-e Set-up Steps

Where only one bank account is used for payments, [gl_bank_accounts] and [gl_bank_account_aba_mappings] need only be populated with the one required bank account's details.

The table gl_bank_accounts contains these fields:

Field Type Description
bank_account_idint(10) unique_id
gl_code_idint(10) Enter as 0 or add gl_code_id to link this to the gl_codes table for reporting. The field is not required for this functionality.
bankvarchar(3) Enter the bank code. E.g. ANZ, CBA.
account_namevarchar(26) Enter the account name. This is displayed on the Payments / Periodic screen in the "Bank Account" field.
account_name_shortvarchar(16) Enter the short account name, this is used to populate aba files.
bsbtinyblob
account_numbertinyblob


The table gl_bank_account_aba_mappings contains these fields:

Field Type Description
bank_account_map_idint(10) unique_id
bank_account_idint(10) unique_id must link to a valid gl_bank_accounts record.
process_typeenum('payables','receipts','becs','bpay') Enter as 'payables'. The other choices are not functional.
company_idint(10) Enter as 0 if you are only using one bank account. Enter a valid gb_companies record only if you are using multiple bank accounts. The [fm_contracts]company_id is used to filter payment records if multiple bank accounts are in use. This means you need to review and update your data to make sure the contract records are all correct and that each company has a matching bank account map record.
apca_numbervarchar(6) Enter the apca number if generating a payments aba file.
aba_descriptionvarchar(12) Enter the description you want for the aba file, e.g. Payments.
aba_self_balancing_flagenum('yes','no')This flag is used in generating the aba file layout. If this flag is set to 'yes', only Detail-Credit rows will be created. If 'no' then a Detail-Debit row is created also. The description of this row reads as 'Payments', 'Receipts' or 'Reimbursements' to reflect the transaction being generated, in this case it would show 'Payments'.

Useful Queries

select * from gl_bank_accounts; 

select * from gl_bank_account_aba_mappings where process_type = 'payables';

select ba.bank_account_id, ba.bank, ba.account_name, ba.account_name_short
, baam.process_type, baam.apca_number, baam.aba_description, baam.aba_self_balancing_flag
from gl_bank_accounts as ba
inner join gl_bank_account_aba_mappings as baam on baam.bank_account_id = ba.bank_account_id 
where baam.process_type = 'payables';
