Overview

The Driver Portal is built on software that is designed to run on mobile devices, but it will work as software running in a browser window.
If the configuration below is done, the software is able to be run as an app.
The benefits of do this are as follows;

  1. You can offer your portal to users as an app rom the Google Play and Microsoft App stores.
  2. The portal app will load with a splash page, which feels nicer to users
  3. Some portal activities can continue even if the phone drops out of cell reception range (but not making claims)
  4. Software updates to the app on the phone happen in the background


The portal is not currently available as an Apple app, but this configuration does give Apple users a nicer experience because

  1. The splash page function works
  2. If you use the Add to Home Screen feature to make a shortcut makes the portal look and feel like an app
  3. The Add to Home Screen feature also removes the browser footer bar

App Store Account

To do this, you will need to create and maintain an account with the Google Play and/or Microsoft App stores.
To proceed, contact your Account Manager to help you with the set-up steps.

[edit]

Set-up Steps

  1. Your Account Manager will raise a development task. We will add the PWA code to your portal software. (PWA stands for progressive web app).
  2. We will prepare the PWA wrapper for you. This is done using a tool located at https://www.pwabuilder.com
  3. The wrapper contains a file called assetlinks.json. This contains a field listing the fingerprint. We store the fingerprint detail in [gb_control]driver_portal_android_app_fingerprints for you.
  4. After testing, we will send you the app file you need to submit to the Google Play or Microsoft Apps store.
There are more details about the App Store set-up here: https://github.com/pwa-builder/CloudAPK/blob/master/Next-steps.md
