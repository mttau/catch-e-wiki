Overview

The Error Report identifies the items that caused the upload to fail. The file is exported in .csv format.
Below is a list of possible errors you may see in the validation file. Correct these issues and Re-Upload the file.
The file may refer to a header records or a row number. The row number is for the record that is affected. Each affected row will have it's own validation message.
If there is more than one issue with a row, you will see a validation message for each row issue.

If you are using executeImportTableData, the Error Report will be emailed to you if running the job generates it.

header - - - File contains duplicate headers: , Three reasons why you might see this error are as follows;
  1. The header row is missing from the upload file. Add the required header row.
  2. The upload file is not in *.csv format. Save into *.csv.
  3. You have blank columns saved in the file. See below for two methods for finding and deleting the blanks.

Method 1

  1. From the cell A1
  2. Click on Ctrl+End
  3. Highlight the blank columns
  4. Hover over the columns and click on Delete from the keyboard or the mouse
  5. Re-save the file

Method 2

  1. From the Home tab, click Find & Select button in the Editing section.
  2. Select Go To Special.
  3. Click the radio button for Blanks and click OK.
  4. The blank cells are now highlighted, find the Cells section of the Home tab and click the Delete arrow to select the Delete options
  5. Select Delete Sheet Columns
  6. Also check and if needed Delete Sheet Rows

-1 -1 Unsupported scientific notation It's likely that you've edited the upload file in Excel and the data has become corrupted.
Excel converts numbers over 15 digits into scientific notation (eg 5.4265E+12) and the data is changed when the file is re-saved.
Re-build your file and import the column into Excel as Text, not Number. This will prevent Excel from modifying the cell.

row_number -1-1 'field' Record does not exist The unique id of the record has been populated in the data file but does not exist in the database.

This can happen if you are uploading a data file to a different environment where the data is not the same.
Check the existing data in the table you are uploading to.
E.g. if uploading to Staging, download the existing file from Staging.

1. For existing records, you can update the records by importing records with an existing unique id
E.g. You are importing a record into [fm_drivers]. The field [fm_drivers]driver_id is populated and already exists in the database.
See more detailed illustrations in the Data File page in the Update existing records section.

2. For new records, leave the unique id field blank. The system will allocate the unique ids on import.
E.g. You are importing a record into [fm_drivers]. The field [fm_drivers]driver_id should be blank.
See more detailed illustrations in the Data File page in the Create new records section.

last_row_number -1 -1 (23000 - 1062 - Duplicate entry 'cell_content' for key 'field' )1. The file contains two records that contain the value for the primary key field, or unique key field, or

2. There is a record in the file that will create or update a record in the table to contain a value that already exists in another table record.

E.g. a [qt_client_defaults] upload contains two records with the same client_id value.

(23000 - 1062 - Duplicate entry '100030' for key 'client_id')

Solution: When importing existing records into the [qt_client_defaults] table, the two fields client_default_id and client_id must both be in the upload file and there can only be one row in the file that contains this combination.

E.g. The table [fm_cards] contains a primary key (card_id) and a unique key (supplier_id,card_number).
Search the file and then the existing table records for the reported value and update the file as required.
An [fm_cards] upload file contains two records with the same supplier_id and card_number values.

(23000 - 1062 - Duplicate entry '100459-12326389' for key 'supplier_id')

Solution: When importing existing records into the [fm_cards] table, the three fields card_id, supplier_id and card_number must all be in the upload file and there can only be one row in the file that contains this combination.

The system will only validate and report the first error of this kind that is found, so re-check your file carefully to remove any other errors like this or you will get a validation report for each one as it is found!
