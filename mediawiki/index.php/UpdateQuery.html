Overview

Update an existing Query stored in the [gb_queries] table.

[edit]

Permissions

To run this API, the nominated 'web-services' role needs to be given permission.
If you are not actively using the API, leave the permission off for better security.
Go to Roles / Apis and check on Queries:Update.

[edit]

URL Examples

https://api.catch-e.com/gb/queries/{query_id}
[edit]

HTTP Method

Use the HTTP Method 'PUT' for consuming this web service.

[edit]

Headers

Key Format Notes Mandatory
Audit-User-Id string Pass a user_id here to create audit records with this user's details.
This can only be done if the authenticated API user has Roles / Apis checked for the permission Audit:UserIdOverride 		No
[edit]

Body (JSON)

Example Value.

{
  "name": "COUNT of contracts with Rego",
  "description": "Count of Active Contracts with Rego",
  "query": "SELECT COUNT(*) FROM fm_contracts WHERE suspend_date = '0000-00-00' AND reg_no != 'ORDER';",
  "super_flag": "no",
  "status_flag": "active"
}

Queries are validated prior to being saved in the [gb_queries] table.
