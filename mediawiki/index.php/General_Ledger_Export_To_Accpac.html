Overview

This option is used to export a sales, purchases of journal file to the AccPac external accounts system.

The standard AccPac export files do not require a general journal export because the Sales Ledger export is split by cost code. Discuss with Catch-e if you require alterations to this method.

Note 20/06/2011 the export files were modified so that PL exports summarised by supplier invoice and gl code, i.e. contract level was removed. The SL exports were left at exporting at the contract level and may need to be modified if requested by future client users. As at 20/06/2011 the SL and GL Accpac exports are not used by any clients of Catch-e.

[edit]

Workflow

  1. Click on the ‘New’ tab
  2. Select Purchases or Sales from the select box
  3. Review Control Items and Control Total
  4. Click ‘Create’
  5. Click ‘Export’
  6. Save the txt file generated and import into the external accounts system
  7. Select the Control Type again, Export and Save if both Debits and Credits exist
  8. Click ‘Edit’
  9. Update the status from ‘pending’ to ‘exported ok’
  10. Add any notes if required.



Note that Credit transactions (i.e. the opposite to normal) are generated as separate files to allow exporting to Accpac separately. There is a single Export ID containing both the debits and credits.

The system will display 'Control Credits' and 'Control Credits' and an overall control total. You can select the Debit or Credit file individually using the "Select Control Type" drop down list per batch.

The ‘create’ action flags the relevant transactions as exported and will not be duplicated in any subsequent ‘create’ actions.

Copies of exported files can be re-created anytime by using the search icon and selecting the relevant file.

Interface File Specification

Purchase Ledger:

Header Row:

  1. H to denote header row
  2. Vendor Code (refer Global Control gl.accpac_pl_export_supplier_code nominating field to use)
  3. Invoice No (22chr CapAlphanumeric)
  4. Invoice Date (dd/mm/yyyy)
  5. Invoice Total Amount including GST
  6. PO number (22chr CapAlphanumeric) if any
  7. Order number (22chr CapAlphanumeric) if any
  8. Invoice transaction description (60 characters) if any

Detail Row:

  1. D to denote detail row
  2. Expense class code (gl code from contract types)
  3. Ex Tax amount
  4. Tax amount
  5. Purchase Item Tax Class
  6. Expense transaction description including the Supplier Ref from the Contracts/Payments screen (up to 60 characters)



Purchase Tax Classes are as follows: Tax Class Description

  1. Taxable Purchasers
  2. Input Taxed Purchasers
  3. GST Free Purchasers
  4. Capital Taxable Purchasers
  5. Capital Non Taxable Purchasers



Note detail rows containing GST are forced to have an exact GST amount based on the current GST rate (e.g. 10%). If a supplier invoice has a non 10% GST amount the detail row is split into 2 rows.

[edit]

Sales Ledger Interface

Header Row:

  1. Contract ID, 6 numerics
  2. Invoice No, 22chr CapAlphanumeric
  3. Invoice Date (dd/mm/yyyy)
  4. Invoice Total Amount

Detail Row:

  1. Item Category, see list below
  2. Ex Tax amount
  3. Tax amount
  4. Sales Item Tax Class, see list below
  5. Quantity,
  6. Unit billing rate (ex GST)



Sales Tax Classes are as follows: Tax Class Description

  1. Taxable Items
  2. GST Free Items
  3. Input Taxed Items
  4. Capital Taxable Items
  5. Capital Non Taxable Items
