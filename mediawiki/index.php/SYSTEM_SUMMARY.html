System Configuration

The system is spread between the Users, Hosting Provider and Catch-e. Each of these entities will have the following:

  • Users:
    • PC or Laptop
    • Web Browser program such as Microsoft Internet Explorer
    • An Internet connection via dial up modem, broadband or network server
    • Communication capabilities via e-mail and telephone
  • Hosting Provider:
    • Clustered server environment
    • Linux Operating System
    • MYSQL database
    • PHP scripting language
    • Firewall protection
    • Data Backup and Restore facilities
    • Technical support and constant overview of the technology communication capabilities via e-mail and telephone
  • Catch-e:
    • Internet connection e.g., via ADSL broadband or Fibre Optic Cable
    • PC’s and Laptops
    • Communication capabilities via e-mail and telephone
    • Support services including backup server
    • Development environment
    • Technical support
    • Technical documentation
    • Consultancy services

Operating System and Browser settings

Catch-e.fleet is designed to run on a variety of operating systems as the browser is usually the critical software for interfacing with Catch-e.fleet. Certain settings may be required to ensure smooth operating of the system.

For detailed instructions refer to Operating System and Browser settings

User Access Levels

User access is controlled by roles and permissions at each screen level. The system has pre defined roles but these may be further modified per database. Example Roles and access levels may be:

  • Super
  • Admin (can manage users).
  • User (standard internal user, no access to finance and lease screen unless contract type is Novated, do not have access to Export Data).
  • Disposals - as per 'User' but also has access to Sale Management (note: Residual value fields are non editable on the Sale/Detail screen).
  • Driver (External Novated Drivers: restricted by fm_drivers table and driver_id field).
  • Fleet Manager (External Fleet Managers: restricted by fm_client table and client_id field).
  • Group Manager (External Group Manager, restricted by fm_client_group table and client_group_id).
  • Demo
  • Manager (Internal Manager. Includes Fleet/Import Data).
  • Accounts (Internal User, has access to finance and lease screens and Card Services. Includes Fleet/Import Data).
  • Finance - as per Accounts but with additional access to Client/Quote defaults and Client/Financiers screens. Includes Fleet/Import Data
  • Warehouse (limited user for viewing maintenance screen only).
  • State Manager (Excludes finance and lease screens, can edit/delete events).
  • Fleet Manager (read only). As per external Fleet Manager access above but with read only access.
  • Procurement (access to all quote screens and setup/reference data - similar to Accounts role. Includes Fleet/Import Data).
  • Introducer. Has access required to create lease quotations, manage clients, suppliers, drivers and access to the following Fleet/Administration/Contract Screens:
    • Vehicle
    • Options
    • Order
    • Events
    • FBT
    • Maintenance
  • Quotes Consultant. Has access to create lease quotations, manage clients, suppliers, drivers and order management with access to the following screens:
    • Fleet/Admin/Quotes:All screens
    • Fleet/Admin/Contracts:Contract, Vehicle, Options, Order,Finance,Billing,Lease, Events, FBT and Maintenance screens.
    • Fleet/Admin/Drivers screens (all)
    • Fleet/Admin/Clients (excludes quote defaults and finance screens)
    • Fleet/Admin/Suppliers
    • Fleet/Admin/Events (all)
    • Fleet/Admin/Order Management
    • System: change password only.
  • Package Employee. External user with access to salary packaging screens and reports.
  • Package Employer. External user with access to salary packaging screens and reports limited to their employees.
  • Package Advisor. External user with access to salary packaging screens and reports limited to employees linked directly to the individual advisor. When an employee is granted web access via the "Login Access" tick box on the Packaging/Employees/Contact screen the linked advisor is notified via an email message.




Manager access is required to issue approvals for suspended vehicles.

Future development will include at each screen permissions per role to be set as:

  • No access
  • View access only
  • Edit and post existing records
  • Create new records, delete records (where applicable), edit and post.

Contingencies and Alternate Modes of Operation

In the event of a disaster the hosting provider is initially responsible for restoring the system and database. If the hosting provider fails to do this, Catch-e may host the system themselves using internal resources or alternatively install the system on the user’s PC or network.

In the event of a user’s Internet connection failing, alternative Internet access should be arranged, which may be in the form of a dial up connection or connection from an alternative location.
