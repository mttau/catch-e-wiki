Overview

Create the contract event emails that you want using standard email text.These can then be generated and stored from the Contracts / Events tab.

Download this request form Contract Event Email Request and send it to Catch-e Support if you want to set-up a Contract Event Email.

Note: This work is typically chargeable: 1.5 hours for new Contract Event Email being set up. 1 hour to modify existing Events.

[edit]

Extras

Use the standard contract event email set-up with an extra setting to;

  1. Create a driver access email event
  2. Create an employee access email event
[edit]

Client Set-up

1. Go to Posting Classes / Details and create the record you want to use. This must have the "Event" check box selected.

2. Download a Contract Event Emails, fill out the contract event email request document and enter the email details you want.

3. Go to the Contract Event Email Placeholders page to see all the available standard placeholders you can use in these emails. It helps us if you populate your document with these placeholders as they are listed.

The document contains details such as

  1. Posting Classes / Details "Class code"
  2. Email Type (Text or HTML). If HTML, we'll need that file too.
  3. Sender Name and Address (this can be either the user that is sending the email or you can nominate a generic name and email address (E.g. Catch-e Support, support@catch-e.com.au))
  4. Recipient Email (you can have more than one, they can be To:, CC: or BCC:)
  5. Subject Line
  6. Body Text
  7. Static attachments (if required)

Send this completed form to Catch-e Support and they will set up the email for you.

[edit]

Catch-e Set-up

  1. Populate gb_templates with the mail message, using pdf_contract_event_email_message as the template name.
    1. Enter the related posting_class_code into the sub_name field
    2. Populate mail_sender_name and mail_sender_address if the sender details are to be generic, otherwise leave blank and the users details will be applied to the sent emails.
  2. Add the template_id into the selected [gl_posting_classes]template_id field
  3. Add the recipients to gb_template_recipients
  4. If attachments are being used, add them to gb_template_attachments
  5. If the email is for a Certificate of Currency, check that the Suppliers/Account field ""COC Template" is populated with the appropriate template.

Userful Queries

select pc.posting_class_code, pc.system_event_flag
, t.template_id, t.name, t.sub_name, t.mail_sender_name, t.mail_sender_address, t.mail_subject, t.mail_message, t.mail_message_format, t.mail_footer_flag
, tr.type, tr.name_from_field, tr.address_from_field, tr.status_flag
from gb_templates as t
inner join gl_posting_classes as pc on pc.template_id = t.template_id
left join gb_template_recipients as tr on tr.template_id = t.template_id
where t.name = 'pdf_contract_event_email_message'
and pc.system_event_flag = 'no'
;
