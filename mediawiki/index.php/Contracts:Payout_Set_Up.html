Show Payout tab

  • The Payout tab is only visible on contracts where the Contract Type has been set up to show the Payout tab.
  • This can be achieved by confirming the following;
1. Go to Fleet / Controls / Accounts / Contract Types
2. Nominate the Contract Type for which the Payout tab will be required
3. In the Details tab, Check on the "Payout Tab" Flag, to make the Payout tab visible in the contracts area, Check this off to disable the function. This can be done by selecting the button.


[edit]

Send emails and create an event

The [gb_control]payout_notifications has multiple parameters you can set to toggle emails and events on or off and are as follows;

ParameterDefault ValueComments
Payout Notification RequestednoSet this to 'yes' if you want to send an email to the recipient/s listed in [gb_templates_recipients]address when a payout record is updated to 'Requested' status.
Payout Notification Requested CC FinanciernoSet this to 'yes' if you want to send an email to the contract's financier. It will send to the address stored in the Suppliers / Financier "Email" field if populated.
Payout Notification OpennoSet to 'yes' if you want to send an email to the user that created the payout record when a payout record is updated to 'Open' status.
Payout Notification AcceptednoSet to 'yes' if you want to send an email to the recipient/s listed in in [gb_templates_recipients]address when a payout record is updated to 'Accepted' status.
Create EventnoSet to 'yes' if you want a contract event to be created when a payout record is created.

If an event is created, the event "Date" is the same as the payout "Date", the event "Value" and "Description" are left blank, "Action" will be 'None' and the "Completed" checkbox will remain unchecked.

Event Posting Class Code Enter the Posting Class Code you want used to create contract events when Create Event = 'yes'

Email messages

The content of the emails trigged by the controls above are stored in gb_templates
You can update the content of the emails by updating the text and using the available payout email placeholders. To finish the set-up, add records to gb_template_recipients to define who will receive the emails below. The recipients must be defined addresses, not user placeholders.

Payout "Status"namemail_subjectmail_message
Requestedpdf_payout_requestedPayout Request for Contract: #contract_id#, Reg: #reg_number#A payout value has been requested by #full_name# for Contract: #contract_id#, Reg: #reg_number#.

Please send Contract Payout advice to #client_code# - #client_name#.

Acceptedpdf_payout_acceptedPayout Accepted for Contract: #contract_id#, Reg: #reg_number#A payout has been accepted for Contract: #contract_id#, Reg: #reg_number#.

Useful queries

SELECT * FROM gb_templates WHERE name IN ('pdf_payout_open','pdf_payout_requested','pdf_payout_accepted');

SELECT t1.*,t2.* 
FROM gb_templates AS t1
LEFT JOIN gb_template_recipients AS t2 ON t2.template_id = t1.template_id
WHERE t1.name IN ('pdf_payout_open','pdf_payout_requested','pdf_payout_accepted');
