[edit]

Schedule payments

You can create scheduler jobs to run the periodic payments automatically. If the supplier is set-up to receive a remittance advice by email, the email will be sent when the job is run.
A scheduled job will not generate a Payments (ABA) file. You must do that manually in the Payments screen.

To Pay
BlankPendingDisplays or select'Email' or 'Post'DisabledDate'Email' or 'PDF'
  1. Select a "Supplier Code" if you want to limit the payment to one supplier
  2. Select 'Pending' status. This disables the "Dated From" field
  3. Select a "Payment Type", one must be selected
  4. If you have multiple bank accounts, select the correct "Bank Account"
  5. Select the required "Delivery" method, one must be selected. The "Output" field will default in 'Email' or 'PDF' to match what you have selected.
  6. Enter the "Dated To" date you want to select all records dated on or before that date for payment
  7. Click on Go
  8. Click on Pay
  9. If you selected a delivery of 'Email' you can review the email attachments then close the email window
  10. If you selected a delivery of 'Post' you can review, save and print the pdf outputs then close the output window
  11. Post or if there is something wrong with the payment, click on Abort to back out of the payment.
To Print a Text(ABA) file
Select or enter Payment NoPostedDisplays or select--All--DisabledDisabledText(ABA)
  1. Select the 'Posted' status. This enables the "Payment No" field
  2. Enter or select a payment no from the lookup. This will disable the "Dated From" and "Dated To" fields and remove their values
  3. If you have multiple bank accounts, select the correct "Bank Account"
  4. Change the output from 'Text(CSV)' to 'Text(ABA)'
  5. Click on Go
  6. Print the aba file.

Changing the status away from 'Posted' will enable the "Dated From" and "Dated To" fields and disable the "Payment No" field.
If status is 'Posted' and "Payment Type" is 'Single Posting Class', then 'Text(ABA)' option is not available.

To Un-Post a payment
Select or enter Payment NoPosted--All----All--DisabledDisabledText(CSV)
  1. Select the 'Posted' status. This enables the "Payment No" field
  2. Enter or select a payment no from the lookup. This will disable the "Dated From" and "Dated To" fields and remove their values
  3. Do not filter any other fields. Filtering any other fields will disable the Un-Post button
  4. Click on Go
  5. Click on Un-Post

Changing the status away from 'Posted' will enable the "Dated From" and "Dated To" fields and disable the "Payment No" field
