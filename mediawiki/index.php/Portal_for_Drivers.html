Overview

This portal is known as the Driver Portal as the early features were tailored to the Driver role, but it now contains features used by the Package Employee role also.

The setup steps below apply to both the Driver Portal and the Employee Portal, which contains additional functions to cater for packaging information.

[edit]

URL Access

Standard Driver Portal URLs are as follows:

CLIENTNAME.driver-portal.catch-e.com
CLIENTNAME.driver-portal.support.catch-e.com
CLIENTNAME.driver-portal.test.catch-e.com
CLIENTNAME.driver-portal.staging.catch-e.com

[edit]

Old Driver Portal Setup

[edit]

Overview

This portal is known as the Driver Portal as the early features were tailored to the Driver role, but it now contains features used by the Package Employee role also.

The setup steps below apply to both the Driver Portal and the Employee Portal, which contains additional functions to cater for packaging information.

Portal for Drivers

[edit]

URL Access

Standard Driver Portal URLs are:

CLIENTNAME.driver-portal.catch-e.com
CLIENTNAME.driver-portal.support.catch-e.com
CLIENTNAME.driver-portal.test.catch-e.com
CLIENTNAME.driver-portal.staging.catch-e.com

[edit]

Client Set-up

  1. Provide Catch-e with the following;
    1. The primary and secondary colour you would like to use in the portal.
    2. Confirm with Catch-e which features should be made available through the Portal i.e. View my balance, Change password and Send an enquiry can be hidden should they be unwanted features.
    3. If you want full bank account numbers displayed or partially hidden.
    4. For the claim lodged confirmation email, provide the email wording and design. Placeholders that can be used in this email can be found in Driver Portal Email.
    5. For the 'Send an Enquiry' email, provide the email wording and design.
    6. For the 'Send an Enquiry' screen feature, provide the list of enquiry reasons you would like to make available.
  2. Check and update Fleet / Controls / Accounts / Contract Types “Claims Tab?” field. This must be checked ON for eligible Contract Types.
  3. Check and update Fleet / Controls / Interfaces / Card Services ‘Reimburse’ interface mappings. Only stored mappings will be available for selection, regardless of the contract’s budgets. Only budgets that are mapped and are also stored in the contract will be available in the portal screens. Note: Where a product uses VMRS Code 'REG' the driver is permitted to input 0.00 for the GST value. In all other cases a GST value is mandatory, and is either system calculated or input by the driver.
  4. For Apple devices where IOS 13 or higher is installed the camera will not work correctly without first setting up Safari settings on the device. Refer to iPhone IOS 13.
[edit]

2 Factor Authentication

To use 2 factor authentication (2FA), follow the steps below. Also review the 2 Factor Authentication Setup page for an overview of this feature.

  1. Go to Setup / Reference Data and select the gb_controls table. Find the record for "2factor_auth_enabled_flag" and set this to 'yes'. Note: This setting controls 2FA at the highest level - if this is set to 'yes', the menu item is exposed to internal system users who can then optionally enrol for 2FA. Visit 2 Factor Authentication Setup for the full details of this feature.
    1. Check and update System / Roles for 'driver'. If the "Mandatory" check box is left un-checked, the driver can optionally choose to use 2FA. If it's checked, the driver is required to use 2FA and will be taken to the enrolment screen on login.
  2. If the driver has a problem using 2FA, 'admin' users can unenroll the driver by un-checking the Drivers / Contact "2FA Enrolled" check box.
[edit]

Global Controls Set-up

  1. The Forgot Username? Forgot Password? set-up details are documented in the Reset Password page.
  2. Populate [gb_control]driver_portal_terms_of_use_text if you want to have a Terms of Use pop-up appear when a driver clicks on Raise a Claim. This pop-up will only appear once each time a driver logs in. E.g. if a driver makes three claims in succession, the Terms of Use pop-up will only appear on the first claim. If the control is left blank, the pop-up does not appear ever.
  3. Populate [gb_control]driver_portal_attach_receipt_text if you want to add text to the Attach Receipt screen. If the control is left blank, there is no additional text.
  4. Set [gb_control]driver_portal_display_bank_details_flag as 'no' to partially hide the driver's bank account number in the Banking Details screen.
  5. Check setting of [gb_control]approval_reimbursement_notification_flag. Claims will not be emailed on confirmation if this is set to 'no'. If 'yes' the email template detailed below must be set-up, or the Portal screens will not function correctly.
  6. If an lodgement confirmation email is to be sent, you need to do the following;
    1. Set up email template [gb_templates]pdf_contract_event_email_message where sub_name = 'PORTAL'. Put in the message the driver is to receive. Refer to the Driver Portal Email page for available placeholders.
    2. Create a Posting Class event to record the email against. The "Class Code" must = 'PORTAL'. This needs to be linked to the email using [gl_posting_classes]template_id.
  7. Set up [gb_control]driver_portal_enquiry_enabled_flag as 'yes' to make the 'Send an Enquiry' button available on the Home screen and then do the following
    1. Create records in the table fm_driver_enquiry_codes to reflect the reasons the client wants.
    2. Set up email template [gb_templates]pdf_driver_event_email_message where sub_name = 'ENQUIRY'. Put in the message your staff should receive. Refer to the Driver Portal Email page for available placeholders
    3. A Driver Event called ENQUIRY must exist if you want to record the emails in the system. The entry must be added into the table fm_driver_event_codes.
  8. Review the message stored in [gb_control]driver_portal_no_claims_text. If the Drivers / Contact field "Driver Claims" is un-checked, the driver is prevented from raising claims in the Driver Portal. When they click on the Raise a claim button, this message is shown and the user is kept on the Home screen.
  9. Review the file size limit allowed for an individual file being attached to a claim [gb_controls]driver_portal_attachment_max_size_in_bytes. The default value is 20971520 (20 MB).
  10. Review the total size limit allowed for the total of all files being attached to a claim [gb_controls]driver_portal _total_attachment_max_size_in_bytes. The default value is 104857600 Bytes (100MB).
  11. Set [gb_controls]driver_portal_use_gl_posting_map_reporting to 'yes' if you want the View my vehicle balance to display values as reported in the Funds Check pop-up screen.
  12. Review the text stored in [gb_controls]driver_portal_view_my_balance_text. You can customise the text to be displayed on the new style View my balance report. Default text is:
    Notes:
    Funds received are those processed at report date
    Pending and requested claims are not included

Where Funds Check is in place

If the Funds Check is already in use, the set-up steps for using the same data on the View my vehicle balance are as follows;

  1. Set [gb_controls]driver_portal_use_gl_posting_map_reporting to 'yes' as noted above. (Set to 'no' if you are not using Funds Check).
  2. Create a record in gl_posting_map_reporting for each Contract Type that should have this feature. Set the [gl_posting_map_reporting]type to 'driver_portal'.
  3. Set the [gb_report_queries]status_flag for both versions of the 'driver_portal' report record to 'active'. One serves the internal users and the other defines the data and report columns available to drivers in the Driver Portal.
  4. Check the values stored in [gb_companies]name as it may be used in the description field for Receipt and Budget transactions. There must be an entry in [fm_contracts]company_id for this to description to appear correctly on the report.

Visit Screen Reports for more details about [gb_report_queries].
