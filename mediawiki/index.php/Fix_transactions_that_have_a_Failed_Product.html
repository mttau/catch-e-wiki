Fix transactions that have a Failed Product

From Help wiki

Jump to: navigation, search

Main Page → Workflow → Fleet Management → Card Services → Fix transactions that have a Failed Product

Qualifiers

  1. You have imported a fuel transaction file and it is at 'Pending' status
  2. The Transactions / Batches screen contains transactions with a Validation of 'Failed Product'

Why this happened

  1. This will happen if the product stored in the transaction is not mapped in the Card Interface.
    1. Either you are using the interface for the first time or
    2. The card provider has added or renamed product codes

Identify the missing products

  1. Go to the Fleet / Card Services → Transactions / Batches tab
  2. Validation - choose 'Failed Product'
  3. Click on
  4. If there's only a couple of failed transactions
    1. Click on Batch-Item of the first record to navigate to the Transactions / Transaction tab, which shows this record in detail.
    2. Note the displayed Product Code
  5. If there are many failed transactions (indicating a number of products that need to be added)
    1. Click on to export a CSV file that lists all the failed products
    2. Sort the spreadsheet by product and consolidate into a list
    3. Use this list to compile the information you need (steps below) to update the Card Interface mapping
  6. Using this information, look at the raw file you imported and other channels to decide what kind of product this is

Find or create the required VMRS codes

  1. Each new product will need a matching VMRS code in the Card Interface mapping
  2. Go to the Fleet / Controls / Maintenance → VMRS codes / Details screen
  3. Search for and note a suitable VMRS record to use with this product
  4. If one can't be found, create a new one

Enter new card interface products

  1. A new interface mapping record needs to be added for each card services product
  2. Go to Fleet / Controls / Interfaces / Card Services
  3. Card Interface - enter or lookup the required code
  4. Navigate to the Card Services / Mapping tab
  5. Click
    1. Product Code - enter the new product code.
    2. Description - enter the suppliers product description
    3. Class - enter the class you want this product to be allocated to
    4. VMRS - enter the VMRS code that is suited to this product
    5. Click on

Re-check your pending card transaction batch

  1. Go to the Fleet / Card Services → Transactions / Batches tab
  2. Click on
  3. If the validations are all fixed
  4. Click on
