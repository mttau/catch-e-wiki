Owl → Add → Populating Vary Button for an Item screen


Replaced by Activate an item


Qualifiers

  1. The item's Status is 'Active'


Populate vary button

  • If any billing records have been posted, show a Vary button. "Start Date" and “Contract ID” will not be editable.
  • If the “Disbursement Type” is ‘Periodic’ and payment records have been posted, “Disbursement Type”, “Supplier” and “1st Payment Due” will not be editable. Also, “Number of Disbursements” cannot become lower than the last posted disbursement record.
  • The fields “Cycles” and “End Date“ cannot become lower than the last posted billing record.
  • If “Pay Cycle” is changed, the stored “Cycles” and “Start Date” values should remain in place. “End Date” should be recalculated. Once “End Date” is recalculated, the user can change “Cycles” and “End Date” fields as required.

Saving an edited Item

On Save, update contract records as required of the below screens .

Billings

1. Lease Term
2. Description
3. Contract Start
4. Contract End
5. Periodic Interval
6. Total Billings
7. Periodic Billings
8. Start Periodic
9. Remaining

Lease

1. Period – Number and Date
2. Budget Amounts

Payments

1: Supplier
2. Supplier Ref
3. Amount
4. GST
5. Periodic Interval
6. Due Day
7. Total Payments
8. 1st Payment Due
9. Final Payment Due
10. Payment Amounts
