Pay Run Process

Jump to: navigation, search

Main Page → Workflow → Salary Packaging → Salary Packaging Processes

  1. Use ToDo list to check required actions
  2. If Employer gets preview reports
    1. Create the client's Pay Run
    2. Print and save required reports
    3. Email reports to client
  3. If Employer requires additions to the Pay Run
    1. Add items to the Pay Run
    2. Go to Employees / Item - create required items and make them active
  4. If Employer requires edits to the Pay Run
    1. Navigate to Pay Run
    2. Hyperlink to record using Contract ID
    3. Go to correct billing and Un-Post
    4. Edit billing record (to change only this pay run)
    5. Vary billing record (to change this pay run and future pay runs)
  5. Invoice added or edited records
    1. Navigate to Pay Run
    2. Create second invoice
  6. If Employer requires deletions from the Pay Run and record should be carried over to next pay
    1. Navigate to Pay Run
    2. Hyperlink to record using Contract ID
    3. Un-Post record to remove items from the Pay Run
  7. If Employer requires deletions from the Pay Run and record should not continue
    1. Navigate to Pay Run
    2. Hyperlink to record using Contract ID
    3. Un-Post record to remove items from the Pay Run
    4. Hyperlink to Item
    5. Suspend Item using prior Pay Run date as End Date
  8. Re-Print reports for Posted Pay Run
  9. Receipt Pay Run
  10. Auto build claims
  11. Auto post payments (Scheduler Job)
  12. Auto post claims/sp (Scheduler Job)
  13. Auto post claims/fm (Scheduler Job for linked Novated Leases)
  14. Disbursements (ABA) (for salary packaged items)
  15. Reimbursements (ABA) (for linked novated leases)
  16. Disbursements BPAY (for salary packaged items)
