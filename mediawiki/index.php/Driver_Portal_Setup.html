Overview

This portal is known as the Driver Portal as the early features were tailored to the Driver role. A variant of the portal can also be used for the Package Employee role also. Different screens and wording apply.

The setup steps below cover both the Driver Portal and the Employee Portal.

Your Account Manager will guide and assist you in setting up your portal.

[edit]

Channel Portals

Separate Driver and Employee Portals can be setup for different Channels.
Each Portal is created with separate URL Access.
Developing a channel portal is chargable and some differences in the setup steps apply.

[edit]

URL Access

Standard Driver Portal URLs are as follows:

CLIENTNAME.driver-portal.catch-e.com
CLIENTNAME.driver-portal.support.catch-e.com
CLIENTNAME.driver-portal.test.catch-e.com
CLIENTNAME.driver-portal.staging.catch-e.com

[edit]

Screen Text

When configuring your portal, review the text shown to users on screens.
Changes can be made to the text to suit your needs.
Visit the Driver Portal Text page for more details.

[edit]

App Store

The portal software can be configured for use as an app in the Google Play and Microsoft App stores.
Visit the Driver Portal App page for this.

[edit]

Access

Drivers / Contact → Access - set up the driver's access email.
Employees / Contact → Access - set up the employee's access email.

If you have Channel Portals, each channel will need a separate access email template.

Login

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gb driver_user_login_alias_enabled_flag Set this as 'yes' to use the driver's "Email" address as their login. The default is 'no', and in this case, the login is the "Driver ID".

Note1: Make sure the control is set to 'yes'. Any setting such as 'Yes' will not work.
Note2: "Driver ID" is always available as a login, even if the email address is enabled.

 yes
gb driver_temporary_password_flag When set to 'yes', the driver is issued with a temporary password when "Access" is checked to enable a Driver Portal login.
A driver can be given a login without an active vehicle contract in this situation.
Set this to 'no' to disable the feature.

This control does not apply to the Employee Portal where a temporary password is a standard feature.

no

View my salary package balance

This screen shows a listing of your packaging transactions.
The display can be restricted to show transactions from a nominated date.

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gb driver_portal_view_my_package_balance_from_date Use this control to restrict the transactions displayed on the ‘Transactions’ screen in the Employee Portal. The default setting is for no restrictions, so all stored transactions will be displayed.
Enter a date in the '"Value" field to show only transactions dated on or after the entered "Value" date. Acceptable date formats are 'dd/mm/yyyy' and 'yyyy-mm-dd'. No other date format will work.

Use the "Effective From" date to specify when the control should be invoked. E.g.

Effective FromValueComment
01/04/2022 Until 01/04/2024, all transactions on or after 01/04/2022 will be displayed.
01/04/2024 01/04/2023 From 01/04/2024, all transactions on or after 01/04/2023 will be displayed.

Raise a claim

This button may be called Raise a claim or Raise a vehicle claim depending on your system's configuration.
1. Review the Claims → Set-up page, this covers the system set-up requirements for the Contracts / Claims tab, as the configuration steps are also necessary for this process.
2. Visit the REIMBURSE page.
3. Where a mapping uses the VMRS Code 'REG' the driver is permitted to input 0.00 for the GST value. In all other cases a GST value is mandatory, and is either system calculated or input by the driver.
4. Catch-e recommends that the Approvals module of the system also be used. This enables you to monitor claims more easily than from the Contract module.

Fuel Claim Scheduler Jobs

Visit the Fleet Management Scheduler Jobs page for details of two standard scheduler jobs you can activate.
Use these jobs to:

  1. Review your current portal contract settings for Fuel / Charging
  2. Automatically create the event required to enable access to the Home Charging screen for eligible contracts.

Note:
Currently only events to enable access to the Home Charging screen are created by the scheduler job.
If you wish to enable the Home Charging Percentage screen for vehicles, you will need to need to determine your own logic for allowing this and update the vehicle's event selection.
If you want to automate the logic you use use for Home Charging Percentage, contact Catch-e Support and send in a Report Request to make changes to the event creation job.

Change password

The Change password button is one of the default menu options. Enable the Temporary Password feature if you want to enforce a password change for users when they first login, or when their password has been re-set.

Hide the Change password button if you are using the Generate a signon token API to by-pass the login screen. If Roles / Details "Single Signon" is checked for the 'driver' or 'employee' role, this button will be hidden.

Setup / Reference Data' → gb_controls
ModuleNameCommentDefault Value
gb temporary_password_flag Set this to 'yes' to direct users to the Change password page when they first login, or when their password has been re-set.
If this is set to 'no', users can change their password at anytime, but only when they choose to do so.

Visit Temporary Password for more details about this feature.

yes
gb driver_temporary_password_flag Set this to 'yes' to direct users to store temporary passwords for drivers in [fm_drivers]generated_passwords, save to Audit History, and update the [gb_user_passwords] to display allocated temporary password in email [gb_templates]pdf_client_login. No change to employee portal.

If this is set to 'no', users can retain to existing behaviour for both Driver Portal and Employee Portal.
Visit Temporary Password for more details about this feature.

no
