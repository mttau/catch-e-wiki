Overview

The vehicle screen is used to select the vehicle required and to display and allow editing of vehicle pricing including discounts etc. See below for a sample vehicle screen with a variant and options selected.

[edit]

Selecting a vehicle

Vehicles are selected by using the drop down field selections. Each selection controls the data to be displayed in the subsequent field, i.e. choosing the make "Holden" will restrict the model list to only Holden models etc. Note that Makes, Models and Variants are accessible only if your system administrator has made them "Active" in the Setup/Reference data section of the system.

Vehicle Photo

If a vehicle photo exists in your data, it will be displayed. There can be more than one image stored or referenced for a variant and they are selected for display in the following priority order; Front, Side, Interior and Rear.

BonusNumThese are stored in qt_variant_bonuses by bonus_type.
Dealer DiscountNumThese are stored in qt_variant_dealer_discounts. The supplier_id (dealership)and client_id are optional in this table. If a preferred dealer is selected, any associated discount in this table for this variant will be used.

If a client code and variant are selected and there is a matching preferred dealer in the qt_variant_dealer_discounts table and the location states match, then the system will pre-select the preferred dealer and discount amount. The following business rules are used when calculating dealer discounts:

  1. Changing the State code or Client Code on the Quotes / Contract tab will lookup the Preferred Dealer from the qt_variant_dealer_discounts table using the variant_id, client_id and state at the current quote_date. It will then look up the discount for that supplier from the qt_variant_dealer_discounts table and recalculate the values for dealer_discount_net and related fields. If there is no Preferred Dealer found then the field on the Quotes / Vehicle tab will be set to blank and the discount used to calculate dealer_discount_net set to zero.
  2. Changing the Variant on Vehicle page will lookup the Preferred Dealer as before, but if no Preferred Dealer is found, the existing Preferred Dealer will be left unchanged.
  3. Changing Preferred Dealer from either the input field itself or from the search pop-up, will lookup the discount for that supplier from qt_variant_dealer_discounts and recalculate the values for dealer_discount_net and related fields. If the supplier has no entry in qt_variant_dealer_discounts a zero discount will be returned.
  4. Changing Preferred Dealer from either the input field itself or from the search pop-up, will give a warning if the new Preferred Dealer is in a different state (using the field [fm_suppliers]state) to the Delivery State.

Quotes / Vehicle - I can't find a variant

If you can't select a new vehicle model, it's likely that the data hasn't been added to your database yet by your provider.
Check that your scheduler job is running OK. This will be either

  1. System / Scheduler → Method → qt / glass / importVehicleData
  2. System / Scheduler → Method → qt / red_book / importVehicleData

If the job is running OK, contact your data provider to find out when the records are expected to be added.
Sometimes there is a delay between seeing variants on the RedBook or Glass's website and your data feed.

If your data provider has confirmed the variant has been added to your feed, contact Catch-e Support and provide a RedBook Code or an NIVC code for investigation.
