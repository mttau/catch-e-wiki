Overview

When you are ready to suspend an item, update the "Status" field from 'Active' to 'Suspended'.
You can suspend an item with a future-dated "End Date", so make sure you update the item’s “Cycles” or “End Date” to show the final payroll date that you want for the this item.
If the "Package Item" is linked to a Contract Type "Group" that is not 'Salary Packaged', no updates listed below will occur.

Data Updates

[edit]
Suspend an item that is not linked to a contract

The following updates are made in all cases

  1. Employees / Item "Status" becomes 'Suspended'
[edit]
Suspend an item that has not been processed on a Pay Run

The following updates are made

  1. Employees / Item "Status" becomes 'Suspended'
  2. Contracts / Contract "Contract Status" becomes 'Suspended'
  3. Contracts / Contract "Suspend Date" is updated to match Employees / Item "Start Date"
  4. Contracts / Lease records are removed
[edit]
Suspend an item that have been processed on a Pay Run

The following updates are made

  1. Employees / Item "Status" becomes 'Suspended'
  2. Contracts / Contract "Contract Status" becomes 'Suspended'
  3. Contracts / Contract "Suspend Date" is updated to match Employees / Item "End Date" + 1 day
[edit]
If the item Cycles and End Date have changed
  1. Contracts / Contract "End Date" is updated to match Employees / Item "End Date"
  2. Contracts / Billings "Total Billings" will be updated and will match Employees / Item "Cycles"
  3. Contracts / Lease the schedule will be updated and the last period will match Employees / Item "End Date"
