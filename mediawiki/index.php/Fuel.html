Overview

A fuel budget can be calculated in a quote by the system when the required data tables are populated.

Calculation

  1. Selecting a variant determines which Fuel Type to use.
  2. Selecting a "Delivery State" enables the Cost per Litre to be calculated. This is [qt_fuel_types]fuel_cpl + gb_controls]fuel_cost_adjustment
  3. The default or selected Fuel Usage enables the Litres per 100KM to be calculated. This is [qt_variants]fuel_{fuel_usage} + gb_controls]qt_fuel_usage.
  4. Selecting the "Annual KMs" enables the Kms to be calculated.
  5. Total Budget can now be calculated. This is Cost per Litre * Litres per 100KM * Kms) / 100


[edit]

Re-calculation triggers

The fuel profile selections and costs are re-calculated if you change any of the following fields:
Quotes / Contract "Effective Date", "Term(Months)", "Annual KMs" or "Delivery State"
Quotes / Vehicle "Variant"

Set-up

[edit]
[gb_controls]

We recommend you review these controls periodically.
Visit the following process page: Review fuel costs

gb_controls]qt_fuel_usage - Use this control to set the default entry for the Quote / Lease Fuel Profile "Fuel Usage" field.
This field is used in the vehicle's fuel consumption calculation and thus, the fuel budget.
This can be set as 'combined', 'metro' or 'country'.
gb_controls]fuel_cost_adjustment - set a scale to adjust the fuel costs up or down for each state.
Quote / Contract "Delivery State" is the state that is used to trigger this scaling.
[edit]
[qt_fuel_types]

This table stores the costs of each fuel type.
We recommend you review these periodically.
Visit the following process page: Review fuel costs

[qt_fuel_types]fuel_cpl - stores the "Cost per Litre" for each fuel type.
[edit]
[qt_variants]

Go to Setup / Reference Data 'qt variants' to review or update the values for selected variants
Contact your Account Manager if you need to review or change your data process.

These values come from data imports from either Glasses or RedBook (your vehicle data provider) and further updated by linked scheduler queries.

[qt_variants]fuel_type_translation_id - stores the variant's fuel type.
[qt_variants]fuel_metro - stores the variant's "Litres per 100KM" for metro fuel use.
[qt_variants]fuel_country - stores the variant's "Litres per 100KM" for country fuel use.
[qt_variants]fuel_combined - stores the variant's "Litres per 100KM" for combined fuel use.
