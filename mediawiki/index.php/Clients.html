Clients are customers of the user of the system. The client screens consist of several screens:

  • Contact: Contains contact details and addresses etc.
  • MLA: Contains contractual information such as Master Lease Agreement date, insurance policy details etc.
  • Quote Defaults Contains client defaults specific for quoting.
  • Contract Defaults: Contains fee defaults and billing defaults which are used on the contract billing screen to setup contract billing (i.e. Initial billing and periodic billing).
  • Approval Defaults: Contains flags and codes used in Approval checks.
  • Prospect: Used to record information specific to the prospecting process.
  • Discounts: Make Discount levels and Variant Rebates.
  • Financiers: Stores financiers, limits, review dates and margin rates.
  • Events: Used to record events per client (e.g. meetings, follow up actions etc).
  • Contact List Used to record list of Client Contacts.
  • Contact Details Used to record details of Client Contacts.
  • Divisions Used to define divisions for this client. The divisions can then be allocated to contracts using contract events.
  • Cost Centres Used to define cost centres for this client. The cost centres can then be allocated to contracts using contract events.
  • Cost Centre The Cost Centre screen to hold extra information about each cost centre
  • Card Fees Used to add a transaction uplift fee per product, per fuel supplier.




New clients can be created by clicking the ‘New’ button or by using the ‘Copy’ button which will copy the details of the current client to a new client. Each client has a unique client id. The client code is user editable. Note: 'Other' fields are NOT copied to the new client.

The header section of these screens contains information about the client:

  • FBT Type: Displays the FBT Type set up for the selected Client
  • Due Off + 30 days: displays how many contracts where Contract End date in the next 30 days (including contracts where Suspend Date in the next 30 days).
  • Orders: displays how many contracts have a status of ‘Order’.
  • Deliveries + 30 days: displays how many orders have a delivery date in the next 30 days
  • Average Fees: Average $ fees for live contracts.
  • Days since last visit: displays number of days since based on system date and visit date.
  • Icon: (top right of screen). This icon appears if a pdf document exists for this client. This document is typically used to display the master lease agreement or authorised signatories etc. Clicking on the icon displays the pdf document. The document needs to be named as the client code (eg AA1.PDF) and uploaded to the ....fleet/administration/clients/documents folder within Catch-e.
