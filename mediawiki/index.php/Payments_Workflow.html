Overview

Below are details of how to perform various payment processes in the Contracts / Payments tab.
Entering a "Suspend Date" on the Contracts / Contract tab will update the "Stop Payments" field if the date is earlier than the stored date.

More Processes

[edit]
Set-up a Payment
  • Go to Payments and select the "Payment Type" you want to create.
  • Click to create a new schedule, check details are correct and
Note: you cannot create a new payment if the contract does not have a set "Term" or "Start Date".
[edit]
Correcting Supplier Details
  • If you have set up a payment schedule with the wrong supplier, the existing schedule needs to be stopped. Use the "Stop Payments" checkbox and date to set all the pending transactions to stopped. Once done, you can create a new, correct schedule.
Note that for Finance, Insurance and Roadside Assistance payments, the supplier record used on the new payment schedule are copied back to their original reference fields ("Financier", "Insurer" or Vehicle "Roadside Supplier").
[edit]
Correcting Payment Details
  • If a payment structure field is incorrect and all "Pending" rows are to be updated, use to change the relevant field and the Schedule will re-build accordingly.
  • If only some of the "Pending" rows are to be updated, use to open the Payment Schedule rows for editing.
  • Posted payments cannot be changed; use to add in an adjusting entry to correct the posted transaction(s).
[edit]
Extend a Finance Lease
  • If a new agreement and reference is required, the existing schedule needs to be stopped or posted. Only one "Active" schedule of each Payment Type (Finance and Insurance) is allowed. Once this is done, a new schedule can be created with any supplier.
A schedule will be tagged as "Inactive" when no Pending transactions exist or the schedule is stopped.
  • If an existing Finance Agreement is being extended, just the contract and edit the "Total Payments" field to create the extension.
[edit]
Delete a row
  • Any "Pending" row in the Payment Schedule can be deleted by checking the Tick-box ON for the row(s) to be deleted; then click the button.
[edit]
Stop Payments
  • If you want to stop the Payments, click on and the Stop Payments check box. Today's date will be inserted into the "Stop Payments Date" field. This can edited to change the timing of the stop. Any payment schedule rows that have a "Due Date" on or before the "Stop Payments" date will be stopped. These stopped rows are then displayed in a light grey colour.
[edit]
Edit an Auto-stop
  • All active payments are automatically put on "Stop" if the vehicle is Suspended, Paid-out, Sold or Terminated. If a multiple of these actions occur, the "Stop Payment Date" is updated with the earliest transaction date.
  • This is done to prevent any extra payments being made for a vehicle. This needs to be taken into consideration when managing a vehicle disposal.
  • If there are final payments to be made; click on and tick OFF "Stop Payments". The "Stop Payments Date" will disappear. Change the "Final Payment Due" date to the desired date and the "Pending" rows will re-build with the required payments.
Alternatively, change the "Stop Payments Date" to reflect the desired stop and any "Pending" rows that should exist before this date will re-build.
