Overview

Go to the Package Item Details page and review the main types of package items that can be configured.
Go to the Package Item Fields page for descriptions of the package item fields that are needed to create items.
These are normally configured by your Account Manager during the setup process.

[edit]

Request New Package Items

Email Catch-e Support to request a new package item and put the following details in your email:
Go to the Package Item Fields page for more details of each field.

Field Setting
supplier_id_default Supplier Code or leave blank
package_item_code Code up to 6 characters
package_name Name of package item
benefit_type 'exempt', 'type-1' or 'type-2'
cap_type 'primary', 'meal' or leave blank if benefit_type = 'exempt'.
gst_flag 'yes' or 'no'
employee_contribution_flag 'yes' or 'no'
total_deduction_default 0 or amount
compress_flag_default 'yes' or 'no'
inertia_flag_default 'yes' or 'no'
cycles_default 0 or number of cycles
disbursement_type_default 'recurring', 'claim' or 'periodic'
supplier_id_edit_flag 'yes' or 'no' (must be yes if there is no default supplier)
substantiation_flag_default 'yes' or 'no'

Suppliers

Set-up the following standard suppliers. Most package items will be linked to one of these suppliers.

FEES - link this supplier to package items where you (the Packager) will be receiving fee income.
EMPDB - link this supplier to package items where the employee will be receiving the disbursement monies. As the "Payee" is nominated to be the 'Employee', a bank account that is linked only to that employee will be required.


Supplier Code Name Supplier Type Payee Pay Method ABA Format Type
FEES {CLIENT NAME} Salary Packaging Supplier Direct Credit Supplier
EMPDB Employee Disbursement Salary Packaging Employee Direct Credit Payment Reference

Sales Export logic

There is a Package Item field that controls which item records will be selected for export to the sales and journal files.
In most situations, you will want to export all packaging transactions. Set [sp_package_items]tax_invoice_flag = 'yes' in this case.
If you are working as a bureau, only your fees should be exported. Set [sp_package_items]tax_invoice_flag = 'no' in this case.

The logic that is applied is shown below :-

  1. If [sp_package_items]tax_invoice_flag = 'yes' set [gl_transactions]status_flag = 'active' for sl records
  2. If [sp_package_items]tax_invoice_flag = 'no' set [gl_transactions]status_flag = 'report-only' for sl records
  3. When creating [gl_transactions] records for jnl, select only records where [gl_transactions]status_flag = 'active' for sl records to include

Useful Queries

Catch-e staff reference:
Home → Client Support → Data Updates → Create Package Items 


# Review package items
select pi.package_item_id, pi.package_name, pi.benefit_type, pi.cap_type, pi.gst_flag, pi.add_gst_flag
, pid.total_deduction_default, pid.compress_flag_default, pid.inertia_flag_default, pid.cycles_default
, pid.disbursement_type_default, CONCAT(s.supplier_code,' ',s.name) AS supplier, pid.supplier_id_edit_flag
, pid.substantiation_flag_default, pay.payee, paym.payment_method, ppm.status_flag as 'pay-combo'
FROM sp_package_items AS pi
LEFT JOIN sp_package_item_defaults AS pid ON pid.package_item_id = pi.package_item_id
LEFT JOIN fm_suppliers AS s ON s.supplier_id = pid.supplier_id_default
LEFT JOIN gl_payees AS pay ON pay.payee_id = s.payee_id AND pay.status_flag = 'active'
LEFT JOIN gl_payment_methods AS paym ON paym.payment_method_id = s.payment_method_id AND paym.status_flag = 'active'
left join gl_payee_payment_methods AS ppm ON ppm.payment_method_id = s.payment_method_id AND ppm.payee_id = s.payee_id 
	AND ppm.process_type = 'item' AND ppm.status_flag = 'active'
ORDER BY   pi.package_name, pid.disbursement_type_default, s.name
;

# Review package items cars
select pi.package_item_id, pi.package_name
, pi.sp_profile_table_name
, pipc.*
FROM sp_package_items AS pi
LEFT JOIN sp_package_item_defaults AS pid ON pid.package_item_id = pi.package_item_id
inner JOIN sp_package_item_profile_cars as pipc on pipc.package_item_id = pi.package_item_id
ORDER BY pi.package_name
;

# Data check

# Script
# Placeholders
select @user_id_last_edit := user_id from gb_users where user_login in ('admin'); # Change to show your login
select @posting_map_id := posting_map_id from gl_posting_maps where posting_map_group = 'salary-packaged';
select @supplier_id_default := supplier_id from fm_suppliers where supplier_code = 'EMPDB';
select @module := 'sp'; 
select @package_item_code := 'code';
select @package_name := 'Add Package Name ';
select @benefit_type := 'exempt';
select @cap_type := null;
select @gst_flag := 'no';
select @employee_contribution_flag := 'no';
select @total_deduction_default := '0';
select @compress_flag_default := 'no';
select @inertia_flag_default := 'yes';
select @cycles_default := '0';
select @disbursement_type_default := 'claim';
select @supplier_id_edit_flag := 'no';
select @substantiation_flag_default := 'no';
select @next_id := max(package_item_id)+1 from sp_package_items;
# create package item
START TRANSACTION;
INSERT INTO sp_package_items (package_item_id, posting_map_id, package_item_code, module, package_name, benefit_type, cap_type, gst_flag, employee_contribution_flag)
VALUES (@next_id, @posting_map_id, @package_item_code, @module, @package_name, @benefit_type, @cap_type, @gst_flag, @employee_contribution_flag);
select @next_id := max(package_item_id)+1 from sp_package_items;
REPLACE INTO gb_unique_ids VALUES('sp_package_items', @next_id);
# create package item default
select @package_item_id := max(package_item_id) from sp_package_items;
INSERT INTO sp_package_item_defaults (package_item_default_id, package_item_id, total_deduction_default, compress_flag_default, inertia_flag_default, cycles_default, disbursement_type_default, supplier_id_default, supplier_id_edit_flag, substantiation_flag_default)
VALUES (uuid_short(), @package_item_id, @total_deduction_default, @compress_flag_default, @inertia_flag_default, @cycles_default, @disbursement_type_default, @supplier_id_default, @supplier_id_edit_flag, @substantiation_flag_default);
# Add audit record for package_item_id
SELECT @record_id := max(package_item_id) from sp_package_items;
SELECT @audit_field_id := audit_field_id FROM gb_audit_fields WHERE table_name = 'sp_package_items' AND field_name = 'package_item_id';
INSERT INTO gb_audit (user_id, timestamp, audit_field_id, record_id, complex_key, value_before, value_after, origin)
values ( @user_id_last_edit, NOW(), @audit_field_id, @record_id,'','', @record_id, 'internal');
COMMIT;
