Overview

Use this tab to create and record the drivers contact details. When a quote is done and the driver record is selected, some of these details will be populated into the quote.
Visit Employees / Contact for the employee version of this page.

2FA Enrolled Check Box If checked, the means the driver has enabled 2 factor authentication (2FA) in the Driver Portal.

You can configure the Driver Portal 2FA feature to be hidden, optional or compulsory.
If it optional, the driver can enable 2FA in the Driver Portal from the 2 Factor Authentication button.
If it's compulsory, they will be navigated to the enrolment screen when they next login.

If the driver has any problems with 2FA and it needs to be re-set it or they don't want to use it anymore, 'admin' users can un-check this field.
The field is role restricted for all other roles, but it can be enabled on request.

[gb_users]2factor_enrollment_status
Driver ClaimsCheck BoxThis field is checked by default when a new driver is created and also displays on the Drivers / Contact tab. Only internal users can see and edit this field.
When checked, the driver can create new claims in the Contracts / Claim tab or the Driver Portal if they have access.
If un-checked, the driver cannot create new claims and an alert message is shown if they try. This text is stored in [gb_controls]driver_portal_no_claims_text and can be modified.
If un-checked, internal users can still create claims in the Contracts / Claim tab, but are alerted before the claim is created: "Driver cannot create claims. Create claim anyway?". [OK] [Cancel]

Validations and Alerts

Alert Comments
Create
Client cannot be empty! Enter the "Client" that this driver is linked to.
Employee No '{employee_no}' already exists for Client '{client_code}'! This warning will be given if the client already has an Employee stored with this "Employee No".
OK - click to accept this warning and save the record with a duplicate "Employee No".
Cancel - click to return to edit mode to update the "Employee No" before saving to remove the duplication.
{Given Name} {Surname} already exists for {Client Name} A driver record already exists with the entered "First Name" and "Surname" for the selected client. Cancel to avoid creating a duplicate record. If the record is for a different driver - find a way to make the name distinctive so the two drivers are not confused.

Please check the address(es) and try again.
