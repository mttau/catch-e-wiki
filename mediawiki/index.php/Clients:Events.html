Overview

Use Clients / Events to record transactions of note for the client.
Client events can be created by scheduler jobs. These jobs can be configured to send and store emails, reports and sms texts. Visit client queries for more details.
Events for all clients can be viewed at Fleet / Administration / Events / Clients.

Event screens also exist for Quotes, Contracts, Drivers, Employees and Suppliers.

[edit]

Workflow

New events are started by clicking on the New button. Click on Create to save the event.
Use the # check box to select an event to Edit or Delete.
Use the # check box to select an event to Attach a document to. Attached documents can be downloaded for viewing by clicking on the paper clip icon .
Every event can be managed by flagging the required action, setting a due date to be completed by and also giving you the flexibility to mark the event as completed or incomplete.
Incomplete tasks are summarised in the KPI Home screen.

Set-up Notes

Contact your Account Manager if you want assistance with your event configuration.

To add or modify events, update records in [fm_client_events].
To make an event work as a Cost Centre event, set [fm_client_event_codes]cost_centre_flag to 'yes'.
To make an event type inactive, set [fm_client_event_codes]status_flag to 'inactive'.

[edit]
Change the type sort order

The "Type" list shown when you create new events is ordered using the field [fm_client_event_codes]client_event_code.
These codes are not displayed on the screens. Update these codes to change your sort order.
Before you do this, check and update any links you may have to scheduler mailQuery jobs.

[edit]
Add a custom list to an event "Value" or "Description" field

If you want to use a defined list of choices in an event "Value" field, add records into the table [fm_client_event_values]. You can also set a default record for the list.
If you want a predefined selection list for the "Description" field, add records into the table [fm_client_event_descriptions]. You can also set a default record for the list.

Event lists used for "Value" and "Description" can be used separately or together.
