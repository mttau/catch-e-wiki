Overview

When a batch is displayed on the Allocate tab, you can print a csv file of the transactions or produce an ABA file to create a client direct debit file for importing into your banking software. The ABA file is available for "Pending" and "Posted" batches.

For the ABA file to be created sucessfully,

  1. Each client within the batch will need to have their banking details populated in the Client / Contract Defaults tab. I.e. "BSB Number", "Bank Account Number" need to be populated and "Payment Method" set to Direct Debit.
  2. The batch must have a zero variance.
  3. A "Collation" method must be selected.
  4. Batches that result in negative lines in the ABA file will not output (you will get a warning message, see below).

Batch Creation

You can create direct debit batches manually, or a scheduler process can be developed to automate the creation of Direct Debit Receipt Batches.

[edit]

Scheduling your Direct Debits

Contact your Account Manager if you want to get a scheduler process set-up to automate batch creation. This work is chargeable.

The expected process when direct debits are scheduled is this:

  1. Report created
    1. User is emailed
    2. Report contains summary of clients and records that will be in the Direct Debit Batch.
  2. Script creates direct debit batch using the following criteria
    1. Clients / Contract Defaults "Payment Method" is 'Direct Debit'
    2. Clients / Contract Defaults BSB and Account Number are populated
    3. Contracts have billing records that are unpaid (choose if only unpaid or also partly paid records are selected)
    4. Contracts have billing dates or terms that meet the selection rules
  3. The direct debit batch will contain the following details
    1. Banking Date is added as today
    2. Bank Account - the specified account is added
    3. Target - as calculated for the batch
    4. Variance - is 0
    5. Collation is added (as specified)
    6. Status is set as 'Pending'
    7. Item No - a separate item is created for each client in the selection
    8. Receipt Type is set as Direct Debit
    9. Client - each client code is stored
  4. The user has received an email alert and must then
    1. Review the pending batch
    2. Print the ABA file
    3. Load the ABA file into banking software for processing
  5. When the bank statement is received
    1. The batch can be posted.
    2. If any reversals occur, they will also need to be processed, or the original batch edited to net off the reversal before posting

Set-up

See also the process page Change a bank account.
The banking details used to create the Receipts (ABA) file for direct debits are stored in the [gb_controls] table.
You can update these details yourselves.
Go to Setup / Reference Data → gb_controls to update the details shown below.

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
gb bank_accounts Receivables
Financial Institution - the three letter code for your bank
Company Name - the name to use in your ABA file.
Bsb - the BSB to use in your file.
Account Number - the account number to use in your file.
gb becs_details Company Name Short - the short name to use in your ABA file
Identification Number - the identification number to use in your ABA file.
Aba File Description - the description to use as your referent in your ABA file.
User Credits Only - set to no to add the self balancing row to the ABA file.
gb aba_identification_number_debit Value - enter the identification number which is assigned by APCA and User Financial Institutions for transacting debit ABA files.
gb aba_user_debits_only_flag Value - is used to determine if your organisation is a full ABA user (allowing debits and credits) or a debit user only.
If your organisation is debit only, any items in the batch that result in a credit will be excluded from the ABA file. 		yes
