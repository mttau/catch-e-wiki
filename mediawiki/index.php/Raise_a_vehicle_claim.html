Overview

This screen is available for use in the Driver Portal.
The Employee Portal contains the same feature, but contains additional rules. Visit Raise a claim to see this variation.

The claim process allows one transaction per approval. This makes the process fast and keeps claim values smaller. It also makes it easier for you to review and approval attached receipts.

Only budgets that have a REIMBURSE mapping record will be available for selection here.
Where a user has lodged a claim using the driver portal, a system generated email can be sent to the user to confirm the claim details.

A Contracts / Event record can also be set-up to record the email. If the user's email address is blank, an event record will be created, but set to incomplete and the note 'Driver email is blank' will be added.

If the user's record field "Driver Claims" is un-checked, the user is prevented from raising claims and receives an alert advising of this if they try. The alert message is customisable.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.
[edit]
Driver can log in but gets an error when they click on Raise a Claim button
  • If your driver or employee can log into the portal, but get an error message if they click on the "Raise a claim" button, this means their access to this function has been disabled.

Check that the user has a suitable contract for raising claims.

  1. Contract Types / Details "Claims Tab" must be checked.
  2. Drivers / Contact "Driver Claims" must be checked.
  3. Contracts / Contract "Contract Start" - check this is populated.
  4. Contracts / Contract "Contract End" - check this has not been reached.
  5. Contracts / Contract "Suspend Date" - check this has not been populated.

See also the Validations and Alerts that are documented for this button.

[edit]
Submit Claim Error

If an error is returned when you "Submit Claim", check that the Card Interface is configured correctly. There can only be one interface where the "Interface Type" = 'Claim' and it needs to be linked to a valid supplier.

[edit]
Attach Receipt Error

Only files with an extension stored in gb_mime_types will be accepted for upload in the driver portal. i.e. if the extension is missing or does not match, the error below is returned.

"Unrecognised or unsupported attachment. Please check your file contains a valid extension and is a supported file type."
[edit]
Unable to load receipts

If the Driver Portal is used in an iframe and the user is trying to attach a receipt whilst working via VPN, their organisations firewall may block this. Advise user to try again when they are not logged into a VPN.

[edit]
Unable to select photo on iPhone

User unable to select a photo to attach - if the users phone is an iPhone with an IOS 13+, they will need to update their settings for this feature to work.

[edit]
Connection Error

A connection error can happen at the end of the claim process if the Claim confirmation email is not completely set-up.

The message will look like this: 

A connection error has occurred while processing your request.
Please press F5 to refresh the screen and try again.

One data set-up issue looks like this:

  1. You have have set the gb_controls "approval_reimbursement_notification_flag" to be 'yes'
  2. There is no Posting Class called 'PORTAL' set up.

Or

  1. The 'REIMBURSE' interface is not set-up
  2. The 'REIMBURSE' interface supplier is not set-up

When the claim is being completed, the system needs the 'PORTAL' posting class to create the email and contract event.

Useful Queries

select * from gb_templates where name = 'pdf_contract_event_email_message' and sub_name = 'PORTAL';
select * from gb_controls where name = 'approval_reimbursement_notification_flag';
select posting_class_code from gl_posting_classes where posting_class_code = 'PORTAL';

select pc.posting_class_id, pc.posting_class_code, pc.name, pc.template_id, t.template_id
from gl_posting_classes as pc
left join gb_templates as t on t.name = 'pdf_contract_event_email_message' and t.sub_name = 'PORTAL'
where pc.posting_class_code = 'PORTAL'
;
select interface_code, i.interface_type, i.description, s.supplier_code, s.name, s.status
from fm_interfaces as i
left join fm_suppliers as s on s.supplier_id = i.supplier_id
where i.interface_type = 'claim'
;
select interface_code, i.interface_type, i.description, ivm.product_code, ivm.description, ivm.posting_class_id, ivm.vmrs_code_id
from fm_interfaces as i
inner join fm_interface_vmrs_mappings as ivm on ivm.interface_id = i.interface_id
left join gl_posting_classes as pc on pc.posting_class_id = ivm.posting_class_id
where i.interface_type = 'claim'
order by ivm.posting_class_id, ivm.vmrs_code_id
;
