Overview

Use scheduler jobs to automate the billing process so that you don't have to run the billings manually from the Billing screens.
You can set-up billing jobs to run for all clients, or restrict the selection to a particular channel, client group or client.

[edit]

Methods

Method Description Type
Combined This job runs scheduled Combined billings. fm / billings / combined
Periodic This job runs scheduled Periodic billings. fm / billings / periodic
Recharge This job runs scheduled Recharge billings. fm / billings / recharges
Recharge Fuel This job runs scheduled Recharge Fuel billings. fm / billings / rechargesFuel


[edit]

Scheduler Mail fields

By default the Mail fields (Send Mail checkbox, Subject, Body, Sender, Recipients etc) are not editable.
When Billing jobs are run, and the "Delivery" parameter is set to 'Email', the respective Client's "A/C Payable" and "CC" will be used as the recipients. The Sender will be the [gb_templates]mail_sender.

If you want to receive a notification email that the job has run, the [gb_scheduler_types]allow_send_mail_flag should be set to 'yes' and [gb_scheduler_types]attachment_required_flag should be set to 'no'. Contact your Account Manager if you would like to do set this up.

Automated billing did not email

Automated billing is done by scheduler jobs, and a likely cause of issues is that the job did not run

  1. Go to the System / Scheduler screen and select "Module' - 'fm', "Class" - 'billings' and click on Go to find your billing jobs.
  2. Check the "Last Message" column for a recorded issue. E.g. 'Periodic billing locked by: {user_login}
  3. Use the Run Now button to run the job immediately.

Billing jobs will not run if user are editing records in the system. Catch-e recommend billing jobs be set to run out of working hours. 5am works well.
