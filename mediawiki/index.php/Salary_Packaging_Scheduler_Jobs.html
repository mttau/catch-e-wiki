Overview

The scheduler jobs below are the recommended base set for Salary Packaging clients.
The Job ID and Query ID's shown are for our Demo environment and will be different in your system.
The name of each job will be the same, unless changed.

Build Budgets
executeScript Add Budget amounts for claims
Use this to add a budget amount for claims you have raised or edited.

Purpose
Update budgets to store the required amount for pending claims.

Selection:

  1. Contracts where linked Item "Disbursement Type" is 'Claim'
  2. Package Item's where module is 'budget' are excluded from this process

Logic

  1. Build a temporary table to store
    1. each contract's pending billing period and date
    2. each pending approval that is in active / pending funds status
    3. if the pending approval is new and unmatched - the number in the Quantity field is used to match the approval with the requested billing period. I.e. 1 - next pending, 2 = 2nd pending.
  2. Update approvals to
    1. add or update external_order_number field to match the required billing period
    2. update the invoice date and odometer date to match the required billing period
    3. if the billing period is next month, the dates are set to be the last day in the current month ( sp/approvals/autoPostClaims only selects approvals dated in the current month)
  3. Update billings to
    1. store PRETAX amount_net and amount_gst that matchs these amounts in the linked pending approvals
    2. update PRETAX amount_net and amount_gst to be 0 if there are no longer any linked pending approvals

Note - you may have to review and update pending records if they have crossed the end of the month

executeQuery Add Budget amount for claims - exceptions

Purpose
Report where approvals cannot be added to the budgets for review and correction/ Logic

  1. Suspend Date has been entered
  2. Approval Qty has multiple values

Job IDQuery IDParentTypeDescription

100185 100225 executeQuery SP - Car Contract Budget Checks

Run Time: Weekdays 5:00am
Delivery Type: Email
Purpose:
List contracts where budgets are not complete for a car contract that is linked to an item.
Selection Criteria:

  1. Item is linked to a car profile and item "Contract ID" is the same and
  2. Contracts / Billing - contains pending periodic budget records

Data Checks

  1. ECM - Contracts / FBT "ECM Flag" is checked and ECM budget is 0, or
  2. ECM GST - Contracts / FBT "ECM Flag" is checked and ECM GST budget is 0, or
  3. FBT or,
    1. Clients / Contract Defaults "Employer FBT Type" is 'Full FBT Payable' or 'Rebatable'
    2. Contracts / FBT "Benefit Type" is not 'Exempt'
    3. Contracts / FBT "FBT Treatment Type" is not 'EV Exempt Reportable'
    4. Contracts / FBT "ECM Flag" is 'No'
    5. FBT budget is 0
  4. FBT
    1. Clients / Contract Defaults "Employer FBT Type" is 'Concessional 30000 Exempt' or 'Concessional 17000 Exempt'
    2. Clients / Quote Defaults "Allow Quoting within Exempt Cap" is un-checked
    3. Contracts / FBT "Benefit Type" is not 'Exempt'
    4. Contracts / FBT "FBT Treatment Type" is not 'EV Exempt Reportable'
    5. Contracts / FBT "ECM Flag" is 'No'
    6. FBT budget is 0
  5. Operating Cost Method - Business Use - contracts are excluded if
    1. Contracts / FBT "Benefit Type" is 'Operating'
    2. Contracts / FBT "Business Use" is 100%
  6. Operating Cost Method - ECMR - contracts are excluded if
    1. Contracts / FBT "Benefit Type" is 'Operating'
    2. Contracts / Events contains an active event for 'EmployeeContribution Required'.
    3. Visit the Employee Contribution for details about this feature.

100170 100212 100177 executeImportTableData SP - Update Item Data

Run Time: Daily 6:00am. See parent job.
Table: sp_employee_package_items

Purpose: Update to match details in contracts when edits have been made.
Selection Criteria:

  1. the linked contract has billing dates and Cycles do not match Last Billing Period
  2. the linked contract has billing dates and Start Date does not match Contract First Billing Date
  3. the linked contract has billing dates and End Date does not match Contract Last Billing Date
  4. the linked contract has billing dates and Deduction Per Pay does not match Next Pending or Last Posted Billing Total
  5. Status
    1. is not 'Accepted' and Contract is order ( Suspend Date is blank and one un-dated billing record has been created or there are no billing records)
    2. is not 'Active' and Contract is active ( Suspend Date is blank and billing records exist)
    3. is not 'Suspended' and Contract is suspended ( Suspend Date is populated)

Updates:

  1. Total Deduction
    1. if the linked contract has billing dates and Compress is 'Yes' and Inertia is 'Yes' and Cycles is less than Pay Cycles per Year - update to show Cycles X Deduction Per Pay
    2. if the linked contract has billing dates and Compress is 'Yes' - update to show Cycles X Deduction Per Year
    3. if the linked contract has billing dates - update to show Cycles X Deduction Per Pay
    4. otherwise, keep the stored Total Deduction
  2. Cycles
    1. if the linked contract has billing dates – update to show Last Billing Period (this can become different if billings are shortened or extended)
    2. if the linked contract billing dates are blank, keep the stored Cycles
    3. if the linked contract has a Suspend Date and there are no billing records, make this 0
  3. Start Date
    1. if the linked contract billing dates are blank, keep the stored Start Date
    2. if the linked contract has no billing records, keep the stored Start Date
    3. if the linked contract has billing dates, use the First Billing Date (this can become different if the first billing is delayed)
  4. End Date
    1. if the linked contract has billing dates, use the Last Billing Date (this can become different if billings are shortened or extended)
    2. if the linked contract billing dates are blank, keep the stored End Date
    3. if the linked contract has a Suspend Date and there are no billing records, make this date blank
    4. otherwise, keep the stored End Date
  5. Deduction Per Pay
    1. If there are pending deductions, show the amount of the next pending billing (item 1). Note 0 value billing records are excluded
    2. If there are no pending deductions, show the amount of the last posted billing (item 1). Note 0 value billing records are excluded
    3. otherwise, keep the stored amount
  6. Status
    1. if the linked contract does not have a Suspend Date and there is an un-dated billing record - update to 'Accepted'
    2. if the linked contract does not have a Suspend Date and there are no billing records - update to 'Accepted'
    3. if the linked contract does not have a Suspend Date and there are pending billings - update to 'Active'
    4. if the linked contract has a Suspend Date - update to 'Suspended'

Job IDQuery IDParentTypeDescription
100174 100216 100166 executeImportTableData SP - Update Item from Vehicle Quote

Run Time: See parent job.
Table: sp_employee_package_items
Process page: Car Details auto updates.
Purpose:

  1. This job updates item records from data in the linked vehicle quote.
  2. Once a quote is added to the Item, the required fields for quoting are calculated from quote data and added into the Item record.

Selection:

  1. Employees / Item "Package Item" is linked to a car profile and the car profile quote_flag = 'yes'
  2. Employees / Item Car Details "Quote ID" is populated and "Contract ID" is not populated
  3. Employees / Item "Start Date" is on or after today's date less 1 month. E.g. if today is 19/05/2023 - "Start Date" must be on or after 19/04/2023
  4. Employees / Item "Status" is 'Quoting' or 'Pending'
  5. Employees / Item "Contract ID" field is blank
  6. Quotes / Contract "Type Code" is linked to the same car profile as the Item
  7. Quotes / Contract "Quote Status" is not 'Cancelled', ' Deleted' or 'Expired'.

Where:

  1. Employees / Item "Total Deduction" is not the same amount as Quotes / Lease Budget Inclusive "Totals" x "Term (Months)"
  2. Employees / Item "Start Date" is less than one month old.


Updates:

  1. "Description" - shows the Contract Type and Variant Short Description. An 'O' is also added if the vehicle is using Operating Cost Method for FBT.
  2. "Total Deduction" = Quotes / Lease Budget Inclusive "Totals" x "Term (Months)"
  3. "Employee Contribution"
    1. If "Use ECM?" is un-checked then 0
    2. Else "FBT Value" x fbt_statutory_rate (20%)
  4. "Cycles" - ROUND((Term (Months) / 12) * pay_cycles_per_year,0)
  5. "End Date" - calculated from "Cycles" and "Start Date" using the employee's "Pay Cycle"
  6. "Deductions Per Pay" - ROUND ("Total Deduction" / "Cycles",2)

  2. Deleted and no longer visible when

    2. When the item's "Benefit Type" is 'Exempt' and "Disbursement Type" is 'Claim' and the invoice record has a total of 0

Purpose

  1. As one of your annual tasks, use the report to review and update employee super amounts as required.

Sort Order

  1. Sort Order (1 = update, 0, matches so no update needed)
  2. Current Rate (lowest to highest)
  3. Client Code, Pay Cycle, Given Name
