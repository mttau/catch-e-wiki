Main Page → Workflow → System Administration → System Maintenance → Data archiving

Qualifiers

  1. Archive data when
    1. it is no longer needed
    2. you have run the Deidentify Driver Data process
    3. you want to improve system performance by reducing data volume

Overview

This process is designed to be run annually with assistance from Catch-e staff.
The required archive intervals in years for various system areas are set in a table called [gb_archive_intervals]
The intervals are set as 0 by default.
This means no records will be archived unless the table is updated and intervals are set.

Client Steps

  1. Review and update your archive settings by doing the following
    1. Export data from a table - download [gb_archive_intervals] and review the intervals that are set in this table. Update to suit your business.
    2. Import data to a table - upload the changed records.
  2. Contact Catch-e Support to raise a task and book the archiving date for you.
    1. We recommend doing this in mid-January, as this is a quiet time of the year.

Catch-e Steps

  1. On the arranged date, Catch-e's System Administrator will create an archive database snapshot
  2. This snapshot will be named like this: archive-database-live-2022-02-24
  3. Data Archiving will only run if a snapshot that is less than 12 hours old exists
  4. Contracts that will be archived are first copied into a table called [fm_contracts_archived]
    1. [fm_contracts_archived] remains available in your database for reference
  5. Data in primary tables are deleted if they meet the archiving criteria
  6. Data in secondary tables (linked using a FKC) are also deleted
  7. Catch-e Staff will confirm the archiving has completed successfully.

Archive Recovery

  1. Archive snapshots can be restored and made available to users in Workbench
  2. Archive data will not be restored into the system
  3. Contact Catch-e Support to arrange for an archive snapshot to be restored
  4. Archive snapshots are then available for 24 hours
