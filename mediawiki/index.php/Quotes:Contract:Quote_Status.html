Overview

The quote's status field is stored on the Quotes / Contract tab. The status defines what stage of the process the quote is in. You can filter on "Status" in the Quotes / Search screen.

Read through the set-up notes for configuration details.

A "Sub Status" can also be used to extend the classification of a quote.

[edit]

--Active--

--Active-- is a filter that groups a number of statuses that indicate the quotes are in-progress. It is a quick way to view all quotes that potentially need follow-up actions by your staff. The grouping can be configured for you.

The default filter on the Quotes / Search screen is --Active--.

--Active-- status is also used to define a Home screen chart; Quotations Active.

[edit]

Status Definitions

The status definitions listed below are as standard in Catch-e. These may be set-up differently for your organisation.

StatusTypeDescription
--Active--
DealerQuote sent to dealer for pricing.
DraftA new quote can be saved with a status of "Draft" if it is still being worked on
OpenProcurement/Manager roles can change the status of a quote when ready. If it is set to "Open" upon saving an email will be sent to the Quote Creator and the Account Manager will be CC"d on this email to advise that the quote is open and ready to send.

When the status of the quote is saved as "Open" from any other status, the system sends an email from the user saving the quote to the user who created the quote advising them it is now ready. The Account Manager is CC'd on this email. The Driver's name from the quote is included in the subject line of this email for easier identification by the recipient.

PendingProcurement/Managers can set a quote to pending so that they are aware it is a work in progress
RequestedThe default status for a new quote is "Requested". Standard users cannot change the status of a requested quote. They can edit a requested quote and re-save it.

When creating a new quote at status "Requested" or saving an existing quote from another status to "Requested", an email can be sent to a procurement officer to make them aware of the request.
To set-up this feature, enter the procurement users email address in the Contract Types/ Details "Quote Request Email To" field on the screen.

ReviewThe quote is being reviewed and is not ready to be sent yet.
Re QuoteA status that is available where there is a parent Quote ID. As an example this would be when the quote is being copied from another.
SentStandard users will have a ‘Sent’ button active for ‘Open’ quotes. When they send a quote to a client (i.e. by email or by printing/faxing etc) they should click the ‘Sent’ button which will update the quote status to sent and update the last edit date/time
--Not Active--
AcceptedIndicates that the quote is proceeding to a signed-up contract. This status can be updated on this list box. It will update automatically if you click on the "Accept" button.
CancelledStatus to be used by Procurement/Managers to effectively cancel a quote (e.g. client changes their mind and no longer want the quote).
DeclinedTo be used if the client declines the quote.
DeletedStatus to be used by Procurement/Managers to effectively delete a quote.
ExpiredThe quote has expired (e.g. has passed its valid til date).
Pricing Received
RecurringUsed for quotes which are regularly re-used, e.g. mine site or client specific specifications and options etc. This type of quote should be copied and converted to a normal quote when a new quote is required.
SampleTo show which quotes are indicative only
--Not Visible--
ExternalStatus used for quotes generated using a Web Service and not available as an option for the Status list box on either the Quotes / Search or Quotes / Contract screens.


Note: We recommend you do not manually change the statuses "Accepted", "Sent" and "Opened", as these are used in conjunction with the "Accept" button.


[edit]

Role Limitations to use Status

The 'User' role has restricted access to quote functionality but can draft, request, send, copy, print and accept quotes.

For example to print a quote for this role the quote must have a status of:

  • Open
  • Sent
  • Declined
  • Accepted


[edit]

Set-up Notes

The default quote status for new or copied quotes can be set using [gb_control]qt_quotes_status_flag_default.

The list box is stored in the table qt_quote_statuses.

A quote status can be made inactive by setting [qt_quotes_statuses]status_flag to 'inactive'. The status will no longer be available in the "Status" list box for either the Quotes / Search screen or Quotes / Contract tab.
Quotes that exist where the status has been changed to inactive will not be altered. They will be visible on the Quotes / Search screen if "Status" filter of --All-- is selected.
These quotes can still be recalled on the Quotes/Contract screen by directly entering the quote id.

Set [qt_quotes_statuses]active_flag to 'yes' to place the status record into the --Active-- group. If configured this way, the status will display by default when the Quotes / Search screen is loaded.
It will also be included in the data used on the Home page chart; Quotations Active.
