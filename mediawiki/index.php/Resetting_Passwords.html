Overview

The Reset Password can be displayed on your user's Login screen.
If this feature is set-up, then users can reset their own passwords if they have forgotten their password or locked themselves out.

Set-up Notes

Follow the following set-up steps to active the password reset feature.
Note that some controls here are shared with the Driver Portal Forgot Username? Forgot Password? feature.

[edit]
User Details

There is a system user called '_password_reset_'.
This user is the sender for password reset emails.
Check the user "Name" and "Email" details are suitable for these emails and update them as required.

[edit]
Role and Expiry Details

The '_password_reset_' user is linked to the role 'password_reset' role.
The Roles / Details "Session Timeout" field determines how long a password recovery email URL is available for. The default setting is 2 days.
Check and update the "Session Timeout" time depending on your preferences.

[edit]

[edit]
Email Message

Run the set-up changes in your Staging environment and review the email message. If you want to change the text of the message, email the changes to Catch-e Support.
This message is also used by the Driver Portal Forgot Username? Forgot Password? feature, so check that the wording is suitable for both your staff and for drivers if using both features.

An email message needs to be configured for the URL email that is sent. Visit Email Setup for standard configuration information for emails;

[gb_templates]name this must be populated as pdf_password_reset
[gb_templates]mail_sender_name can be left blank, as the reset email is being sent by the user '_password_reset_', it will use that "Name". Like other emails, a default name can be entered that will override '_password_reset_' "Name".
[gb_templates]mail_sender_address can be left blank, as the reset email is being sent by the user '_password_reset_', it will use that "Email". Like other emails, a default can be entered that will override '_password_reset_' "Email".
[gb_templates]mail_subject enter a suitable subject
[gb_templates]mail_message enter a suitable message, but it needs to contain the placeholder "#password_reset_url#" for the reset URL link to appear. You can also add the URL's timeout timestamp "#signon_timeout#" if that suits.

[edit]
Event Logs

A log of password reset requests are stored in the gb_eventlog table.

Additional information can be found in the gb_user_access_log table, where all reset attempts are recorded and defined as 'reset'.

[edit]
Useful Queries
SELECT * FROM gb_controls WHERE name LIKE 'password_%';
SELECT full_name,email FROM gb_users WHERE user_login = '_password_reset_';
SELECT mail_sender_name,mail_sender_address FROM gb_templates WHERE name = 'pdf_password_reset';
# template details can be blank, the email will use _password_reset_ Name and Email

SELECT * FROM gb_eventlog WHERE narrative LIKE 'RESET%' ORDER BY timestamp DESC LIMIT 10;
SELECT * FROM gb_user_access_log WHERE access_type IN ('reset','fiel','fail-hash') ORDER BY timestamp DESC LIMIT 10;

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.
[edit]
403 Forbidden

If you get a '403 Forbidden' message when you are logging in, you are trying to log in from a Geo-blocked location.
To login, you will need to either

  1. Run a VPN connection from an unblocked location (like Australia)
  2. Arrange for the IP addressg to be whitelisted. This can only be done if you are running a static IP address. Go to the process page Whitelist an IP address for the steps to do this.
[edit]
Invalid token

If a user clicks on the URL link sent by the Forgot Password? feature and gets this error, it means either

  1. They have clicked on the link previously (it will only work once) or
  2. Their email system has triggered the URL while running security tests for phishing (thus using up the one-time link)

In both cases, the easiest way to manage this is to go to Users / Password and Issue a temporary password to the user.

[edit]
My browser tells me to change this password

Your browser has a monitoring tool enabled that checks the passwords you enter against databases that store known stolen credentials.

If your browser alerts you about this, it means your login password is no longer secure.
You can confirm this by using a site like haveibeenpwned.com/Passwords to check this.

Below are our recommendations if you get this message:

  1. Change your password immediately
  2. Use a good password manager to help you generate and maintain strong, unique passwords for each different system or website you use.
  3. Maintain a unique password for your Catch-e login.
  4. Do not use your Catch-e login password on any other system or website.
