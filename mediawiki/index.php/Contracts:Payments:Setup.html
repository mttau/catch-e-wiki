Client Set-up

To utilise this process, you need to set-up supporting information:

[edit]

Classes

Each class used to create payment schedules needs to be updated to become accessible to this function.

  1. Navigate to Posting Classes and
    1. "Payment" - check on the check box. Only checked classes can be added to the Contract Types / Payments tab.
    2. "Supplier" - add a supplier code if you want a default supplier for 'Sundry Payments'. See also the process page Configure a Sundry Payment.
[edit]

Contract Types / Payments

  1. Review the Posting Class records you want to add. Make sure "Payments" is checked.
  2. Go to Contract Types / Payments
    1. Add the required records for each payment type you want to create for the selected contract type.
    2. The recommended entries are detailed in the Contract Types / Payments page.
    3. Check that each record has the required GL Code for your accounting system.
    4. To use a specific GL Code for a particular supplier, go to Suppliers / Account GL Code and enter the required GL Code there.
[edit]

Contracts / Payments tab

Contract Types can be set-up with any combination of types you want to use.

  1. If no records are mapped in Contract Types / Payments for a given Contract Type, then the Contracts / Payments tab is not visible for those contracts.
  2. Only mapped records can be selected for use in the Contracts / Payments tab. E.g. If a Contract Type only has 'Finance' mapped, only 'Finance' is available for selection.
  3. Set-up either 'Finance' or 'Finance - Principal and Interest' on a Contract Type, not both.
[edit]

Management Fees

If you want to create payment schedules for Management Fees

  1. Review the existing contracts and fees. All contract management fees will need to be updated to work in the same way.
  2. Enter a default supplier in [gb_controls]payments_mfee_default_supplier_id
  3. Check and set [gb_controls]report_budget_fees_as_actual_flag as 'no'. If it is 'yes', budget and actual values will be added together in Driver Reports.
[edit]

Emailing Payment Advices

If you are going to email payment advices to suppliers, check that;

  1. Suppliers / Detail "Contact" and "Email" are filled in correctly.
  2. Do a test run of the emails and check the email header and body text is as you would like. They can be amended for you by Catch-e support.
[edit]

Using the ABA file

If you are going to use the ABA file to create banking payments, check that

  1. Suppliers / Account "Payment Delivery Type" is set correctly.
  2. Suppliers / Account "BSB Number" and "Bank Account Number" have been entered correctly.
  3. Do a test generation of the ABA file and check the details are set-up correctly.These can be amended for you by Catch-e support.

Export files - Finance - Principal and Interest

Setup / Reference Data → gb_controls
ModuleNameCommentDefault Value
glpl_export_finance_reporting_flag This control is only applicable when
  1. you are exporting transactions to Xero
  2. you are using the "Payment Type" 'Finance - Principal and Interest'.

When the flag is set to ‘no’

  1. The export file will itemise the classes ‘PRI' (Principal) and 'INT’ (Interest). The class 'FIN' (Finance) is ignored.
  2. Contract Types / Payments "G" must be set to 'N' for INT, PRI and FIN for the "Type" 'Finance - Principal And Interest'
  3. Contracts /Payments - when a 'Finance - Principal And Interest' payment is saved, the GST setting is checked. If the 'FIN' class GST is populated, the record cannot be saved. You will see a validation message.

Where the flag is set to ‘yes’

  1. The export file will only include the ‘FIN’ class and if configured, will include GST.

Catch-e Set-up

[edit]
Enabling Un-Post for exported payments

By default, only the role 'admin' can un-post an exported payment batch.
This can be updated in Role Restrictions for you by your Account Manager if you want to modify which roles have that access.

[edit]

Payment Advice Email

Each payment type has it's own email template where the wording can be customised.
The payment advice is configured to email to the contact whose details are stored in the Suppliers / Detail "Contact" and "Email" fields.
This can be configured to email to Suppliers / Account "Contact" and "Email" instead.
Contact Catch-e Support to change the configured email contact. We recommend you use the same contact details on all remittance advice templates.

Payment Advice Footer

The payment advice footer returns details stored in gb_companies for the record where [gb_companies]default_flag = 'yes'. If more than one record is set to 'yes', a system error is returned.
Error Message : pdf_company::pdf_company Error reading default company from gb_companies!

The footer contains template fields as follows:
[gb_companies]name
[gb_companies]address1, [gb_companies]address2, [gb_companies]city postcode
Phone: [gb_companies]phone

[edit]

Channels

For clients using multiple channels, the Remittance Advice can be updated to hold footer details details for the specific channel if linked to a separate company.
When a channel template is created, the footer will display the linked company’s details.
If no linked company is found, the selected company details will be used instead.
Follow these links to review channel and company set-up details: Company, Channels Setup, Payments Setup

Useful Queries

# Show templates and recipient details
select t.template_id,t.name,tr.type,tr.template_recipient_id, tr.name_from_field, tr.address_from_field, tr.status_flag 
from gb_templates AS t 
left join gb_template_recipients AS tr ON t.template_id = tr.template_id
WHERE t.name IN ('pdf_periodic_finance','pdf_periodic_insurance','pdf_periodic_management_fee'
,'pdf_periodic_roadside_assistance','pdf_periodic_sundry_payment');

# Show remittance advice footer details
select ch.channel_code, coy.name, coy.address1, coy.address2, coy.city, coy.state, coy.postcode, coy.phone
from gb_companies as coy
left join fm_channels as ch on ch.company_id = coy.company_id
;

# Show suppliers with payment transactions and their contact details
select supplier_code, name, contact, email, account_contact, account_email, payment_delivery_type, cpstr.payment_type
from fm_suppliers as s 
inner join fm_contract_payment_structures as cpstr on cpstr.supplier_id = s.supplier_id
GROUP BY s.supplier_id, cpstr.payment_type
;

# Show clients by channel and company with payment footer company name
select 
ch.channel_code, chcoy.name as channel_company
, cl.client_code, coy.name as contract_company
, IF(t.channel_id is not null, chcoy.name, coy.name) as template_footer_company
from fm_contracts as c 
inner join fm_clients as cl on cl.client_id = c.client_id
left join gb_companies as coy on coy.company_id = c.company_id 
left join fm_channels as ch on ch.channel_id = cl.channel_id
left join gb_companies as chcoy on chcoy.company_id = ch.company_id
left join  gb_templates as t on t.name IN ('pdf_periodic_finance','pdf_periodic_insurance','pdf_periodic_management_fee'
,'pdf_periodic_roadside_assistance','pdf_periodic_sundry_payment') and t.channel_id = ch.channel_id
group by cl.client_code
order by template_footer_company, channel_code, client_code
;
