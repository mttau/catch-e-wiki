Total Finance Payment Calculation: Monthly Payments Fee

The reason this error appears is due to the calculated gross payment value in Catch-e does not match the Maclease quote engine's calculation.

Error message; 

Total Finance Payment Calculation: Please confirm value in the following field: 
* Monthly Payments Fee

This can happen for one of the following reasons;

  1. Macquarie can set Financier Monthly Administration Fee that is expected to be charges on your finance agreements. This fee can be configured to populate into the Quote / Finance Monthly Payments "Fee" field in Catch-e. If the configuration doesn't match your arrangement with Macquarie, this mans it can be populating in, but is not required, or vice versa. If you think this is the issue;
    1. Contact your Maclease Account Manager to check if the if the Financier Monthly Administration Fee is applicable and how much it should be.
    2. Then check the records in Setup / Reference Data 'qt_financier_posting_map_defaults' for Macquarie to see how the "Fee" is set up.
  2. There is a mis-match between Macquaries's expected Total Finance Payment and the Quote / Finance Monthly Payments "Repayment Inc GST" amount in Catch-e. This is sometimes due to a rounding calculation of 1 to 2 cents that causes the application to fail. If you think this is the issue;
    1. Go to the Quote / Finance tab
    2. Open this calculator
    3. Enter the values in the section Catch-e Gross Payment Calculation from the Quotes / Finance Tab.
    4. Click on Print to export the stored JSON file that was submitted to Macquarie. The values you require are in the last section labelled "quotation".
    5. Enter the values in the section Maclease Gross Payment Calculation from the JSON Payload Output. You can export this using the Print button.
    6. The calculator will show you the value Maclease is expecting and what you can do to try to match the value in Catch-e
    7. If the value cannot be rounded to be exactly the same as the value Maclease is expecting, you may need to enter the quote application directly into the Maclease system.
    8. Contact Catch-e Support if you have any questions.

Driver Previous Address Incomplete error

This error will appear when the set up criteria have not been met, as the driver has resided at their current address for less than 2 years. When this is the case, various fields become mandatory and MUST be populated for a successful application to be passed to Maclease. These fields are listed below;

  1. Address line 1
  2. Address line 3
  3. Overseas Address Flag , where 'Country' does not equal 'Australia'
  4. Postcode where Overseas Address Flag equals 'No'. A 4-digit number must be entered
  5. State where Overseas Address Flag equals 'No'
  6. Country where Overseas Address Flag equals 'Yes'
  7. Years at Address must be a value between 0 and 99 and at least 1 non-zero value for “years_at_address” or “months_at_address”.
  8. Months at Address must be a value between 0 and 11 and at least 1 non-zero value for “years_at_address” or “months_at_address”. 
   Overseas Address Flag - This is not a field in Catch-e but an element that defaults into the application file on submission of an application.  

  It will be populated as 'NO' when the 'Country' entry is 'Australia' and 'YES' for all other countries.

Brokerage Value

Catch-e users will have access to the Maclease Partner API once they have been set up as an 'Introducer' by Macquarie Leasing. They will be required to be set up under the "Default Rate" plan.

This will have an affect when the passing the 'brokerage_value' as Maclease will validate the amount to be less than the maximum commission brokerage configured in the Macquarie system.

The "Refinance" option will be omitted from this rule.

Where this validation breached, the following error message will be displayed, "EXCEEDS_MAX_THRESHOLD" - refer specific comments in list below.

Solving the Maclease Validation Error

The listed error messages above typically appear as a suffix in the error displayed, with the prefix identifying the Maclease Element causing the issue. The identified Maclease Element can be found in the JSON file. This can be produced by selecting 'Print' from the Quotes / Finance Tab.

For example, the following error is displayed on 'Apply':

Go to the exported JSON file from the quote. Based on the error reading 'REQUIRED', it implies the data is missing. 'State' and 'Postcode' are associated with addresses, therefore you will be looking for an 'address' element that is missing these descriptions.

Return to the Drivers / Finances tab and enter the missing data. Once saved and no other errors are triggered, the application will be successful.

This is only one of many Maclease errors that can be triggered. If you apply the prefix:suffix logic that is displayed in the error, you will be able to navigate through the potential validation errors.
