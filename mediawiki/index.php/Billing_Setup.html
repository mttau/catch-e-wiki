Overview

There are a range of system and client-level settings that enable you to arrange the billing process to suit your needs.
Combined, Periodic and Recharge invoicing can be set-up to run automatically from the Scheduler.
The general settings are described below;

[edit]

Clients / Contract - "Billing Type" default and exclusions

The Clients / Contract Defaults "Billing Type" field determines your preference for which billings are used for the client.
It is set when a client is created.
You can set a preferred default for this field in [global_controls]client_billing_type_default.
You can also exclude entries from the list of choices by adding these entries to [gb_controls]client_billing_type_exclusions.
E.g. you could set a default of "Combined" and prevent the use of "Any".

[edit]

Screen Displays and Execution

[edit]
Execute on navigation

The billing screens are set-up to run the default filters when you first navigate to the screen. I.e. all records dated today or earlier.
This can be very slow if you run your billing monthly and also need to change the filters.

You can change this so that the records do no populate onto the screen until you click on the "Go" button.
Update by changing the setting [gb_controls]billing_auto_search_flag.

[edit]
Date display

Control the filters displayed on Billing/Periodic and Billing/Combined screens. The standard settings are 'no' and usually only one date called "Dated" is displayed.

If [gb_controls]billing_invoice_from_date_flag is set to 'yes', the screens will display an "Invoice From" date. This allows population of a 'From' date on your invoice header (if configured) but has no bearing on the transaction selections in the screens.

If [gb_controls]billing_invoice_date_override_flag is set to 'yes', billing screens will show "Invoice To" and "Invoice Date" filters ("Invoice Date" replaces "Dated") to allow you to select and invoice billings with a "Periodic" date that is ahead of the "Invoice Date".

[edit]

Invoice Formats

[edit]
Invoice Templates

Your invoice templates are configured to display your details and can be updated as required.

[edit]
Channel Invoice Templates

If you have arranged for separate Invoice Templates for one or more of your Channels, before selecting any filters in the header section of this screen, you must select the invoice template from the list box at the bottom right corner of the screen. The list will contain (for example on the Periodic Billing screen) "Standard Periodic Invoice" (default value) and any Channel templates will be listed below it (eg. "CHANNEL Periodic Invoice").

Once the Channel template has been selected, the "Channel Code" filter in the header section will be read only and will display the relevant Channel Code accordingly. All other Client selection filters (Group Code, Client Code, Reg No) will now only return Billing records linked to the selected Channel once you click the Go button.

[edit]

Billing Transaction Selections

[edit]
Create Credit Notes

If configured to do so, separate credit notes can be created for negative value transactions. This needs to be set-up and is not the default configuration.

[edit]
Itemise Adjustments

If the Billing Adjustments Itemised flag is selected, any adjustments made to a contract will be shown separately on the Periodic or Combined Billing Tax Invoice.

[edit]
Remove Zero value records from Combined

Remove zero value lease charges for periodics, fees and initials from the combined invoice template with the following setting [gb_controls]billing_combined_show_zero_lease_charges_flag

[edit]
Consolidate Toll Charges on Combined

On the combined invoice, Toll charges can be set-up to consolidate into one transaction.
When the combined billing screen displays Tolls (posting_class_code 'TOL'), they are consolidated to one row per vehicle (also on the PDF/CSV outputs).
This generates one row per contract in fm_billing when posting and the description reads "Consolidated Toll Costs".
To turn on toll consolidation, update the setting [gb_controls]billing_combined_consolidate_tolls_flag

If toll consolidation is used, the combined billing template will require testing and possible alteration to accomodate this change.
It's also important to note that any prior toll billings will display the description 'Consolidated Toll Costs' in invoice reprints and in the Receipting screens instead of their original detailed description.
However, the detailed description is always held in the Contracts/Maintenance screen regardless of this flag.

[edit]

Billing Emails - read receipts

All billing emails have read receipts switched on by default. This is a coded preference.
Contact Catch-e Support if you would like to have this switched off. They will arrange a change request task to have this done for you.
