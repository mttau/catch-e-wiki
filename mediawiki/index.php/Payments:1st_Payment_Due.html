Finance

Below are the data population rules that are applied to a finance payment, unless edited by the user.

  1. Contract / Finance "Finance Due Day" is populated if
    1. Contract / Finance “Finance Start” is entered or edited AND
    2. Suppliers / Account “Pay Due” has a valid stored value. Visit the Finance Due Day page for full details.
  2. Contracts / Payments "Periods Deferred" updates from
    1. Contracts / Finance "Finance Deferred" if this is populated
    2. Otherwise leave this blank or enter a value if required
  3. Contracts / Payments "Due Day" updates from
    1. Contracts / Finance "Finance Due Day" if this is populated
    2. Otherwise the default value is '1'
    3. Enter a different value if required
  4. Contracts / Payments "1st Payment Due" date updates from
    1. Contracts / Finance "1st Payment Date" if this is populated
    2. Otherwise, find the first instance of the "Due Day" that occurs after the "Contract Start" date AND if "Periods Deferred" is populated, increase the "1st Payment Date" by that number of months.

Subsequent payments will scheduled using the appropriate "Due Day" if the "Periodic Interval" is 'Monthly'. Other intervals such as 'Weekly' and 'Fortnightly' will cascade from the "1st Payment Date" that is recorded.

Insurance

The '1st Payment Due' date is populated as follows (in preference order);

  1. Populate with the Contracts / Insurance "Start Date", if this field is blank;
  2. Populate with the "Due Day" date that occurs in the month following the Contracts / Billing "Contract Start" date.

Notes;

  1. "1st Payment Due" can be edited to use any desired date.
  2. Changing the "Due Day" and/or the "Periods Deferred" will update the "1st Payment Due" if no schedule exists and no "Start Date" [Insurance tab] exists.
  3. Subsequent payment dates will be scheduled using the appropriate "Due Day" if payments are Monthly (or fit a "Due Day" profile); others such as Weekly and Fortnightly will cascade from the "1st Payment Date" used.

Roadside Assistance

The '1st Payment Due' date is populated as follows (in preference order);

  1. Populate with the Contracts / Vehicle "Start Date" and populate the "Due Day" with the Contracts / Vehicle "Start Date" DAY, if this field is blank;
  2. Populate with the "Due Day" date that occurs in the month following the Contracts / Billing "Contract Start" date.

Notes;

  1. "Due Day" default is '1'
  2. "1st Payment Due" can be edited to use any desired date.
  3. Changing the "Due Day" and/or the "Periods Deferred" will update the "1st Payment Due" if no schedule exists and no "Start Date" [Insurance tab] exists.
  4. Subsequent payment dates will be scheduled using the appropriate "Due Day" if payments are Monthly (or fit a "Due Day" profile); others such as Weekly and Fortnightly will cascade from the "1st Payment Date" used.
