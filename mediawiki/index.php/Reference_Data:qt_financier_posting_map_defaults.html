Overview

This table contains fees and defaults for various Financier / Contract Type combinations. These default into quotes when the selected combination is chosen.
More granular settings can be achieved by storing entries for combinations of Financier / Contract Type / Term.

Most of the [qt financier posting map defaults] table fields are shown in the List tab, but not all.
Go to the Detail tab to edit existing records so you can see and update all the available fields.

Further Notes about using Term

If you do not use Term as a variable, any quote using the stored Financier / Contract Type combination will apply the default costs. See MAC in the table below.

If you use Term as one of your variables, the quote will select the record where the set term is equal to, or lower than the term entered in the quote.
E.g. If you raise a quote with a term of 23, the system will apply values from [qt_financier_posting_map_defaults] where the term equals 12, not 24.
If you use Term as a setting, a record where Term = 1 will be required. See STG in the table below.

Action Buttons

Button Description
Press 'Go' to execute your chosen filtering. The page will refresh with the selected transactions and these can be viewed page by page. If 'Go' is greyed out, then the current filtering has been executed.
Press 'Edit' to adjust a record. This button is only available for the most recent record (i.e. when "Last Effective Date" is checked). We recommend only using this to correct errors in a record you have created.
Press 'New' to create a new record. Do this when making a change to the values that will default into a quote. Creating a new record for each change will allow you to filter and review changes you have made to each Financier / Contract Type / Term combination over time.
