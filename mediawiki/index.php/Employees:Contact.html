Overview

Use this tab to create and record contact details when you are catering for package employees.
When a quote is done and the employee record is selected, some of these details will be populated into the quote.
Visit Drivers / Contact for the driver version of this page.

Action buttons

Field Description
View
Press 'New' to create a new employee.
Edit Click to edit an existing Employee record
Press 'Copy' to create a new employee and copy selected employee data into it. It copies all the data but employer information. Useful when a employee changes employers. List of fields that are copied can be viewed here.
Click to copy the existing record. Fields that will default across from existing record are ‘Salutation’, ‘Given Name’, ‘Middle Name’, Surname’, Work Phone’, ‘Mobile’, ’Email’, ’Address Details’ and ‘Reports By’. ‘Last Login’ will default to ‘Never’.

Home and postal addresses are copied and stored in newly created gb_address records.

Create When you have finished creating a new record, click to create.
Edit
Save When you have finished editing the record, click to save.
Cancel Click to discard your current changes.

Validations and Alerts

Alert Comments
Create or Save
Client cannot be empty! Enter the "Client" that this driver is linked to.
Employee No '{employee_no}' already exists for Client '{client_code}'! This warning will be given if the client already has an Employee stored with this "Employee No".
OK - click to accept this warning and save the record with a duplicate "Employee No".
Cancel - click to return to edit mode to update the "Employee No" before saving to remove the duplication.
{Given Name} {Surname} already exists for {Client Name} An employee record already exists with the entered "First Name" and "Surname" for the selected client. Cancel to avoid creating a duplicate record. If the record is for a different employee - find a way to make the name distinctive so the two employees are not confused.
Cannot send due to an invalid email address!
Please check the address(es) and try again.
Search for 'email address' in the online help to see all the validation rules.		 This validation error occurs when a valid email address is missing. Update employees contact details with a valid email address and resend the email.
Access
Cannot grant login access

A valid 'Email' must be entered first!

Enter the user's email address before checking the "Access" check box.
