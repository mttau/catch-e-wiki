Overview

The field "Amount Financed" can be validated using a Present Value Formula if the associated Contract Types/Details field "Validate Amount Financed" is selected.
The tolerance can be set and used in the calculation of the Present Value. This is controlled by the Global Control amount_financed_tolerance - the default allowance is set to zero. But for example you could have tolerance of +/- $5. When the "Amount Financed" value is edited, it will give an error if the value is not within tolerance.

If the "Validate Amount Financed" check box is not selected, then the validation will not be performed.

View the attached worksheet to see this validation in action.

[edit]

Present Value

Two Present Value formulas are used depending on the group the Contract Type belongs to.

[edit]
Operational

This formula is applied when Contract Type "Group" is not Novated. I.e. Fleet Managed, Insurance, Lease and Non Billing.

Amount Finance = -PV(Cost of Funds/12, Term, Payment, Net RV, Advance).
Where:

Payment = "Regular Payment"
Term = "Term"
Advance = "Advance/Arrears" (Advance = 1, Arrears = 0)
Cost of Value = "Cost of Funds"
Net Value = "Residual Value"

[edit]
Novated

This formula is applied when Contract Type "Group" is Novated.
The key difference is that the calculation also uses "Months Deferred" as a factor.

Amount Finance = -PV(Cost of Funds/12, Term, Payment, Net RV, Advance) - Deferred Interest.

Where:
Payment = "Regular Payment"
Term = "Term"
Advance = "Advance/Arrears" (Advance = 1, Arrears = 0)
Cost of Value = "Cost of Funds"
Net Value = "Residual Value"
Deferred Interest = "Deferred Interest"
