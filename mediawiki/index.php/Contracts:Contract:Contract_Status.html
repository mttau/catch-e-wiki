Overview

The contract's status and sub status is stored on the Contract, Order and Terminate tabs and defines where in your process the contract is. You can filter on these as "Status" and "Sub Status" in the Contracts / Search tab.

The "Contract Status" records you can select are configurable. Each status can be set-up to show a particular set of sub statuses.
"Contract Sub Status" functionality depends on how the selected "Contract Status" record has been configured.

Read also the Set-up section of this page to gain a full understanding of this function and it's features.

[edit]

Renaming, Hiding and Display Order

"Contract Status" and "Contract Sub Status" can both be re-labelled or hidden.
Each list presents in record order, but this can be modified to suit your business.

[edit]

'Suspended' or 'Terminated' date validation

If you update the "Contract Status" to 'Suspended' or 'Terminated' on the Contract, Order or Terminate tab, you are alerted to enter the "Suspend Date" or the "Terminated" date if the required date field is blank. The record changers cannot be saved until this is done.

If the contract has active payment schedules, entering these dates will also trigger the Stop Payment process that updates payment records.

[edit]

Linked tables

"Contract Sub Status" can be linked to one of three sub tables to deliver the desired "Contract Sub Status" information.
If you change from one linked status to another, the "Contract Sub Status" will change to display the value that is linked to the stored or selected "Contract Status".
E.g. If the Status/Sub Status is 'Ordering'/'Delivered' and the Status is changed to 'Active', the sub status will display as blank (as no sub status has been selected and "Order Status" is no longer linked).

[edit]

Auto-updating Status

If the Order or Due Off Status are being used as the "Contract Sub Status", those tables can be configured to change the "Contract Status" to another one when a particular sub status is selected,
E.g. If the Status/Sub Status is 'Ordering'/'New' and the "Order Status" is changed to 'Cancelled', the "Contract Status" will be change to "Terminated".

[edit]

Linked to Order Status

If the "Contract Status" is linked to Contracts/Order "Order Status", then both the "Contract Sub Status" and "Order Status" will display the existing due off list and data.
Any changes to the "Order Status" will update and display in "Contract Sub Status" and visa versa.
The "Order Status" can be configured to move the "Contract Status" to a new status if a particular "Order Status" is selected.
E.g. Selecting an "Order Status" of 'Settled', can move the "Contract Status" to 'Active'.

[edit]

Linked to Sub Status

If the "Contract Status" is linked to Contracts/Contract "Contract Sub Status", then a unique list box is displayed.

[edit]

Linked to Due Off Status

If the "Contract Status" is linked to Contracts/Terminate "Due Off Status", then both the "Contract Sub Status" and "Due Off Status" will display the existing due off list and data.
Any changes to the "Due Off Status" will update and display in "Contract Sub Status" and visa versa.
The "Due Off Status" can be configured to move the "Contract Status" to a new status if a particular "Due Off Status" is selected.
E.g. Selecting a "Due Off Status" of 'Sold', can move the "Contract Status" to 'Terminated'.

[edit]

Not linked

If the "Contract Status" is not linked to a sub status table, then "Contract Sub Status" will have a blank list box and no record will be stored or displayed.

Set-up

Below are shots of the default deployment of statuses with some suggestions. This can be modified to suit your processes.
For all tables, records will not be displayed in the list box where the status_flag is 'inactive'.
Contracts that are edited where these records are stored will have them silently removed on Save.

[edit]
fm_contract_statuses

The first entry in the fm_contract_statuses table is default status of new contracts when they are accepted or created.
Below are two illustrations of how you can configure this table. The first is the default set-up.

  1. Update the field display_order if you want to change the order that the statuses are shown to users in the drop list.
  2. The display_order only works in the Contracts / Search "Status" field.
[edit]
Linked to Order and Due Off Statuses

Choose this configuration when

  1. You want to see the Order Statuses and Due Off Statuses on the Contracts / Search tab.
  2. You want to update the "Contract Status" from selected Order Statuses and Due Off Statuses.



[edit]
Linked to Sub Status

Choose this configuration when

  1. You want to set a "Contract Sub Status" that can remain with the contract over it's life.
  2. You want to run processes that use "Contract Sub Status" on a separate path to Order Statuses and Due Off Statuses
  3. You use the Order Management and Due Off Management screens and processes and don't need to repeat that information on the Contracts / Search tab.



[edit]
fm_order_statuses

If you link the first status to fm_order_statuses, remember that "Order Status" also defaults on acceptance to the record with an order_status_code of 'NEW'. So this combination needs to make sense.
Hint: If you populate the contract_status_id_update field of the 'SETTLED' record to '100001', the "Contract Status" will automatically update to 'Active' when you select this sub status. This function only works if you have linked [fm_contract_statuses] to Order and Due Off Statuses.

  1. Update the field display_order if you want to change the order that the sub statuses are shown to users in the drop list.
  2. The display_order only works in the Contracts / Search "Status" field.



[edit]
fm_contract_sub_statuses

This table is not populated by default. You can advise us of the configuration you want.

  1. Populate contract_status_id if you want to link a Sub Status to a particular Contract Status.
  2. When the contract is updated to a new Contract Status, linked Sub Status records will be silently removed.
  3. Leave contract_status_id as 0 if you want to use the Sub Status on any Contract Status.
  4. When contract_status_id is 0, the chosen Sub Status will remain with the contract unless a user changes it.
  5. Update the field display_order if you want to change the order that the sub statuses are shown to users in the drop list.
  6. The display_order only works in the Contracts / Search "Status" field.



[edit]
fm_due_off_statuses

Hint: If you populate the contract_status_id_update field of the 'SOLD' record to '100003', the "Contract Status" will automatically update to 'Terminated' when you select this sub status. This function only works if you have linked [fm_contract_statuses] to Order and Due Off Statuses.

  1. Update the field display_order if you want to change the order that the sub statuses are shown to users in the drop list.
  2. The display_order only works in the Contracts / Search "Status" field.
