Email pay run reports

From Help wiki

Jump to: navigation, search

Main Page → Workflow → Salary Packaging → Pay Runs → Email pay run reports

Qualifiers

  1. Your pay run reports have been set-up
  2. Your client has allocated pay run reports
  3. A client contact is allocated to receive the 'Pay Run Reports' email
  4. You have invoiced a client's pay run
  5. It's time to email the pay run reports.
  6. Review the Pay Run Processing SP - Pay Run Process Report to monitor pay run dates.


Process

  1. Navigate to Packaging / Pay Runs
  2. Client Code - enter or lookup your client's code
  3. Pay Cycle - select the pay cycle you need (it will default in if there is only one).
  4. Status - select 'Posted'
  5. Period Date - enter or pick the date of the pay run
  6. - the screen will refresh and return the pay run records
  7. Output - choose 'Pay Run Reports'
  8. Format - 'Email' will default here. This Email can use these placeholders.
  9. - the alerts below will only be shown if there is a set-up issue
    1. Alert: The specified pay run client does not have any assigned Pay Run Reports
      1. Go to Clients / Packaging Defaults "Pay Run Reports" and select the required reports
    2. Alert: The specified pay run does not have any linked Pay Run Report "To" recipients.
      1. Go to Clients / Contact Details and allocate allocate a recipient for the 'Pay Run Reports' email.
  10. View Mail Queue - pop-up screen to review the recipient(s) and attachment(s) to be sent in the email.
    1. Batch-Item - click on hyperlink to review email text. This can be edited if the email is in text format.
    2. - an email with relevant attachments will be sent to the recipient(s)
    3. - the email is cancelled
