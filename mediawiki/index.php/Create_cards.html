Overview

This service allows you to create new card records.
It performs the same function as creating a card in the Cards / Card Details tab does.

[edit]

Permissions

To run this API, the nominated 'web-services' role needs to be given permission.
If you are not actively using the API, leave the permission off for better security.
Go to Roles / Apis and check on Cards:Create .

This API is not configured for external use but can be made available on request.
Contact Catch-e Support to request this update.


[edit]

HTTP Method

Use the HTTP Method 'POST' for consuming this web service.

[edit]

URL Examples

https://api.demo.catch-e.com/authenticate

https://api.{{environment}}.catch-e.com/authenticate

https://api.test.catch-e.com/fm/cards

Input Fields (Header)

Key Format Notes Mandatory
Audit-User-Id Number User ID that you want to show as the user in audit records.
To use this Key, the authorised role must have the permission Audit:UserIdOverride enabled. 		No


[edit]

Input Fields (Body)

JSON Field Format Notes Mandatory
contract_id Number The Contract ID the card should be linked to. Yes
card_number Number Card number provided by the supplier. Yes
supplier_id Number The supplier the card is linked to. [fm_suppliers]supplier_id. Yes
account_numberNumberAccount number of the card No
posting_flag Text This updates [fm_cards]posting_flag and defaults to 'yes' if it is not specified. No
recharge_flag Text This updates [fm_cards]recharge_flag. If this Key is not specified the value will default from the linked Contract Types / Expense-Approvals "R" fild for the ‘F - Fuel’ record. No
client_id_recharge Number This updates [fm_cards]client_id_recharge. If this Key is not specified, the Client ID that is attached to the Contract will be used. Cannot be blank if “Posting Enabled” flag = ‘yes’.No
history OnlyText ([fm_cards]history_only_flag) – This defaults from Clients / Contract Defaults "History Only".No
date ActivatedText ([fm_cards]activated_date).No
date InactivatedText([fm_cards]inactivated_date).No
expiry DateText([fm_cards]expiry_date).No
card_sub_status_id Number Pass the sub status you want for the card. I.e. [fm_cards]card_sub_status_id. No
status_flag Text Pass the status of the card. The default entry is 'active'. No
notes Text Notes to be stored with the attachmentNo
Audit-User-Id Number User Id to use for audit purposes. This header requires the Audit: UserIdOverride permission to be associated with your role.No
