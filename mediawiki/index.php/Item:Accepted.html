Overview

When you are ready to make your item active, update the "Status" field from 'Pending' to 'Accepted' or 'Active' to create the contract.

  1. If the item already has a stored "Contract ID", no updates will occur.
  2. If the "Package Item" is linked to a Contract Type "Group" that is not 'Salary Packaged', no updates will occur.
[edit]

Pending to Accepted

Update an Item to 'Accepted' if it will proceed, but final paperwork is required. The following updates will occur;

  1. a contract record will be created
  2. 'Driver Change' and a 'Registration Change' events will be created
  3. If "Substantiation' is checked, a 'Substantiation' event will be created
  4. If the Employee is linked to a "Cost Centre", a 'Cost Centre Change' event will be created
  5. the created contract will populated into the item's "Contract ID" field
[edit]

Accepted to Active

When you're ready to make the item active, update the "Status" from 'Accepted' to 'Active. the following updates will occur;

  1. a contract start and end date will be populated
  2. the billing schedule will be created
  3. Create contract
  4. Update billings
  5. If Item “Disbursement Type” = ‘Periodic’, create payment schedule.
  6. create the payment schedule is to do the following;
  • Update item to active status
  • Navigate to the contract
  • Navigate to payment tab, click on New, click on Save
  • Populate budget amounts for created budget schedule in the Contract / Lease tab based on logic supplied in the Budget_amounts_calculation
[edit]

Pending to Active

Update an item's "Status" from 'Pending' to 'Active' if all the information is in place. All of the updates listed above will occur and make the contract active

Contract Events

Two or three contract events are created when an item is accepted.
For each event created, the following field updates are standard:

  1. "Date" is left blank
  2. "Description" is left blank
  3. "Action" stores as '100001. I.e. the first record stored in the [fm_contract_event_actions] table.
  4. "Due Date" is blank.
  5. "Completed" will populate with the default that is set in the linked Posting Classes / Detail "Event Completed" field.
  6. "User" is the person who saved the item record.


Event TypeDescriptionBusiness Rules
REG Registration Change An event record is created..
"Value"is populated from "Contract ID"
DRIVER Driver Change An event record is created.
"Value" is populated from Item “Employee”
SUB Substantiation An event record is created if the Item “Substantiation” check box is checked.
"Value" is blank.
It can be configured to populate if you configure a custom list for this event and set a default value.

The "Description" custom list will not populate here, but if configured, will be available when editing the event later.

CC Cost Centre An event record is created if the Employee has a stored "Cost Centre".
