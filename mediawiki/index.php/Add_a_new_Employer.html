Qualifiers

  1. You are providing Salary Packaging services
  2. You have completed the set-up required for Salary Packaging


[edit]

Process

[edit]

Contact

  1. Navigate to Clients / Contact
  2. Click on
  3. Client Code - each code must be unique.
  4. Name - enter the employer's name.
  5. Contact 1 Name - enter the contact name. This person receives vehicle lease emails.
  6. Contact 1 Email - enter the contact email.
  7. A/C Payable - enter the accounts payable name.
  8. A/C Payable Email - enter the accounts payable email. This person receives vehicle lease invoices.
  9. Payroll Name - enter the payroll contact name.
  10. Payroll Email - enter the payroll contact email address. This person receives pay run reports and packaging invoices.
  11. A/C Manager - select from drop list
  12. Sales - select from drop list.
[edit]

Packaging Defaults

  1. Navigate to Clients / Packaging Defaults

Payroll Details

  1. Pay Cycle - select up to three pay cycles for your client.
  2. Payroll Date - enter a payroll date that occurs before the first payroll that you will be processing.
  3. Pay Run Start - enter the date you want the pay run process to start on. This usually means emailing pay run data reports to the Employers to confirm.
  4. Active - check to make the payroll active.

Banking Details

  1. Client Bank Account - check this if you have a separate bank account set-up to hold this employer's salary packaging transactions. Leave this un-checked if the standard company bank account is being used. (If this is un-checked, no further details are required in this section.)
  2. Financial Institution - display only field shows the name of the bank. This is identified from the recorded bsb.
  3. Bank Account Name - used in the Disbursements (ABA) file
  4. Bank Account Short Name - used in the Disbursements (ABA) file
  5. BSB Number - used in the Disbursements (ABA) file
  6. Bank Account Number - used in the Disbursements (ABA) file
  7. APCA Number - enter this if you need to show the APCA number on this client's Disbursements (ABA) file
  8. ABA Description - enter the description that should be used in the aba file that becomes the reference field on bank statements.
  9. ABA Self Balancing - check or un-check to set your preference to show or hide a self balancing row in aba files.

Pay Run Processing

  1. Processing State - Select the state that the payroll processing is done in. Public Holidays that occur in this state will be incorporated into the timing rules for generating the Payroll Deduction Report. I.e. if a public holiday falls within the cut off days, the report will be sent a day earlier.

Report Codes

  1. Pre Tax - if entered, the code will show on the Payroll Summary Report. This will show in brackets after the heading. E.g. Pre Tax (1-123)
  2. Post Tax - if entered, the code will show on the Payroll Summary Report. This will show in brackets after the heading. E.g. Post Tax (1-124)
  3. FBT - if entered, the code will show on the Payroll Summary Report. This will show in brackets after the heading. E.g. FBT (1-125)
  4. GST - if entered, the code will show on the Payroll Summary Report. This will show in brackets after the heading. E.g. GST (1-126)
  5. ITC - if entered, the code will show on the Payroll Summary Report. This will show in brackets after the heading. E.g. ITC (1-123)

Quote Defaults

The Employer's pay cycles and dates need to be entered before any processing can be done.
Without these details, the Employees / Item records cannot be created as the Start Date list requires this information.

  1. Navigate to Clients / Quote Defaults
  2. Pay Cycle - select the pay cycle frequency for this employer's 1st payroll
  3. Payroll Date - enter or pick one of the dates the 1st payroll occurred on. (The payroll dates need to be in the past. The system calculations do not work if these dates are in the future.)
  4. Pay Cycle 2 - select a pay cycle frequency if the employer has a 2nd payroll
  5. Payroll Date 2 - enter or pick one of the dates the 2nd payroll occurred on
  6. Pay Cycle 3 - select a pay cycle frequency if the employer has a 3rd payroll
  7. Payroll Date 3 - enter or pick one of the dates the 3rd payroll occurred on
  8. Payroll Cut Off Days - enter the number of days before a pay run that a payroll requires for processing. I.e. no new items should be added to the pay run after this time.
