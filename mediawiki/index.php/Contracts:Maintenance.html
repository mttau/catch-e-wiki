Overview

The Maintenance screen offers a summary view of a contract's transactions. Transactions can originate from the following areas:

  • Approvals (such as maintenance, services, fuel, transport and others)
  • Payments (finance, insurance and management fee)
Payment transactions are displayed if they have a "Posted" or "History only" status. The "Due Date" of the payment is displayed on the Maintenance screen and in the Maintenance report.
Finance transactions are only displayed and reported if the Finance checkbox Report Finance Payments? is checked, Insurance transactions are always displayed.
Management Fee transactions are displayed when payments are "Posted".
Single Posting Class – they are displayed if they have a status of ‘Posted’ and ‘History Only’.
  • Reg/CTP Renewals
  • Card Services
  • Contract Events
With the creation of a Driver Change Event and the entry of an odometer reading, an "Odometer Reading Update" entry is created against the maintenance screen and is shown as a valid entry
  • Vehicle Transfers
Where a vehicle is moved between contracts the Maintenance History can be shared where specific data relationships exist.

Filters

Filter Type Description [table]field
FromDateEnter or pick a date you want to filter transactions from.
ToDateEnter or pick a date you want to filter transactions to.
ClassCheck ListThe default selection is '--All--'. Click on the field to pull down the list of classes that can be checked. Check on only the classes you want to display transactions for and click Go.

This filter works independently of the "Exclude Fuel & Tolls & Insurance" check box.
E.g. If you check the Fuel check box here, "Exclude Fuel & Tolls & Insurance" will need to be un-checked, or that will keep the fuel transactions hidden.

Exclude Fuel & Tolls & InsuranceCheck BoxChecked by default. When checked the named classes will be excluded from the selection. These are often set as Fuel, Tolls and Insurance.
Configure your standard exclusion list in [gb_controls]contract_maintenance_exclude_checkbox_classes.If you want to set up a posting class , Please contract our support team
SupplierDrop ListThe default selection is '--All--'. Select a supplier to show only their transactions. Only suppliers that are linked to stored transactions for this contract will be shown.
All HistoryCheck BoxChecked by default. When selected, transactions from other contracts will be displayed on a grey background if they are believed to be the same vehicle.
Visit All History for the rules used to determine this.

Field Descriptions

Field Type Description [table]field
OdoCheck BoxWhen checked the odometer reading is treated as 'valid' and is used in reporting and current odometer reading calculations.
CTextClass: The transaction type is allocated to a particular Class, e.g. M = Maintenance. Refer to Posting Classes for further details.
RTextRecharge: Y = Yes, N = No. The recharge flag denotes if the transaction was recharged to the customer. The recharge flag defaults from the Contract Types table depending on the Class. The user may override the default at the point of data entry if permitted.
DescriptionText(40)Shows a description of the transaction. If the record is an imported fuel file, it includes the timestamp of the fuel fill.
QtyNumQuantity. Typically used to track the quantity of tyres used.
SupplierLinkSupplier code for the transaction. This link navigates you to the Supplier's record. When hovering over the link - the full name of the supplier is displayed in the message line at the bottom left of screen.
ReferenceLinkReference number of the transaction. If the transaction is an Approval the Reference is an Invoice Number and the link will navigate you to the relevant Approval. If the transaction is a Payment, the Reference is a Payment No and the link will navigate you to the Payment screen. Payment No is displayed in Payment Advice PDF's and can be used to located the transaction on the Fleet / Payments / Periodic screen.

Some Reference numbers do not display as a hyperlink and have been saved (or uploaded) directly to the fm_maintenance table. E.g. a Fuel transaction, an "Odometer" reading or "History only" maintenance transactions.

AttachmentDocuments attached to an approval can be opened on this screen by clicking on the . External users can only see and open attachments in the following two situations:
  1. If Contract Types/Details "Disclose Running Costs" check box is checked on. This is a likely scenario for a Novated Lease product.
  2. If Contract Types/Details "Disclose Running Costs" is checked off and the particular transaction row has a Recharge flag set to 'Y'. This is a likely scenario for an Operating Lease product.
App #LinkApproval Number of the transactions. The link will navigate you to the Approvals/Approval tab or the Contracts/Approval tab. If the approval is a claim ("Reimbursement?" flag is checked on), you are navigated to the Contracts/Claim tab. The link on these claim approvals also works for Drivers, as they also have access to the Contracts/Claim tab, but no other external users. I.e. Fleet Managers do not have this link.
CostNumNet of GST value of the transaction

Screen Legend

Legend Description
PendingIndicates that transactions in red are pending and have not yet been posted. Pending Insurance transactions are not displayed in the Maintenance screen.
PostedIndicates that transactions in black have been posted
BillingIndicates the transaction will be recharged but the recharge invoice has not yet been raised. These records have the following settings
  1. [fm_maintenance]recharge_flag is 'Y'
  2. [fm_maintenance]status_flag is 'recharge'
History Only This legend is shown for three situations.
  1. The record is part of an approval that was posted using the History Only button. This can be done if the transaction is to record information or past costs. These transactions are not transferred to your accounting system in export files.
  2. The record is an odometer reading.
  3. The record belongs to a contract that has the same VIN number. I.e. an older contract for the same vehicle.
    This can be useful if you are reviewing the maintenance costs done for a vehicle.

Output Options

Report NameDescription
MaintenanceChoose this to get a report of the filtered records that are displayed on the screen. This report will only select and show the records that are shown. E.g. if "Exclude Fuel and Tolls" has been checked, then those transactions are excluded from screen and this report. Select from either a PDF or Text(CSV) version.
Go to Standard Reports / Maintenance Inc Payments to run a fleet-wide version of this report.

If an external user is viewing the screen, some records you can see and report on may be hidden from them.
Visit "Disclose Running Costs" to see the rules that govern this.

FuelChoose this output to get a report of the contract's fuel usage and performance. Select from either a PDF or Text(CSV) version.

Go to Reporting / Standard Reports / Fuel Performance to run a fleet-wide version of this report.

Action Buttons

Button Description
View
Print a Maintenance or Fuel Performance PDF or Text(CSV).
Click to transfer costs from one contract to another
Click to manually add an odometer reading to the contract. The reading is added with a Class Code of ‘ODO’ and a VMRS Code of ‘FBT’.
The description will default to the description stored for the VMRS Code 'FBT'. This is something like 'Odometer Reading Update', but you can update this to show the wording you want.
The user's login is recorded as the "Reference" in the record. Once saved, the reading is stored with a status of ‘History Only’, as it is not required for cost or accounting reports.

Odometer readings are validated when they are being saved. If the value is not within 20% of the last valid reading, the user is alerted to confirm the reading is correct.

If the user is a driver updating their own vehicle's odometer record, they are asked to acknowledge a declaration that confirms the reading is true and correct. The text notes that these entries have audit records. These records have a timestamp appended to the description field, so it is easy to see when the reading was created.

This button is available if you have an IMEI Interface set-up.
Edit
When you have finished editing the record, click to save.
Cancel if you don't want to save your current edits.
