Overview

Exporting whole tables will generate a ‘dump’ of all records within the selected table. If no records are present, a file containing simply column headings will be created which may be used as a template for importing whole tables. The export whole tables process consists of the following steps:

  1. Select Table: Drop down list of available tables
  2. Create File: The system creates a temporary file
  3. Download: The system downloads the file in .csv format


[edit]

Set-up Notes

The tables that can be imported or exported can be restricted by updating entries in gb_tables.
This table also controls the sort order that the tables list displays in, if you have tables you use often, you can adjust gb_tables to put these at the top.
If an export is long running, it will be stopped if it exceeds the timeout stored in [gb_controls]whole_table_export_timeout
File size limits to the export file process apply.
