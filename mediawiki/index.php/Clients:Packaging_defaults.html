Overview

This tab is visible if the Packaging module is configured for your system.
Enter your client's packaging details here.

[edit]

Process Pages

Add a new Employer
[edit]

Header Buttons

Button Description
Click to create a new client. Three fields in the Clients / Contact tab are mandatory. These are: "Client Code", "A/C Manager" and "Sales".
Click to copy an existing client record. All existing record details except for the "Client Code" are copied to the new client record.

Field Descriptions

Field Type Description [table]field
Payroll Details
Pay Cycle List Select the client's pay cycle from the list. This field can also be seen and edited on the Quote Defaults tab.
A client can have up to three pay cycles recorded. See also the Start Periodic page.
The list of available pay cycles is limited to records where billing is enabled. Visit the Pay Cycle Setup page for more details

The first pay cycle row will be used as the default choice in vehicle quotes if the selected driver does not have a recorded pay cycle.
Once set as the default, it cannot be removed, only updated. The second and third pay cycles can be updated or removed.

The pay cycle field is read only when the client has contracts containing pending budgets for this pay cycle.
The field becomes editable when the contracts have no pending budgets.

[fm_clients]pay_cycle_id
[fm_clients]pay_cycle_id2
[fm_clients]pay_cycle_id3
Payroll Date Date Enter a payroll date here for the selected pay cycle.
For new clients, add a payroll date that has passed as the system needs a prior date to calculate the available Employees / Item "Start Date" values.

If your client's payroll is 'Monthly' and the payroll date is the last day of the month, add a prior payroll date that lands on the 31st of the month.
E.g. 31/03/2024, not 30/04/2024. System calculations will always select the last day of the month for the selected pay cycle if this is done.

The payroll date field must be populated when the client has contracts containing pending budgets for the same pay cycle.
You will be alerted if you try to remove the date.
The date can be removed when the contracts have no pending budgets.

If a pay cycle is removed the payroll date will also be removed.

[fm_clients]payroll_date
[fm_clients]payroll_date2
[fm_clients]payroll_date3
Pay Run Start Num Use this field to record a 'Start' for pay run processing. This can be used to calculate when to send pay run preview reports to your clients.
Leave this as 0 if this is not part of your process.

If a pay cycle is updated or removed the "Pay Run Start" value will be cleared.

[sp_client_packaging_defaults]pay_run_start
[sp_client_packaging_defaults]pay_run_start2
[sp_client_packaging_defaults]pay_run_start3
Active Check Box Check this on to make the pay cycle active and available for use. This can only be done once a "Pay Cycle" and "Payroll Date" have been entered.

Once checked, the pay cycle can be selected in the Employees / Contact "Pay Cycle" list.

Uncheck to make the pay cycle inactive. Do this when a pay cycle is being wound down.
Inactive pay cycles can't be added to new employee records and are removed when an employee record is edited.
When an employee's stored pay cycle is inactive, new items cannot be created.

[fm_clients]pay_cycle_id_active_flag
[fm_clients]pay_cycle_id2_active_flag
[fm_clients]pay_cycle_id3_active_flag

If this is un-checked, except for the fields "BSB" and "Account Number", the values set in the other banking fields will be disabled and the data flushed.

Taxable Sales List Enter value to determine how GST is treated for this client.
This field name can be renamed. 		[sp_client_packaging_defaults]taxable_sales
Processing State List Choose the appropriate state for the payroll.
This field name can be renamed. 		[sp_client_packaging_defaults]pay_run_processing_state
Weekend Treatment List Choose the timing of when reports can be run, based on payroll sequence and if the date falls on a weekend or not. The options are defined as follows.
  • ‘Leave as weekend’ – reports will generate on weekends
  • ‘Pay on Friday’ - reports due on the weekend will run on the Friday before the weekend
  • ‘Pay on Monday’ - reports due on the weekend will run on the Monday after the weekend

This field name can be renamed.

[sp_client_packaging_defaults]pay_run_weekend_treatment
Delivery Type List This field sets the output that is applied when the client's pay run is invoiced.
Choose from either 'Email' (the default) or 'Post'.
If 'Email' is chosen, then reports will be emailed when the pay run is invoiced.
If 'Post' is chosen, a PDF will be launched.

Validations and Alerts

Below are alerts that you may receive if a record cannot be saved.

Alert Comments
Field entry
BSB Number could not be matched to a valid financial institution A valid BSB must be entered
Bank Account Number must be number and up to 9 digits A valid Bank Account Number must be entered i.e. no non-numeric characters and only up to 9 digits long
APCA Number is a 6 digit number The APCA Number must be 6 digits long and cannot contain other characters. Correct or remove the entry.
Save
Bank Account Number must be numeric and up to 9 digitsThe Account Number must be between 6 and 9 digits long and cannot contain other characters. Correct or remove the entry.
BSB Number cannot be empty if a Bank Account Number has been entered. Enter the BSB Number or remove the Bank Account Number.
The BSB could not be matched to a valid financial institution. The first 2 or 3 numbers of the BSB need to be the same as a stored [sp_financial_institutions]financial_institution_number.
Visit BSB Number Checker to confirm the code you are trying to use.
If the code is correct, add the required record to your [sp_financial_institutions] table.
Payroll Date cannot be blank for the '{pay_cycle_code}' Pay Cycle. If a client has packaged contracts with pending budgets for the named pay cycle, the "Payroll Date" must be stored.
Add the payroll date and then save your changes.

Set-up Notes

  1. This tab is not visible unless configured by Catch-e Support.
  2. Visit Role Restrictions → Clients for set-up details.
  3. "Pay Cycle" - check and update [gb_controls]fm_clients_pay_cycle_default to store your most popular "Pay Cycle" entry.
  4. "Client Bank Account" - check and update [gb_control]sp_client_packaging_defaults_bank_account_enabled_flag_default to store your preferred setting for this field.
  5. "Taxable Sales" - check and set a default value for this field in [gb_control]sp_client_packaging_defaults_taxable_sales_default.
