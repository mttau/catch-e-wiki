Overview

This page details the structure of the disbursements aba file.

[edit]

Record Selection

  1. Contracts / Contract "Company" is the same as the selected company or
  2. Contracts / Contract "Client" is the same as the selected client
  3. The contract is linked to an Employees / Item record
  4. The transaction is
    1. a claim or
    2. a payment
  5. The transaction "Payment Method" is 'Direct Credit'
  6. If the "Payee" is
    1. 'Supplier', then the suppliers "BSB Number" and "Bank Account Number" must be populated correctly
    2. Otherwise, the "Payee" can be 'Employee' or 'Driver' and the transaction must contains a stored Bank Account.
  7. The transaction is posted
  8. The transaction has not already been paid on a previous aba file

Negative Transactions

When negative Claims and/or Payments have been raised, they will be included / excluded as detailed below.

  1. When "Payee" = 'Supplier' and the Supplier's "ABA Format Type" = 'Supplier'
    1. Include all transactions (both negative and positive) in data set
      1. If batch total is positive
        1. Create the batch
        2. Group and Sum transactions by Supplier
      2. If batch total is negative
        1. Exclude records, no batch is created
        2. Negative transactions will remain pending until the next positive batch's total amount exceeds the total amount of the pending negative transactions.
  2. For all other set ups
    "Payee" = 'Supplier' and the Supplier's "ABA Format Type" = 'Payment Reference' *
    "Payee" = 'Employee' and the Supplier's "ABA Format Type" = 'Supplier'
    "Payee" = 'Employee' and the Supplier's "ABA Format Type" = 'Payment Reference' *
    1. Negative amounts will be excluded
      1. If batch total is positive
        1. Create the batch
          1. Payments will be grouped by the Payment Schedule
          2. Claims will be grouped by Contract
      2. If the batch total is negative
        1. Exclude records, no batch is created
        2. * Negative transactions will remain pending until the next positive batch's total amount exceeds the total amount of the pending negative transactions.

Note:

  1. Currently, all Employees / Item records are linked to contracts where the Contract Type "Group" is 'Salary Packaged'
  2. This will be relaxed in future when all claim and payment records have populated "Payee" and "Payment Method" data.
[edit]

Record Grouping

Suppliers / Account "ABA Format Type".

  1. 'Payment Reference' creates a separate ABA file row for each eligible claim and payment transaction.
  2. 'Supplier' groups claims or payments being paid to a supplier into one totalled row. This grouping will only be done when the claim or payment "Payee" is also set as 'Supplier'.

[gb_becs]transaction_date(yyyymmdd)_disbursements_[gb_becs]becs_id_[sp_client_packaging_defaults]bank_account_name.aba

  • When the selected "ABA Type" = 'Company', the name of the ABA file will be constructed using the Client details.

  • These will display on the file as follows.

Detail Record - Self Balancing

If the "ABA Type" is 'Company' and [gl_bank_account_aba_mappings]aba_self_balancing_flag is 'no', this record is not generated.
If the "ABA Type" is 'Client' and Clients / Packaging Defaults "ABA Self Balancing" is unchecked, this record is not generated.
