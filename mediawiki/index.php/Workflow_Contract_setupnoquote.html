[edit]

Create the Contract

Go to Fleet / Administration / Contracts: click and create contract details.
This will create a contract with "ORDER" as the Registration Number.

[edit]

Place and Manage the Vehicle Order

  • Contracts/Order - update Order Information and click to generate a Dealer Order in an Excel format if required.
  • Use Fleet / Administration / Order Management and Fleet / Administration / Reports / Orders to manage vehicles on order.
  • Update the Order status to "Delivered" when the vehicle has been delivered, or if it is already in your fleet.
[edit]

When Delivered

  • Contracts/Vehicle – and enter vehicle information - some recommended basics are as follows:
  • Registration Managed/Expiry Date and Flag and Rego State (for Registration reporting)
  • FBT Value, Start Odometer Reading and Date (for FBT reporting).
  • VIN and Engine No
  • Service Interval and Months (for Service Due and Overdue reporting).
  • Contracts/Finance – edit and enter the Financier code (if not copied from the quote), Finance reference number, finance start date, end date, 1st payment date etc.
  • Contracts/Events - enter the followings events:
1. Create a Registration Change event to put in the Registration plate number. Use the vehicle delivery date as the event date.
2. Create a Driver Change event and put the delivery date in as the event date.
If the Driver is new, they will need to be created in Fleet/Administration/Drivers before you can create the Driver Change event.
If the vehicle is a Pool vehicle, we recommend a "Pool" driver be created. This enables a custodian to be allocated and the useage of the vehicle to be clear.
3. Create a Cost Centre event if applicable

Where the event date is left blank, the system date will be used as the default entry on 'Save'.

If the Cost Centre is new, it will need to be added to Fleet/Administration/Clients/Cost Centres before the event can be created.
  • Contracts/Billing - enter the Lease Term, the Contract start date and Contract Type (contract end will auto calculate).
Even if the vehicle is owned, having a Lease Term enables Fleet Management reporting. Contract Type is an essential field.
[edit]

If Invoicing a Client (n/a for internal fleets)

  • Contracts/Billing
  • Set the periods deferred as required (often ‘2’).
  • Set the Start period to the month you want to start invoicing – e.g. if delivered in December then start invoicing 1/1/2008.
  • Contracts/Lease - check the lease budgets being billed – they should be as per the quote although ECM budgets may need to be entered for reporting.
  • For ECM reporting, if you want the FBT reports to show the ECM amounts used in the quote you need to add them as ECM budgets on the Lease tab. If you edit the lease screen before the invoicing you can for example enter the optimal monthly ECM as per the quote. If the quote only shows ECM by pay cycle or annually, divide the annual amount by 12 to get the average monthly ECM amount to enter on the lease screen. There are 2 class codes used for this: “Employee Contribution” – enter the debit amount. The 2nd class is “ECM Clearing” – enter the same amount as a credit . These debit and credit entries mean that the tax invoice to the client is the correct amount (i.e. finance and budget costs) and the system uses the EmployeeContribution amount on the FBT reports.
  • Contracts/Insurance – edit and set the billing type to ‘Driver Reimbursement’ if Driver's own insurance is used.
This is necessary for correct budget vs. actual reporting, see Billing Type for full details.
  • Contracts/Maintenance – create an opening odometer reading for an existing vehicle.
This improves Fleet Management Reporting and Fuelcard reading validations.
