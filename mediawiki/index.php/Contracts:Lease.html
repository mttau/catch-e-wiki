Overview

The Contracts / Lease tab shows the budget amounts for each period of the contract. When you navigate to this tab, you will see the budgets for the next pending period. This record has an asterix showing next to the date in the "Period" field . E.g. '17-01/10/2010*'

You can change the period in the "Period" list box to see previous (posted) periods or future (pending) periods. You can change the budget amounts for pending records, but not posted ones, as they have been billed.

You will normally see a budget rows for each budget that is stored in the linked Contract Type / Accruals tab. So a 'Fleet Managed' contract may display only three budget rows while a 'Novated' contract may have more than 10 budget rows.

When billing process runs, pending records will be billed if the contract is not suspended. If a contract is suspended and the pending record is dated before the entered Contracts / Contract "Suspend Date", it will still be billed. Any pending records dated on or after the "Suspend Date" will not be billed. This allows you to future-date a "Suspend Date" on a contract, or to insert adjustments as required.

If you need to update the contract budgets for multiple contracts, consider using the Import Data / Contract Budgets feature.

GST rounding:
When the individual budgets are added together to get the total GST there can be a perceived rounding error where the GST total is not exactly 10% of the total net value.

In these cases the GST on the highest budget (usually finance) is amended to get the right total. When quotes are accepted the original values are kept but if the lease screen is edited and the "Overridden" tick box is off then this GST recalculation will occur.

Entering a Lease

In the Billing screen, the fields "Lease Term", "Contract Start", "Contract End", "Contract Type" and "Start Periodic" must be populated before the Lease screen details can be created.
Click to enter the required budgets in Period 1 - the system then populates the remaining periods with the same values.
If the Quotations module is being used, the budget values from the quote will show in the Lease screen and no editing should be required (if the quote budgets are correct!).

[edit]

Editing a Lease

You can only edit unbilled scheduled lease periods. For example, if a contract has been billed for 16 periods, then periods 17 on can be edited.

If a period is edited, all the remaining periods will be updated with the changed values. So, if a budget variation is needed, edit the period when the variation should occur and the change will flow on to all subsequent periods.

If you only want to edit one period, edit the selected period and then adjust the subsequent period to reflect the required ongoing value.

Another approach is to use the Insert function to add in the change as a secondary record in the period, so no other changes are required.

[edit]
Editing out of sequence

It's possible to un-post a scheduled record and have later records that remain posted. In this situation, if this earlier record is edited, it will still update all remaining scheduled records regardless of where in the billing sequence they are.
For example, period records 1, 2, 4 and 5 are invoiced and period 3 was un-posted and is now pending. Any changes made to period 3 will also affect period 6 onwards. However, if period 6 is changed, period 3 will remain unchanged, as changes to scheduled records only update future scheduled records.

Adjusting a Lease(Insert)

Go to the process page Adjust a lease period billing for the steps to do this.
You can adjust a single lease period by adding an adjustment record.
These are numbered sequentially after the inital billing. E.g. If the adjustment is against the 4th billing period, the first adjustment will be identified as 4.2, the second 4.3, and so on.
You can also identify adjustments in the Lease screen as the Note column displays "Adjustment".

To make an adjustment, select the billing row you want to adjust and then click on .

When an adjustment row has not been billed:

  • You can the record - it will only change the selected adjustment row. Note the difference here in behaviour to regular schedule rows where an edit updates all future scheduled billing.
  • You can the record. Again, note that you cannot delete a scheduled billing.

You can make multiple adjustments against a period if you need to. I.e. you can click on Period 4, or Period 4.2 and in both cases, the new adjustment 4.3 will be created.

You can make adjustments with a 0, postive or negative value.

  • If the adjustment has a value, a tax invoice or an adjustment note will be generated within the standard billing process. The invoice will be described as usual, except the billing period will display with the suffix (E.g. 4.2). From this, you will be able to identify that the transaction is an adjusting entry.
  • If the adjusment results in a 0 value total, no invoice will be generated in the billing process. Specifically - the billing routine is run and the 0 value transaction is posted. When this is bundled with later transactions of value, you will not notice anything. If you run the billing only for a 0 value adjustment, the warning message "Nothing found" will appear - ignore this and the transaction anyway. If this adjustment has been made against a Novated Contract, the billing period will not be included in the Driver Statement under 'Months Life to Date so far' and 'Months this FBT year so far', as the billed period has been reversed.

Click here to view the Release Notes pdf about this feature. Any adjustments you want to make must be run before you change a Contract's "Periodic Interval" in the Billing screen. Changing the interval will re-write all unposted billing records the new interval.

[edit]

Generating Credit Notes

If configured to do so, a credit note can be generated when you edit a lease period or insert an adjustment that has a total negative value. If this happens, the "Note" field is updated to display 'Credit' when the record is saved.

If a credit note is created, it will be invoiced even if the contract has been suspended.

This needs to be set-up and is not the default configuration.


[edit]

Missing a Budget?

If a budget line has been added to the "Contract Type" after the lease began, it will not be displayed in the Lease screen. If you need to use the missing budget, you'll need to do the following;

  1. Go to the Billing screen and click "Vary" followed by "Save" and "OK to Save any Changes".

Once you have done this, the missing budget line will be displayed on the Lease screen. Go to The effect of "Save" when changing "Contract Type" to see how this works.

For workflow tips, go to the Budget Variations page.

we recommend that you print or export the Lease Schedule Report before you make any billing or lease changes on existing contracts.

Troubleshooting

Visit the main Troubleshooting page for a list of all the available problem-solving tips.


[edit]
The Finalise button is disabled
[edit]
Enable rules
  1. The contract belongs to the 'Novated' group.
  2. A "Suspend Date" has been entered on the contract.
  3. There are no transactions in 'Pending' or 'On Hold' status in the Payments and Maintenance tabs.
  4. At least one period has been billed.

Even if all the conditions for using the Finalise button have been met, sometimes required data is missing if the contract has been migrated into the Catch-e System.
In these situations, the Finalise feature cannot be used and you will need to finalise the contract manually.

[edit]
The Un-Post button is disabled
  1. The selected period is pending
  2. The selected period contains adjustment records. These must be uposted in reverse sequence, and the records must then be deleted to then enable the previous record to be unposted.
E.g. Unpost Period 4.3, then delete it.
Unpost Period 4.2, then delete it.
Period 4 can then be unposted.
