Overview

This report screen is designed for formal reports which can be output to PDF and are available to external users such as Fleet Managers.
The filter fields are dynamic, so later fields are updated to show only available choices. For example, if a particular client is selected then the system will refresh displaying only that client’s relevant details.

Selected reports can be scheduled to email directly to clients periodically.
Reports can be configured to show channel logos if required.

More reports are available in the Reports / Standard Reports screen for internal users.

Report Filters

The report filters generally are described below. Not all filters are used on all reports.
Each report page contains the specific details of that report including its particular filters.

The report filters filters are inactive (greyed out) if they do not match your data.
E.g. If you use the FBT Operating Report and select a client that has no Operating Method contracts, then the remaining filters will not be electable and the report cannot be run.

Troubleshooting

Contract Type - vehicles will not show up on reports unless they have been assigned a “Contract Type”. The “Contract Type” governs a lot of the business rules and treatment of the contract – and are also helpful in managing the vehicle.

Start Date - reports are date-driven, so if the vehicle has no “Start Date”, then it will show up on all reports regardless. If you don’t know when a vehicle started, put in an arbitrary date to improve reporting.

Suspend Date - the “Suspend Date” is the field that the system uses to indicate that the vehicle is inactive for reporting or approvals. Vehicles will continue to display on reports until a "Suspend Date" is entered.
