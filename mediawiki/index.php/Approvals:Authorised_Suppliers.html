Authorised Suppliers

When an maintenance row is added to an Approval the combination of Posting Class and Supplier is to be validated against the Authorised Suppliers table as soon as the Posting Class is selected.

If there is a match for the Class and no matching Suppliers (in fm_client_supplier_posting_classes) all users are to be presented with a pop up warning message:

Class Code "Maintenance" is not allowed for this Supplier in Client Approval Defaults. Please review.

Click ‘OK’ to continue. At this stage this is merely a warning to the user.

If there is no match for the Class users can continue to enter the Approval details as usual.

[edit]

Email Notification

When the ‘Save’ button is clicked for an approval (regardless of whether it’s a new approval or updating an existing one) where there is a match for the Class and no matching Suppliers (in fm_client_supplier_posting_classes) an email notification is created for Client Contact 1. (This is similar functionality to Approval Limit exceeded email notification). The users are presented with a pop up warning message:

Class Code "Maintenance" is not allowed for this Supplier in Client Approval Defaults. Click OK to send email to Client Contact. Click Cancel to review approval details.

Clicking ‘OK’ will allow the Approval to proceed and send an email notification to Client Contact 1 email address.

Clicking ‘Cancel’ will present a second warning message (Click OK to save this Approval without sending an email to Client Contact. Click Cancel to review approvals details.) where clicking ‘OK’ will allow the approval to be saved. Click ‘Cancel’ once more to leave the Approval in edit mode (No email sent yet). When user is ready to ‘Save’ the approval again the validation should be triggered again.

Email Setup

A Template entry named ‘approvals_authorised_supplier_class_exception_notification’ that contains the wording for the email notification to your client.

The email notification can be set up in either plain text or html and uses applicable placeholders as shown here.

PlaceholderDescription[Table]Field
#reg_no# Returns the Reg No from the Contract [fm_contracts] reg_no
#short_description# Returns the Vehicles Short Description [qt_variants] short_description
#approval_id# Returns the Approval ID [fm_maintenance] approval_id
#supplier_name# Returns the Supplier name for the Supplier requesting the Approval [fm_suppliers] name
#approval_exceptions_approval_total# Returns the total Approval amount (Incl GST) SUM( [fm_maintenance] total_cost)
#approval_exceptions_html#
OR
#approval_exceptions_text# 		Returns the Approval Job Row/s that are the subject of the exception [fm_maintenance] description
#approval_exceptions_job_summary_html#
OR
#approval_exceptions_job_summary_text# 		Returns the first row from each Job included in the Approval when at least 1 row is the subject of the exception [fm_maintenance] description
