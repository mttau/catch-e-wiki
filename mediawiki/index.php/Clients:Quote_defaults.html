Overview

The Quote Defaults screen contains fields used specifically in the quotation process:

[edit]

Header Buttons

Button Description
Click to create a new client. Three fields in the Clients / Contact tab are mandatory. These are: "Client Code", "A/C Manager" and "Sales".
Click to copy an existing client record. All existing record details except for the "Client Code" are copied to the new client record.

Pay Cycle Drop List Choose the pay cycle you want to populate into new quotes for this client.
The default value for this field is 'Monthly', but this may be changed. The list of available pay cycles is limited to records where billing is enabled.
Visit the Pay Cycle Setup page for more details.

The pay cycle field is read only when the client has packaged contracts containing pending budgets for the same pay cycle.
The field becomes editable when the contracts have no pending budgets.

This field can be renamed. Access to this field can be role restricted.

Payroll Date Date Enter one of the client's pay run dates for this payroll here.
If this field is populated, the "Start Periodic" date for new contracts can be calculated. Visit Start Periodic for full details.

The payroll date field must be populated when the client has packaged contracts containing pending budgets for the same pay cycle.
You will be alerted if you try to remove the date. The date can be removed when the contracts have no pending budgets.

If a pay cycle is removed the payroll date will also be removed.

This field can be renamed. Access to this field can be role restricted.

Financier Lookup Enter or lookup a financier's "Supplier Code" here if you want to set a default entry for the client.

This field will show a default entry if the client is linked to a channel and the Channels / Channel Defaults "Financier" field is populated.

When creating new quotes, Quotes / Finance "Financier" will be populated in the following preference order

  1. If Clients / Quote Defaults "Financier" is populated, use this financier.
  2. If Contract Types / Details "Quote Financier" is populated, use this financier.
  3. Lookup or directly enter a financier.

Visit the Financier page for configuration details.

Validations and Alerts

Below are alerts that you may receive if a record cannot be saved.

Alert Comments
Edit
Table {table_name} already locked by user: {user_login} The table cannot be edited while another user is editing it. Wait and try again later. Visit System Locks for more details.
Field entry
Pay cycle already in use! When adding Pay Cycles for a client, each one must be different. E.g. if the client has two fortnightly payrolls, select 'Fortnightly' and 'Fortnightly2'.
Select a different pay cycle, or leave the field blank.
Save
Payroll Date cannot be blank for the '{pay_cycle_code}' Pay Cycle. If a client has packaged contracts with pending budgets for the named pay cycle, the "Payroll Date" must be stored.
Add the payroll date and then save your changes.
Payroll Date cannot be the 15th of the month when Pay Cycle is 'Bi-Monthly' or 'Twice Monthly'. When a Pay Cycle is set to 'Bi-Monthly' or 'Twice Monthly', the second payroll date of the billing sequence is always assumed to be the 15th of the month.
Update the Payroll Date field to record the other payroll date that is to be used.
